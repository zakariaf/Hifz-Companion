## Goal

Two domain-blind selection leaves exist under `packages/features/lib/src/design_system/`: a named **`CyclePresetPicker`** — a single-select list of the five named cycles (7-Manzil weekly khatm · 1 juz/day · ½ juz/day · 2 juz/day · Custom) plus a labelled **Pure-cycle** conservatism toggle, the visible face of the engine's cycle ceiling rendered as a choice a teacher recognizes — and a reusable generic **`SettingsPicker<T>`** single-choice pattern (M3 `RadioListTile` / selectable `Card` / single-select sub-page) for one mutually-exclusive display-only preference (language · calendar · numerals · term-set · theme · muṣḥaf-riwāyah). Both draw their interaction states from the T02 `MihrabStateLayer` and render every label/number through the T01 `numberFormatFor(locale)` path; both take a fixture selection + an `onSelected` callback and **write nothing** — they emit a choice, never an `EngineConfig`, a `due_at`, a stored instant, or a re-typeset glyph. Selected-state and per-locale (fa/ckb/ar × four appearances) goldens prove the M3 single-select discipline, the no-slider rule, and the no-reward/no-recommendation invariants. The onboarding/Settings screens that host them and the Riverpod stores that persist the choice are E11/E16.

## Implementation notes

These are domain-blind display leaves: correctness is **structural** — the controls are M3 single-select (no slider anywhere), they emit a choice and mutate nothing, every label/number flows through T01, and the state/focus/Semantics come from T02. The feature-layer wiring that maps a chosen preset → `EngineConfig` and persists it (E11/E16) is **out of scope**; this task ships the leaf + a fixture-driven preview + goldens.

TEST-FIRST (correctness-critical, the no-slider / no-mutation invariant): write `cycle_preset_picker_test.dart` and `settings_picker_test.dart` first — the **`find.byType(Slider)` → `findsNothing`** assertion, the "selection routes through `onSelected`, the widget writes nothing" assertion, and the "calendar option re-render leaves the fixture instant unchanged" assertion must exist and fail before either widget is implemented.

1. **Location & exports.** Create under the `features` design-system namespace (E10-T02 convention):
   - `packages/features/lib/src/design_system/pickers/cycle_preset_picker.dart` — the `CyclePresetPicker` leaf (the five named presets + the Pure-cycle toggle; the Custom *reveal* is a collapsed affordance only — the four bounded Custom fields' full editor is the feature layer E11/E16, this leaf shows the "Custom" option and a disclosure, not the editor).
   - `packages/features/lib/src/design_system/pickers/settings_picker.dart` — the generic `SettingsPicker<T>` single-select pattern.
   Export both through the `features` barrel under `design_system`; keep `lib/src/` internals private. REUSE SPDX header (`GPL-3.0-or-later`) on every new file. `dart format` + `dart analyze --fatal-infos` clean.

2. **`CyclePresetPicker` is a domain-blind leaf.** Its inputs are display data only: an `enum CyclePreset { weeklyKhatm, oneJuzPerDay, halfJuzPerDay, twoJuzPerDay, custom }`, the localized name/subtitle for each (passed in as `l10n.*`-sourced strings, **not** authored here), the current selection, a `bool pureCycleEnabled`, and two callbacks `ValueChanged<CyclePreset> onPresetSelected` / `ValueChanged<bool> onPureCycleChanged`. It imports **no** `EngineConfig`, no engine, no store, no `due_at`. Render the presets as a radiogroup of `RadioListTile`/selectable `Card` rows (`SegmentedButton` only if a short variant is ever needed); the Pure-cycle toggle is a labelled `Switch.adaptive` selected state matching T05's teacher-toggle treatment, with a fidelity subtitle ("follow my cycle exactly — no reordering"). **No `Slider`, no numeric dial, no `target_R`, no D/S/R, no retention percentage anywhere in the tree.**

3. **`SettingsPicker<T>` is the generic single-choice pattern.** `SettingsPicker<T>({ required List<SettingsOption<T>> options, required T selected, required ValueChanged<T> onSelected })` where `SettingsOption<T>` is `{ T value; String label; String? subtitle; }`. It renders a radiogroup of ≥48dp rows, selected = radio glyph **and** filled-emphasis text (shape AND color), focus ring from `MihrabFocusRing`. It is preference-agnostic: language/calendar/numerals/term-set/theme/muṣḥaf-riwāyah are all just `List<SettingsOption<T>>` the feature layer builds. The leaf computes nothing and stores nothing — `onSelected` fires and the widget is done. For the **muṣḥaf-riwāyah** option set, the label carries the named edition + riwāyah ("Ḥafṣ ʿan ʿĀṣim — Madani muṣḥaf"); the leaf never re-typesets a glyph, never applies UI numerals/fonts to a glyph page, and never calls an edition "the Quran" absolutely.

4. **Numbers and dates through T01 only.** Every numeral in a preset name/subtitle ("7-Manzil", "every 7 days", "1 juz / day") and every numeral/date in a Settings option label ("Hijri ۱۴۴۷", "Jalālī ۱۴۰۵") is produced by `numberFormatFor(locale)` / `CalendarPresenter` (T01) and the mixed run is FSI/PDI-isolated (`isolateLtr` for the digit token). No `n.toString()`, no ASCII-digit splice, no hardcoded English "Manzil/Juz" — term-set names are `l10n.*` ARB strings (authoring owned by the localization pipeline; this task references keys).

5. **RTL & token discipline.** Logical `start`/`end` insets only (`EdgeInsetsDirectional`); the selected affordance/leading radio at start, the disclosure/chevron at end; spacing via `space.*`, colors via `color.*` roles, type via `type.body`/`type.caption`, target via `touch.min` — no inline hex/pt/dp, no `EdgeInsets.only(left:/right:)`. The selected/pressed/focused/disabled visuals come from `MihrabStateLayer` (T02); the focus ring is `MihrabFocusRing`. Read direction via `Directionality.of(context)`.

6. **Register the state sets into the T02 gallery.** Add a `ComponentStateMatrix` for each picker (`CyclePresetPicker`: default / a non-default preset selected / Pure-cycle on / focused / Custom-row focused; `SettingsPicker`: a representative option set with selected / pressed / focused, plus a calendar-option specimen and a muṣḥaf-riwāyah specimen) so `pumpComponentMatrix` renders the full set with no host screen. Specimen builders receive fixture display data only.

7. **Pitfalls to avoid:**
   - **A slider / dial / `target_R` / D/S/R / retention %.** The single most important reject: a `Slider`, a "due cards per day" stepper exposed as the primary control, an FSRS number, or any retention percentage in either tree. The cycle is a **named** choice; the test asserts `find.byType(Slider)` is `findsNothing`.
   - **Mutating anything.** Neither leaf writes an `EngineConfig`, a `due_at`, a stored instant, or a preference; selection only fires the callback. A calendar option must be a pure re-render — the fixture instant is unchanged after selection (asserted).
   - **Re-typesetting / mislabelling the muṣḥaf.** The muṣḥaf-riwāyah option never draws a glyph, never mirrors/translates/applies UI numerals to a page, and never reads "the Quran" absolutely — it names the edition + riwāyah.
   - **A reward/celebration on selection.** No glow/pop/sparkle/confetti/streak/badge/"optimal!"/exclamation on picking a preset or toggling Pure-cycle; selection is an M3 state layer (assert the celebratory tokens — which do not exist in E06 — are absent).
   - **"Disable smart features" framing for Pure-cycle.** It reads as fidelity ("follow my cycle exactly, no reordering"), never as making the app worse; run the framing past the adab conscience-check.
   - **Hardcoded English term-set / ASCII digits / physical-side insets.** Term-set names are ARB strings (ckb provisional), numerals via T01, layout logical-only — the `features/**` physical-side and ASCII-digit greps are gates.
   - **Building the Custom editor or the host screen.** The Custom four-field editor, the onboarding cycle step, and the Settings list are E11/E16 — this leaf shows the "Custom" option + a disclosure affordance, nothing more.

## Acceptance criteria

- [ ] `cycle_preset_picker.dart` exposes a `CyclePresetPicker` leaf rendering the five named presets (7-Manzil weekly khatm · 1 juz/day · ½ juz/day · 2 juz/day · Custom) as a single-select radiogroup plus a labelled Pure-cycle `Switch.adaptive`, taking display data + `onPresetSelected`/`onPureCycleChanged` callbacks; it imports no `EngineConfig`, engine, store, or `due_at`, and emits a choice only.
- [ ] No `Slider`, numeric retention dial, `target_R`, D/S/R value, or retention percentage exists anywhere in either picker's tree (asserted by `find.byType(Slider)` → `findsNothing` and an absence check).
- [ ] The Pure-cycle toggle is framed as **fidelity** ("follow my cycle exactly — no reordering"), passes the adab conscience-check, and its selected `Switch.adaptive` state matches the T05 teacher-toggle treatment.
- [ ] `settings_picker.dart` exposes a generic `SettingsPicker<T>` single-choice pattern (`List<SettingsOption<T>>` + `selected` + `onSelected`) of ≥48dp radiogroup rows; selected state is the radio glyph **and** filled-emphasis label (shape AND color, never hue alone); the muṣḥaf-riwāyah option names the edition + riwāyah and never re-typesets/mirrors/translates a glyph page.
- [ ] Selecting any option (preset or setting) routes solely through the callback — neither leaf mutates an `EngineConfig`, a `due_at`, a stored instant, or a preference; a calendar option's selection is a pure re-render leaving the fixture instant unchanged.
- [ ] Every numeral/date in a label renders through `numberFormatFor(locale)` / `CalendarPresenter` (T01) in the locale digit set (Extended Arabic-Indic fa/ckb, Arabic-Indic ar), mixed runs FSI/PDI-isolated; term-set labels are `l10n.*` ARB strings (ckb provisional), never hardcoded English or ASCII digits.
- [ ] Both leaves draw enabled/pressed/disabled/focused/selected from `MihrabStateLayer` + `MihrabFocusRing` (T02); all visual values come from named `color.*`/`type.*`/`space.*`/`touch.min` tokens (E06); layout is logical `start`/`end` only — no inline hex/pt/dp, no physical-side inset.
- [ ] Each option is a ≥48×48dp / 44×44pt target with a localized `Semantics` name + selected value, grouped as a radiogroup (`inMutuallyExclusiveGroup`), with a visible focus ring per WCAG 2.2 SC 2.4.7; states mirror under RTL.
- [ ] No reward/celebration on selection (no glow/pop/sparkle/confetti/streak/badge/"optimal!"/exclamation); selection is a quiet M3 state layer.
- [ ] Both pickers register a `ComponentStateMatrix` into the T02 gallery and are golden-tested via `pumpComponentMatrix`; every new file carries the REUSE SPDX header; `dart format` + `dart analyze --fatal-infos` clean.
