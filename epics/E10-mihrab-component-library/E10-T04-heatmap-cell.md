## Goal

One domain-blind leaf widget exists under `packages/features/lib/src/design_system/heatmap/`: a `HeatmapCell` — the single retention square that the E15 whole-Quran grid composes 604 times. It paints the calm **single-hue green-receding-to-neutral lightness ramp** (`color.heatmap.strong → faded`, by token name, **never red**), applies **VSUP-style muting** so a never-recited or self-rating-only page renders suppressed toward the faded neutral and saturates only as confident reviews accumulate, encodes its state **three ways at once** (ramp colour + a locale-numeral value/range + a plain transcreated label + optional decay texture), and carries the **min-leaning juz roll-up** variant with its weakest-page badge at the logical start. It is a stateless renderer fed a display-only view model and an optional `onTap`; it reads `card.R` / `sourceConfidence` / the min-leaning aggregate from a fixture (never recomputes them), draws no Quran glyph, shows no raw D/S/R or crisp percentage, and is never a streak, score, or scoreboard tile. The whole-Quran `GridView`, the page-detail sheet, the upcoming-load forecast, the Progress route, and the Riverpod read model that streams real engine state are **E15 progress-and-heatmap** — this task owns the cell + roll-up badge leaves and their strength/uncertainty/locale golden matrix with a grayscale/deuteranope readability check, nothing more.

## Implementation notes

**TEST-FIRST (correctness-critical: the honesty mappings).** Four behaviours a reviewer must trust before any pixel ships: (1) the ramp is single-hue lightness, **never red**, and magnitude order survives grayscale/deuteranope; (2) a low-confidence/never-recited input renders **muted**, and a single self-rating cannot reach the top tier; (3) every cell carries a locale-numeral value/range **and** a plain label (colour never alone); (4) the juz roll-up is **min-leaning** and surfaces a weakest-page badge, never a mean, and no cell renders a raw `R`/D/S/crisp-% or a "safe to drop"/scoreboard state. Write the widget + mapping tests in the **Tests** section first, then the widget — and assert the negatives (no red, no percentage, no streak/celebration, no mean) as hard as the positives.

1. **Location & shape.** The cell and its small collaborators live beside the other E10 components in the `features` umbrella package:
   - `packages/features/lib/src/design_system/heatmap/heatmap_cell.dart` — the `HeatmapCell` leaf (page cell + juz roll-up variant).
   - `packages/features/lib/src/design_system/heatmap/heat_level.dart` — a tiny `enum HeatLevel { strong, good, fair, weak, faded }` (display-blind; maps 1:1 to `color.heatmap.{strong,good,fair,weak,faded}`). The engine's `card.R → HeatLevel` classification lives in E04/E15, **not** here.
   - `packages/features/lib/src/design_system/heatmap/heatmap_cell_data.dart` — the immutable, domain-blind view model (below).
   - `packages/features/lib/src/design_system/heatmap/weakest_page_badge.dart` — the calm weakest-page marker.
   Export the public surface through the `features` barrel under the `design_system` namespace; keep `lib/src/` internals private. REUSE SPDX header (`GPL-3.0-or-later`) on every new file. All widgets are `StatelessWidget` (focus/press handled by standard M3 plumbing) — **no Riverpod, no store, no engine/date/persistence/`quran` import**.

2. **The view-model contract (display-only, engine-blind).** `HeatmapCellData` carries only what the cell renders, never anything it would compute:
   - `HeatLevel level` — the engine-classified band (page) or the **min-leaning** `rollUp` (juz). The cell never derives it from `R`.
   - `String localizedValue` — the retrievability percentage **or range**, already formatted in locale numerals via the T01 `numberFormatFor(locale)` path (e.g. a range "۸۰–۹۰٪" or "estimated — not yet recited"); the leaf never calls `intl` itself and never splices ASCII digits.
   - `String label` — the transcreated plain band label ("strong" / "softening" / "ready for revision"), an `AppLocalizations` key resolved by the caller/specimen, never a hardcoded literal.
   - `bool everReviewed` and `double sourceConfidence` — the two VSUP inputs (`self ≈ 0.5`, `teacher = 1.0`); the cell reads them to choose the muted blend, it never writes or recomputes them.
   - `bool isJuzRollUp` + `int? weakestPageId` — the juz variant flag and the weak-page the badge names; null on a page cell.
   - `bool showDecayTexture` — opt-in third channel for the decaying end.
   The constructor and the consumer (`onTap`) are the only inputs; there is **no** `card.R`, `due_at`, `DateTime`, or store handle on the type.

3. **The ramp — single-hue lightness, by token name, never red.** Resolve `level → color.heatmap.{strong..faded}` through the E06 `HeatmapTokens` `ThemeExtension` (`Theme.of(context).extension<...>()`), never an inline hex and never a hue axis. `strong` is the calm base green; the decaying end (`weak`/`faded`) is a **muted neutral**, not a second hue and **never** `color.semantic.*` red — the cell must reference no alarm-red token (a grep over the file proves it). The ramp re-tones per appearance via the theme; the encoding and roll-up are appearance-invariant.

4. **VSUP muting — resolve the blend at the token layer, never a per-widget alpha.** When `!everReviewed` or `sourceConfidence` is low, suppress the fill toward `color.heatmap.faded` / the surface neutral via the `HeatmapTokens` muting helper (a named blend step, not an invented per-widget opacity). A **never-recited** page renders at the faded-grey tier **regardless of an optimistic prior**; a **self-rating-only** page reads less saturated than a teacher-confirmed one and **cannot** resolve to the `strong` tier — encode that as an explicit clamp in the level→fill resolution so a reviewer can read the rule. This is the visual analogue of the engine's conservative priors (the cell honours the rule; it does not own the priors — domain-scheduling-engine-rules).

5. **Redundant encoding — colour + number + label (+ optional texture), bidi-safe.** The fill is reinforcement, never the message: render `localizedValue` and `label` inside/under the cell so the state is legible in greyscale, to a colour-blind ḥāfiẓ, and to a screen reader. The numeral run is bidi-isolated (FSI/PDI) via the T01/eng-rtl helper so a Latin-script range or page number never reorders inside the RTL label. When `showDecayTexture`, paint a subtle pattern (the third colour-independent channel) over the decaying end only — a restrained `CustomPainter`, no glyph, no decoration that reads as an alarm. Lower ramp steps may sit below the 3:1 non-text floor (they live inside labelled tap-through cells); `strong` as the glance anchor must clear 3:1 — assert this in the contrast test, do not restate the 03 §7 audit.

6. **The cell square, target, and selected/focus state.** Constrain the tappable square to `touch.min` (≥48dp) min width/height; gutters between cells are the grid's job (E15) at `space.1` (4dp), but the cell documents the `space.1`/`space.4` expectation it is built for. When `onTap != null` the cell is a button; wire the **selected** + **focused** states through `MihrabStateLayer` / `MihrabFocusRing` (E10-T02) — a visible `color.outline` ring on focus (SC 2.4.7) that follows the component under RTL, **not** a physical side. There is **no** celebratory selected/hover effect (no glow/pop/sparkle) — assert its absence.

7. **The juz roll-up variant + weakest-page badge.** When `isJuzRollUp`, the cell's fill uses the supplied **min-leaning** `level` (the leaf renders it; the min-leaning aggregate is computed upstream — never average inside the widget), and a `WeakestPageBadge(pageId: weakestPageId)` is positioned with `PositionedDirectional(start: 0, top: 0)` so it sits at the logical **start** under `Directionality.rtl`. The badge is calm — a quiet dot/marker from the heat ramp's weak step or a neutral, **never** `color.semantic.*` red, never an alarm exclamation. The badge shows only when a weak link exists (`weakestPageId != null`).

8. **`Semantics` — value + label as words, in the active locale.** Wrap the cell so a screen reader hears one merged phrase like "Juz ۱۳: needs revision, weakest page ۲۵۳" (page cell: "Page ۲۵۳: ready for revision, estimated ۸۰–۹۰٪") in the active locale via `MergeSemantics`; expose `button`/`selected`/`focusable` flags through the T02 `Semantics`-flag helper; `ExcludeSemantics` any purely decorative texture. The announced strings are `AppLocalizations` keys (ar template + fa/ckb transcreation), bidi-isolated; never a raw `R`/percentage spoken as a bare number.

9. **Register the state set into the T02 gallery.** Add a `ComponentStateMatrix` for the cell with named specimens covering the rule surface: `strong`, `good`, `fair`, `weak`, `faded` (the five ramp steps); `muted_never_reviewed` and `muted_self_rating_only` (the two VSUP cases); `juz_rollup_with_weak_badge` (the min-leaning variant); and `selected` / `focused`. Specimen builders receive only `HeatmapCellData` + a no-op `onTap`; no engine/date import. `pumpComponentMatrix` produces the per-locale goldens — re-implement no matrix loop.

10. **Pitfalls to avoid:**
    - **Building the grid / sheet / screen.** No 604-cell `GridView`, no `JuzTile` grid container, no page-detail sheet, no upcoming-load forecast, no `ProgressScreen` route, no `retentionMapProvider` — all **E15**. Any grid/sheet/route/store leaking into this PR is a review reject, filed against E15.
    - **Recomputing engine state.** No `card.R`, `due_at`, `sourceConfidence`, or juz aggregate is computed in the widget — they arrive on `HeatmapCellData`. No `DateTime.now()`; the cell shows no date at all.
    - **A red / rainbow / traffic-light ramp.** Green→red, jet/rainbow, a saturated alarm-red decay end, or any `color.semantic.*` red on the cell/badge — forbidden; assert the file references no alarm-red token and the ramp is single-hue lightness.
    - **Averaging the juz.** A mean (or any aggregate) inside the widget that hides one rotting page — forbidden; the leaf renders the supplied min-leaning `level` and surfaces the weakest-page badge.
    - **A raw number / crisp percentage / D/S/R.** A bare "87%", an `R`/D/S value, or a "safe to drop"/"mastered"/"100% green" cell state — forbidden; the value is a localized band/range shown as the prediction it is; assert none renders.
    - **Colour as the sole channel.** A cell with no label/value — forbidden (SC 1.4.1, A3); the grayscale/deuteranope test fails it.
    - **A streak / score / celebration.** No streak tile, points, leaderboard, "completion %", confetti, or green-flash on the cell or its selected state (R3/C6; C-041, C-044) — assert absent; there is no E06 token for such an effect.
    - **Physical-side layout or a hardcoded `Directionality`.** No `EdgeInsets.only(left:/right:)` / `Positioned(left:)` / `Alignment.centerLeft/Right` (the `features/**` grep is a gate); the badge uses `PositionedDirectional(start:)`, the leaf reads direction from context, the harness supplies `Directionality.rtl`.
    - **A hardcoded label / ASCII digit / inline hex.** Every label/value is an ARB key + the T01 numeral path; every colour is a `color.heatmap.*` token by name — no literal, no raw ASCII digit, no inline hex survives the gates.
    - **A Quran glyph or decoration.** The cell draws no āyah and tints no glyph — the muṣḥaf lives only in the E13 reader; the optional texture is an abstract pattern, never a glyph.
    - **An `Ahem`/placeholder font in the goldens.** This library's goldens load the real bundled UI fonts (epic DoD); `Ahem` is forbidden here.
