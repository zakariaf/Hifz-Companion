## Goal

The pure-UI infrastructure every later E10 component reuses exists under `packages/features/lib/src/design_system/`: a `#Preview`-equivalent **gallery harness** that renders any component's full named state set in isolation (no host screen, no route, no store), the shared **M3 interaction-state model** (`MihrabStateLayer` over a role color + a visible focus ring per [07-components §6](../../docs/design-system/07-components.md)) that every interactive leaf draws its enabled/pressed/disabled/focused/selected states from, and a single reusable **golden-matrix scaffold** (`pumpComponentMatrix`) that pumps any specimen across fa/ckb/ar × the four appearances (Light · Sepia · Dark · Night) on the real bundled UI fonts under the throwing-`HttpOverrides` offline bootstrap. T03–T10 reuse this one scaffold and this one state model instead of hand-duplicating per-component matrices. No domain logic, no engine read, no glyph — pure UI infrastructure.

## Implementation notes

This is UI infrastructure: correctness here is *structural* (the state model is M3-correct and RTL-mirrored; the matrix is exhaustive and deterministic; the offline guard is installed), so the harness's own tests and a couple of reference specimens are the contract. Build it to be **reused unchanged** by T03–T10 — any per-component matrix that re-implements the pump loop is a review reject.

1. **Location.** The design system lives inside the `features` umbrella package (E01/E06). Create:
   - `packages/features/lib/src/design_system/state/mihrab_state_layer.dart` — the shared M3 interaction-state model.
   - `packages/features/lib/src/design_system/preview/component_gallery.dart` — the `#Preview`-equivalent gallery harness types (`ComponentSpecimen`, `ComponentStateMatrix`, the `ComponentGallery` viewer).
   - `packages/features/test/support/golden_matrix.dart` — the reusable `pumpComponentMatrix` golden scaffold (test-support, not shipped).
   - `packages/features/test/support/offline_test_bootstrap.dart` (or extend the repo-wide `test/test_setup.dart`) — the library entry point that calls `useOfflineTestPolicy()` and `loadMihrabUiFonts()`.
   Export the public `state/` and `preview/` surface through the `features` barrel under the `design_system` namespace; keep `lib/src/` internals private; `test/support/` is never exported. REUSE SPDX header (`GPL-3.0-or-later`) on every new file.

2. **The shared interaction-state model (`MihrabStateLayer`).** Define a `MihrabInteractionState` enum `{ enabled, pressed, disabled, focused, selected }` and a `MihrabStateLayer` helper that, given a role color from `ColorScheme` and the active state(s), returns the M3 **state-layer overlay** (a translucent `on`-color over the role color at the M3 opacities) — never an ad-hoc per-component opacity or a bespoke hue (§6 anti-pattern). Provide a `WidgetStateProperty`/`WidgetStatesController`-driven `overlayColor` builder so leaves consume it through standard M3 `ButtonStyle`/`InkWell` plumbing, and a `MihrabFocusRing` (a `FocusableActionDetector`/`Focus`-aware outline drawn in `color.outline`, ≥2dp, following the component via `Directionality.of(context)`, **not** a physical side) so SC 2.4.7 is structural. Expose a `Semantics` flag mapping (`enabled`/`selected`/`focusable`) helper so leaves announce state non-visually. **Disabled = dimmed role + no state layer**, styled as *waiting, not error* (the disabled grade band reads "reveal to grade", §6). No `selected`/`pressed` path may add a glow/scale/celebratory effect — assert its absence (adab; epic DoD).

3. **The gallery / `#Preview` harness.** A `ComponentSpecimen` is `{ String name; WidgetBuilder build; }`; a `ComponentStateMatrix` is an *ordered, named* set of specimens (e.g. the grade band's `enabled/pressed/disabled/focused`, the page card's six states) — the analog of a Swift `#Preview` set. `ComponentGallery` is a dumb scrollable View that renders a passed `ComponentStateMatrix` as a labeled vertical list, each specimen in a captioned cell, inside the composed `ThemeData` and a context-derived `Directionality` — so a developer (and the golden scaffold) renders **any component's full state set in isolation, with no host screen, no `go_router` route, and no Riverpod store**. The gallery references no engine/date/persistence type; specimen builders receive only display data. The state-label captions are `l10n.*` keys / dev placeholders, never user-facing claim copy.

4. **The reusable golden-matrix scaffold (`pumpComponentMatrix`).** A single test helper that takes a `ComponentStateMatrix` (or one specimen) and, for each of `fa`/`ckb`/`ar` × `MihrabAppearance.{light, sepia, dark, night}`, pumps it under `Directionality(textDirection: TextDirection.rtl)` inside the matching composed `ThemeData` (E06-T06) and the locale's `AppLocalizations`, with `numberFormatFor(locale)`/`CalendarPresenter` (T01) supplying any numerals/dates, then `expectLater(find.byType(ComponentGallery), matchesGoldenFile('goldens/<component>/<state>__<locale>__<appearance>.png'))`. Pin `devicePixelRatio`, `physicalSize`, and theme; disable animations; load the **real bundled Vazirmatn/Estedad via `FontLoader`** in `loadMihrabUiFonts()` (never `Ahem` — Sorani extra letters پ چ ژ ڤ ک گ ڕ ڵ ۆ ێ ە ھ and Persian/Arabic digits must actually render). Also run the matrix once at **200% text scale** (`MediaQuery` `textScaler: TextScaler.linear(2.0)`) so reflow-not-clip is provable per locale (ckb is the longest transcreation). Tag the goldens `@Tags(['golden'])` for the pinned-OS lane; masters regenerate only via local `--update-goldens`.

5. **The accessibility-gate hook.** Add a `meetsLibraryGuidelines(tester)` helper that runs `androidTapTargetGuideline`, `labeledTapTargetGuideline`, and `textContrastGuideline` against the pumped tree (09 §10), plus a `assertColorIndependent(...)` grayscale/deuteranope readability check stub the cell/decay components (T03/T04) fill in — so the same gate code is called once per component and aggregated in T10, never re-written per task.

6. **The offline bootstrap.** Make `test/support/offline_test_bootstrap.dart` the library-wide `setUp` entry: it calls `useOfflineTestPolicy()` (installs the throwing `_ThrowingHttpOverrides` from `docs/engineering/11-testing-strategy.md` §7 — a stray socket is a loud, named failure, not a silent 400) and `loadMihrabUiFonts()`. Every E10 widget/golden suite imports and calls it; **no** E10 component opens a socket, so none opts out (only the asset-downloader test does, and that is not in this epic). Building the themes and loading the bundled fonts are the only IO.

7. **Pitfalls to avoid:**
   - **Building a component.** No page card, grade band, heat-map cell, picker, banner, reminder row, or confirm gate is authored here — this task ships *only* the state model, the gallery harness, and the golden scaffold. A real component leaking into this PR is a review reject, filed against its owning task (T03–T09).
   - **A reward/celebration state.** A `pressed`/`selected` path that adds a glow, scale-up, sparkle, confetti, or a "success" haptic/motion tier — there is no such token in E06; state feedback is a quiet state layer + (at most) one restrained haptic tick. Assert the absence.
   - **A hardcoded app-wide `Directionality` or an LTR assumption.** The harness wraps each specimen in `Directionality.rtl` *deliberately* (the sanctioned manual island for a host-less leaf) and reads direction via `Directionality.of(context)`; it never assumes RTL in the state-layer/focus-ring geometry, and never uses `EdgeInsets.only(left:/right:)` / `Alignment.centerLeft/Right` (the `features/**` physical-side grep is a gate).
   - **Re-implementing the matrix per component.** T03–T10 must call `pumpComponentMatrix`; if a per-component test re-writes the locale×appearance loop, the scaffold has failed its purpose — keep the API small and total.
   - **`Ahem` / a placeholder font in the goldens.** This library's goldens load the real bundled UI fonts (epic DoD); the font-independent `alchemist` trick is allowed only for pure-layout goldens elsewhere, never here.
   - **A `package:quran`/QPC-glyph reference or any `TextStyle` that re-typesets a Quran string.** The harness is chrome-only; no muṣḥaf glyph ever appears in the gallery or a specimen (the wall is E06-T05).
   - **An engine/date/persistence import.** The gallery and specimens take display data only; `numberFormatFor`/`CalendarPresenter` (T01) are the single number/date path, called by the caller, not re-implemented.

- [ ] `component_gallery.dart` exposes `ComponentSpecimen`, `ComponentStateMatrix`, and a dumb `ComponentGallery` View that renders a passed state matrix as a labeled list in isolation — no `go_router` route, no Riverpod store, no engine/date/persistence/`quran` import; specimen builders receive display data only.
- [ ] `packages/features/test/support/golden_matrix.dart` exposes `pumpComponentMatrix(...)` that renders any `ComponentStateMatrix` across `fa`/`ckb`/`ar` × `{light, sepia, dark, night}` (and a 200%-text-scale pass) under `Directionality.rtl` on the **real bundled Vazirmatn/Estedad** (`FontLoader`, never `Ahem`), pins DPR/size/theme, disables animations, and is `@Tags(['golden'])`.
- [ ] `meetsLibraryGuidelines(tester)` runs `androidTapTargetGuideline` + `labeledTapTargetGuideline` + `textContrastGuideline`, and an `assertColorIndependent(...)` grayscale check exists for T03/T04 to fill — both reused by T10, not re-written per task.
- [ ] `test/support/offline_test_bootstrap.dart` installs the throwing `HttpOverrides` (`useOfflineTestPolicy()`) and `loadMihrabUiFonts()`; it is the single entry T03–T10 call; no E10 component opts out.
- [ ] The public `state/` + `preview/` surface is exported through the `features` barrel under `design_system`; `lib/src/`/`test/support/` internals stay private; every new file carries the REUSE SPDX header; `dart format` and `dart analyze --fatal-infos` are clean.
- [ ] No component (page card, grade band, heat-map cell, picker, banner, reminder row, confirm gate) is built in this task; the only widget shipped is the generic `ComponentGallery` viewer.

## Tests

In `packages/features/test/design_system/` (Flutter widget + golden tests on E01's lanes; goldens load the **real** bundled UI fonts via `FontLoader`, never `Ahem`; all suites call `offline_test_bootstrap` in `setUp`):

- `mihrab_state_layer_test.dart` (widget/unit) — for a reference `FilledButton`/`InkWell` styled through `MihrabStateLayer`: the `enabled` overlay is transparent; `pressed`/`focused`/`selected` resolve to the **M3 state-layer overlay over the role color** (assert the resolved `overlayColor` is the role's `on`-color at the M3 opacity, not a bespoke `Color`); `disabled` dims the role and adds **no** state layer; the `MihrabFocusRing` appears on focus (`findsOneWidget`) drawn in `color.outline`, and on RTL it follows the component (assert it is not pinned to a physical side); the `Semantics` node carries the right enabled/selected flags. **A celebratory-state guard:** no glow/scale/sparkle widget and no haptic/celebratory-motion call appears on `pressed`/`selected` (`findsNothing`).
- `component_gallery_test.dart` (widget) — a fixture `ComponentStateMatrix` of three dummy specimens renders as a labeled list with each specimen's caption present; the gallery pumps without a `MaterialApp` route or any provider (assert it builds host-less under a bare `Directionality.rtl` + `ThemeData`); it imports no engine/date/persistence/`quran` symbol (verifiable by an import-scope check).
- `golden_matrix_scaffold_golden_test.dart` (`@Tags(['golden'])`) — `pumpComponentMatrix` over a small **reference specimen set** (one enabled + one disabled + one focused styled box) produces the full fa/ckb/ar × Light/Sepia/Dark/Night matrix (12 frames) plus the 200%-text-scale pass, each `matchesGoldenFile` on the pinned-OS lane — proving the scaffold itself is deterministic, RTL-correct, real-font, and reflow-safe before any component depends on it. (These are reference frames for the scaffold, not a shipped component.)
- `library_guidelines_test.dart` (widget) — `meetsLibraryGuidelines(tester)` passes on the reference specimens (tap target ≥48dp, labeled, text contrast ≥4.5:1 in each appearance); `assertColorIndependent(...)`'s stub is exercised on a redundantly-encoded fixture so T03/T04 inherit a working check.
- **Offline guard:** every suite above runs under the shared throwing-`HttpOverrides` bootstrap; a negative test asserts that a deliberate `HttpClient()` attempt inside a specimen throws the named `StateError` — proving the guard is live for the whole library. Building the themes and loading the bundled fonts is the only IO.
