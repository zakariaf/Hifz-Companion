## Goal

A calm, neutral evidence-certainty badge exists as a domain-blind leaf under `packages/features/lib/src/design_system/`: a sealed `EvidenceGrade` enum (`ma`/`rct`/`exp`/`cs`/`obs`/`text`/`trad`) parsed once from the bundled `CLAIMS.md` data (never the raw `[..]` tag), a **pure** `String certaintyLabel(EvidenceGrade, CertaintyStrings)` mapping (no `BuildContext`, no I/O, no model) that returns a lay confidence-*about-the-evidence* phrase, a `CertaintyLabel` widget that renders that phrase in **one neutral container for every grade** (never a traffic-light color, never color alone, no star rating, no "proven", no retention/Quran promise), and the always-reachable plain-words `CertaintyLegend`. `[TRAD]` reads as named scholarship paired with its source, issues no fiqh ruling, and is not ranked above the empirical grades. The mapping is unit-tested test-first for all seven grades; the badge ships a seven-grade × three-locale (fa/ckb/ar) golden matrix proving the non-traffic-light styling. This task builds the *renderer only* — the science screen that hosts it and the CLAIMS register it reads are E19.

## Implementation notes

**TEST-FIRST:** write `certainty_label_test.dart` (the seven-grade mapping unit suite, §Tests) **before** the `certaintyLabel` body and the widget. The mapping is the correctness-critical core (a wrong phrase ships straight to a ḥāfiẓ as a methodology claim); the seven cases must exist and fail before the switch is filled.

1. **Location & files.** Pure-UI leaf inside the `features` umbrella design-system namespace (same anatomy as T02):
   - `packages/features/lib/src/design_system/certainty/evidence_grade.dart` — the sealed `enum EvidenceGrade { ma, rct, exp, cs, obs, text, trad }` with a `static EvidenceGrade parse(String tag)` that strips `[ ]`/whitespace, lowercases, and `firstWhere`s by `name`, **throwing a typed register-integrity error** on an unknown grade (no silent default — it is a release-blocking data defect; see `template.dart`). Order the values best→weakest among empiricals, with `trad` last (named scholarship, not ranked above them).
   - `packages/features/lib/src/design_system/certainty/certainty_strings.dart` — an immutable `CertaintyStrings` value type holding the six lay phrases + `semanticPrefix`, built at the call site from `AppLocalizations` (so the mapping stays pure and testable across fa/ckb/ar). No `BuildContext` stored.
   - `packages/features/lib/src/design_system/certainty/certainty_label.dart` — the pure `String certaintyLabel(EvidenceGrade, CertaintyStrings)` function **and** the `CertaintyLabel` + `CertaintyLegend` widgets.
   Export the public surface through the `features` barrel under `design_system`; keep `lib/src/` internal. REUSE SPDX header (`GPL-3.0-or-later`) on every file. `dart format` + `dart analyze --fatal-infos` clean.

2. **The pure mapping (`certaintyLabel`).** Total, never-throwing over the sealed enum via an exhaustive `switch` expression (no `default`, so adding a grade is a compile error, not a silent fallthrough). Each arm returns the localized phrase from `CertaintyStrings`. `[RCT]` and `[EXP]` both map to the same "a single controlled study" phrase (per §5). The function takes the strings *in* — no `AppLocalizations.of(context)` inside it — so the unit test passes plain fixture strings and the golden test passes the real localized ones. Describe the **evidence**, never the user's Quran; no arm contains "safe", "mastered", "proven", a percentage, or a star.

3. **The ARB strings.** Add seven `l10n.*` keys for the phrases plus the legend rows and `semanticPrefix` to the ar template ARB and the fa/ckb transcreations (authored under the localization pipeline / E19 — this task references keys, never hardcodes a literal). Phrases are **transcreated** per locale (the calm-honest register must survive), not literally translated. `nullable-getter: false` key coverage; no hardcoded user-facing string. (If the keys do not yet exist in this branch, stub them via a local fixture `CertaintyStrings` for the widget tests and file the ARB authoring against E19's localization task — but the production widget reads `AppLocalizations`.)

4. **The neutral badge (`CertaintyLabel`).** A `StatelessWidget` taking `{ required EvidenceGrade grade, required CertaintyStrings strings }`. **One neutral container for all seven grades** — read `color.surface` / `color.text.secondary` (and the `type.caption` token) by name from the E06 design-system tokens via `Theme.of(context)`; **never pick color by grade**, never a `color.semantic.warning`/success fill, never red-weak/green-strong. Strength is carried by the **text phrase** only (optionally a single neutral shape/icon, same for all grades — never color-as-signal). Wrap in `Semantics(label: '${strings.semanticPrefix}$phrase')` so the strength is read out as text, never inferred from color; use `ExcludeSemantics` on any decorative shape. Logical insets (`EdgeInsetsDirectional` / `space.*`) only. If a literal grade name is shown as a glossed secondary, isolate any Latin run with the `eng-rtl-and-bidi-layout` `isolateLtr` helper. The badge is **non-interactive** (no tap, no focus ring, no state layer beyond a static surface) — it is a read-only label, not a control.

5. **The plain-words legend (`CertaintyLegend`).** An always-reachable `StatelessWidget` rendering one calm row per grade — a `CertaintyLabel` beside its localized plain-words explanation ("a meta-analysis pools many studies; an experiment is a single controlled test; a traditional source is named scholarship") — ARB-sourced, transcreated, **never** a "★★★★★" key. The `[TRAD]` row reads as named scholarship, deferring to the teacher and the *sanad*; it issues no ruling and is not visually elevated above the empirical rows. Reflows (not truncates) at 200% text scale (ckb is the longest transcreation).

6. **Pitfalls to avoid:**
   - **Passing the raw `[MA]`/`[TRAD]` tag into the badge**, or rendering it as the label — the widget takes the sealed enum; parsing happens once (E19) via `EvidenceGrade.parse`.
   - **A traffic-light / color-coded badge** — green for `[MA]`, red/amber for `[OBS]`/`[TRAD]`. The container is the *same* neutral surface for all seven; color is never the signal (WCAG 1.4.1; voice §2).
   - **A star rating, a "proven" badge, a confidence percentage, or any retention/Quran promise** — all forbidden by §5 / C-016 / C-017; the phrase is a quiet statement, never a guarantee.
   - **Ranking `[TRAD]` above the empirical grades** or styling it as a fiqh verdict — it is named scholarship paired with a source, methodology only, issues no ruling (C-046, CLAIMS scope clause).
   - **A `BuildContext`/`AppLocalizations`/I/O call inside `certaintyLabel`** — it must stay pure; strings are injected.
   - **A silent fallback for an unknown grade** — `parse` throws a typed error (release-blocking data defect), it never renders a default badge.
   - **Hardcoded hex / inline `TextStyle` color** or an English-first literal — tokens by name, ARB-sourced transcreated strings only.
   - **Conflating this evidence grade with the recite grade band (T05)** — Again/Hard/Good/Easy is a different axis entirely.
   - **A network/`google_fonts`/model call to "explain" a grade** — fully offline, bundled, static.

## Acceptance criteria

- [ ] `evidence_grade.dart` exposes a sealed `enum EvidenceGrade { ma, rct, exp, cs, obs, text, trad }` (best→weakest empirical order, `trad` last) and `EvidenceGrade.parse(String)` that strips brackets/whitespace, lowercases, matches by `name`, and **throws a typed register-integrity error** on an unknown grade — no silent default.
- [ ] `certaintyLabel(EvidenceGrade, CertaintyStrings)` is a **pure** function (no `BuildContext`, no I/O, no model), total over the sealed enum via an exhaustive `switch` (no `default`), returning the §5 lay phrase per grade; `rct` and `exp` map to the same "single controlled study" phrase; no arm contains "safe"/"mastered"/"proven"/a percentage/a star.
- [ ] `CertaintyLabel` renders the phrase in **one neutral `color.surface`/`color.text.secondary` container identical for all seven grades** — color is never picked by grade, no traffic-light fill, no warning/success color; strength is in the text only; text ≥ 4.5:1 across light/sepia/dark.
- [ ] The badge is non-interactive (no tap target, no focus ring, no reward/celebration state); it carries a localized `Semantics` label (`semanticPrefix` + phrase) so the grade is read out as text, with decoration under `ExcludeSemantics`.
- [ ] `[TRAD]` renders as "named traditional scholarship" paired with the source by name/number, issues no fiqh ruling, and is not styled as more authoritative than the empirical grades.
- [ ] `CertaintyLegend` is an always-reachable plain-words legend (one calm row per grade, no "★★★★★"), ARB-sourced and transcreated per fa/ckb/ar, reflowing without truncation at 200% text scale.
- [ ] All user-facing copy is `AppLocalizations` keys (ar template + fa/ckb transcreation), no hardcoded literal; any Latin source name / literal grade name is bidi-isolated (`isolateLtr`); logical `start`/`end` insets only.
- [ ] No network/`google_fonts`/model call anywhere; the grade and wording come from the bundled register/binary; the file imports no engine/date/persistence/`quran` symbol; REUSE SPDX header present; `dart format` + `dart analyze --fatal-infos` clean.

## Tests

In `packages/features/test/design_system/certainty/` — all suites call T02's `offline_test_bootstrap` (`useOfflineTestPolicy()` throwing-`HttpOverrides` + `loadMihrabUiFonts()`) in `setUp`; goldens reuse `pumpComponentMatrix` on the **real** bundled UI fonts (never `Ahem`), `@Tags(['golden'])` for the pinned-OS lane.

- `certainty_label_test.dart` (**unit, `package:test`-style, written FIRST**) — the correctness-critical mapping:
  - `certaintyLabel` returns the expected phrase for **all seven** grades (`ma`/`rct`/`exp`/`cs`/`obs`/`text`/`trad`) against a fixture `CertaintyStrings`; `rct` and `exp` return the *same* "single controlled study" phrase.
  - No phrase contains the banned substrings "safe", "mastered", "proven", "%", or "★" (a copy-guard over the fixture-rendered output) — encoding the §5/adab anti-patterns as an assertion.
  - `EvidenceGrade.parse` round-trips every tag form (`"[MA]"`, `"MA"`, `" ma "`) to the right enum; an unknown tag (`"[XYZ]"`, `""`) **throws** the typed register-integrity error (no default).
  - The `switch` is exhaustive — adding a grade is a compile error (documented; verified by the no-`default` structure).
- `certainty_label_widget_test.dart` (widget) — `CertaintyLabel` for each grade renders the phrase as primary text, carries the `Semantics` label (`semanticPrefix` + phrase), exposes **no tap target / no focus ring** (it is read-only), and draws the literal grade name (if shown) bidi-isolated; the resolved container color is the **same** neutral `color.surface` for `ma` and for `obs`/`trad` (assert byte-equal fills — the non-traffic-light guard at the widget layer); `meetsLibraryGuidelines(tester)` passes (labeled, text contrast ≥ 4.5:1 in each appearance).
- `certainty_label_golden_test.dart` (`@Tags(['golden'])`) — the **seven-grade × three-locale (fa/ckb/ar) golden matrix** via `pumpComponentMatrix` across the four appearances (Light/Sepia/Dark/Night) plus a 200%-text-scale pass, each `matchesGoldenFile('goldens/certainty_label/<grade>__<locale>__<appearance>.png')` — proving the neutral (non-traffic-light) styling is identical across grades and the ckb (longest) transcreation reflows without clipping. A `CertaintyLegend` golden (fa/ckb/ar × four appearances) proves the plain-words legend has no star key and reflows at 200%.
- `assertColorIndependent(...)` (the T02 grayscale/deuteranope stub) over the seven badges — confirms that with hue removed the grade is still read from the text, never lost (the badge is by construction monochrome-neutral, so this passes trivially and documents it).
- **Offline guard:** every suite runs under the shared throwing-`HttpOverrides`; a negative check asserts a deliberate `HttpClient()` attempt inside a specimen throws the named `StateError`. The only IO is composing themes and loading the bundled UI fonts.
