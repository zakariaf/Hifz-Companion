## Goal

Two domain-blind banner leaves exist under `packages/features/lib/src/design_system/banners/`: a calm **`CatchUpBanner`** that renders the engine's **already-computed** re-spread plan in the empathy ŌåÆ honest fact ŌåÆ path ŌåÆ choice order ("you missed N days ŌĆö here is an M-day catch-up plan that still completes your cycle"), and an **`EmptyState`** family with three variants ŌĆö welcoming **first-run**, calm **all-done / nothing-due** terminal, and **neutral silent welcome-back** after a gap. Both are *states* of a host surface (the Today list, E12), not screens; both compose a pre-built read model (`CatchUpPlan` / `EmptyStateModel`) fed by a fixture and render every count/date through the T01 `numberFormatFor(locale)` / `CalendarPresenter` path, drawing interaction states from T02. They compute **no** catch-up math (that is E04), open **no** socket, and mutate **no** state ŌĆö `CatchUpBanner` emits the user's choice through a callback, the empty states emit at most one gentle next-step callback. Per-locale (fa/ckb/ar ├Ś four appearances) goldens plus copy-invariant assertions prove the no-shame floor: no red overdue pile, no broken-streak, no "you're behind" scold, no "Welcome back! N days", no confetti on all-done. The Today list these sit inside, its Riverpod store, and the ARB string authoring are E12 / the eng-* skills.

## Implementation notes

These are domain-blind display leaves: correctness is **structural** ŌĆö they compose an already-computed read model, compute no catch-up math, mutate nothing, render every count/date through T01, take their interaction states from T02, and pass the copy-invariant lint. The Today list container, its loading/populated sequencing, the budget-feedback line, the Riverpod store, and the ARB string authoring are **out of scope** (E12 / eng-* skills); this task ships the leaves + fixture-driven previews + goldens.

TEST-FIRST (correctness-critical ŌĆö the no-shame / no-mutation / no-compute invariants): write `catch_up_banner_test.dart` and `empty_state_test.dart` first. The **`assertNoBannedPhrase`** copy-invariant (no guilt/fear/loss / streak / "Welcome back ŌĆ” N days" / "safe to drop" / exclamation / emoji), the **"renders the fixture `CatchUpPlan`, computes nothing and emits a choice only"** assertion, the **"all-done renders no confetti/celebration widget and no celebratory haptic/motion call"** assertion, and the **"silent welcome-back exposes no greeting string"** assertion must exist and fail before either widget is implemented.

1. **Location & exports.** Create under the `features` design-system namespace (E10-T02 convention):
   - `packages/features/lib/src/design_system/banners/catch_up_banner.dart` ŌĆö the `CatchUpBanner` leaf.
   - `packages/features/lib/src/design_system/banners/empty_state.dart` ŌĆö the `EmptyState` family (the three variants).
   Export both through the `features` barrel under `design_system`; keep `lib/src/` internals private. REUSE SPDX header (`GPL-3.0-or-later`) on every new file. `dart format` + `dart analyze --fatal-infos` clean.

2. **`CatchUpBanner` is a domain-blind leaf over a pre-built plan.** Its single input is a display read model ŌĆö `CatchUpPlan({ required int missedDays, required int planDays, required List<CatchUpPlanItem> items, ... })` where each `CatchUpPlanItem` carries only display data (page/juz identity, track, a decay hint) and a flag marking FAR/manzil mandatory items ŌĆö plus the localized empathy/fact/path strings (passed in as `l10n.*`-sourced strings, **not** authored here), and one callback `ValueChanged<CatchUpChoice> onChoice` over an `enum CatchUpChoice { startPlan, adjust, defer }`. It imports **no** engine, no `EngineConfig`, no store, no `due_at`, and never reads `DateTime.now()`. Render in the fixed order ŌĆö empathy line ŌåÆ honest-fact line (the N-days count via T01) ŌåÆ the path line (the M-day plan via T01) ŌåÆ the plan rows (reusing the T03 page-card contract, most-decayed/prayer-critical first as the controller ordered them) ŌåÆ the user-owned choices ŌĆö on a calm `surfaceContainer`/`Card` (Level 0ŌĆō1), copy in `type.body`. Any referenced decay reads as `color.heatmap.weak`/`faded`, never red. `haptic.confirm` fires on `startPlan` only, paired with a visible change; **no** success/reward haptic, **no** confetti/fanfare.

3. **`EmptyState` is a three-variant family over a pre-built model.** Input is `EmptyStateModel` discriminated by `enum EmptyStateKind { firstRun, allDone, silentResume }` carrying only the localized fact line, an optional one gentle next-step (label + callback) for `firstRun`, and nothing for `silentResume`. **`firstRun`**: the calm fact ("Your revision will appear here once you begin") + one gentle next step framed as invitation (the step *into* cold-start is owned by E11; this surface only frames the calm invitation). **`allDone`**: a single calm closing line in `color.text.secondary` ("Today's revision is complete") ŌĆö informational, no confetti/streak/badge/exclamation. **`silentResume`**: renders *nothing special* ŌĆö it is the absence of a reproach; the host resumes into the ordinary Today screen, and this variant exposes **no greeting string** (no "Welcome back", no day-count). A pure empty state stays plainly neutral ŌĆö the empathy-then-path register is reserved for the `CatchUpBanner`. Render on a flat `surface` / Level 0ŌĆō1 `surfaceContainer`, `type.body`/`type.caption`, secondary color, generous `space.*` whitespace; no red, no mascot/hero illustration.

4. **Counts and dates through T01 only.** Every count ŌĆö "N days missed", "an M-day plan", "N pages", a juz index in a plan row ŌĆö and every date (a last-reviewed/next-due in a plan row) is produced by `numberFormatFor(locale)` / `CalendarPresenter` (T01), placed in an **ICU `plural`** message where it bears a count (Arabic supplies all six CLDR categories), and the mixed run is FSI/PDI-isolated (`isolateLtr` for the digit token). No `n.toString()`, no ASCII-digit splice, no hardcoded English "days/pages/Juz" ŌĆö all strings are `l10n.*` ARB keys (authoring owned by the localization pipeline; this task references keys).

5. **RTL & token discipline.** Logical `start`/`end` insets only (`EdgeInsetsDirectional`); the empathy/fact/path lines and plan rows read startŌåÆend; spacing via `space.*`, colors via `color.*` roles (calm `surfaceContainer`, `color.text.secondary`, `color.heatmap.weak`/`faded` for any decay hint ŌĆö never `color.semantic.error`/red), type via `type.body`/`type.caption`, target via `touch.min`, motion via `motion.duration.short` + `motion.curve.standard`, haptic via `haptic.confirm` ŌĆö no inline hex/pt/dp/ms, no `EdgeInsets.only(left:)`. The choice/next-step controls draw their states from `MihrabStateLayer` + `MihrabFocusRing` (T02). Read direction via `Directionality.of(context)`.

6. **One calm `Semantics` announcement per surface.** The `CatchUpBanner` announces, in the user's locale, the empathyŌåÆfactŌåÆpath summary and its choices as one merged container (`MergeSemantics` over the copy run, `ExcludeSemantics` on decorative decay swatches); each `EmptyState` variant announces its state and single next step (the `silentResume` variant announces nothing, by construction). Choice/next-step controls are Ōēź48dp with a `labeledTapTargetGuideline`-passing label.

7. **Register the state sets into the T02 gallery.** Add a `ComponentStateMatrix` for `CatchUpBanner` (a short plan / a longer plan / a plan whose rows include a mandatory FAR item / focused-choice) and one for `EmptyState` (firstRun / allDone / silentResume), so `pumpComponentMatrix` renders the full set with no host screen. Specimen builders receive fixture display data only.

8. **Pitfalls to avoid:**
   - **A red overdue pile / broken-streak / "you're behind" scold.** The single most important reject: any `color.semantic.error`/red fill, a "N overdue" count styled as alarm, a streak-reset, or a fault-opening line ("you're behind", "you let this slip", "N days lost"). The banner opens with empathy and renders decay as green receding to muted neutral; the test `assertNoBannedPhrase` and a no-red-fill golden guard this.
   - **Greeting the gap.** The `silentResume` variant must expose **no** "Welcome back" / "you haven't opened the app in N days" string and no day-count; a test asserts the variant produces no greeting text node.
   - **Computing the catch-up math.** The banner renders the controller's pre-built `CatchUpPlan` and never re-derives the spread, the order, or which pages are mandatory; it imports no engine and reads no clock.
   - **Dropping a FAR/manzil item to shorten the plan.** Mandatory items are rendered, never silently elided or labelled droppable; a test asserts a fixture mandatory item appears in the rendered plan.
   - **Celebration on all-done / on accepting a plan.** No confetti/fanfare/streak-repair/badge/exclamation on the all-done state or on `startPlan`; acknowledgment is copy + a `motion.duration.short` fade; a celebration guard asserts no glow/scale/sparkle widget and no celebratory haptic/motion call (the celebratory tokens do not exist in E06).
   - **Manufacturing concern in a pure empty state.** `firstRun` and `allDone` stay plainly neutral ŌĆö do not borrow the empathy-then-path / hard-news register where there is no hard news.
   - **Transactional/pressuring framing.** No "come back tomorrow", streak-at-risk, FOMO, urgency, or "upgrade/premium/unlock" anywhere.
   - **Hardcoded English / ASCII digits / physical-side insets.** Strings are ARB keys (ckb provisional), numerals via T01, layout logical-only ŌĆö the `features/**` physical-side and ASCII-digit greps are gates.
   - **Building the Today list / store.** The finite FarŌåÆNearŌåÆNew list, its loading/populated sequencing, the budget-feedback line, the StreamProvider, and the "start the plan" mutation are E12 ŌĆö these leaves emit a choice/next-step callback and nothing more.
