## Goal

The whole Mihrab library is proven accessible and golden-stable **together**, as one release-blocking CI lane, not component-by-component. A single aggregate suite under `packages/features/test/design_system/library_accessibility/` runs every shipped component (the page card + track chip + decay indicator, the heat-map cell, the grade band + teacher toggle, the certainty label, the cycle-preset + Settings pickers, the catch-up banner + empty-state family, the reminder row + destructive-confirm gate) through the T02 helpers ŌĆö `meetsLibraryGuidelines(tester)` for `textContrastGuideline` (A1, 4.5:1 text / 3:1 non-text) across all four appearances, `androidTapTargetGuideline` + `iOSTapTargetGuideline` (A6, 48dp / 44pt) and `labeledTapTargetGuideline` (A7) on every interactive leaf, the filled-in `assertColorIndependent(...)` grayscale/deuteranope check (A3) on the heat-map cell and decay indicator, a 200%-text-scale reflow pass (A4) in the longest `ckb` transcreation, and the RTL focus/reading-order assertion (A8) ŌĆö plus the per-locale (fa/ckb/ar) ├Ś four-appearance golden matrix for the whole library on the real bundled UI fonts under the throwing-`HttpOverrides` offline guard. A new `.github/workflows` golden+a11y lane (the pinned-OS golden job) and the banned-import gate make every one of those checks a build that either passes or fails. This task **builds no component and authors no widget** ŌĆö every leaf already carries its own ┬¦10 acceptance criteria; this is the composed gate that fails the build, the *last* place a11y is enforced, never the first place it is considered.

## Implementation notes

This task is a **gate**, not a feature: its correctness is that it *fails the build* when any component breaks an a11y/honesty/RTL/offline invariant, and stays green otherwise. Build it to run the library **as it is** through T02's shared helpers ŌĆö any re-implementation of the pump loop, the guideline set, or the offline bootstrap is a review reject (it belongs in T02). Author **no** widget; if a component is missing a label or a redundant channel, that is filed back against its owning task (T03ŌĆōT09), not patched here.

1. **Location.** Create the aggregate suite under `packages/features/test/design_system/library_accessibility/`:
   - `library_accessibility_test.dart` ŌĆö the widget-tier a11y gate (A1/A6/A7 + the cross-cutting honesty/no-gamification assertions); runs in the **fast** CI lane.
   - `library_golden_matrix_golden_test.dart` (`@Tags(['golden'])`) ŌĆö the whole-library per-locale ├Ś four-appearance golden matrix (A3/A4/A8 ride here); runs in the **golden** (pinned-OS, Linux-only) lane.
   - `library_offline_guard_test.dart` ŌĆö the negative test proving a stray socket in any specimen throws the named `StateError` under `useOfflineTestPolicy()`.
   Every file imports and calls `test/support/offline_test_bootstrap.dart` (`useOfflineTestPolicy()` + `loadMihrabUiFonts()`) in `setUp`, and carries the REUSE SPDX header (`GPL-3.0-or-later`). No file under `lib/` is added ŌĆö this task ships test + CI only.

2. **The component registry.** Build one `List<ComponentStateMatrix>` (a `librarySpecimens()` collector in `test/support/`) that imports each component's already-defined state matrix (the named `ComponentStateMatrix` each of T03ŌĆōT09 registered into T02's gallery) ŌĆö the page card's six states, the heat-map cell's strength/uncertainty/selected set, the grade band's enabled/pressed/disabled-until-revealed/focused set, the certainty label's seven-grade set, the two pickers' selected sets, the banner + empty-state family, and the reminder row + destructive-confirm states. The aggregate iterates this one list so adding a component to the library automatically extends the gate; a component with no registered matrix is a deliberate test failure (`expect(librarySpecimens(), isNotEmpty)` plus a per-component presence assertion).

3. **The a11y widget gate (`library_accessibility_test.dart`, fast lane).** For every specimen in the registry, pump it through `pumpComponentMatrix`'s single-frame path and call `meetsLibraryGuidelines(tester)` (T02) ŌĆö which runs `androidTapTargetGuideline`, `iOSTapTargetGuideline`, `labeledTapTargetGuideline`, and `textContrastGuideline` ŌĆö once per `fa`/`ckb`/`ar` ├Ś `{light, sepia, dark, night}`. **TEST-FIRST for the honesty invariants** (these are correctness-critical and cheap to assert): add cross-cutting guards that sweep the *whole* rendered library and `expect(..., findsNothing)` for ŌĆö any raw `R`/D/S numeral or a bare retention `%` in a card/cell (the engine-honesty leak); the strings "safe to drop"/"mastered"/"done"/"finished" in any locale (the decay axiom); any streak/score/badge/confetti/celebratory-motion widget (no gamification); a greenŌåöred (alarm) heat-map pair (the cell is never a scoreboard). These guards must exist and pass before the lane is wired green.

4. **Color-independence (A3) ŌĆö fill the T02 stub.** Implement `assertColorIndependent(tester, finder)` (the stub T02 shipped) so it captures the heat-map cell and the decay indicator's rendered frame, applies a **grayscale** transform and a **deuteranope** color-matrix transform, and asserts the strength ordering still reads ŌĆö i.e. the lightness ramp + the numeric retention value + the label remain distinguishable when hue is removed (SC 1.4.1; 08 ┬¦2ŌĆō┬¦5). Assert the text weakest-pages equivalent / per-cell label is present as the non-visual channel. Run it over T04's cell at several strength/uncertainty levels and T03's decay indicator; a cell that loses its weak-vs-strong distinction in grayscale fails the gate.

5. **The golden matrix (`library_golden_matrix_golden_test.dart`, golden lane).** Run `pumpComponentMatrix` over the **whole** registry, producing `goldens/library/<component>/<state>__<locale>__<appearance>.png` across `fa`/`ckb`/`ar` ├Ś `{light, sepia, dark, night}` on the real bundled Vazirmatn/Estedad via `FontLoader` (never `Ahem`), pinned DPR/physicalSize/theme, animations off, `@Tags(['golden'])`. Add the **200%-text-scale** pass (`MediaQuery` `textScaler: TextScaler.linear(2.0)`) for the reflow gate (A4) ŌĆö captured in `ckb` (the longest transcreation) where a fixed-width chip or two-column row breaks first ŌĆö asserting no clip/overflow. The RTL focus/reading-order gate (A8) is asserted on a representative composed row by walking the `Semantics` traversal order and `expect`-ing it runs endŌåÆstart (right-to-left) matching visual order. Masters regenerate only via local `--update-goldens`; CI verifies, never blesses.

6. **The CI lane (`eng-add-ci-check`).** Wire the suite into `.github/workflows`:
   - The **fast** job runs `flutter test packages/features/test/design_system/library_accessibility/library_accessibility_test.dart` (and the offline-guard test) on the pinned `subosito/flutter-action@v2` `flutter-version`.
   - The **golden** job runs `flutter test --tags golden` (whole-library matrix) on the **one pinned Linux runner** and pinned `flutter-version` ŌĆö never macOS/Windows (documented cross-OS golden drift).
   - Confirm the **no-network** gate's three layers cover the new code: the throwing-`HttpOverrides` bootstrap is the suite's `setUp`, the dependency allow-list still rejects analytics/ads/backend SDKs over the resolved graph, and the `flutter analyze` banned-import scope (`assets/` is the only socket) stays green over the new test files. Map this lane in the gateŌåÆjob table as the design-09 ┬¦10 (A1ŌĆōA8) half of the localization+accessibility release gate (PRD ┬¦20.5). No `continue-on-error`; the release job re-runs it and blocks publish on red.

7. **Pitfalls to avoid:**
   - **Authoring or patching a component here.** No page card / cell / band / label / picker / banner / row / gate is built or fixed in this PR; a missing label or redundant channel is filed back against its owning task (T03ŌĆōT09). This task only *runs* the library.
   - **Re-implementing T02's helpers.** `pumpComponentMatrix`, `meetsLibraryGuidelines`, the offline bootstrap, and the `MihrabAppearance`/`ThemeData` plumbing are imported and called, never re-written; the only new body is `assertColorIndependent`'s fill-in and the cross-cutting honesty guards.
   - **Skipping Sepia.** All four appearances are gated, with Sepia audited hardest (the documented low-saturation contrast trap, 09 ┬¦3); a matrix that silently omits an appearance defeats A1/A2.
   - **`Ahem` / a placeholder font.** The library goldens load the **real** bundled UI fonts (Sorani extra letters + Persian/Arabic digits must render); the font-independent trick is forbidden here (it is for pure-layout goldens elsewhere).
   - **Treating any gate as advisory.** `continue-on-error: true` on a gate, a floating Flutter SDK (re-renders goldens), running goldens on macOS/Windows, or `--update-goldens` in CI each make the gate assert nothing ŌĆö all refused.
   - **An engine/date/persistence import or a clock read.** The aggregate renders specimens fed display data only; any specimen date/number comes from injected `CalendarPresenter`/`numberFormatFor` (T01), never `DateTime.now()`; no glyph / `package:quran` reference appears (the muß╣ŻßĖźaf wall is E06-T05 / E13).
   - **A green check that serves no named gate.** Every assertion maps to an A-row or a non-negotiable; do not invent a release-blocker with no doc behind it (`eng-add-ci-check` ┬¦1).
