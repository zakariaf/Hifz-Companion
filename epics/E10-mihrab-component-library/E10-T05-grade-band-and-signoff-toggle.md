## Goal

Two domain-blind leaf widgets exist under `packages/features/lib/src/design_system/`: a `GradeBand` — a row of four large thumb-zone `FilledButton`s for **Again / Hard / Good / Easy**, rendered in the four canonical interaction states (enabled · pressed · disabled-until-revealed-styled-as-waiting · focused) drawn from the shared `MihrabStateLayer` (E10-T02), with the localized grade verbs ("needed help" / "minor mistakes" / "recited clean" / "effortless") laid out RTL and bidi-safe, each button ≥56dp tall / ≥48dp wide and ≥8dp apart, every button carrying a `Semantics` label that states its **verdict + consequence** ("again — review again soon"); and a `TeacherSignoffToggle` — a labelled `Switch.adaptive` ("Teacher present"), **off by default**, with autonomy-supportive copy ("for your teacher to confirm"). Both are stateless renderers fed display data and callbacks only: the band emits a chosen `GradeChoice`, the toggle emits a `bool`. No celebration on any grade. The reveal-on-tap surface, stumble-line tapping, the sacred-text grade guard, the undo affordance, and the recite route that *hosts* these leaves are **E12** — this task owns the band and toggle leaves and their per-locale enabled/pressed/disabled/focused goldens, nothing more.

## Implementation notes

**TEST-FIRST (correctness-critical copy + state mapping).** The grade-verb ↔ `GradeChoice` mapping, each button's verdict+consequence `Semantics` label, the disabled-as-waiting (not error) styling, and the off-by-default toggle are the behaviours a reviewer must trust. Write the widget tests in the **Tests** section first, then the widgets — and assert the *absence* of celebration as hard as the presence of correctness.

1. **Location & shape.** Both leaves live beside the other E10 components in the `features` umbrella package:
   - `packages/features/lib/src/design_system/grade/grade_band.dart` — the `GradeBand` row.
   - `packages/features/lib/src/design_system/grade/grade_choice.dart` — a tiny `enum GradeChoice { again, hard, good, easy }` (display-blind; the engine's `G=1..4` mapping lives in E04/E12, not here).
   - `packages/features/lib/src/design_system/grade/teacher_signoff_toggle.dart` — the `TeacherSignoffToggle` leaf.
   Export the public surface through the `features` barrel under the `design_system` namespace; keep `lib/src/` internals private. REUSE SPDX header (`GPL-3.0-or-later`) on every new file. Both widgets are `StatelessWidget` (or hold only ephemeral focus/press state via standard M3 plumbing) — **no Riverpod, no store, no engine/date/persistence/`quran` import**.

2. **`GradeBand` contract.** Constructor: `GradeBand({ required bool enabled, required ValueChanged<GradeChoice> onGrade, ... })`. `enabled == false` is the **disabled-until-revealed** state. The widget owns *no* knowledge of *why* it is disabled (the reveal lives in E12); it renders the waiting state and a quiet hint when disabled, and emits the chosen `GradeChoice` via `onGrade` when enabled. It applies **no sacred-text guard** and performs **no capping** — that guard is E12/domain-grading-pipeline; the band is a dumb four-choice control. There is exactly **one** `onGrade` callback; the band never persists, never recomputes, never advances a route.

3. **Four `FilledButton`s, thumb-zone sized, logical RTL order.** Build the row from four `FilledButton`s in a logical-inset `Row`/`Wrap` (`EdgeInsetsDirectional`, gaps via `space.2` ≥8dp), each constrained to **≥56dp tall and ≥48dp wide** (§5; satisfies §6/A6 with margin). Order them in *logical* DSL order `[again, hard, good, easy]` — under the harness's `Directionality.rtl` they render right-to-left automatically (eng-rtl-and-bidi-layout: the geometry flips, not a manual reversal). At 200% text scale the band may wrap to a second row (use `Wrap`, never clip/truncate a verb); ckb is the longest transcreation and must reflow, not overflow.

4. **Localized grade verbs, bidi-safe, from ARB.** Each button's label is an `AppLocalizations` key — `gradeAgainVerb` ("needed help"), `gradeHardVerb` ("minor mistakes"), `gradeGoodVerb` ("recited clean"), `gradeEasyVerb` ("effortless") — transcreated for fa/ckb/ar (the ARB values are owned by the localization pipeline; this task references keys and provides the **ar template** intent, never a hardcoded literal). Any mixed-script run inside a verb is bidi-isolated (FSI/PDI) via the T01/eng-rtl helper. The verbs map to the fluency reading in **C-024** (effortless = automaticity); they are calm competence words, never praise/shame (adab; C-045).

5. **State layers from the shared model — no bespoke opacity, a visible focus ring.** Style the buttons through `MihrabStateLayer` (E10-T02): `enabled` = full role color; `pressed` = + M3 pressed state layer (a quiet visual tick only — **at most one restrained haptic**, never a celebratory pattern); `focused` = the `MihrabFocusRing` (`color.outline`, ≥2dp, following the component via `Directionality.of(context)`, **not** a physical side); `disabled` = **dimmed role + no state layer**, styled as *waiting, not error*, with a calm hint string `gradeBandWaitingHint` ("reveal to grade") shown beside/under the dimmed band (§6). Reference `color.*` / `type.*` / `space.*` / `touch.min` by name; no inline hex/pt/dp.

6. **Per-button `Semantics`: verdict + consequence.** Wrap each button so a screen reader hears the **verdict and its consequence** as one merged phrase in the active locale — `gradeAgainSemantics` ("again — review again soon"), `gradeHardSemantics`, `gradeGoodSemantics`, `gradeEasySemantics` — plus the M3 enabled/disabled state flag. The *Again* consequence copy stays calm per **C-003** (a weak join to revisit, never "you failed / lost the page"). When disabled, the band exposes the waiting hint to assistive tech, not a "button unavailable" dead-end. Use `MergeSemantics` per button; `ExcludeSemantics` any decorative divider.

7. **No celebration — assert the negative.** On *Good* or *Easy* there is **no** confetti, chime, streak bump, glow, scale-up, sparkle, or celebratory haptic/motion tier — advancing is the host's (E12) concern and is quiet; the band itself emits only the `GradeChoice` (§5 anti-pattern; R3/C6; C-045). There is no token in E06 for such an effect; the test asserts none is referenced.

8. **`TeacherSignoffToggle` contract.** Constructor: `TeacherSignoffToggle({ required bool teacherPresent, required ValueChanged<bool> onChanged })`. Render a **labelled `Switch.adaptive`** ("Teacher present", key `teacherSignoffLabel`) in a **≥48dp row** with logical insets; default-off is the *caller's* initial value but the component documents and is tested at `teacherPresent: false` as the canonical default (off by default — adab; §7). The supporting/`Semantics` copy is autonomy-supportive — `teacherSignoffSupporting` ("for your teacher to confirm") — **never commanding** ("you must have a teacher"), and the `Semantics` announces label + state ("Teacher present, off"). The leaf only emits `onChanged(bool)`: it **does not** write `review_log`, set `source = teacher`, change confidence, mark a badge, or switch profiles (all E12/domain-grading-pipeline). The `Switch` is non-directional and is **never mirrored**.

9. **Register state sets into the T02 gallery.** Add a `ComponentStateMatrix` for the band — named specimens `enabled`, `pressed`, `disabled`, `focused` (the four canonical states) — and for the toggle — `off` (default) and `on` — so a developer renders the full state set host-less and `pumpComponentMatrix` produces the per-locale goldens. The specimen builders receive only display data (an `enabled` bool, a `teacherPresent` bool, no-op callbacks); no engine/date import.

10. **Pitfalls to avoid:**
    - **Building the recite flow.** No reveal-on-tap surface, no masked muṣḥaf page, no stumble-line hit areas, no undo affordance, no route, no grade *persistence*, and **no sacred-text guard / Hard-cap** — those are E12. A reveal surface or a guard leaking into this PR is a review reject, filed against E12.
    - **A reward/celebration on a grade.** Any glow/pop/sparkle/confetti/success-chime/celebratory-haptic on Good/Easy — forbidden; assert its absence (adab; epic DoD).
    - **The toggle doing the override.** Writing `review_log`, setting `source = teacher`, changing `sourceConfidence`, drawing a teacher-sourced badge, or switching profiles inside this leaf — all E12/domain-grading-pipeline; the leaf emits a `bool` and nothing else.
    - **`SegmentedButton` instead of four `FilledButton`s.** §6 explicitly chooses the four large buttons for thumb-zone size over `SegmentedButton`; do not "tidy" the band into a segmented control.
    - **Commanding or default-on toggle copy.** "You must have a teacher" / a default-on switch — both violate §7/adab; copy is autonomy-supportive and the documented/tested default is off.
    - **Physical-side layout or a hardcoded `Directionality`.** No `EdgeInsets.only(left:/right:)` / `Alignment.centerLeft/Right` (the `features/**` grep is a gate); the leaf reads direction from context, the harness supplies `Directionality.rtl`.
    - **A hardcoded verb/consequence string or ASCII digit.** Every label/`Semantics` string is an ARB key (ar template + fa/ckb transcreation); no literal survives the localization gate.
    - **An `Ahem`/placeholder font in the goldens.** This library's goldens load the real bundled UI fonts (epic DoD); `Ahem` is forbidden here.
    - **A Quran glyph.** Neither leaf draws or re-typesets any āyah — the muṣḥaf lives only in the E13 reader.
