## Goal

Two domain-blind Settings/onboarding leaf widgets exist under `packages/features/lib/src/design_system/components/`, each fed a fixture view model and a callback, mutating nothing. (1) `ReminderRow` — a single **off-by-default** toggle, a reminder-time picker shown **only once enabled** (rendered through T01's `CalendarPresenter`/`numberFormatFor` so the time appears in the user's calendar + locale numerals), an optional **framed-as-help** catch-up-note toggle, and one calm honest sentence; it *configures* a reminder and exposes the chosen `(enabled, TimeOfDay, catchUpNote)` to its caller — it does **not** schedule the notification (E18) and reads no clock. (2) `DestructiveConfirm` — a two-step gate for an irreversible erase-all / wipe-profile / abort-and-discard that states the concrete consequence in plain words, makes the safe **Cancel / Keep my data** path the visually-primary `FilledButton` and default focus, puts the destructive trigger in the hard-to-reach top-start corner with the safe action low in the thumb band, and surfaces a confirm callback only after the second deliberate gesture; the wipe write, the backup card, and the host screens are E16/E17. Both ship with the full preview state matrix and per-locale (fa/ckb/ar × Light/Sepia/Dark/Night) goldens through T02's `pumpComponentMatrix`, with **off-by-default** and **safe-path-primary** asserted as goldens.

## Implementation notes

These are user-facing-copy-bearing leaves, so correctness here is **adab + dark-pattern correctness** (off-by-default, safe-path-primary, calm blameless copy, no soft-delete claim) and is pinned by widget assertions and goldens. TEST-FIRST for the two correctness-critical invariants: write the **off-by-default** test (a freshly-built `ReminderRow` from a null/absent config has the toggle off and the time picker absent) and the **safe-path-primary** test (Cancel is the `FilledButton`, holds default focus, and the destructive trigger is not in the bottom band) *before* the widget bodies — both must exist and fail first. Build both as **dumb Views fed a fixture VM + callbacks**; neither persists, schedules, wipes, nor reads a clock.

1. **Location.** Both components live in the `features` umbrella package's design system (E01/E06/T02). Create:
   - `packages/features/lib/src/design_system/components/reminder_row.dart` — the `ReminderRow` leaf + its `ReminderRowState` value (`{ bool enabled; TimeOfDay time; bool catchUpNote; }`) and `ReminderRowCallbacks` (`onEnabledChanged`, `onTimeChanged`, `onCatchUpNoteChanged`).
   - `packages/features/lib/src/design_system/components/destructive_confirm.dart` — the `DestructiveConfirmSheet` leaf + a `DestructiveConfirmSpec` value (`{ DestructiveAction action; }` where `action ∈ { eraseAll, wipeProfile, abortDiscard }`, selecting the consequence string + blast-radius step) and an `onConfirmed`/`onCancelled` callback pair.
   Export both through the `features` barrel under the `design_system` namespace; keep `lib/src/` internals private. REUSE SPDX header (`GPL-3.0-or-later`) on every new file; `dart format` + `dart analyze --fatal-infos` clean.

2. **`ReminderRow` — off by default, opt-in is one tap (ui-reminder-row; §9).** A `SwitchListTile.adaptive` whose value comes straight from the injected `ReminderRowState.enabled` (the fixture default is **`false`**); turning it on is a single deliberate tap, turning it off again is a single tap. No pre-ticked state, no status-quo bias. The switch is the visually-primary control with a `Semantics` label + on/off value (localized via `l10n.*`), ≥`touch.min`. The row reads `Directionality.of(context)` and lays out with `EdgeInsetsDirectional`.

3. **`ReminderRow` — time picker shown only once enabled, rendered through T01 (ui-numerals-calendar-text; §10).** When `state.enabled` is true, reveal a reminder-time control whose displayed value is formatted through `CalendarPresenter`/`numberFormatFor(locale)` (T01) into the user's calendar + locale numeral set, FSI/PDI-isolated; tapping it raises the platform time picker and reports the chosen `TimeOfDay` via `onTimeChanged`. The leaf **never** constructs a `Jalali`/`HijriCalendar`, **never** concatenates ASCII digits, and **never** reads a clock — the "today/now" edge is E18's job; this leaf only displays the configured value. When disabled, the time control and the catch-up toggle are absent (not merely greyed), so the off state is the simplest.

4. **`ReminderRow` — optional framed-as-help catch-up-note toggle + one honest sentence (§10; C-042/C-043).** Below the time picker (only when enabled), an optional `Switch`-style catch-up-note toggle labelled as **help** ("a calm note when a few days build up", help-framed, fully silenceable) — never "N days lost", never a red overdue pile, never a streak. Under the row sits **one** plain transcreated line stating the honest local-only fact ("a neutral reminder at a time you choose, silence it anytime — nothing is sent"). All strings are `l10n.*` keys (ar template + fa/ckb transcreation), traced to C-043/C-042, and pass the banned-phrase voice gate; no exclamation mark, no countdown, no guilt/fear/loss. **The row schedules nothing** — it surfaces `(enabled, time, catchUpNote)` to the caller; the `NotificationScheduler` and the persisting notifier are E18/E16.

5. **`DestructiveConfirm` — concrete, irreversible consequence, both halves (ui-destructive-confirm; §8).** The sheet states the *full* consequence in plain words per `DestructiveAction`: for `eraseAll`, "permanently erases all hifz records for everyone on this device — it cannot be undone, and because there is no server nothing is recoverable elsewhere"; for `wipeProfile`, the single-profile equivalent; for `abortDiscard`, the discard-this-draft equivalent. Never a bare "Are you sure?". Copy leads with calm acknowledgment then the honest fact then the choice (§4), is `l10n.*` (ar template + fa/ckb transcreation), and passes the adab + banned-phrase gate (no "you'll lose your hifz", no command, no shame).

6. **`DestructiveConfirm` — safe path visually primary + default focus; destructive one deliberate step away (§9; §5).** **Cancel / Keep my data is the primary `FilledButton`** and holds the default focus + the visible `MihrabFocusRing` (T02); the destructive action is a plainer, secondary affordance (text/outlined) the user must deliberately reach — never the bright/default button. Layout is logical: the safe action sits **low in the thumb band** (`AlignmentDirectional`/`EdgeInsetsDirectional`, above the nav inset), the destructive trigger and its entry point sit in the hard-to-reach **top-start** corner. Both controls ≥`touch.min`.

7. **`DestructiveConfirm` — second deliberate gesture sized to the blast radius (§11; thumb-zone margin).** A whole-device `eraseAll` warrants a stronger second step (a second confirm screen / a *type-the-word* / a *hold-to-confirm*) so it cannot fire on one stray tap; `wipeProfile`/`abortDiscard` may need only the one consequence dialog. Use the **minimum** step that defeats an accidental tap — never escalate past it (that would be obstruction). `onConfirmed` fires only after the second gesture completes; `onCancelled` (or a barrier dismiss) is always one easy tap. The leaf **performs no wipe** — it reports the confirmed intent; the transactional erase (`review_log` rows truly deleted, persist-then-republish) is E17/`eng-create-riverpod-store`.

8. **`DestructiveConfirm` — quiet completion, reduce-motion, RTL, semantics (ui-destructive-confirm §8/§9; A8).** No celebratory motion or haptic and no alarming red flourish on confirm; at most a single non-repeating `haptic.warning` at the warning step. Any transition obeys `MediaQuery.disableAnimations` (cross-fade / instant cut). Localized `Semantics` announce the full consequence **and which button is destructive**; the focus ring is on the **safe** action. All tokens (`color.semantic.warning`, `color.outline`, `type.*`, `space.*`, `touch.min`, `haptic.warning`) referenced by name from E06 — no inline hex/dp.

9. **Preview matrix + goldens via T02.** Register each component's named state set as a `ComponentStateMatrix`: `ReminderRow` → `{ off (default), on, on+catchUpNote }`; `DestructiveConfirm` → `{ eraseAll-step1, eraseAll-step2 (type/hold), wipeProfile, abortDiscard }`. Render them through `ComponentGallery` and golden them with `pumpComponentMatrix` (fa/ckb/ar × Light/Sepia/Dark/Night, plus the 200%-text-scale pass for ckb reflow). These goldens are the visual proof of **off-by-default** (the reminder's default frame shows the switch off and no time picker) and **safe-path-primary** (the confirm frame shows Cancel as the filled/focused button).

10. **Pitfalls to avoid:**
    - **Scheduling or persisting in the leaf.** No `flutter_local_notifications`, no `NotificationScheduler` call, no Riverpod write, no `DateTime.now()` — the row surfaces the config to its caller; E18 schedules, E16 persists. A scheduler/clock reference in this PR is a review reject.
    - **Wiping in the leaf.** No DAO/repository call, no `review_log` mutation — `DestructiveConfirm` reports the confirmed intent only; the transactional wipe is E17. (And never model a "soft-delete" flag anywhere — the data layer truly deletes; §8.)
    - **Pre-ticking the reminder / defaulting it on.** The off state is the protective default; a default-on or pre-checked toggle is a sneaking dark pattern (§9/§11).
    - **Making Cancel the buried/greyed option or the destructive button the bright/default one.** That is interface-interference (§11) and inverts the safety asymmetry; assert Cancel is primary + focused in a widget test.
    - **A destructive control in the bottom thumb band.** §5 forbids it; the destructive trigger is top-start, the safe action is low.
    - **Guilt/fear/loss/streak/escalation copy, or a celebratory erase.** Any "you'll lose your hifz", "N days lost", "don't break your streak", countdown, exclamation mark, or confetti/fanfare on erase is a release-blocking adab breach (C-043; domain-adab).
    - **Raw ASCII digits or a hardcoded Gregorian time.** The reminder time goes through T01's `numberFormatFor`/`CalendarPresenter`, FSI/PDI-isolated — never `"$hour:$minute"`, never the Arabic block shown to a Persian reader.
    - **`EdgeInsets.only(left:/right:)` / `Alignment.centerLeft/Right`.** Banned by the `features/**` physical-side grep; the top-start / thumb-band placement is logical.
    - **A glyph / `package:quran` reference.** Neither leaf renders Quran text; the muṣḥaf is E13.
