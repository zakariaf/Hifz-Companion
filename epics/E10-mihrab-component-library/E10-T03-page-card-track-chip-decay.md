## Goal

Three domain-blind leaf widgets exist under `packages/features/lib/src/design_system/page_card/`: a non-interactive `TrackChip` (pairs a tradition-tied color family with a localized **sabaq / sabqi / manzil** term-set label — color **and** text, never color alone, never alarm-red, never a count/badge), a `DecayIndicator` (the same fact encoded three ways — a single-hue lightness ramp swatch + a filled/half/hollow glyph + a localized "solid / holding / needs revision" label, ≤ `space.4`, no "safe to drop" level), and the `PageCard` row that composes them: a flat `ListTile`/`Card` at elevation Level 0–1 with the four M3 list-item slots (leading = chip + decay swatch; headline = "Page N · Juz M" in locale numerals, bidi-isolated; supporting = optional next-due / weak hint; trailing = a mirroring chevron), rendered across all **six states** (default · weak · due-today · pulled-forward · done · locked) drawn from the shared `MihrabStateLayer` (E10-T02). The whole row is one ≥48dp tap, wrapped in `MergeSemantics` so a screen reader hears one localized phrase ("Page ۲۵۳, Juz ۱۳, far-revision, needs revision"). All three are stateless renderers fed a domain-blind `PageCardViewData` and an `onOpen` callback only — the row reads streamed state, never re-derives a schedule, never shows `R`/D/S or a percentage, and **draws no Quran glyph**. This task ships the leaves and their per-locale six-state golden matrix; the Today list container that orders these rows Far→Near→New and the recite route they tap into are **E12**.

## Implementation notes

**TEST-FIRST (correctness-critical honesty + mapping).** Four things a reviewer must trust are encoded as tests written *before* the widgets: (a) the row **never** renders `R`/D/S/a percentage/"safe to drop"/"mastered" (negative honesty guard), (b) the decay band reads correctly under grayscale + deuteranope (color is never the sole channel), (c) the "Page N · Juz M" headline renders in locale numerals and stays bidi-isolated (no "30 of 7"), and (d) the six states map to the right emphasis + indicator and the whole row is one labelled ≥48dp tap. Write the **Tests** section first, then the widgets — and assert the *absence* of a score/streak/glyph as hard as the presence of correctness.

1. **Location & shape.** All three leaves live beside the other E10 components in the `features` umbrella package, under one `page_card/` folder:
   - `packages/features/lib/src/design_system/page_card/track_chip.dart` — the `TrackChip` leaf.
   - `packages/features/lib/src/design_system/page_card/decay_indicator.dart` — the `DecayIndicator` leaf.
   - `packages/features/lib/src/design_system/page_card/page_card.dart` — the `PageCard` row.
   - `packages/features/lib/src/design_system/page_card/page_card_view_data.dart` — the display-blind view data (`PageCardViewData`, `TrackFamily`, `DecayLevel`, `CardState`).
   Export the public surface through the `features` barrel under the `design_system` namespace; keep `lib/src/` internals private. REUSE SPDX header (`GPL-3.0-or-later`) on every new file. All three widgets are `StatelessWidget` (focus/press handled by standard M3 plumbing via `MihrabStateLayer`) — **no Riverpod, no store, no engine/date/persistence/`quran` import**.

2. **`PageCardViewData` is the domain-blind contract (no `R`, no engine type).** Define an immutable value type holding only what the row *renders*: `int page`, `int juz` (raw ints — the locale numeral formatting happens at render via `numberFormatFor`, never pre-baked ASCII), `TrackFamily track` (`{ far, near, neww }`), `String trackLabel` (the already-localized regional term-set string), `DecayLevel decay` (`{ solid, holding, needsRevision }`), `String decayLabel` (already-localized "solid"/"holding"/"needs revision"), `CardState state` (`{ defaultState, weak, dueToday, pulledForward, done, locked }`), and an optional `String? supportingHint` (already-localized "next: in N days" / "weak line N", or null). It carries **no** `R`/D/S/`due_at`/percentage field — the decay band is mapped from `R` (min-leaning) by E12 *before* it reaches this leaf, so the leaf has no number to leak. The mapping `R → DecayLevel` and the `ReviewItem → PageCardViewData` assembly are E12/domain-scheduling-engine-rules, **not** this task.

3. **`TrackChip` — non-interactive, color AND text.** Render a label-only `Chip` (or a styled `Container` with the same metrics; no `onPressed`, no `onDeleted`, no `selected`/`onSelected`) in `type.label`, `space.1` glyph↔text, at the row start. Resolve the tradition-tied color family from tokens by name: `far` → the `color.accent.green` *maintenance* family, `near` → a secondary neutral-tinted family, `neww` → a tertiary neutral-tinted family — **never** an alarm-red for any track, never `color.semantic.warning`/danger on the chip. The label is the passed `trackLabel` term-set string (ckb's longer terms **wrap**, never `TextOverflow.ellipsis` — no truncation of sacred-adjacent vocabulary). Wrap in `ExcludeSemantics` (the chip is part of the row's merged phrase, not a separate node). No count/XP/streak/stacked badge.

4. **`DecayIndicator` — triple-redundant, ≤ `space.4`, no "safe to drop".** Render a tiny swatch + glyph: color from the single-hue ramp (`solid` → `color.heatmap.strong`, `holding` → the mid ramp step, `needsRevision` → `color.heatmap.faded`/`…weak`, a **muted neutral** — never red/amber), glyph by shape (`solid` filled, `holding` half, `needsRevision` hollow/receding), size capped at `space.4` so it stays a quiet indicator not a gauge. The `decayLevel` enum has **exactly three** values — there is structurally **no** `safeToDrop`/`mastered`/`done`-revising value (C-019). Wrap the swatch in `ExcludeSemantics`; the localized `decayLabel` is carried into the row's supporting line and merged `Semantics`, not drawn as a separate visual chip here. Encoding is identical across the four appearances and fa/ckb/ar (only the ramp re-tones).

5. **`PageCard` — flat row, four logical slots, one tap.** Build a flat `Card` at elevation Level 0–1 (token, no decorative shadow, no brand tint) wrapping a `ListTile`:
   - `leading`: a `Row(mainAxisSize: min)` of `TrackChip` then `SizedBox(width: space.2)` then `DecayIndicator` — both labels inside the row, at the logical **start** (right in RTL).
   - `title`: the headline. Format `page`/`juz` via `numberFormatFor(locale)` (T01), compose the localized "Page N · Juz M" through the `AppLocalizations` key (`l10n.pageJuz(page, juz)`), and **bidi-isolate** the mixed run (FSI/PDI via the T01/eng-rtl helper) so it never reorders. `type.body`. Never splice raw ASCII digits.
   - `subtitle`: `supportingHint == null ? null : Text(hint, …)` in `type.caption` + `color.text.secondary`.
   - `trailing`: an auto-mirroring directional chevron (`Icons.chevron_right` / the `Icons.arrow_*` family that respects ambient direction) at the logical **end** (left in RTL).
   - `onTap: onOpen` — the **single** action; the whole row is one `touch.min` (≥48dp) tap. The chip/swatch are not separate `onTap`s.
   Use `EdgeInsetsDirectional`/`AlignmentDirectional` only; no `EdgeInsets.only(left:/right:)` (the `features/**` grep is a gate). Reference `color.*`/`type.*`/`space.*`/`touch.min`/elevation by name; no inline hex/pt/dp.

6. **The six states drive emphasis + indicator only — never the page art.** Map `CardState` to emphasis through `MihrabStateLayer` (E10-T02), not bespoke opacity: *default* = `surface`; *weak* = a quiet `color.semantic.warning` **outline** + the calm `supportingHint` (never alarm-red, never a shame signal — C-003); *due-today* = the ordinary due emphasis; *pulled-forward* = rendered **identically to any due item** (no "the algorithm chose this" affordance, no badge — C-016); *done* = a checked, dimmed, removable row (reads as status, not broken — §6); *locked* = a small lock affordance signalling a teacher's `manual_lock` (a human override, not an alarm). Any focus/press emphasis uses the M3 state layer + the `MihrabFocusRing` (`color.outline`, SC 2.4.7) over the role color, following the component under RTL.

7. **One merged localized phrase.** Wrap the whole row in `MergeSemantics` so a screen reader hears one node — "Page ۲۵۳, Juz ۱۳, far-revision, needs revision" — composed from `AppLocalizations` in the active locale (fa/ckb/ar) with the correct `TextDirection`; the chip/swatch glyphs are announced as **words** (via the merged label), decorative dividers are `ExcludeSemantics`. The state (e.g. *weak*, *done*, *locked*) is part of the spoken phrase, not color-only. No `Semantics` string is a hardcoded literal — every one is an ARB key (ar template + fa/ckb transcreation).

8. **Register the six-state set into the T02 gallery.** Add a `ComponentStateMatrix` for the page card — named specimens `default`, `weak`, `dueToday`, `pulledForward`, `done`, `locked` — built from fixture `PageCardViewData` values (no-op `onOpen`), so a developer renders the full state set host-less and `pumpComponentMatrix` produces the per-locale goldens. The specimen builders receive only display data; no engine/date import.

9. **Pitfalls to avoid:**
   - **Building the Today list.** No finite/capped sliver list, no Far→Near→New section headers, no all-done/catch-up sequencing, no `go_router` route, no store — those are E12. A list container or route leaking into this PR is a review reject, filed against E12.
   - **A leaked number / "safe to drop".** Any rendered `R`, D, S, `due_at`, percentage, or a "safe to drop"/"mastered"/"you can stop revising" string is a non-negotiable violation; the `DecayLevel` enum has no such value and the negative honesty test asserts no such text renders.
   - **Color alone on the chip or swatch.** A track encoded by color without its term-set label, or a decay band by hue without glyph+label — both fail SC 1.4.1 and the grayscale/deuteranope check.
   - **Alarm-red anywhere.** An alarm-red track, a red/amber "danger" decay end, a red weak outline, a downward-trend arrow, or a falling-grade animation — all forbidden (08 §3; the weak state is a *quiet* warning outline).
   - **A gamification surface.** A streak, count, XP, badge, score, confetti, or celebratory green tint on the card — assert its absence (R3/C6; C-044/C-045).
   - **A second tap target.** The chip or swatch made separately tappable/focusable — there is exactly one `onTap` (the row) and the inner labels are `ExcludeSemantics`.
   - **Physical-side layout or a hardcoded `Directionality`.** No `EdgeInsets.only(left:/right:)` / `Alignment.centerLeft/Right` (the `features/**` grep gate); the leaf reads direction from context, the harness supplies `Directionality.rtl`.
   - **Raw ASCII digits or a hard-spliced headline.** The headline numbers go through `numberFormatFor(locale)` and the run is an isolated ARB placeholder, never `'Page ' + page.toString()`.
   - **An `Ahem`/placeholder font, or a QPC glyph.** Goldens load the real bundled UI fonts (epic DoD); the row draws **no** Quran glyph and re-typesets no āyah — the muṣḥaf lives only in the E13 reader.
