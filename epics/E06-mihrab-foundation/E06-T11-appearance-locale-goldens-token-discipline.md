## Goal

Two release-blocking gates close the visual half of the Mihrab foundation. **(1) Goldens:** the skeleton presentation widgets — `MihrabScaffold` + the RTL five-tab `NavigationBar` (E06-T08), and `MihrabCard`/list row, the `FilledButton`/`SegmentedButton` styling, and the `AppearanceSwitcher` (E06-T09) — are screenshot in **all four appearances** (Light · Sepia · Dark · Night) × **all three RTL locales** (`fa` · `ckb` · `ar`), on the **real bundled Vazirmatn/Estedad** UI fonts, tagged `@Tags(['golden'])` and pinned to E01's Linux golden lane, so any physical-direction (left/right) leak, clipped Sorani letter, or appearance-recolour regression changes pixels and fails CI before merge. **(2) Token discipline:** a no-arg gate script greps the `features/**` widget code for raw hex `Color(0x…)`/`Colors.*`, off-scale dp, bare `Duration`/`Curve` literals, physical-side `left`/`right` insets, and `letterSpacing` on the Arabic UI runs — and exits non-zero on any hit, so every value provably resolves to a named token (`ColorScheme`/`TextTheme`/`ThemeExtension`) and no widget hardcodes a value the design system owns. This task ships no widget of its own: it promotes E06-T08/T09's per-widget specimens into the full pinned-OS golden run and adds the source-discipline grep. Per [design-system 02 §9](../../docs/design-system/02-material-and-platform-foundations.md) and [05 §3](../../docs/design-system/05-layout-spacing-touch.md).

## Implementation notes

This task authors **tests and a gate script**, not UI — the widgets under test are E06-T08/T09's, the themes E06-T03/T06's. Correctness here is *visual/structural* (the golden masters are the contract) and *lexical* (the grep is sound only if it cannot be trivially bypassed).

1. **Golden test file & layout.** Add `packages/features/test/design_system/goldens/skeleton_appearance_locale_golden_test.dart` (mirrors the `design_system` source subtree E06-T09 uses, eng-write-dart-test §11). REUSE `GPL-3.0-or-later` SPDX header; `@Tags(['golden'])` at file top so it runs only in the Linux golden job. Master images under `packages/features/test/design_system/goldens/` in a stable, self-describing naming scheme: `<widget>__<appearance>__<locale>.png` (e.g. `mihrab_navigation_bar__night__ckb.png`) so a diff names exactly the failing cell.

2. **Real bundled fonts, never `Ahem`** (eng-write-dart-test §6/§7). In `setUpAll`, `FontLoader` the real bundled Vazirmatn (`type.family.ui`) and Estedad (`type.family.uiFallback`) faces E06-T04 declares, `await loader.load()`. The whole point of these goldens is to actually *draw* Persian digits and the Sorani extra letters (پ چ ژ ڤ ک گ ڕ ڵ ۆ ێ ە ھ) — `Ahem` would render every glyph as a square and hide a tofu, a broken Arabic join, or a clipped diacritic. (The font-independent block strategy permitted for *some* RTL goldens is **not** used here, eng-rtl-and-bidi-layout §10 — Sorani coverage must be exercised on the real face.)

3. **Pin the render, kill nondeterminism** (eng-write-dart-test §6). Fix `tester.view.devicePixelRatio` and `tester.view.physicalSize` to one compact phone size (the only required form factor, 05 §2); set `debugDisableShadows`/animations off; pump under the composed `ThemeData` for the appearance, not a default theme. No `pumpAndSettle()` on anything indefinite — `pump()` a fixed frame (the skeleton chrome does not animate; E06-T07's transitions are not screenshot here). Inject any "today"/display data as literals — no wall clock, no `DateTime.now()`, no real DB, no asset loader.

4. **Drive the 4×3 matrix as data, not 60 copy-pasted bodies.** Iterate `for (final appearance in MihrabAppearance.values) for (final locale in const [Locale('fa'), Locale('ckb'), Locale('ar')])`, pumping each skeleton widget (or a small specimen `Column` grouping the related primitives, reusing E06-T09's specimen widgets) inside `MaterialApp`/`Theme` carrying `themeDataFor(appearance)` (E06-T06), `locale: locale`, the generated `localizationsDelegates`, and an explicit `Directionality(textDirection: TextDirection.rtl)`. One `matchesGoldenFile` per (widget, appearance, locale) cell. Five skeleton surfaces (scaffold+nav, card, buttons, segmented, switcher) × 4 × 3 ⇒ the frame count the epic deliverable names; keep specimens minimal so a diff is legible.

5. **Assert the RTL geometry the goldens lock** (eng-rtl-and-bidi-layout §2; 05 §3). The headline invariant the night/ckb frames must show: the nav's **Today destination renders at the trailing/right edge** and Settings at the leading/left (the `Directionality` flip, never a hand-reversed list); leading affordances at `start`, chevrons at `end` (auto-mirrored); ckb's longer transcreated labels **reflow** within the same insets (the `AppearanceSwitcher` to a radio group, E06-T09) with **no clip, no overflow stripe, no tofu**. The goldens are the visible proof; the structural offset assertions live in E06-T08/T09's widget tests, not re-done here.

6. **The token-discipline grep — file, scope, rules.** Add `tool/check_token_discipline.dart` (a no-arg Dart gate per eng-add-ci-check §10; a `.sh` is acceptable but Dart gives typed errors and the REUSE header trivially). It scans **only the widget/presentation Dart under `packages/features/lib/src/**`** (the design system + feature views), skipping generated files (`*.g.dart`, `*.drift.dart`, `*.freezed.dart`, `*/generated/*`), test files, and the token-*definition* files themselves (the `ThemeExtension`/`ColorScheme`/`TextTheme` builders in `design_system/theme/**` are *allowed* to name raw hex/dp/`Duration` — that is the one file each family is owned in, 02 §9; the grep's job is to stop a *consumer* widget from inlining one). Encode that allowlist explicitly (e.g. a `theme/` path prefix) so the gate is auditable. Each violation prints `file:line: <rule>` and the script exits non-zero on any hit.

7. **The five grep rules** (each maps to a 02 §9 / 04 / 05 anti-pattern):
   - **Raw hex color:** `Color(0x…)` / `Color.fromARGB(…)` / `Colors.<name>` outside the theme-definition allowlist → use a `ColorScheme` role or `MihrabColors` extension (02 §2/§9).
   - **Off-scale dp:** a numeric literal inside `EdgeInsets*`, `SizedBox(width:/height:)`, `Padding`, `Gap`, `BorderRadius.circular(n)`, or a bare `const SizedBox(height: 16)` that is not resolved through `SpacingTokens.space*` (05 §1; the `EdgeInsets.all(13)` anti-pattern). Heuristic-flag numeric inset/box literals; the fix is `theme.extension<SpacingTokens>()!.spaceN`.
   - **Bare `Duration`/`Curve`:** a literal `Duration(…)`/`Curves.*` in a widget → use `MotionTokens` (06 §1; chrome here does not animate, but the rule guards the family).
   - **Physical-side inset:** `EdgeInsets.only(left:/right:)`, `Alignment.centerLeft/Right`, `Positioned(left:/right:)` → `EdgeInsetsDirectional`/`AlignmentDirectional`/`Positioned.directional` (05 §3; eng-rtl-and-bidi-layout §2 — *coordinate with the existing `features/**` physical-side grep; reuse it rather than duplicate the pattern if eng-rtl-and-bidi-layout §10 already lands one*).
   - **Arabic-run `letterSpacing`:** a `letterSpacing:` with a non-zero literal in the UI widget tree (04 §6/§8 — Arabic joins; tracking breaks the join). Zero is allowed; non-zero is a violation.

8. **Make the grep sound, then prove it red-first** (eng-add-ci-check §1; eng-write-dart-test test-first habit). The gate is only worth wiring if a planted violation fails it. Add a tiny fixture under `tool/test/fixtures/token_discipline/bad_widget.dart.txt` (a `.txt` so it is not compiled/analyzed) containing one of each banned pattern, and a `tool/test/check_token_discipline_test.dart` that runs the checker against the fixture and asserts it reports all five and exits non-zero — and against a clean fixture and asserts it exits zero. This is the gate's own regression test; without it the grep can silently rot to matching nothing.

9. **CI wiring** (eng-add-ci-check §2/§6/§9). Golden job: ensure the new `@Tags(['golden'])` file is picked up by the existing `flutter test --tags golden` step on the pinned Linux runner and pinned `flutter-version`; add no new job if the tag step already globs `packages/features/test/**`. Restraint job: add a step `dart run tool/check_token_discipline.dart` (no args), **required**, and add the gate→job-table row. The release job re-runs both. Never `continue-on-error`; never run `--update-goldens` in CI.

10. **Pitfalls to avoid:**
    - **`Ahem` (or any placeholder font) in these goldens** — it hides exactly the tofu/clip/broken-join these frames exist to catch; load the real bundled faces (eng-write-dart-test §6).
    - **Running the skeleton goldens on macOS/Windows CI** — documented cross-OS pixel drift; Linux-only pinned runner (eng-add-ci-check §6).
    - **`--update-goldens` in CI** (auto-blessing) — the gate would assert nothing; regenerate locally, review the image diff in the PR.
    - **A grep so broad it flags the token-definition files** (the `ThemeExtension`/`ColorScheme` builders *must* name raw values) — scope the allowlist to the owning `theme/` subtree, or **so narrow it never fires** (a checker with no red-first proof is theatre) — the fixture test guards both ends.
    - **Hand-reversing the nav list "to put Today on the right"** in a specimen — declare logical order and let RTL place it; a golden that bakes in a manual reversal hides the bug it should catch (eng-rtl-and-bidi-layout §2).
    - **Hardcoding an English/ASCII string or ASCII digit in a specimen** — specimens use `l10n.*` keys and locale-formatted numbers (the ASCII-digit grep, eng-rtl-and-bidi-layout §10, is a sibling gate; do not reintroduce a violation in a test).
    - **Screenshotting a muṣḥaf glyph / QPC asset** — none appears in chrome; the wall is E06-T05, these are presentation widgets only.
    - **`pumpAndSettle()`** on an indefinite indicator, reading `DateTime.now()`, or touching the network — none belong in a deterministic golden (eng-write-dart-test §8/§9).
