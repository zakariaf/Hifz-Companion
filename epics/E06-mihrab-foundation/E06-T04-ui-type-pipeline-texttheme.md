## Goal

The Mihrab UI type pipeline exists as one auditable file tree: `type.family.ui` = **Vazirmatn** and `type.family.uiFallback` = **Estedad** are bundled faces (declared in `app/pubspec.yaml`, files under `app/assets/fonts/`), runtime font fetch is disabled (no `google_fonts` in the resolved graph), and a custom `TextTheme` maps the six-step Arabic-script ramp onto the M3 role slots — `type.display` → `displayLarge/Medium`, `type.title` → `titleLarge/Medium`, `type.body` → `bodyLarge/Medium`, `type.label` → `labelLarge`, `type.caption` → `bodySmall` — at the design-system sizes (28/22/16/14/13 sp), every entry carrying `height` ≈ 1.5–1.6 and `letterSpacing: 0`, with the zero-spacing applied as a component-theme default so a stray value cannot creep onto an Arabic run. The `type.numeral` rule is stated and wired to the `l10n` `numberFormatFor(locale)` path (it is *not* a size). The bundled font **version is deliberately not locked here** — the per-codepoint Sorani no-tofu gate that pins it is E09's. No `ThemeData` is composed in this task (that is E06-T06); this task ships the `TextTheme` builder and the type tokens it reads, fully unit- and golden-tested in isolation.

## Implementation notes

The type metrics are correctness-adjacent (a clipped diacritic or a non-zero Arabic letter-spacing is a visible defect), so the size/height/letter-spacing values and the slot mapping are pinned in unit tests **before** the `TextTheme` builder code (see Tests).

1. **Location.** The design system lives inside the `features` umbrella package (E01: "the `features` umbrella … the design system lives inside"). Create the type tree under `packages/features/lib/src/design_system/type/`:
   - `type_tokens.dart` — an immutable `TypeTokens` value type (the `type.*` token values: the six sizes in sp, the two weights, `lineHeightArabic`, `letterSpacingArabic`, the two font-family names) with a `const` constructor, `copyWith`, and `///` docs; the single owner of the `type.*` values per the README token map.
   - `mihrab_text_theme.dart` — `buildMihrabTextTheme(TypeTokens tokens)` returning a `TextTheme` with the six slots populated and the `type.*` → slot mapping applied.
   - `type_numeral.dart` — the `type.numeral` routing helper that defers to `l10n`'s `numberFormatFor(locale)` (re-exported, not re-implemented — E09/`l10n` owns the formatter).
   Export the public surface (the builder, `TypeTokens`, the numeral helper) through the `features` barrel under a `design_system` namespace; keep `lib/src/` internals private. REUSE SPDX header on every new file.

2. **`TypeTokens` value type.** Fields carry units in the name per coding standards: `displaySizeSp` 28, `titleSizeSp` 22, `bodySizeSp` 16, `labelSizeSp` 14, `captionSizeSp` 13; `weightSemibold` (`FontWeight.w600`), `weightMedium` (`FontWeight.w500`), `weightRegular` (`FontWeight.w400`); `lineHeightArabic` (a range applied as `TextStyle.height` — use **1.5** for the dense label/caption end and up to **1.6** for body/prose, pinned as named constants, never a magic literal in a widget); `letterSpacingArabic` = **0.0**; `fontFamilyUi` = `'Vazirmatn'`, `fontFamilyUiFallback` = `'Estedad'`. `const` ctor, `copyWith`, `///` on every public member (`public_member_api_docs` is an error). Cite design-system 04 §4/§6 in the doc comment for the size/height numbers.

3. **`buildMihrabTextTheme`.** Build a `TextTheme` whose slots map per design-system 02 §4 / 04 §4: `displayLarge`+`displayMedium` ← `type.display` (28 sp, Semibold); `titleLarge`+`titleMedium` ← `type.title` (22 sp, Medium); `bodyLarge`+`bodyMedium` ← `type.body` (16 sp, Regular); `labelLarge` ← `type.label` (14 sp, Medium); `bodySmall` ← `type.caption` (13 sp, Regular). Every populated `TextStyle` sets `fontFamily: tokens.fontFamilyUi`, `fontFamilyFallback: [tokens.fontFamilyUiFallback]`, `height: <lineHeightArabic for that slot>`, `letterSpacing: tokens.letterSpacingArabic` (0.0). Do **not** set `color` here — color comes from the `ColorScheme` `onSurface` uniformly (M3 `Typography.material2021` applies it; E06-T03/T06 own color). Slots the ramp does not name (`headline*`, `labelMedium/Small`, `displaySmall`, `titleSmall`) inherit the nearest mapped slot's family/height/spacing so no component ever falls back to bare Roboto metrics on an Arabic run.

4. **Zero letter-spacing as a component-theme default (not only per-style).** Beyond setting `letterSpacing: 0` on each `TextStyle`, the design-system 04 §6 rule is that zero-spacing is encoded so "a stray value cannot creep in." Provide the hook E06-T06 wires into `ThemeData`: the component themes that draw Arabic chrome (`textTheme`, plus the text styles inside `FilledButtonThemeData`/`SegmentedButtonThemeData`/`NavigationBarThemeData`/`ListTileThemeData` where E06-T09 styles them) resolve their label style from `buildMihrabTextTheme`, never an ad-hoc `TextStyle(letterSpacing: …)`. In this task, expose the `TextTheme` and a documented note at the builder declaration that downstream component themes must source their label style from it; the actual `ThemeData` composition is E06-T06.

5. **The `type.numeral` rule is wiring, not a size.** `type_numeral.dart` does not define a `TextStyle`; it states and routes the rule that any digit-bearing UI string is formatted through `l10n`'s `numberFormatFor(locale)` (Extended Arabic-Indic `-u-nu-arabext` for fa/ckb, Arabic-Indic `-u-nu-arab` for ar) before composition — never ASCII concatenation (eng-rtl-and-bidi-layout rule 7; design-system 04 §5). Re-export / thin-wrap the `l10n` helper; do **not** re-implement the formatter (E09/`l10n` owns it). The `///` doc states the rule and that per-locale correctness is exercised by E09.

6. **Bundled-font declaration (audit artifact).** The `flutter: fonts:` block in `app/pubspec.yaml` declares `Vazirmatn` and `Estedad` pointing at `app/assets/fonts/` (E01-T02 already reserved this block with placeholder/actual `.ttf`s and **no `google_fonts`**). This task confirms the **family names** used in `TypeTokens` match the declared family names exactly (a mismatch yields silent fallback) and that Regular + Medium + Semibold weight files (or the variable face's weight axis) are present for each. Do **not** pin a font version, do **not** add the Sorani-coverage gate (E09), and do **not** add `google_fonts`.

7. **TEST-FIRST ordering.** Write `type_tokens_test.dart` and `mihrab_text_theme_test.dart` (Tests section) asserting the exact sizes, weights, slot mapping, `height` range, `letterSpacing == 0`, and `fontFamily`/`fontFamilyFallback` **before** the builder body, so the pinned ramp is a frozen contract, not a value read back off whatever was typed.

8. **Pitfalls to avoid:**
   - Routing any Quran/QPC glyph or a muṣḥaf string through `buildMihrabTextTheme` or `TypeTokens` — the UI type pipeline never touches scripture (the wall is structural; E06-T05 proves it). No `package:quran` import appears in this tree.
   - A non-zero `letterSpacing` (or M3's default tracking) leaking onto an Arabic run — it shatters the cursive join; assert `== 0` on every populated slot.
   - Copying the Latin/Roboto `material2021` metrics (tight ~1.2 line-height, 12–14 sp secondary text) onto Arabic slots — clips marks; the floor is `type.caption` 13 sp and `height` ≥ 1.5.
   - Adding `google_fonts` or any runtime fetch — offline-by-construction; the E01 no-network/allow-list gate must stay green with this code included.
   - A `fontFamily` string in `TypeTokens` that does not byte-match the `app/pubspec.yaml` family name (silent fallback to the system font — caught by a golden, but assert the names too).
   - Locking the bundled font version, or asserting Sorani coverage here — both are E09's; this task only declares the faces and the numeral rule.
   - Hardcoding a user-facing string, an ASCII digit, or a raw hex/dp/`Duration` in this tree — there is no copy here at all, and numbers are sizes (sp constants), never displayed digits.
   - Setting `color` inside the `TextTheme` styles — color is the `ColorScheme`'s job (E06-T03/T06); a colored `TextStyle` here breaks the four-appearance recolor.

## Tests

In `packages/features/test/design_system/type/` (Flutter Testing; runs on E01's fast lane). **Write the token/mapping assertions first:**

- `type_tokens_test.dart` — asserts the frozen ramp: `displaySizeSp == 28`, `titleSizeSp == 22`, `bodySizeSp == 16`, `labelSizeSp == 14`, `captionSizeSp == 13`; weights w600/w500/w400; `letterSpacingArabic == 0.0`; `lineHeightArabic` within `[1.5, 1.6]`; `fontFamilyUi == 'Vazirmatn'`, `fontFamilyUiFallback == 'Estedad'`; `copyWith` round-trips and is non-mutating.
- `mihrab_text_theme_test.dart` — over `buildMihrabTextTheme(const TypeTokens.defaults())`: each mapped slot has the expected `fontSize` and `fontWeight`; **every** populated slot has `letterSpacing == 0`, `fontFamily == 'Vazirmatn'`, `fontFamilyFallback == ['Estedad']`, and `height` in `[1.5, 1.6]`; no slot sets `color`. A parameterized case iterates all 15 `TextTheme` slots and asserts none carries a non-zero `letterSpacing` (the Arabic zero-spacing invariant) and none resolves to a non-Mihrab family.
- `type_numeral_test.dart` — asserts the helper returns a non-null `NumberFormat` for each of `fa`/`ckb`/`ar` and that fa/ckb yield Extended Arabic-Indic and ar yields Arabic-Indic digits for a sample integer (the routing is correct; exhaustive per-locale/per-calendar numeral correctness is E09's suite, referenced in a comment).
- `mihrab_text_theme_golden_test.dart` (`@Tags(['golden'])`) — a small specimen widget (one `Text` per slot showing fa/ckb/ar sample words, including the Sorani extra letters پ چ ژ ڤ ک گ ڕ ڵ ۆ ێ ە ھ) pumped under `Directionality(textDirection: TextDirection.rtl)` with the **real bundled Vazirmatn/Estedad** loaded via `FontLoader` (never `Ahem`); `matchesGoldenFile` on the pinned-OS lane proves no clipped mark, no tofu visible at this stage, and zero-spacing rendering. (The locking Sorani no-tofu gate is E09; this golden is the visual smoke that the faces load and render.)
- Offline guard: the suite runs under the shared throwing-`HttpOverrides` bootstrap (E01-T06) — the type pipeline makes zero network calls; building the `TextTheme` and loading the bundled fonts touches no socket.
