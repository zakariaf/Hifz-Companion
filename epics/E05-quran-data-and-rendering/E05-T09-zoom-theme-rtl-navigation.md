## Goal

The reader frame, paging, and edition chrome land in `packages/quran` (transform/paging) and the `/quran` reader feature (chrome). The glyph+overlay `Stack` from E05-T07/T08 is wrapped in a single uniform `Transform.scale` (origin `Alignment.topRight` for RTL) for zoom and a single `ColorFiltered` for sepia/dark — **no** per-theme font swap, **no** reflow on zoom, and the reader's zoom is **independent** of `MediaQuery.textScalerOf` (OS chrome text-scale). Page navigation uses an **RTL-aware `PageView`/`PageController`** whose `reverse` honours direction so page 1→2 advances right-to-left; each swipe rebuilds with a new `pageNumber`/geometry only — the glyph content is **never mirrored, reordered, or re-shaped**. Around the page, the muṣḥaf edition's `riwayah`/`displayName` is shown as ordinary shaped `type.*` UI chrome with locale numerals (`intl`, fa/ckb/ar), never calling the page "the Quran" absolutely; the byte-for-byte / Tanzil·QUL·KFGQPC attribution text is surfaced for About/Credits. This task ships the presentation transform, the paging direction, and the edition chrome only — the glyph layer (T07), the overlay painter (T08), the verified data/fonts (T05/T06), and the 604-page visual-diff (T11) are owned elsewhere.

## Implementation notes

TEST-FIRST (correctness-critical): the two seams that can corrupt the sacred path are **zoom-must-not-reflow** and **paging-must-not-mirror-the-glyphs**. Write, **before** the frame/paging body: (a) the widget test asserting that changing `zoom` (and pumping under `MediaQuery(textScaler: TextScaler.linear(2.0))`) produces an **identical** glyph-line tree — the page scales, never re-flows, and never reads OS text-scale; and (b) the test asserting the `PageView.reverse` is `true` under RTL `Directionality` (page 1→2 advances right-to-left) while each rendered glyph `Text` keeps its own `TextDirection.rtl` and the glyph layer is never wrapped in a mirror/`Transform(flipX)`. Both must exist and fail first.

1. **Files & packages.**
   - The transform frame + paging live in `packages/quran` (muṣḥaf rendering — enforced by `tool/check_quran_isolation.sh`): `packages/quran/lib/src/render/mushaf_reader_frame.dart` (the zoom/`ColorFilter` wrapper around T07's `Stack`) and `packages/quran/lib/src/render/mushaf_page_navigator.dart` (the RTL-aware `PageView`). Both are dumb widgets — no network, no clock, no IO. Barrel-export from `lib/quran.dart`.
   - The **chrome** (riwāyah label, "Page N · Juz M" locator, About/Credits attribution) lives in the `/quran` **reader feature** (`features/lib/src/quran_reader/widgets/`, scaffolded with **eng-add-feature-module**) — it is shaped UI text and *does* localize, so it sits in `features`, not in the `packages/quran` sacred-path module. The reader-state provider (current page, zoom, theme) is the `/quran` Riverpod notifier (authored with **eng-create-riverpod-store** at the reader composition root); this task **reads** `pageNumber`/`zoom`/`readerTheme` from it and does not author the store's persistence.

2. **The transform frame** — exactly per engineering 08 §5:
   ```dart
   Widget mushafReaderFrame(ImmutableGlyphPage page, OverlayLayer overlays,
       ReaderTheme theme, double zoom) {
     return ColorFiltered(
       colorFilter: theme.glyphColorFilter,        // sepia/dark = ONE filter, not a font
       child: Transform.scale(
         scale: zoom,                              // uniform; no re-flow, no per-line restyle
         alignment: Alignment.topRight,            // RTL origin
         child: Stack(children: [page, overlays]), // T07 glyph layer + T08 overlay sibling
       ),
     );
   }
   ```
   - Exactly **one** `Transform.scale` (uniform `scale: zoom`, `alignment: Alignment.topRight`) and exactly **one** `ColorFiltered` wrap the *combined* glyph+overlay `Stack`. Overlays transform *with* the glyphs because they are inside the wrapped `Stack` (T08 applies no transform — the contract is honoured here).
   - `theme.glyphColorFilter` is a `ColorFilter` resolved from the reader theme (light = identity/no-op, sepia, dark). There is **one font per page** — dark mode is a colour filter, **never** a "dark font" and never a per-theme `fontFamily`. No `TextStyle`/`fontFamily`/`fontFamilyFallback` is touched here.
   - Zoom is the reader's **own** value (from the reader-state provider). The frame **never** reads `MediaQuery.textScalerOf(context)` and never multiplies the muṣḥaf by OS chrome text-scale — the page enlarges by `Transform.scale` so printed line breaks never move (PRD §18; engineering 08 §5).

3. **The RTL-aware page navigator** — engineering 08 §3 (geometry rebuild, no reflow) + ui-mushaf-page-view step 6 + eng-rtl-and-bidi-layout:
   ```dart
   final controller = PageController(initialPage: initialPage - 1);
   PageView.builder(
     controller: controller,
     reverse: Directionality.of(context) == TextDirection.rtl, // page 1→2 advances right-to-left
     itemCount: edition.pageCount,                              // 1..pageCount, never literal 604
     onPageChanged: (i) => reader.goToPage(i + 1),              // a new pageNumber only
     itemBuilder: (context, i) {
       final pageNumber = i + 1;
       final page = resolveVerifiedGlyphPage(pageNumber); // T05/T06 verified data → T07 ImmutableGlyphPage
       return mushafReaderFrame(page, overlaysFor(pageNumber), reader.theme, reader.zoom);
     },
   );
   ```
   - `reverse` reads `Directionality.of(context)` — **never** a hardcoded `TextDirection.rtl` (all three locales are RTL, but the direction is derived, not assumed; eng-rtl-and-bidi-layout step 2/3). The page-turn *direction* mirrors; the **glyph content does not** — each page is still drawn glyph-only with its own `TextDirection.rtl` (T07), and the glyph layer is **never** wrapped in a mirror, `Transform(flipX)`, rotation, or reflection.
   - A swipe/jump rebuilds with a **new `pageNumber` → new geometry → new dedicated font** only (a different `ImmutableGlyphPage`); the renderer never re-typesets, never reflows, and never recomputes a break. `itemCount`/the page bound come from `edition.pageCount` (T01), never a literal `604`.
   - If `resolveVerifiedGlyphPage(pageNumber)` returns no verified page (asset unverified — the integrity layer's verdict), the navigator shows the calm "refuse to render Quran text" chrome state (this task's `l10n`, see note 5), never an unverified page and never a substitute font.

4. **The edition / locator chrome** (reader feature, shaped UI text):
   - **Riwāyah label** → render `edition.displayName` (e.g. *"Ḥafṣ ʿan ʿĀṣim — Madani muṣḥaf"*) as ordinary shaped `type.*` UI text (the chrome *does* go through the shaper). It is laid out with logical insets (`EdgeInsetsDirectional`/`AlignmentDirectional`). The page is **never** labelled "the Quran" absolutely (R2). No tafsīr/translation/commentary is added.
   - **"Page N · Juz M" locator** → the page number and juz index are formatted via `numberFormatFor(locale)` (pinned `-u-nu-arabext` for fa/ckb → `۰۱۲`, `-u-nu-arab` for ar → `٠١٢`) and placed as **isolated ARB placeholders** inside the localized message (`"{page} از {total}"` style), each mixed run FSI/PDI-isolated (`isolateLtr` for the known-direction digit run) — **never** `"Page " + n.toString()`, never a raw ASCII digit in chrome. The page index/juz come from the verified reference data (T05), **not** recomputed. The `604` total traces to **C-031**.
   - **About/Credits attribution** → surface the byte-for-byte provenance + the three separate attributions: Tanzil Uthmani text (CC BY 3.0, link to tanzil.net), QUL (Tarteel) page/line/word layout, KFGQPC QPC V2 per-page fonts (unmodified redistribution). This is the honest-provenance half of **C-048**; it is plain shaped chrome copy (any link clearly leaves the app), not the sacred path.

5. **Strings via the ARB pipeline (fa/ckb/ar).** The riwāyah label is data (`displayName` from the edition) but every *surrounding* chrome string this task introduces — the locator message, the About/Credits attribution lines, the zoom/theme control `Semantics` labels, and the calm "this page could not be verified" refuse-to-render copy — lands as ARB keys for fa/ckb/ar (authored with **eng-add-localized-string**; base/template `app_ar.arb`, transcreated fa/ckb), with locale-numeral placeholders and FSI/PDI isolation. Any religiously-loaded copy (the attribution framing, the refuse-to-render wording) is run past **domain-adab-and-religious-integrity** before it ships.

6. **Zoom/theme controls.** The reader's zoom (in/out, the muṣḥaf's own control, **not** OS text-scale) and the light/sepia/dark toggle are interactive chrome around the page: each carries a `Semantics` label (eng-accessibility) and meets thumb-zone/contrast norms; each writes to the reader-state provider (current page/zoom/theme) through its single mutation path (the store is **eng-create-riverpod-store**'s, not authored here — this task wires the control to it). The control changes `zoom`/`theme` *values*; the frame (note 2) turns them into the transform.

7. **No network, no IO, no clock, no AI.** The frame, the navigator, and the chrome open no socket, read no file at render time, call no `DateTime.now()`, and touch no AI/ASR/audio. Verified glyph pages/geometry/fonts arrive from T05/T06 via the `/quran` providers; the riwāyah/attribution text is static edition data + ARB strings.

8. **Pitfalls to avoid:** a **per-theme `fontFamily`/font swap** for dark mode (dark = a `ColorFilter`, never a "dark font"); letting `MediaQuery.textScalerOf` reach the muṣḥaf (it must zoom by `Transform.scale`, never reflow with OS chrome text-scale); a **non-uniform** scale, a per-line restyle, or any width-driven re-layout on zoom (reflow corrupts the printed page); wrapping the glyph layer in a mirror / `Transform(flipX)` / rotation / reflection for RTL (the muṣḥaf is **never** mirrored — eng-rtl-and-bidi-layout step 4; only the `PageView.reverse` direction mirrors); **hardcoding** `reverse: true` instead of reading `Directionality.of(context)`; a `PageView` whose `itemCount`/bound is a literal `604` instead of `edition.pageCount`; re-typesetting or recomputing a break on page change (a new page is a new font+geometry, never a reflow); `"Page " + n.toString()` / a raw ASCII digit / a hard-spliced number in any chrome string; re-rendering the page's **own printed ayah numbers / juz markers / sajda signs** through `intl` to "match the UI numerals" (those are the immutable glyph layer — ui-numerals step 9); labelling the page "the Quran" absolutely or adding any tafsīr/translation (R2); a badge/counter/confetti/glow/ornament over an āyah, or any piety/wuḍūʾ gate to open the page (R3, adab); placing the riwāyah/attribution chrome inside `packages/quran` (the sacred-path module ships no localized chrome — it belongs in `features`); applying a `Transform`/`ColorFilter` twice (T08 applies none; this frame applies exactly one of each); importing `dio`/`http`/`HttpClient`/`drift` in the `packages/quran` frame/navigator files.

## Tests

`packages/quran/test/render/mushaf_reader_frame_test.dart` and `packages/quran/test/render/mushaf_page_navigator_test.dart`, plus the chrome at `features/test/quran_reader/quran_reader_chrome_test.dart` (mirror the source paths), `flutter_test` (the bootstrap installs E01's throwing `HttpOverrides`; this suite must never trip it — the reader is offline). The frame/navigator are pumped over a small **synthetic** `ImmutableGlyphPage`/`PageGeometry`/`MushafEdition` (placeholder opaque glyph codes, a 5-line page, `pageCount` 7), so unit/widget cases are fast, deterministic, and Ahem-free. Required cases, the first two written FIRST:

- **Zoom does not reflow / no OS text-scale (TEST-FIRST, central case)**: pump the frame at `zoom: 1.0`, `1.5`, `2.0`; the glyph-line widget tree (count, order, per-line `glyphCodes`, per-line `softWrap: false`/`maxLines: 1`) is **identical** at every zoom — only an enclosing `Transform.scale` differs. Pumping under `MediaQuery(textScaler: TextScaler.linear(2.0))` yields the **same** tree (the frame never reads `MediaQuery.textScalerOf`) — the muṣḥaf zoom is independent of OS chrome text-scale (PRD §18).
- **Paging is RTL, glyphs are not mirrored (TEST-FIRST)**: under RTL `Directionality`, the `PageView.reverse` is `true` (page 1→2 advances right-to-left) and is `false` under LTR `Directionality` (proving it is derived, not hardcoded); each rendered glyph `Text` keeps `TextDirection.rtl`; the glyph layer is wrapped in **no** mirror / `Transform(flipX)` / rotation; `itemCount == edition.pageCount` (7 here, never 604).
- **One scale + one filter over the combined Stack**: the frame applies exactly one `Transform.scale` (uniform, `alignment: Alignment.topRight`) and one `ColorFiltered` around the glyph+overlay `Stack`; an overlay marker (a `currentAyah` box from T08) scales and recolours **with** the glyph layer (one transform covers both); no second transform/filter exists.
- **No per-theme font swap**: rendering the frame in light, sepia, and dark resolves the **same** `fontFamily` per page (`qpcFontFamily(page)`) — only `theme.glyphColorFilter` differs; no `fontFamily`/`fontFamilyFallback` is touched by this task (grep + widget assertion).
- **Page change = new pageNumber/geometry, no reflow**: swiping/jumping to a new page builds a new `ImmutableGlyphPage` with that page's font/geometry; the prior page's lines are not re-typeset or recomputed; `onPageChanged` reports the 1-based `pageNumber`; no width/`TextPainter` line computation runs on any page change.
- **Locator numerals per locale (chrome test)**: the "Page N · Juz M" run renders Extended Arabic-Indic (`۰۱۲`, U+06F0 range) under fa and ckb and Arabic-Indic (`٠١٢`, U+0660 range) under ar — asserted by codepoint; the run is FSI/PDI-isolated (an isolate control surrounds the formatted token); no ASCII digit appears in the rendered chrome (a per-locale golden + a codepoint assertion).
- **Riwāyah / About copy is sect-neutral (chrome test)**: the chrome renders `displayName` (e.g. *"Ḥafṣ ʿan ʿĀṣim — Madani muṣḥaf"*), never the bare string "the Quran"/"القرآن" as an absolute label; the About/Credits surface lists Tanzil/QUL/KFGQPC attributions and the byte-for-byte provenance line; no tafsīr/translation string is present.
- **Refuse-to-render state**: when the resolver yields no verified page, the navigator shows the calm localized refuse-to-render chrome (fa/ckb/ar) and **never** a substitute font or an unverified page; the state carries a `Semantics` label.
- **RTL chrome layout + a11y**: the riwāyah label and locator lay out with logical `start`/`end` insets (no physical `left`/`right`); the zoom/theme controls each carry a `Semantics` label and meet the touch-target norm; the chrome golden runs under RTL `Directionality` for fa/ckb/ar.
- **Offline guard**: the whole suite runs with the throwing `HttpOverrides` installed and never trips it; no real network, no `rootBundle` text/font load, no clock, no randomness, no Ahem.

No 604-page real-font golden in this task — that is the E05-T11 visual-diff gate (which renders **this** frame across light/sepia/dark + a zoom step in RTL with the real KFGQPC fonts on min-OS iOS+Android). Here the glyph page/geometry/edition are synthetic fakes so the unit/widget/chrome suites stay fast and deterministic.
