## Goal

A golden harness renders **all 604 muṣḥaf pages** with the **real KFGQPC per-page glyph fonts** (and the fa/ckb/ar UI fonts) loaded via `FontLoader` in `setUpAll` — never the Ahem placeholder, which draws squares and proves nothing — under an RTL `Directionality`, fixed DPR/surface/theme/animations, diffed against the scholar-approved reference muṣḥaf images within a **tight pixel tolerance**; any diff fails the build. The harness includes the test-vector spot-checks of [engineering 08 §6](../../docs/engineering/08-quran-data-and-immutable-rendering.md): each of the **15 sajda marks** at its reference position, **per-surah ayah numbering / end markers**, and **basmala present/absent** correctly (absent only for At-Tawba / Sūrat al-Tawba, sūra 9). The 604-page run is tagged `@Tags(['golden'])` and scheduled in CI two ways — the pinned-Linux `golden` job (stable masters) **and** the on-device `journeys` job on **min-iOS + min-Android** (the cross-platform fidelity proof that gate 2 / [PRD §11.3.4, §20.2, R1](../../docs/PRD.md) requires) — plus a light/sepia/dark + zoom-step matrix. The qualified-ḥāfiẓ on-device proof ([PRD §20.8 / gate 8](../../docs/PRD.md)) stays a **separate human release gate handled at E20**; this task documents that the goldens are *seeded from scholar-approved references*, not auto-captured, and guards regression only.

## Implementation notes

TEST-FIRST (correctness-critical, outranking covenant): the visual-diff *is* the test, and its job is to fail when the sacred text moves. Before wiring any CI job, stand up the 604-page golden harness against the committed reference masters and prove it **red** on a deliberately corrupted page (a shifted/dropped diacritic, a missing glyph rendering as tofu, a misplaced sajda mark, a wrongly-present basmala on At-Tawba) and **green** on the faithful render — a golden gate that cannot go red asserts nothing.

1. **Files** (all in `packages/quran`, the only place QPC-glyph rendering is allowed — enforced by `tool/check_quran_isolation.sh`):
   - `packages/quran/test/golden/mushaf_page_golden_test.dart` — the 604-page sweep + the light/sepia/dark + zoom matrix, `@Tags(['golden'])`.
   - `packages/quran/test/golden/mushaf_spot_checks_golden_test.dart` — the sajda / ayah-numbering / basmala test-vector spot-checks, `@Tags(['golden'])`.
   - `packages/quran/test/golden/_golden_font_loader.dart` — the shared `loadRealKfgqpcFontsForGoldens()` + `MushafGoldenHarness` helper (the `pumpWidget` target wrapping T07's `MushafPageView` in T09's RTL/zoom/theme frame).
   - `packages/quran/test/golden/goldens/mushaf/page_001.png … page_604.png` — the committed reference masters, **seeded from scholar-approved muṣḥaf references** (a `goldens/REFERENCE_PROVENANCE.md` notes their origin and that they are not auto-captured screenshots).
   - `integration_test/mushaf_visual_diff_test.dart` (app package) — the on-device subset run, reusing the same harness + masters, for the min-iOS + min-Android legs.
   - The new `golden` + `journeys` job steps in `.github/workflows/ci.yaml` (REUSE SPDX `GPL-3.0-or-later` header on any new `tool/*.sh`/`*.dart` helper).

2. **Load the REAL fonts, never Ahem** ([eng-write-dart-test §6](../../docs/engineering/11-testing-strategy.md), [eng-add-ci-check §6](../../docs/engineering/11-testing-strategy.md)). In `setUpAll`, build a `FontLoader` per `QPC_P###` family over the verified bundled font bytes (the same bytes T06 registers at runtime) plus the fa/ckb/ar UI families, and `await loader.load()`:
   ```dart
   Future<void> loadRealKfgqpcFontsForGoldens() async {
     for (var page = 1; page <= edition.pageCount; page++) {
       final loader = FontLoader(qpcFontFamily(page))      // QPC_P001 … (T06's convention, one definition)
         ..addFont(_goldenFontBytes(page));                // real KFGQPC glyph file — NEVER Ahem
       await loader.load();
     }
     for (final ui in kBundledUiFontFamilies) { … }        // fa/ckb/ar chrome fonts
   }
   ```
   Ahem renders every glyph as a solid square, so a golden of squares would pass while a diacritic is misplaced — the exact corruption this gate exists to catch (08 §6 pitfalls). This is the `golden`/fidelity tier; the font-independent Ahem/colored-block strategy used for *layout* RTL goldens is **forbidden** here.

3. **Pin everything so only a real change moves pixels** ([11 §5](../../docs/engineering/11-testing-strategy.md)). Fix `tester.view.devicePixelRatio`, `tester.view.physicalSize`, the theme, and disable animations; pump under `Directionality(textDirection: TextDirection.rtl)`. The CI `golden` job pins **one Linux runner** and an explicit `subosito/flutter-action@v2` `flutter-version` (the README baseline, e.g. `3.38.0`) with `channel: stable` — a floating SDK silently re-renders goldens and breaks the gate's meaning. Never run the fidelity goldens on macOS/Windows in CI (cross-OS drift is documented and real); the cross-platform proof is the on-device leg, not the shared masters.

4. **The 604-page sweep**, exactly per 08 §6 / 11 §5:
   ```dart
   @Tags(['golden'])
   void main() {
     setUpAll(loadRealKfgqpcFontsForGoldens);
     for (var page = 1; page <= edition.pageCount; page++) {  // pageCount, not a literal 604 — R2 swappability
       testWidgets('muṣḥaf page $page renders pixel-faithfully', (tester) async {
         await tester.pumpWidget(MushafGoldenHarness(pageNumber: page));   // T07 view in T09 RTL frame
         await expectLater(
           find.byType(MushafPageView),
           matchesGoldenFile('goldens/mushaf/page_$page.png'),
         );
       });
     }
   }
   ```
   Drive the loop bound from `edition.pageCount`, not a hardcoded `604`; the glyph lines come from T07's `assemblePage` (grouped by `page-{p}-line-{l}`, never by verse, `fontFamilyFallback: const []`) so a missing glyph surfaces as **visible tofu** that the diff fails on.

5. **Tight tolerance, not "close enough"** (08 §6 pitfalls; domain skill Do/Don't). Configure `matchesGoldenFile` against a deliberately **tight** comparator — the default `LocalFileComparator` is exact-bytes; if a per-pixel-channel threshold comparator is used (to absorb only sub-perceptual raster noise on the on-device legs), pin the threshold low and document why. The failure mode (a shifted mark, a broken ligature) is *small in pixels but catastrophic in meaning*; the sacred text does not get a loose tolerance. Record the chosen tolerance and its justification in `goldens/REFERENCE_PROVENANCE.md`.

6. **The light/sepia/dark + zoom matrix** (T09's transform frame). For a representative page set (every page is swept at the base theme; the theme/zoom matrix runs over a covering sample — e.g. one page per juz plus the sajda/numbering/basmala vector pages — to keep the master count tractable while still proving the transform never disturbs the glyph layer), capture goldens at: light, sepia (`ColorFiltered`), dark (`ColorFiltered`), and one zoom step (uniform `Transform.scale`, RTL `Alignment.topRight` origin). The assertion is that zoom/theme **transform the rendered layer, not the text** — printed line breaks must not move between light and the zoom step (08 §5); a per-theme font swap or an OS-text-scale reflow would change the glyph layout and fail.

7. **The sajda / ayah-numbering / basmala spot-checks** (08 §6 table; PRD §11.3.5), in `mushaf_spot_checks_golden_test.dart`:
   - **Sajda marks (15)**: for each of the 15 sajda āyāt, render its page and golden-assert the sajda overlay (T08's `MushafOverlayPainter` coordinate rect) sits at its **reference position** — drive the 15 `(surah, ayah, page, line)` references from the bundled QUL/sajda dataset, never a hardcoded list, and assert the count is exactly 15.
   - **Ayah numbering / end markers**: golden-assert per-surah ayah counts and the end-of-ayah markers render at their reference positions against the masters; the data-of-record is the bundled layout dataset (counts are *read*, never recomputed — 08 §3, [PRD §6.1](../../docs/PRD.md)).
   - **Basmala present/absent**: golden-assert the basmala renders at the head of every sūra **except At-Tawba (sūra 9)**, and is **absent** there; drive "which sūra omits the basmala" from the dataset, and assert that exactly one sūra (sūra 9) omits it. (Al-Fātiḥa's basmala-as-āyah-1 and An-Naml's mid-sūra basmala are part of the reference render, not special-cased in code — they are whatever the QUL masters show.)

8. **Schedule it in CI two ways** ([11 §8](../../docs/engineering/11-testing-strategy.md) gate→job map; eng-add-ci-check §6):
   - `golden` job (pinned Linux, `flutter test --tags golden`): the 604-page sweep + spot-checks + theme/zoom matrix against the committed masters — fast, deterministic, stable-master regression gate.
   - `journeys` job (on-device): the **min-Android** leg via `reactivecircus/android-emulator-runner@v2` at `api-level: 24` running `flutter test integration_test/mushaf_visual_diff_test.dart`, and the **min-iOS** leg on a pinned min-iOS simulator runner — the cross-platform fidelity proof PRD §11.3.4 / §20.2 names. Add both to the **gate 2 row** of the PRD-gate→job mapping table; both are **required, never `continue-on-error`**.

9. **CI never blesses** (08 §6 / 11 §5 / eng-add-ci-check §9). The CI run only *verifies* against committed masters; masters are regenerated **locally** with `flutter test --update-goldens`, reviewed in the diff like any code change, and — because the masters are seeded from scholar-approved references — a master change is a deliberate, reviewed act, not an auto-capture. Never add `--update-goldens` to a CI step; a self-blessing gate asserts nothing.

10. **Offline by construction** (C-048; eng-write-dart-test §8). The harness keeps E01's throwing `HttpOverrides` installed and never trips it — fonts and masters are local; rendering a page touches no network. `packages/quran` imports no networking symbol (`dio`/`http`/`dart:io HttpClient`); the file passes `tool/check_quran_isolation.sh` and `flutter analyze`'s banned-import scope.

11. **Pitfalls to avoid:** loading **Ahem** (or any font-independent/colored-block trick) for the fidelity goldens — a golden of squares passes while diacritics are wrong; a **loose tolerance** ("close enough" for the sacred text); a **floating SDK** or `channel: stable` without a pinned `flutter-version` (silently re-renders masters); running the fidelity goldens on **macOS/Windows in CI** (cross-OS drift); `--update-goldens` **in CI** (self-blessing); hardcoding **604** instead of looping `edition.pageCount` (breaks R2 swappability); hardcoding the 15 sajda positions / the basmala-omitting sūra instead of reading the dataset; **re-typesetting** or re-shaping anything to draw the sajda/numbering overlay (markers are coordinate rects over the immutable glyph layer — 08 §4); treating this automated gate as **sufficient** and skipping the §20.8 scholar review (it stays an E20 release gate); duplicating T10's **checksum** hashing here (this is the *pixel* gate); marking any page **optional/skippable** to keep the master count down (every page is swept at base theme — nothing is "safe to drop").

- [ ] All muṣḥaf-fidelity goldens load the **real KFGQPC + fa/ckb/ar fonts via `FontLoader`** in `setUpAll`; **no Ahem**, no colored-block/font-independent substitute; a golden of squares is impossible.
- [ ] Every golden test is `@Tags(['golden'])`, pumps under `Directionality(textDirection: TextDirection.rtl)`, fixes `devicePixelRatio`/`physicalSize`/theme, and disables animations; the comparator tolerance is **tight** (documented in `goldens/REFERENCE_PROVENANCE.md`), not loose.

## Tests

This task *is* a test suite (the visual-diff gate). The artifacts and cases:

- **`packages/quran/test/golden/mushaf_page_golden_test.dart`** (`@Tags(['golden'])`, `flutter_test`): the `for page in 1..edition.pageCount` sweep, real fonts via `FontLoader` in `setUpAll`, RTL `Directionality`, fixed DPR/size/theme, animations off, `matchesGoldenFile('goldens/mushaf/page_$page.png')` per page. Runs only in the pinned `golden` CI job.
- **`packages/quran/test/golden/mushaf_spot_checks_golden_test.dart`** (`@Tags(['golden'])`): the test-vector spot-checks — `'all 15 sajda marks render at their reference position'` (data-driven over the bundled sajda dataset, count == 15, each overlay rect golden-matched), `'per-surah ayah numbering and end markers match the reference'`, `'basmala present at every sūra head except At-Tawba (9)'` (exactly one omitting sūra, == sūra 9). Reference positions/counts are **read from the dataset**, never hardcoded.
- **Theme/zoom matrix** (in the page golden file or a sibling): `light` / `sepia` / `dark` / `zoomStep` goldens over the covering sample; a `'zoom and theme do not move printed line breaks'` assertion comparing the glyph layer's line geometry across light and the zoom step.
- **`integration_test/mushaf_visual_diff_test.dart`** (app package, real stack after `IntegrationTestWidgetsFlutterBinding.ensureInitialized()`): the on-device subset re-running the same harness + masters; scheduled in `journeys` on **min-iOS + min-Android** (the cross-platform proof of PRD §11.3.4 / §20.2). Pump explicit durations; never `pumpAndSettle()` an indefinite indicator.
- **Negative / red-on-corruption proof** (a local, documented harness, not a CI master): render a page with a deliberately shifted diacritic, a forced-tofu glyph (missing-font case), a misplaced sajda overlay, and a basmala forced onto sūra 9 — assert each fails `matchesGoldenFile` against the faithful master. This proves the gate can go red.
- **Offline guard:** every suite keeps E01's throwing `HttpOverrides` installed and never trips it; no network, no `--update-goldens` in CI.
- **No engine golden vectors here** — this task asserts *pixels*, not scheduler math; the FSRS vectors/INV properties stay in `engine/`. **No checksum hashing here** — the text/layout/604-font SHA-256 gate is E05-T10; this is its pixel-diff complement.
