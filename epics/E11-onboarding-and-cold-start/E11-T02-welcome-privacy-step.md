## Goal

The first onboarding step — `WelcomeStep` — exists as a leaf view under `packages/features/lib/src/onboarding/widgets/`, rendered by E11-T01's `onboarding_screen.dart` as step 0 of the resume-safe sequence. It states the intent (the app is built free, as *ṣadaqah*) and the **perceptible privacy covenant** as plain, checkable facts: no account, no microphone / it never records audio, no data leaves the device, one-time download then airplane-mode forever ([CLAIMS C-048](../../docs/science/CLAIMS.md)) — plus the servant-to-the-teacher framing (an aid to revision and to your teacher, not a replacement, not a fatwa — [CLAIMS C-046](../../docs/science/CLAIMS.md)). Every string is calm, exclamation-free, reverent, and passes the adab gate; one "Continue" affordance advances the captured-state controller to the language step. The view is dumb — it writes no persisted state, opens no socket, and reads no clock; it only renders localized copy and calls the step controller's `next()`.

## Implementation notes

This is a **copy-and-presentation** task, not correctness-critical engine work, so no test-first golden-vector is required; but the adab conscience pass and the per-locale ARB coverage are the load-bearing parts and are gated below.

1. **File (leaf view under E11-T01's module).** `packages/features/lib/src/onboarding/widgets/welcome_step.dart`, one primary public type `WelcomeStep` (a `ConsumerWidget` or plain `StatelessWidget` — it holds no state of its own). It is rendered by `onboarding_screen.dart` as the step-0 page; it receives the controller (or a `VoidCallback onContinue`) from the screen, never reaches into the captured state, and calls `onContinue()` (which the screen wires to the step controller's `next()`). Per `eng-add-feature-module`, the View is dumb: no repository/engine call, no `try/catch`, no DB, no http, no `DateTime.now()`.

2. **Copy structure (intent → covenant → servant → continue), all from `l10n.*`.** Compose four calm blocks, every string an `AppLocalizations` getter (no literal survives the `features/**` grep):
   - **Intent / *ṣadaqah*:** a warm, plain line that the app is offered free as a continuing charity (*ṣadaqah jāriyah*) — never "free trial," "premium later," or any SKU/transaction word (voice §7). Key e.g. `onboardingWelcomeIntent`.
   - **Privacy covenant (C-048), stated as checkable facts:** distinct lines — no account; **it never records audio / there is no microphone**; nothing about you leaves this phone; one public download once, then it works in airplane mode forever. Phrase as facts true by construction ("the app cannot listen," "try airplane mode after setup"), never "we value your privacy" (design §5). Keys e.g. `onboardingWelcomePrivacyNoAccount`, `onboardingWelcomePrivacyNoMic`, `onboardingWelcomePrivacyOnDevice`, `onboardingWelcomePrivacyOfflineAfter`.
   - **Servant-to-the-teacher (C-046):** one sentence framing the app as an aid to revision and a servant to the user's teacher — not a replacement for oral correction (*talaqqī*), not a fatwa. It credits the human, claims no authority, speaks for no one. Key e.g. `onboardingWelcomeServant`; its `@description` carries "methodology/religious framing — needs native + scholar review."
   - **Continue:** one verb-in-idiom affordance (e.g. "Continue" → `onboardingWelcomeContinue`) that calls `onContinue()`; no second "skip"/"upgrade"/"sign in" action.

3. **ARB authoring (template-first, transcreated, provisional where religious).** Author every key in `lib/l10n/app_ar.arb` (the `ar` template/base content language) first, each with an `@description` giving translator/scholar context; transcreate `fa` and `ckb` against it (not literal translation) so the calm-reverent register survives — Persian respectful-warm (*šomā*/honorific), Arabic imperatives softened to statements of readiness, ckb canonical-encoded (U+06D5 ە for AE, U+06A9 ک for kaf; no ZWNJ/Teh-Marbuta hack) and flagged provisional. The servant line and any methodology-adjacent privacy phrasing stay "needs scholarly review" in their `@description` until the per-locale gate clears them (E11-T10).

4. **No interpolation, no numerals here — but keep the discipline.** The welcome carries no count, page number, or date, so there is no `plural`/`numberFormatFor` path in this task; if any value is ever added it goes through the locale-numeral + FSI/PDI path (`eng-add-localized-string` §5–§7), never ASCII concatenation. Do not invent a "604 pages" or "N minutes" figure here — those belong to later steps and trace to their own CLAIMS rows.

5. **Layout: RTL-native, reverent, undecorated.** Logical start/end insets only (no hard left/right); the app-wide `Directionality` (RTL for all three locales) is inherited from the shell. Type/spacing/color by token name (`type.body`/`type.title`, `space.*`, calm `color.text.primary`/`secondary`) — no inlined hex/pt, no saturated fill, no warning color (nothing is wrong here). Quiet, aniconic identity only: no mascot, no cartoon Quran, no figurative imagery; an optional, dismissible calm framing is *allowed* but **not** added in this task unless already specced. **No muṣḥaf glyph is rendered on this screen** — the welcome precedes the verified download, so no āyah/page text may appear, and the screen must not name an edition or call the bundled muṣḥaf "the Quran."

6. **Accessibility.** Each block carries a localized `Semantics` label via E08-T02's `labeled(...)` / `mergedItem(...)` helpers so a screen reader voices intelligible, merged phrases in the active locale; the "Continue" control is `button: true` with a localized label, hit target ≥48dp, visible focus ring; reading order runs start→end (right→left), top→bottom. No information is conveyed by color alone (there is no state here to encode).

7. **Pitfalls to avoid.** A hardcoded English literal surviving in an fa/ckb/ar build (the `features/**` grep catches it); any exclamation mark or emoji in the copy (banned-phrase lint); any transactional word — "free trial," "premium," "upgrade," "unlock," "sign up" (voice §7); a guilt/fear/loss or command framing — "don't lose your hifz," "you must," "you should" (voice §3, §6); naming an edition or calling the bundled muṣḥaf "the Quran" (R2 — that belongs to E11-T03, and even there it is *named*, never absolute); the app claiming to *grade*, *verify*, or *approve* recitation, or issuing any ruling (R6 — credit the teacher, not the app); opening a socket, showing a download spinner, or reading a clock from this view (the real download is E11-T04); writing any persisted state from the View (single-write-path rule — the welcome captures nothing); rendering any muṣḥaf glyph before the verified download; locking the `ckb` servant/religious copy before native + scholar review.

## Tests

Dart tests run in CI on every PR (`flutter test`); per-locale goldens load the **real** bundled UI fonts (never `Ahem`) and pin the runner. No test asserts a coverage percentage.

- `features/test/onboarding/welcome_step_test.dart` — **widget**: pump `WelcomeStep` inside a minimal `MaterialApp` under each of `Locale('ar')`, `Locale('fa')`, `Locale('ckb')` with `Directionality.rtl`; assert the intent, the four privacy-covenant facts (no account / no mic / on-device / offline-after), and the servant line each render via their localized `l10n.*` value (the captured ARB string for the locale), and that no English literal leaks in a non-`ar` build; assert no `'!'` appears in any rendered welcome string (the exclamation-mark guard) and no transactional token ("premium"/"upgrade"/"unlock"/"trial") appears.
- `features/test/onboarding/welcome_step_test.dart` (same file) — **interaction**: tapping the "Continue" control invokes the injected `onContinue()` exactly once (recorded via a fake callback / overridden controller); the View performs no other side effect (no write, no navigation push).
- `features/test/onboarding/welcome_step_semantics_test.dart` — **widget**: assert via the semantics tree (`SemanticsTester`/`tester.getSemantics`) that each copy block exposes an intelligible, merged localized `Semantics` label and the "Continue" control is `button: true` with a localized label; `meetsGuideline(labeledTapTargetGuideline)` passes over the step.
- `features/test/onboarding/welcome_step_golden_test.dart` — **golden** (`@Tags(['golden'])`): `matchesGoldenFile` of `WelcomeStep` under fa, ckb, and ar on the **real** bundled UI fonts, pinning RTL geometry and the calm, undecorated layout per locale (ckb checked for wrap-not-truncate of the longer Sorani strings).
- **Offline / localization gates**: a throwing `HttpOverrides` is installed in these widget tests (proving the welcome opens no socket); a banned-import grep over `features/onboarding/widgets/welcome_step.dart` finds no `http`/`dio`/`HttpClient`, no `drift`/DAO, no microphone/ASR/model, and no `DateTime.now()`; `app_ar.arb` key-coverage compiles with `nullable-getter: false`; the `features/**` hardcoded-string grep, the ASCII-digit grep, and the physical-side (left/right) grep stay green; fa/ckb/ar values present for every welcome key (ckb flagged provisional); the banned-phrase copy-lint passes in all three locales.
