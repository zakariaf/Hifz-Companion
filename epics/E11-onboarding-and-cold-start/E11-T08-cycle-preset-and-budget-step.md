## Goal

One onboarding step lands inside the E11-T01 module: a **named cycle-preset pick** ‚Äî **7-Manzil weekly khatm ¬∑ 1 juz/day ¬∑ ¬Ω juz/day ¬∑ 2 juz/day ¬∑ Custom**, plus the **Pure-cycle** toggle framed as fidelity ‚Äî composing E10's `CyclePresetPicker`, paired with a **daily time-budget** input. The step is a dumb View that captures exactly two fields of E11-T01's resume-safe `OnboardingState` ‚Äî `cyclePreset` (the named choice; Custom carries four bounded fields) and `dailyBudgetMinutes` ‚Äî by calling the controller's `setCyclePreset` / `setDailyBudget` commands; it never writes a row (the placement commit is E11-T09). The captured preset maps to **`EngineConfig` only** (`farCycleDays` / `dailyBudget` / `nearCeilingDays` / `newLinesPerDay` / `pureCycleMode`) at commit ‚Äî there is **no `target_R` dial, no FSRS D/S/R, no "recommended for you," no readiness %**. Pure-cycle is one explicit flag; Custom reveals exactly four labelled, bounded steppers. The whole step is RTL by geometry, numerals in the locale set, a radiogroup by `Semantics`, with per-locale goldens on the real bundled fonts.

## Implementation notes

1. **Files** (in the E11-T01 onboarding module, `packages/features/lib/src/onboarding/`):
   - `widgets/cycle_preset_step.dart` ‚Äî the dumb View for this step, composing E10's `CyclePresetPicker` (the named single-select + Pure-cycle toggle + Custom-editor slot) and a daily-budget input row.
   - `widgets/custom_cycle_editor.dart` ‚Äî the Custom reveal: exactly four bounded, labelled steppers (far-cycle days ¬∑ near-window juz ¬∑ new-lines/day ¬∑ daily-budget minutes), shown only when `CyclePresetId.custom` is selected; each field maps 1:1 to an `EngineConfig` field.
   - No new feature-scope provider file here; the step reads from and calls into **E11-T01's `onboardingControllerProvider`**. The View never writes ‚Äî per the feature-module View-never-writes rule.

2. **Captured-state fields.** This task writes exactly two fields of E11-T01's `OnboardingState`: `cyclePreset` (a named `CyclePreset` value ‚Äî the preset id plus, for Custom, the four bounded numbers, plus the `pureCycleMode` flag) and `dailyBudgetMinutes` (a bounded `int`). Selecting a preset calls `setCyclePreset(...)`; toggling Pure-cycle and editing the budget call the same controller path; the controller republishes the captured state via `copyWith`. There is **no `(D, S)`, no `due_at`, no `target_R`, no engine call, and no persisted row** anywhere in this step ‚Äî the captured `CyclePreset` is mapped to `EngineConfig` and committed only at E11-T09.

3. **Named choices only ‚Äî never a slider (the central rule here).** Render the presets through E10's `CyclePresetPicker` as a radiogroup of named cards/segments using `CyclePresetId` (`sevenManzilWeekly`, `oneJuzPerDay`, `halfJuzPerDay`, `twoJuzPerDay`, `custom`); the labels/subtitles are localized term-set strings the View builds from `AppLocalizations` + the active term-set. There is **no `Slider`**, no `target_R` dial, no "due cards/day" number, and no FSRS D/S/R surfaced anywhere in the tree (PRD ¬ß15.1; ui-cycle-preset-picker rule 1).

4. **The preset ‚Üí `EngineConfig` mapping is the picker's entire effect.** Use the skill's `farCycleDaysFor(CyclePresetId)` mapping (7-Manzil‚Üí7, 1 juz/day‚Üí30, ¬Ω juz/day‚Üí60, 2 juz/day‚Üí15); the chosen preset sets `EngineConfig.farCycleDays` (the cycle ceiling the trust clamp reads), and the daily budget sets `EngineConfig.dailyBudget`. The mapping lives at the feature boundary (toward the commit, E11-T09); this View only captures the named choice. The picker **computes no `due_at`, reorders nothing, reads no R** (engine ¬ß6; PRD ¬ß7.6).

5. **Pure-cycle is one explicit flag, framed as fidelity.** The toggle sets the captured preset's `pureCycleMode` (‚Üí `EngineConfig.pureCycleMode = true` at commit): fixed rotation, SR ordering off, zero pull-forward. Its label is fidelity copy ("follow my cycle exactly, no reordering"), **never** "disable smart scheduling" / "turn off smart features" (PRD ¬ß7.11; ui-cycle-preset-picker rule 3). Toggling it flips **exactly one** flag and changes nothing else.

6. **Custom is four bounded fields, nothing more.** When Custom is selected, reveal `custom_cycle_editor.dart` ‚Äî four labelled steppers/selects with sane bounds: far-cycle length (days) ¬∑ near-window size (juz) ¬∑ new-lines/day ¬∑ daily-budget (minutes). Each maps 1:1 to an `EngineConfig` field; there is **no** raw retention target, no "advanced math" pane, no D/S/R (PRD ¬ß15.1; ui-cycle-preset-picker rule 4). Bounds reject out-of-range input rather than coercing silently.

7. **Daily budget input.** A single bounded minutes input (stepper/select), defaulting to the region's tuned constant, formatted via `numberFormatFor(locale)` and described with an ICU `plural` budget-minutes string. The **honest budget line** (PRD ¬ß12.2: if the chosen scope can't fit the budget) is informational and offers raise-budget / lengthen-cycle ‚Äî it is never a red shame-pile and never silently lets pages rot. Computing whether the scope fits is the engine's job at the commit/first-day step (E11-T09); this View only captures the chosen minutes and renders the calm advisory copy if surfaced.

8. **No celebration, no recommendation, no readiness number.** Picking or switching a cycle is a quiet factual change ‚Äî no confetti/streak/badge/score, no "optimal!" praise, no "you're behind" pressure, no "recommended for you" inference, and no exclamation marks; a selected state is an M3 state layer over a role color, never a glow/pop (design-principles ¬ß2; ui-cycle-preset-picker rule 6). The step shows **no** retention %, no seeded D/S/R, no "you're N% ready" (C-009).

9. **RTL + bidi + numerals.** All geometry is logical (`EdgeInsetsDirectional` / `AlignmentDirectional`); no `EdgeInsets.only(left:/right:)` survives the `features/**` grep. Preset subtitles mix a name and a number ("full Quran every €∑ days," "Juz €∑") ‚Äî route each mixed run through the `bidi.dart` helper (`isolate`/FSI for unknown direction; `isolateRtl` for a known-RTL run) so it never reorders its line. Every numeral (cycle length, near-window juz, new-lines, budget minutes) formats via `numberFormatFor(locale)` (fa/ckb Extended Arabic-Indic, ar Arabic-Indic) ‚Äî never ASCII digits (ds-12 ¬ß3/¬ß4; ui-cycle-preset-picker rule 7).

10. **Strings (eng-add-localized-string).** New keys authored in `lib/l10n/app_ar.arb` first, transcreated for fa/ckb/ar:
    - `onboardingCyclePresetStepTitle`, `onboardingCyclePresetStepBody`
    - preset labels + subtitles: `cycleSevenManzil` / `cycleSevenManzilSub({days})`, `cycleOneJuzPerDay` / sub, `cycleHalfJuzPerDay` / sub, `cycleTwoJuzPerDay` / sub, `cycleCustom`
    - `pureCycleFidelityLabel` (fidelity copy) and its `Semantics` toggled/not-toggled values
    - Custom field labels: `customFarCycleDays`, `customNearWindowJuz`, `customNewLinesPerDay`, `customDailyBudgetMinutes`
    - `dailyBudgetLabel`, `dailyBudgetMinutes({count})` (ICU `plural`, all six Arabic CLDR categories), and the calm honest-budget advisory `budgetTightAdvisory` (empathy ‚Üí fact ‚Üí path)
    - each option's `Semantics` selected/not-selected value; the continue/confirm affordance label.
    Preset / term-set labels are **regional term-set** strings (manzil/juz vocabulary) via ICU `select`/region-override, methodology-adjacent ‚Üí run past the adab/scholar gate; ckb values canonical-encoded (U+06D5 €ï, U+06A9 ⁄©) and flagged "needs native + scholar review" in `@description`; no literal in `features/**`.

11. **TEST-FIRST note.** Correctness-critical for this step = the **named-choice / EngineConfig-only / no-target_R / no-readiness** invariants. Write the widget/unit tests in ¬ßTests asserting (a) `farCycleDaysFor` maps every named preset exactly (7/30/60/15), (b) selecting a preset routes through the controller and writes **only** `cyclePreset` (no engine/`due_at`/`target_R`/instant write, over a fake controller/clock recording writes), (c) the tree contains **no `Slider`** and no D/S/R/retention-% string, and (d) Pure-cycle flips **exactly one** flag ‚Äî **before** wiring the View. The pure-UI radiogroup/Custom-reveal geometry is golden-tested, not test-first.

12. **Pitfalls to avoid:**
    - Any `Slider`, `target_R` dial, "due cards/day" number, or surfaced FSRS D/S/R ‚Äî breaks PRD ¬ß15.1 / design-principles ¬ß3 (the single hardest rule here).
    - A "recommended"/"best"/"optimal for you" badge on a preset, confetti/streak/badge on selection, or an exclamation mark.
    - Labeling Pure-cycle "turn off smart scheduling" / implying the app is now worse, instead of fidelity.
    - Adding a fifth Custom field, a raw retention target, or an unbounded input to the Custom editor.
    - Writing the preference in the View, computing `EngineConfig`/`due_at` here, reaching into a global, calling the engine, or calling `DateTime.now()` ‚Äî the E11-T01 controller captures it; the EngineConfig commit is E11-T09's single write path.
    - A red/alarm "you're behind" budget pile instead of the calm empathy‚Üífact‚Üípath advisory.
    - Hardcoded left/right insets or a hardcoded app-wide `Directionality`; ASCII digits; hard-splicing the mixed-script subtitle instead of one isolated ICU message.
    - Showing a readiness %, a seeded D/S/R, or any retention probability (C-009 / C-016 ‚Äî the guarantee is a *cycle*, never a number).
