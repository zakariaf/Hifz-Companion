## Goal

The third and final cold-start capture pass exists as a genuinely **skippable** per-juz "when memorized" input rendered inside the onboarding flow. For each *held* juz, the user may ‚Äî never must ‚Äî say roughly when they finished memorizing it, as either a `CalendarDate` (picked in their own calendar ‚Äî Solar-Hijri JalƒÅlƒ´ / Hijri Umm al-QurƒÅ / Gregorian, with locale numerals) **or** a coarse "years ago / months ago" band that resolves to a `CalendarDate` against the injected `today`. The captured value lands in the controller's sparse `Map<int, CalendarDate> memorizedOn` (E11-T06's controller, written via `setMemorizedOn` / `clearMemorizedOn`) and is fed downstream ‚Äî by the E11-T09 commit ‚Äî into `coldStartCard`'s `memorizedOn` stale-time decay branch so a juz finished years ago is aged back toward reactivation. Skipping is a first-class path with no nag, no asterisk, no completion pressure; a juz simply stays absent from the map. This task captures and renders only ‚Äî it invents no `(D, S)`, opens no `db.transaction`, and reads no clock other than the injected `CalendarDate`.

## Implementation notes

This step is **not** test-first as a whole (it is a capture/render widget, not correctness-critical arithmetic), with one exception called out below: the band ‚Üí `CalendarDate` resolution and the "skip leaves the map untouched" invariant are deterministic and **are pinned by a unit test written before the widget body**, because a wrong `addDays`/band offset would silently mis-age a juz through `coldStartCard`.

1. **File placement** (`packages/features/lib/src/onboarding/widgets/`): add `when_memorized_input.dart` ‚Äî a dumb `ConsumerWidget` (or `StatelessWidget` reading the controller) named `WhenMemorizedInput`, taking the `int juz` it belongs to. It renders under the existing per-juz confidence row from E11-T06; it does **not** own a step of its own in `OnboardingStep` (PRD ¬ß12.1 places "when memorized" on the same screen as confidence ‚Äî it is a sub-control of the `confidence` step, not a fourth pass). One primary public type per file, `lower_snake_case`.

2. **No new controller, no new state field.** Reuse E11-T06/T01's `onboarding_view_model` and its existing `Map<int, CalendarDate> memorizedOn` plus the commands `setMemorizedOn(int juz, CalendarDate date)` and `clearMemorizedOn(int juz)`. This widget reads `ref.watch(...).memorizedOn[juz]` to show the current value (or its absence) and calls those commands on change/clear. It writes **no** persisted row and opens **no** `db.transaction` ‚Äî the seeded card is E11-T09's single write path.

3. **Two input modes, both resolving to one `CalendarDate`.** Offer the user a choice (provision-of-choice, voice ¬ß6):
   - **a precise date** via a date picker bound to the active `CalendarSystem`. Do **not** call Flutter's Gregorian `showDatePicker` directly into the UI as the user's truth ‚Äî convert through the `CalendarPresenter` so a JalƒÅlƒ´/Hijri user picks in *their* calendar; the value stored is always the `CalendarDate` serial-day (`epochDay`), the *display* is the converted, numeral-remapped label;
   - **a coarse band** ‚Äî a small row of chips ("this year", "1‚Äì2 years ago", "3‚Äì5 years ago", "more than 5 years ago", or a months-ago equivalent) that resolves to a representative `CalendarDate` by `today.addDays(-n)` against the injected `today` (e.g. "2 years ago" ‚Üí `today.addDays(-730)`). The band is the low-friction default; the precise picker is for users who remember exactly. Either mode ends as one `CalendarDate` in `memorizedOn[juz]`.

4. **TEST-FIRST (the deterministic core):** the band ‚Üí `CalendarDate` mapping is a pure function ‚Äî `CalendarDate memorizedDateForBand(StaleBand band, CalendarDate today)` ‚Äî defined in this widget's file (or a small sibling) and **pinned by a unit test before it is written**. The test fixes `today` to an explicit `CalendarDate.ymd(...)` and asserts each band resolves to the expected serial day via `addDays` only (no `Duration`, no `DateTime.difference`). The "skip" / "clear" path is likewise pinned: invoking clear (or never touching the control) leaves `memorizedOn` with no entry for that juz ‚Äî never a sentinel `0`/epoch-zero/"unknown" date.

5. **Render the date through `CalendarPresenter`, then localize.** When a value exists, show it via `ref.watch(calendarPresenterProvider)` (domain-calendars-and-hifzdate) ‚Üí `presenter.format(date, system, locale)`; remap its digits downstream to the locale set and bidi-isolate the run (eng-rtl-and-bidi-layout ¬ß7/¬ß8). Never construct a `Jalali`/`HijriCalendar` in this widget, never concatenate ASCII digits, and never display the raw `epochDay`/`DateTime`. If the chosen calendar is Hijri, the Umm al-QurƒÅ honesty note already governs at the presenter; do not key any deadline or guarantee to this date (it only ages `S`).

6. **"Today" is injected.** The band resolution and any "max = today" bound on the precise picker read `ref.watch(clockProvider).today()` (or `todayProvider`) as a `CalendarDate`. **No `DateTime.now()` / `Calendar.current` / `TimeZone.current`** anywhere in the file ‚Äî a grep over the module is part of the gate. The picker's *upper* bound is `today` (you cannot have memorized a juz in the future); there is no sensible lower bound to enforce beyond the calendar's own range guard.

7. **Skip is first-class, structurally.** The default state of the control is "no date" and that is a complete, valid answer ‚Äî the row shows a quiet, optional affordance (a plain "Add when you memorized this (optional)" invitation or a **Skip**), never a required field, never a red "missing date", never a nag that re-prompts. A set value shows a small **clear/remove** affordance so the user can return to the skipped state. Advancing past the confidence/when-memorized screen with an empty `memorizedOn` map is the normal, expected path (most users will skip).

8. **RTL by construction; the band chips and date row are chrome.** Logical `start`/`end` geometry (`EdgeInsetsDirectional`, `AlignmentDirectional`); the band chips read start‚Üíend (right‚Üíleft); the date label is the converted, numeral-remapped, bidi-isolated run from rule 5. Design comes by token name (`color.*`/`type.*`/`space.*`/`motion.*`); the optional affordance and clear control are ‚â•48 dp `touch.min`. No part of this widget touches the mu·π£·∏•af glyph layer (it renders chrome and a date, never Quran text).

9. **Strings deferred but keyed.** All copy ‚Äî the "optional" invitation, **Skip**, the band labels, the clear/remove label, and the Semantics announcements ‚Äî is an `AppLocalizations` key consumed through E09's pipeline and authored/transcreated (ar template + fa/ckb) in **E11-T10**. Do not inline a literal; if a label is needed to compile, add the key in the ARB now and let T10 own the transcreated values. The band term-set ("years ago" register) is regional and review-pending; ckb wraps rather than truncates.

10. **Pitfalls to avoid:** storing or displaying a raw `DateTime`/instant or the bare `epochDay` instead of a `CalendarPresenter`-formatted, numeral-remapped label; constructing a `Jalali`/`HijriCalendar` in the widget or inferring the calendar from `Locale.current` instead of reading the explicit `CalendarSystem`; resolving a band with `Duration`/`DateTime.difference(...).inDays` instead of `CalendarDate.addDays` (DST off-by-one); calling `DateTime.now()` for "today" or the picker bound; making the date a **required** field, re-prompting a skipped juz, or rendering a skipped juz as a red "missing date" / "0%"; running the `coldStartCard` aging math or seeding `(D, S)` in the widget (that is the engine, at E11-T09); opening a `db.transaction` or calling a DAO here; surfacing a "this juz has decayed N%" verdict or any readiness number; hard-splicing an ASCII digit into the date/band string instead of an isolated ARB placeholder; hardcoded left/right or a hardcoded `Directionality`.

## Tests

`package:flutter_test` + Riverpod test harness; runs in CI under both TZ pins; fixtures are explicit `CalendarDate.ymd(...)` values with `clockProvider` overridden to a fixed `today` ‚Äî no hidden clock.

- **`packages/features/test/onboarding/when_memorized_band_test.dart`** (written FIRST ‚Äî the deterministic core):
  - **Band ‚Üí date is `addDays`-pure:** with `today` fixed (e.g. `CalendarDate.ymd(1447, 6, 16)` style), each `StaleBand` resolves to the documented serial day via `addDays` only; assert the resulting `epochDay` integers exactly (no `closeTo` needed ‚Äî these are integers).
  - **DST/timezone-immunity:** the same band resolves to the byte-identical `CalendarDate` under `TZ=Asia/Tehran`, `TZ=Pacific/Kiritimati`, and `TZ=UTC` and across a DST-change date (no `Duration`/`DateTime.difference` path exists to drift).
  - **Skip/clear semantics:** never touching the control leaves `memorizedOn` with no key for that juz; `clearMemorizedOn` after a set returns to no-key (never a `0`/epoch-zero sentinel); a precise pick and a band pick are interchangeable end states (both land one `CalendarDate`).
- **`packages/features/test/onboarding/when_memorized_input_test.dart`** (widget): pumping `WhenMemorizedInput(juz: 13)` over a fake controller ‚Äî choosing a band calls `setMemorizedOn(13, expectedDate)`; tapping clear calls `clearMemorizedOn(13)`; with no value the row shows the optional/Skip affordance and **no** required-field/error state; the widget issues no `Navigator.push` and makes no repository/engine call.
- **Calendar-render assertions:** with a stored value and `CalendarSystem.jalali` (fa) the row shows the date in JalƒÅlƒ´ with Extended Arabic-Indic digits; with `CalendarSystem.hijriUmmAlQura` it shows an Umm al-QurƒÅ label (and an out-of-range value falls back to a Gregorian label rather than throwing); with `CalendarSystem.gregorian` (ar) it shows Arabic-Indic digits ‚Äî **no** ASCII digit appears in any rendered date/band string (per-string assertion).
- **Per-locale goldens** ‚Äî `packages/features/test/onboarding/goldens/when_memorized_{ar,fa,ckb}.png` on the **real** bundled UI fonts (never `Ahem`), pinned runner: the band chips + the rendered date row read start‚Üíend (right‚Üíleft) in all three locales, numerals in the locale set, ckb wrapping (not truncating) its longer band labels. (Composed into the screen-level per-locale goldens in E11-T10.)
