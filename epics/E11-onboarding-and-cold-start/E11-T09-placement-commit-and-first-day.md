## Goal

One repository method — the placement commit — turns the controller's captured `OnboardingState` into a seeded, durable profile. For every held page (expanded from the captured held-juz set in muṣḥaf order) it calls the engine's `coldStartCard(pageId, JuzConfidence, today, {memorizedOn})` with the captured `JuzConfidence`, the optional captured `memorizedOn`, and the injected `today`, assembling the `CardSeed` list; it then commits those seeds **plus** the `cycle_config` row through E03's single `seedColdStart` outer transaction — **persist-before-republish, all-or-nothing**. Only after that `Future` resolves does the onboarding controller republish "ready" and the first generated day get built and handed off; a kill mid-flow leaves no half-seeded state, and a failed commit republishes nothing. The placement summary the handoff lands on is a calm informational surface — "we'll start by revising everything you hold once, and adjust as you recite" — never a celebration. This task is test-first: the seed→store→first-day commit, the persist-before-republish ordering, and the failed-commit proof are written before the repository body.

## Implementation notes

TEST-FIRST (correctness-critical): this is the largest single write in the app and the *sanad*-load-bearing one. Write the repository suite — the seed-fan-out (one `coldStartCard` call per held page, never per juz), the persist-before-republish ordering, the failed-commit-republishes-nothing case — **before** the repository body, and the cold-start → first-day `integration_test` journey alongside it. They must exist and fail first.

1. **The repository method (the single write path).** Add `seedColdStart`-routing in the `data` package's onboarding/profile repository — e.g. `packages/data/lib/src/repositories/cold_start_repository.dart`, method `Future<void> commitPlacement(PlacementInput input)`. It (a) expands the captured held-juz set into the page-id list **in muṣḥaf order** via the read-only `page` reference table (juz → pages), (b) calls `engine.coldStartCard(pageId, input.confidence[juz]!, input.today, memorizedOn: input.memorizedOn[juz])` for **each held page** (per page, not per juz — a juz spans ~20 pages, each gets its own seeded `Card`), (c) maps each returned `Card` to a `CardSeed` row, (d) builds the `Profile` row and the `cycle_config` row from `input.cyclePreset`/`input.dailyBudgetMinutes`, then (e) calls E03's `seedColdStart(profile, seeds)` — the one `db.transaction`. The method returns only after the commit is durable (WAL + `synchronous=FULL`). `PlacementInput` is an immutable `models`/feature value assembled from `OnboardingState` (no `DateTime`, carries the injected `today` as a `SerialDay`/`CalendarDate`).

2. **`coldStartCard` is called, never re-derived.** The repository imports the pure `engine` and invokes the seeder; it does **not** hold a copy of `_coldStartSeed`, compute `S`/`D`, or apply the stale-time decay itself (that is the `memorizedOn` branch inside `coldStartCard`). Pass the captured `memorizedOn[juz]` (a `CalendarDate`/`SerialDay`) through only when present; absence means no stale-time aging. The seeded `Card.dueAt` is `today` for every held page (the calibration pass) — the commit asserts this on the assembled seeds rather than setting it.

3. **Un-held juz produce no row.** Pages of an un-held juz are simply absent from the seed list (they stay `UNMEMORIZED` by absence of a `card` row) — never seeded with a "0%"/placeholder card and never written as a sentinel. The `card` schema invariant `CHECK (track='UNMEMORIZED' OR due_at IS NOT NULL)` holds because the commit writes only memorized cards, each with a non-null `dueAt`.

4. **`cycle_config` rides the same transaction.** The captured `CyclePreset` + `dailyBudgetMinutes` (E11-T08, already validated as `EngineConfig` inputs) map to the `cycle_config` row (`cycle_type`, `far_cycle_days` = the cycle ceiling, `daily_budget_minutes`, `pure_cycle_mode`, `term_label_set`, …). It is inserted **inside** `seedColdStart`'s transaction so cards + config commit atomically — there is no window where a profile has cards but no cycle ceiling, or vice versa. No `target_R` is written; the preset is a named value only.

5. **The controller command + handoff (persist-before-republish).** On the onboarding controller (E11-T01), add `Future<void> commitAndBuildFirstDay()`: it assembles `PlacementInput` from the captured state + `ref.read(clockProvider).today()`, `await`s `coldStartRepository.commitPlacement(input)`, and **only after that returns** flips the controller to a terminal `placed` view-state and lets the router/redirect resolve to the first generated day (the controller does **not** navigate — it publishes state; the redirect guard / View routes). The first day is `engine.buildToday(cards, today)` read through the normal Today `StreamProvider` over the now-committed `card` rows — i.e. the seed re-emits the stream, which *is* the first day; the commit does not separately compute and cache a day plan. A `commitPlacement` failure surfaces as a calm retry state on the controller, never a partial advance.

6. **Mid-flow kill safety, by construction.** Because nothing is written before `commitPlacement` and the write is one `seedColdStart` transaction, an app kill before commit leaves an empty/un-onboarded device (the E07 redirect guard re-routes to `/onboarding` for a clean restart — E11-T01's "abandon = clean restart" contract); a kill *during* commit rolls the transaction back (WAL). There is never a profile with a partial card set. Document this *sanad*-safe property in the repository method's `///` doc comment.

7. **The summary surface is calm, informational, terminal.** The post-commit surface (composing E10's welcoming first-run empty state) renders the C-009 behavioral promise as copy — "we'll revise everything you hold once, and adjust as you recite" — plus a quiet entry into the first day. **No** confetti / streak / badge / completion-% / "you're N% ready" / praise theatre / "Welcome back!"; motion is informative, not celebratory (design-principles §2). All copy is `AppLocalizations` keys (authored/transcreated in E11-T10); no inlined literal, no seeded `D`/`S`/`R` shown.

8. **Pitfalls to avoid:** a raw `insertAll` loop in the controller or repository instead of routing through E03's `seedColdStart` (breaks the single transaction); calling `coldStartCard` once per *juz* instead of once per *page* (mis-seeds 600+ cards); hardcoding the `_coldStartSeed` `(D, S)` table or computing stability in `data` (the engine owns it); republishing "ready"/building the first day **before** the commit resolves (persist-after-publish — the exact inversion the test catches); a missing `await` inside `seedColdStart` (the Drift footgun — data loss); writing a `card` row for an un-held juz or a stored "0%"; storing a `DateTime` for `due_at` instead of a `CalendarDate` serial integer; `DateTime.now()` anywhere in the path (use the injected clock); a celebration/score on the summary; navigating from inside the controller; surfacing a seeded `D`/`S`/`R` or readiness % on the handoff.

## Acceptance criteria

- [ ] A single repository method (`commitPlacement` in the `data` cold-start repository) expands the held-juz set to page ids in muṣḥaf order, calls `engine.coldStartCard(...)` **once per held page** with the captured `JuzConfidence`, optional `memorizedOn`, and injected `today`, and commits the resulting `CardSeed` list **plus** the `cycle_config` row through E03's `seedColdStart` outer transaction.
- [ ] The commit is **persist-before-republish**: the controller's `commitAndBuildFirstDay()` `await`s the repository commit and only **after** it resolves flips to the terminal `placed` state and lets the router resolve the first day; a unit test proves a failed commit republishes nothing and advances nothing.
- [ ] The whole seed + `cycle_config` write is **one `db.transaction`** (all-or-nothing, auto-rollback); every query inside is `await`-ed; no raw insert loop bypasses `seedColdStart`; a mid-commit failure leaves zero `card` rows for the profile.
- [ ] Every seeded `Card` is memorized with a non-null `dueAt = today` (the calibration pass) and `track` derived by the engine from the seeded `S`; un-held juz produce **no** `card` row; the `CHECK (track='UNMEMORIZED' OR due_at IS NOT NULL)` invariant holds for all written rows.
- [ ] The repository imports the pure `engine` and **calls** `coldStartCard`; it hardcodes no `_coldStartSeed` `(D, S)` table, computes no `S`/`D`/`R`, and applies no stale-time decay itself (the `memorizedOn` branch is the engine's); the UI/handoff surfaces no seeded `D`/`S`/`R` or readiness %.
- [ ] `due_at` is stored as a `CalendarDate` serial integer; "today" enters only via the injected `clockProvider`; `DateTime.now()`/`Calendar.current`/`TimeZone.current` appear nowhere in the path (grep-clean).
- [ ] The first generated day is the normal Today `StreamProvider` re-emission over the committed `card` rows (`engine.buildToday`), not a separately cached day plan; the controller publishes state and does not navigate.
- [ ] The handoff lands on a calm informational summary (C-009 behavioral copy via E11-T10 keys) with **no** confetti/streak/badge/completion-%/praise/"Welcome back!"; every string is an `AppLocalizations` key, RTL by geometry, no inlined literal.
- [ ] Every public declaration carries a `///` doc comment; the repository method's doc comment states the persist-before-republish + "no half-seeded state on a kill" *sanad*-safe property; the path passes `dart format` + analyzer/lint and the `data`-imports-no-`http` / engine-stays-pure gates.

## Tests

Test-first. Repository/ordering units are `package:test`/`flutter_test` with Riverpod overrides and an in-memory or fault-injecting persistence handle (no widget pump for the ordering logic); the cold-start → first-day journey is `integration_test` over the **real** Drift/SQLite + asset stack. The throwing `HttpOverrides` offline guard wraps all of it. Run under both TZ pins; `today` is a literal injected `CalendarDate`/`SerialDay`, never a wall clock.

- **`packages/data/test/repositories/cold_start_repository_test.dart`** (written first):
  - **Seed fan-out per page, not per juz:** given a captured held-juz set (e.g. `{1, 30}`) with `JuzConfidence` per juz, `commitPlacement` calls a recording `engine.coldStartCard` exactly once **per page** of those juz (count matches the juz→page expansion from the `page` reference table), each with the right `(pageId, JuzConfidence, today, memorizedOn?)`; un-held juz contribute zero calls and zero seeds.
  - **`memorizedOn` plumbing:** a juz with a captured `memorizedOn` passes that `SerialDay` to `coldStartCard`; a juz without it passes `memorizedOn: null` (the skip-date path, E11-T07) — and neither path computes decay in `data`.
  - **Calibration + invariant:** every assembled `CardSeed` is memorized with `dueAt == today` and a non-null `dueAt`; no seed targets an un-held page; the `cycle_config` row carries the captured preset/budget and no `target_R`.
  - **Persist-before-republish ordering:** a recording handle whose `seedColdStart` closure asserts, at call time, that the controller has **not** yet republished `placed` and the Today stream has not yet re-emitted — proving persist strictly precedes publish.
  - **Failed commit republishes nothing:** a handle wired to throw inside `seedColdStart`; the controller surfaces a calm retry state, stays on the commit step, exposes no `placed` state, and the store holds zero `card`/`cycle_config` rows for the profile (transaction rolled back). Set, and re-attempt-after-failure, both proven.
  - **One transaction / await discipline:** the write goes through E03's `seedColdStart` (asserted, not a raw `insertAll` loop); a static check / review-grep for un-awaited queries inside the transaction body.
- **`integration_test/cold_start_to_first_day_test.dart`** (one of the four PRD journeys, real stack): drive coverage → confidence → optional date → cycle/budget → commit; assert 600+ `card` rows exist after commit with `dueAt = today`, the first Today screen renders the seeded pages grouped manzil → near → new in recitation order (`engine.buildToday`), and the summary surface shows the calm informational copy with **no** celebration affordance. A kill-and-relaunch variant (commit, terminate, relaunch) finds the seeded profile intact and lands on Today, not back in onboarding.
- **Offline guard** — the throwing `HttpOverrides` proves `commitPlacement` and the first-day build open no socket (the only permitted download was E05's, already complete by E11-T04).
