## Goal

Every onboarding ARB key the sibling steps deferred is now authored and locked: the `ar` template (`lib/l10n/app_ar.arb`) carries the source-of-truth copy, and `fa`/`ckb` are transcreated against it (not literally translated), with all term-set / religious-vocabulary entries marked "needs native + scholar review" and `ckb` canonical-encoded and verified to **wrap, never truncate**. Every onboarding control across the eight steps is `Semantics`-labelled with its state, grouped or toggle-flagged, and carries a visible focus ring; geometry is RTL-native and numbers/dates render in the locale numeral set throughout. The whole flow is pinned by widget tests per step and a consolidated per-locale (ar/fa/ckb) golden suite rendered on the **real bundled UI fonts** (never `Ahem`) with deuteranope/grayscale checks; the skip-date and fail-closed-download paths, the seed→store→first-day commit, and a throwing `HttpOverrides` offline guard on every post-download step are all asserted green. The l10n-completeness and per-screen a11y audit gates pass for the onboarding feature.

## Implementation notes

TEST-FIRST (correctness-critical): the l10n-completeness assertions (no missing key across ar/fa/ckb, all six Arabic plural categories present on every count-bearing onboarding message, no hardcoded user-facing literal under the onboarding feature, no ASCII digit in any onboarding label/`Semantics`), the skip-date branch, the fail-closed download-state rendering, and the offline `HttpOverrides` guard on every post-download step are pinned by tests written **before** the ARB values and the consolidated goldens are finalized — they must exist and fail (or flag the gap) first.

1. **ARB authoring lives in the l10n pipeline, not in the feature.** All onboarding keys land in `lib/l10n/app_ar.arb` (the `l10n.yaml` `template-arb-file`) first, then `lib/l10n/app_fa.arb` and `lib/l10n/app_ckb.arb` as transcreations diffed against `ar`. Read every string only through the generated `AppLocalizations` (`l10n.*`) in `packages/features/lib/src/onboarding/**`; `nullable-getter: false` keeps a missing/typo key a compile error. This task does **not** add new widgets — it fills the keys the siblings referenced and corrects any inline literal that slipped through.

2. **One ICU message per interpolation; never concatenate.** Each interpolated onboarding string (the download progress line, "Juz {n}", the budget "{minutes} minutes", the converted "memorized on {date}", the placement summary's held-page count) is one full ICU message with `{placeholder}` slots — the opposite-direction value stays an isolated, formatted placeholder, never a hard-spliced substring (RTL word order will not survive concatenation).

3. **Every count is an ICU `plural` with all six Arabic CLDR categories.** Any count-bearing onboarding message — the held-page/juz count in the placement summary, a "{n} pages to revise" first-day line if surfaced, the catch-up-free "first day" count — is an ICU `plural` defining `zero`/`one`/`two`/`few`/`many`/`other` in `app_ar.arb`; a missing category is a release blocker. The count is locale-numeral-formatted before placement.

4. **Numerals and dates render through `numberFormatFor(locale)` / the calendar presenter.** Juz numbers, budget minutes, the download percentage, and any page count format through `numberFormatFor(locale)` with the pinned numbering system (`-u-nu-arabext` for fa/ckb, `-u-nu-arab` for ar) then inject as a placeholder; the optional "when memorized" date arrives already converted from `domain-calendars-and-hifzdate`'s `CalendarPresenter` (default Jalālī for fa) and is only re-mapped to locale digits and isolated here — never converted in the feature. No ASCII digit is ever concatenated into a localized string or a `Semantics` label.

5. **Mixed runs go through the one bidi-isolation helper.** "Juz {n}", "{page} of {total}", the riwāyah label "Ḥafṣ ʿan ʿĀṣim — Madani muṣḥaf" beside RTL copy, the converted date, and any user-typed profile name are wrapped via the `l10n/bidi.dart` helper — `isolateLtr`/`isolateRtl` when direction is known, `isolate` (FSI…PDI) otherwise. The riwāyah edition string is named explicitly and is itself a localized chrome string, never "the Quran" in the absolute (R2).

6. **Term-sets and `ckb` stay provisional.** The sabaq/sabqi/manzil track vocabulary and the four traditional grade verbs (if surfaced in the cycle-preset / summary copy) are an ICU `select`/region-override, not edited base strings; the `ckb` values are canonical-encoded (U+06D5 ە for AE, U+06A9 ک for kaf — no stray ZWNJ, no Teh-Marbuta-for-AE) and every term-set/religious-vocabulary `@description` carries "needs native + scholar review." The Sorani register and religious terminology are not inferred from fa/ar.

7. **Consolidated Semantics/RTL pass across all eight steps.** Audit every onboarding control for: a `Semantics` label that announces **state** ("Juz ۱۳, held", "Shaky — needs regular revision", "Download ready", "Skip this date"), grouping (`MergeSemantics` so a number + glyph read as one node) or a toggle/selected flag where the control is a choice, a visible focus ring (`color.outline`) for keyboard/switch-control, a ≥48 dp hit target, and redundant color + shape + label encoding (never hue alone). Direction comes from the locale (`supportedLocales: [ar, fa, ckb]` + `GlobalWidgetsLocalizations`); back/next/progress mirror via the auto-mirroring `Icons.arrow_*`; no `EdgeInsets.only(left:/right:)` / `Alignment.centerLeft/Right` / `Positioned(left:/right:)` / hardcoded app-wide `Directionality` survives the `features/**` grep. (The only permitted manual `Directionality` is the language-pick preview rendering a sample in the candidate locale's direction.)

8. **`ckb` wraps, never truncates.** Sorani terms run longer than Persian/Arabic; every onboarding control that holds text (the step headings, the cycle-preset rows, the Solid/Shaky/Rusty options, the buttons) lays out so the `ckb` string wraps to a second line rather than clipping or ellipsizing — the per-locale `ckb` golden is the regression guard, and the bundled UI font's Sorani coverage (پ چ ژ گ ڤ ڕ ڵ ۆ ێ ە) is exercised by rendering on the real font.

9. **The conscience pass is the first gate.** Run every consolidated string through domain-adab-and-religious-integrity before it locks: the welcome states C-048 (no account / no microphone / nothing leaves the device / works in airplane mode) and C-046 (servant to your teacher, no ruling) plainly; the confidence step shows no readiness % and no `D`/`S`/`R`; the un-held juz copy is calm "not held," never "missing"/"0%"; the summary is calm and informational, never a celebration; no string speaks *for* the Quran or overrides a future teacher. The banned-phrase lint (guilt/fear/loss, "you must/should/don't", "safe to drop"/"mastered", exclamation/emoji, "upgrade/premium") runs on all three locales and is release-blocking.

10. **Pitfalls to avoid:** hand-editing `fa`/`ckb` first or literal-translating so register/tone drifts; an inline `Text('…')`/`tooltip:`/`semanticsLabel:` literal left in any onboarding widget; a `count == 1 ? … : …` ternary or `"$count "` splice instead of an ICU `plural`; omitting `few`/`many` from an Arabic plural; concatenating ASCII digits or using the Arabic-Indic block for fa/ckb (or vice-versa); converting a calendar in the feature layer instead of rendering the presenter's output; a hardcoded `Directionality` or a physical left/right inset; mirroring a non-directional glyph or (forbidden) any muṣḥaf glyph; a `ckb` heh+ZWNJ AE hack or a clipped/ellipsized Sorani term; locking a term-set or religious string without the provisional flag; rendering a golden with `Ahem` instead of the real bundled fonts; a `Semantics` label that names a control but not its **state**; a focus ring or ≥48 dp target missing on any onboarding control.

## Tests

Test-first; `package:flutter_test` + the Riverpod test harness with in-memory fakes (controller, repository, asset loader, injected clock) for the widget/golden tiers; the real Drift/asset stack only in the cold-start `integration_test` journey. The throwing `HttpOverrides` offline guard is installed via the shared bootstrap and asserted on every post-download step. Files mirror the source/step names under `packages/features/test/onboarding/`.

- **`packages/features/test/onboarding/l10n_completeness_test.dart`** (written first):
  - **Key coverage:** every onboarding `l10n.*` key referenced under `onboarding/**` resolves in ar, fa, and ckb (no null/missing); a deliberately removed `fa`/`ckb` value fails the assertion.
  - **Arabic plural completeness:** every count-bearing onboarding ICU `plural` message in `app_ar.arb` defines all six CLDR categories; a message missing `few`/`many` fails.
  - **No hardcoded literal / no ASCII digit:** a scan of the onboarding feature finds zero string literals passed to `Text`/`tooltip:`/`semanticsLabel:`/`label:` and zero ASCII digits in any onboarding label or `Semantics` string.
  - **Canonical `ckb` + provisional flag:** `ckb` values use U+06D5/U+06A9 (no ZWNJ, no Teh-Marbuta-for-AE); every term-set/religious-vocabulary entry's `@description` carries the provisional review marker.
  - **Banned-phrase lint:** no never-ship pattern (guilt/fear/loss, "you must/should/don't", "safe to drop"/"mastered", exclamation/emoji, "upgrade/premium") appears in any onboarding string in any locale.
- **`packages/features/test/onboarding/onboarding_semantics_test.dart`** (widget, per step): each onboarding control exposes a `Semantics` node whose label announces its **state** (welcome privacy facts; language/riwāyah selected; download `awaitingFirstDownload`/`downloadInterrupted`/`ready`; "Juz ۱۳, held"; "Shaky — needs regular revision"; "Skip this date"; the chosen cycle preset + budget; the calm summary), carries the toggle/selected flag where it is a choice, shows a focus ring, and meets `meetsGuideline(androidTapTargetGuideline)` + `labeledTapTargetGuideline` (≥48 dp).
- **`packages/features/test/onboarding/skip_date_path_test.dart`** (widget): the optional "when memorized" step renders its skip affordance, skipping advances without a `memorizedOn` value (the `memorizedOn` map gains no entry for that juz), and the `Semantics` announces the skip; a provided date is isolated, locale-numeral, and in the user's calendar.
- **`packages/features/test/onboarding/download_states_test.dart`** (widget): the fail-closed download step renders calm `awaitingFirstDownload`, `downloadInterrupted` (with a non-blaming Retry, no exclamation), and `ready` states; no muṣḥaf glyph renders before `ready`; the interrupted/awaiting copy passes the adab gate (no fault/blame).
- **Per-locale consolidated goldens** — `packages/features/test/onboarding/goldens/onboarding_<step>_{ar,fa,ckb}.png` for each of the eight surfaces, on the **real** bundled UI fonts via `FontLoader` (never `Ahem`), pinned DPR/size/theme, animations off, `@Tags(['golden'])`: each surface reads start→end (right→left); numerals render in the correct locale block; mixed runs do not reorder; **`ckb` wraps rather than truncates**; a deuteranope/grayscale check confirms every choice/state control is distinguishable without hue.
- **Offline guard** — the throwing `HttpOverrides` bootstrap asserts every post-`ready` step (coverage, confidence, when-memorized, cycle preset + budget, summary) opens **no** socket; only the download step's own test (E11-T04's) exercises the single permitted download via its mock, resetting `HttpOverrides.global` in its own `setUp`.
- **Seed→store→first-day commit** — the cold-start `integration_test` journey (shared with E11-T09) drives the full flow on the real Drift/asset stack: capture → confidence → optional date → cycle/budget → `commitPlacement` (`coldStartCard` per held page → `seedColdStart` transaction) → the first generated day; a failed commit republishes nothing (persist-before-republish), and the post-commit Today emission is the first day. This task asserts the localized/RTL/a11y surface of that journey; the commit semantics are owned by E11-T09.
