## Goal

The first cold-start capture pass exists as a fast juz-level tap grid in `packages/features/lib/src/onboarding/widgets/coverage_capture_grid.dart` — a `GridView` of 30 selectable cells in **muṣḥaf order** (juz ۱ at the start/right, juz ۳۰ at the end/left in RTL) where **one tap = held / not-held**. Tapping a cell adds that juz to the controller's `coverage` set (E11-T01's `toggleJuz`); un-toggling removes it. A juz that is simply not selected stays `UNMEMORIZED` by absence — drawn as a calm, un-emphasized cell, **never** alarm-red / "missing" / "0%". Each cell carries a locale-numeral juz label **plus** a selected/check glyph (never selection by color alone), is `Semantics`-labelled "Juz ۱۳, held" / "Juz ۱۳, not held" with a toggle flag and a visible focus ring, and is a ≥48 dp target. The widget is a dumb capture surface: it writes no rows, invents no `(D, S)`, reads no clock, and renders no muṣḥaf glyph.

## Implementation notes

TEST-FIRST (correctness-critical): the held/un-held toggle semantics, the muṣḥaf-order RTL placement (juz ۱ at the start), the absence-not-"0%" rule, and the per-cell `Semantics` announcement are pinned by a widget suite written **before** the grid body — the toggle, the "un-toggle removes from `coverage` (becomes `UNMEMORIZED` by absence, not a `0`/missing value)", the reading-order, and the announced-state cases must exist and fail first.

1. **File & placement.** The grid is a leaf widget under the E11-T01 module: `packages/features/lib/src/onboarding/widgets/coverage_capture_grid.dart`, one primary public type (`CoverageCaptureGrid`, a `ConsumerWidget`). The `coverage` step host inside `onboarding_screen.dart` (E11-T01) mounts it; this task does **not** add a route, a cursor step, or a captured-state field (all E11-T01's) — it renders and drives the existing `coverage` state.

2. **Reads/writes only through the controller.** The widget `ref.watch`es the E11-T01 controller for the `Set<int> coverage` and calls its `toggleJuz(int juz)` command on tap. It holds **no** local `coverage` copy, no `setState` mutation of capture state, and **no** `data`/DAO/`db.transaction`/`coldStartCard` call — it is a dumb View over the single source of truth (eng-add-feature-module View-never-writes; eng-create-riverpod-store single source). A held juz is membership in the set; an un-held juz is **absence** — never stored as `0`, `"missing"`, or a percentage anywhere.

3. **Grid geometry — muṣḥaf order, RTL by construction.** Use a `GridView.count`/`GridView.builder` of 30 cells laid out so cell index 0 = juz ۱ lands at the **start (right)** and index 29 = juz ۳۰ at the **end (left)** under the ambient RTL `TextDirection` (locale-derived, **never** a hardcoded `Directionality` — eng-rtl-and-bidi-layout §2). Mirror the heat-map's 30-juz muṣḥaf-order grid (design-system 07 §8) so the spatial map matches the book the ḥāfiẓ carries. All insets/spacing are logical (`EdgeInsetsDirectional`, the `space.*` scale); no `EdgeInsets.only(left:/right:)`, no `Alignment.centerLeft/Right`, no `Positioned(left:/right:)` (CI-grep-banned in `features/**`).

4. **Cell encoding — redundant, never color alone.** Each cell shows: the juz number formatted via `numberFormatFor(locale)` with the pinned numbering system (`-u-nu-arabext` for fa/ckb, `-u-nu-arab` for ar) wrapped through the bidi-isolation helper (`isolate`/`isolateLtr`) so the "Juz N" run never reorders inside RTL chrome; **plus** a selected-state glyph (a check / filled `Icons.*`) that appears when held and is absent when un-held; **plus** the held/un-held color family from tokens (`color.*`). Held vs un-held is distinguishable by **shape + glyph + label**, not hue alone (SC 1.4.1) — readable in grayscale and for a deuteranope. The un-held cell is the calm baseline appearance (`color.surface`/`color.outline`), **not** an alarm-red/"0%"/"missing" tile (design-system 07 §8 anti-patterns).

5. **Touch & focus.** Each cell is a single ≥48 dp `touch.min` target (a `Semantics`/`InkWell`/`GestureDetector`-wrapped tile, large enough at all three locale font metrics); a visible focus ring (`color.outline`) shows for keyboard/switch-control users. Spacing keeps adjacent cells from merging into one ambiguous tap.

6. **Semantics.** Each cell is one labelled, toggleable node: `Semantics(label: <localized "Juz ۱۳">, toggled: held, ...)` announcing "Juz ۱۳, held" / "Juz ۱۳, not held" (the juz number in the locale numerals, the held/not-held phrase from `AppLocalizations`). Use the toggle/selected flag so a screen-reader user hears the state, and `MergeSemantics` so the number + glyph read as one control. The juz-number-in-`Semantics` path uses the same locale numeral formatter (no ASCII leak into the announcement). (Label strings are keyed now and transcreated in E11-T10.)

7. **Strings deferred but keyed.** Introduce no inline literal: the "Juz {n}" template, the "held"/"not held" `Semantics` phrases, and the step's calm heading/helper line are `AppLocalizations` keys consumed through E09's `gen_l10n` pipeline; add the keys to the ARB now and let **E11-T10** own the ar template + fa/ckb transcreated values (term-sets review-pending, ckb wraps not truncates). The juz number is an isolated, locale-numeral placeholder in the ICU message — never a hard-spliced ASCII substring.

8. **No muṣḥaf glyph, no network, no clock, no AI.** The grid renders only chrome (numbers, a check glyph, token colors) — **never** a KFGQPC page glyph (the heat-map/coverage grid is chrome, not the sacred layer; eng-rtl-and-bidi-layout §9). It touches no socket (the core pack is already verified `ready` by E11-T04 — the grid is only reachable post-`ready`), reads no clock (`DateTime.now()` appears nowhere), and runs no model/microphone/inference — it captures juz-level self-report taps only.

9. **Pitfalls to avoid:** ordering the grid LTR (juz ۱ at the left) or hardcoding a `Directionality`; encoding held/un-held by color alone (no glyph/label); drawing an un-held juz as alarm-red, "missing", "0%", or a failure tile; storing un-held as a `0`/"missing" value instead of mere absence from `coverage`; mutating a local `coverage` copy via `setState` instead of the controller's `toggleJuz`; splicing ASCII digits into the "Juz N" string or the `Semantics` label; rendering a muṣḥaf glyph or any seeded `D`/`S`/`R`/"% ready" in a cell; adding a streak/badge/confetti when the grid fills; opening a `db.transaction`/DAO/`coldStartCard` call from the widget (E11-T09 is the only writer); tap targets under 48 dp; missing the toggle flag or focus ring on a cell.
