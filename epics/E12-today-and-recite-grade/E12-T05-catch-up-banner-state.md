## Goal

After a gap, the Today list shows a calm, supportive catch-up state instead of a blank or a red overdue pile. This task wires the **`catchUp` variant** of the controller's sealed `TodayState` (from E12-T02) onto the E12-T03 list: a dumb `CatchUpBanner` renders the engine's **already-computed re-spread plan** — most-decayed and prayer-critical first, FAR/manzil mandatory — in the **empathy → honest fact → concrete M-day plan → user-owned choice** order, composing the E10 catch-up-banner leaf on a calm `surfaceContainer`. The View never computes the spread or the "N days missed" count: it renders the controller's pre-built `CatchUpPlan` and the injected `CalendarDate` "today". Plan numerals (missed days, the M-day horizon, page/juz on each row) render in the locale set, mixed runs bidi-isolated; `haptic.confirm` fires only when the user accepts the plan; any M-day-plan framing number traces to a CLAIMS row. The boundary with E12-T04 is explicit: when there is **nothing** to catch up on, that task's silent-resume handles the gap — this task fires **only** when a backlog plan exists.

## Implementation notes

This task is presentation-only: no engine call, no `review_log` write, no `DateTime.now()`, no spread math, no new mutation type. It renders the `catchUp` read-model variant the E12-T02 controller already computed. The catch-up copy is **correctness-critical for adab** (it is the app's gentlest moment in its hardest conversation) → its wording, order, and the no-shame invariants are pinned **test-first** by widget tests + the never-ship banned-phrase lint before any visual polish.

1. **Files** (in the `features` umbrella package, `lib/src/today/`, per eng-add-feature-module):
   - `widgets/catch_up_banner.dart` — a dumb `CatchUpBanner extends ConsumerWidget` taking a pre-built `CatchUpPlan` (the E10 leaf composed here; start from the `ui-catch-up-banner/template.dart` scaffold). No state of its own; no engine; no clock; no spread math.
   - Reuse the E12-T03 / E10 page-card row to render `plan.rows` (each `CatchUpPlanRow` carries `pageNumber`, `juzNumber`, a calm `decay` band, and `isMandatory`) — do **not** author a new row widget.
   - Wire the `catchUp` branch into `today_screen.dart`'s `switch` over `TodayState` (the dumb View from E12-T01/T03); add only the new branch, do not re-architect the switch.
2. **Render order is the contract — empathy → fact → plan → choice** (ui-catch-up-banner Pattern 1; voice §4). Lay the banner out top-to-bottom (start-aligned) as: (1) a calm, non-blaming acknowledgment line; (2) the honest fact — `plan.missedDays` stated plainly, never amplified, never a standalone reproach; (3) the concrete plan header — "here is an M-day catch-up that still completes your cycle" using `plan.spreadDays`, followed by the engine-ordered `plan.rows` (most-decayed/prayer-critical first — render in the order received, **never re-sort in the View**); (4) the user-owned choices. The order is asserted by a widget test, not left to layout chance.
3. **Calm visual register — never an alarm** (ui-catch-up-banner Pattern 3; 07-components §1). Render on a flat `surfaceContainer` (Level 0–1) `Card` with `elevation: 0`; copy in `type.body`, `color.text.secondary` for the supporting lines; padding on the `space.*` grid. **No** `color.semantic.error`/red fill, **no** saturated warning fill, **no** alarm icon, **no** countdown. Any decay a row references reads as the heat-map green receding to a muted neutral (`color.heatmap.weak` → `faded`), never red — copy and color tell the same calm story.
4. **The choices are autonomy-supportive — options, never one mandate** (ui-catch-up-banner Pattern 6; voice §6). Render the user's real choices as quiet `TextButton`/`OutlinedButton`s (≥48dp `touch.min`): **start the catch-up plan**, **adjust its length/budget** (deep-link to the E16-owned cycle/budget settings via `go_router`), and **defer** (dismiss back to the ordinary day). Copy is phrased as invitation ("You can start the plan…"), never "You must…". There is no fourth choice that silently drops pages.
5. **Accept is the one mutation, through the single write path** (ui-catch-up-banner Pattern 10; eng-create-riverpod-store). "Start the plan" calls a controller method (e.g. `ref.read(todayControllerProvider.notifier).acceptCatchUpPlan()`) that persists any state change transactionally **before** republishing — this task does not write directly in the widget. Fire `haptic.confirm` (`lightImpact`, OS-gated) **only** on accept, paired with the on-screen state change. "Adjust" navigates (no mutation here); "defer" dismisses (no mutation, no write). If accepting the plan is purely a navigation into the normal day for this slice (no persisted field yet), keep the single-write-path contract documented at the call site so E16/E04 wiring stays clean — and still no `DateTime.now()`.
6. **FAR/manzil is never dropped — assert it, don't implement it** (PRD §7.9, §7.12). The keep/drop and ordering decisions are the engine's; the View renders `plan.rows` as received, including every `isMandatory` row. Add a widget-test invariant that the rendered plan contains every mandatory (FAR/manzil) row the engine put in `CatchUpPlan`, and never labels any row "safe to drop". The banner must never thin the plan to look shorter.
7. **Never greet the gap; one state only** (ui-catch-up-banner Pattern 5; ui-empty-state Pattern 1). The forbidden greeting "Welcome back! You haven't opened the app in N days" must not exist as an ARB key or a literal. The `catchUp` state fires **only** because the controller built a plan; the gap-with-nothing-to-catch-up case is E12-T04's silent resume. Do not compute "days since last open" anywhere in the View — `plan.missedDays` arrives pre-built from the controller's injected `CalendarDate`.
8. **No celebration on accept or completion** (ui-catch-up-banner Pattern 7; 06-motion §2). Starting the plan (and later clearing it) fires no confetti, no fanfare, no "well done", no streak-repair animation. Acknowledge with copy + a `motion.duration.short` (150ms) + `motion.curve.standard` fade only; honor reduce-motion (degrade to an instant cut). `haptic.confirm` is the only haptic; **no** success/reward pulse, **no** nag buzz for the missed days.
9. **Strings via the ARB pipeline, transcreated for fa/ckb/ar** (eng-add-localized-string; voice §8). Every line — the empathy acknowledgment, the honest-fact line (with `missedDays`), the plan header (with `spreadDays`), the three choice labels — is an `l10n.*` key in `app_ar.arb` (base/template language) + transcreated `fa`/`ckb` (ckb flagged pending native + scholar review). Soften the Arabic imperative into a statement of readiness; warm-respectful Persian register; no hardcoded user-facing string; no generic "overdue" label. Any sabaq/sabqi/manzil/cycle term the plan names uses the localized term-sets. Count-bearing lines (N days, M-day plan) use the plural/placeholder message form, not glued fragments.
10. **Locale numerals + RTL by geometry** (ui-catch-up-banner Pattern 9; eng-rtl-and-bidi-layout). `plan.missedDays`, `plan.spreadDays`, and each row's page/juz render via `intl` `NumberFormat` for the active locale (Extended Arabic-Indic ۰۱۲۳ for fa/ckb, Arabic-Indic ٠١٢٣ for ar) — never concatenated ASCII. Wrap every mixed Latin/numeral run (a page number, "Juz N") in bidi isolation (FSI/PDI) so a count never breaks the RTL line. Layout is `EdgeInsetsDirectional`/`AlignmentDirectional` only (no `left`/`right`); the one template serves fa/ckb/ar and ckb's longer transcreated copy reflows within the same insets.
11. **Accessibility** (E08 conventions). One calm `Semantics` announcement frames the banner — the empathy + honest fact + plan-header read as the situation, and the choices announce as buttons with labels and a visible focus ring; the plan's rows announce as the E10 page-card merged phrases. ≥48dp targets; reduce-motion honored.
12. **Pitfalls to avoid:** re-sorting / re-computing the re-spread in the widget (it arrives ordered from the engine); computing "N days missed" or reading `DateTime.now()` in the View (it's `plan.missedDays` from the injected clock); a red/`color.semantic.error` fill, alarm icon, or countdown; dropping or thinning a FAR/manzil row to shorten the plan; a "Welcome back, N days"/"you're behind"/"N days lost" greeting or reproach; a single mandated fix or a silent page drop instead of real choices; any celebration on accept (confetti/streak-repair/`!`/success haptic); a mandate word ("must/should/don't") in any choice label; "safe to drop"/"mastered"/"done" anywhere; both this state and E12-T04's silent resume rendering for the same gap.

## Tests

All deterministic, offline by construction (no surface here opens a socket), on the real bundled fonts, with a seeded `CatchUpPlan` and the injected `CalendarDate` (never a real clock). **Test-first** on the copy/order/adab invariants — those tests are written and failing before the banner is built.

- `features/test/today/catch_up_banner_test.dart` (widget) — **written first**:
  - **order invariant**: given a seeded `CatchUpPlan`, the rendered tree presents empathy → honest-fact → plan-header → rows → choices in that vertical (start) order;
  - **engine-order invariant**: the plan's rows render in exactly the order supplied (a seeded plan whose rows are deliberately not page-sorted proves the View does not re-sort);
  - **FAR-mandatory invariant**: every `isMandatory` row in the seeded plan is present in the tree; no row is dropped or labelled "safe to drop";
  - **calm-register invariant**: the banner uses `color.text.secondary` on a `surfaceContainer`, **not** `color.semantic.error`/red, and renders no alarm icon;
  - **choice invariant**: exactly the three user-owned choices render, each ≥48dp announcing a button role; "adjust" pushes the expected E16 settings route (a fake `go_router`/navigator records the push) and writes nothing; "start the plan" calls the controller's accept method exactly once and fires `haptic.confirm` once (a fake haptic boundary records it); "defer" dismisses and writes nothing (a recording repository/double asserts zero unexpected writes);
  - **no-greeting invariant**: no "welcome back"/"N days"/"you haven't opened"/"you're behind" text appears in the tree across fa/ckb/ar.
- `features/test/today/catch_up_banned_phrase_test.dart` (unit) — runs the never-ship banned-phrase lint over every ARB value this task adds in fa/ckb/ar (guilt/fear/loss: "you're behind", "N days lost", "you'll lose your hifz"; mandate words "must/should/don't"; "safe to drop"/"mastered"/"done"/"finished"; `!`; emoji) and fails on any hit; asserts the forbidden "welcome back … N days" key does not exist.
- `features/test/today/catch_up_numerals_test.dart` (widget) — `missedDays`, `spreadDays`, and a row's page/juz render in the locale numeral set per locale (fa/ckb ۰۱۲۳, ar ٠١٢٣), never ASCII; mixed Latin/numeral runs are FSI/PDI-isolated and do not break the RTL line.
- `features/test/today/catch_up_banner_golden_test.dart` (golden) — per-locale (fa/ckb/ar) goldens of the catch-up state: the empathy→fact→plan→choice banner with the engine's re-spread rows (including a mandatory FAR row), on real bundled fonts; a calm-register golden proves no red/alarm fill; a ckb-reflow golden proves no truncation; a reduce-motion variant proves the acknowledgment fade degrades to instant.
- Offline guard: the suite runs under an `HttpOverrides` that fails any socket open (eng-write-dart-test) — proving the catch-up state makes no network call. (The cold-start → buildToday → grade → next → catch-up-after-a-gap integration journey that also exercises this state end-to-end lives in E12-T09.)
