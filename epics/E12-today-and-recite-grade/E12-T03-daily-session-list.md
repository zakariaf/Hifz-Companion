## Goal

The populated Today renders as a single **finite, time-budget-capped** `CustomScrollView` grouped into three sliver sections — **Far (manzil) → Near (sabqi) → New (sabaq)**, recited old-before-new — that **visibly ends**. Each section has a quiet, localized term-set header; each row is an E10 page-card assembled by the feature layer from one `DueItem` of the controller's pre-built day (E12-T02), placed as one ≥48dp tap into the recite route, with no Quran glyphs and no D/S/R on the row. The list is RTL by geometry (`EdgeInsetsDirectional`/`AlignmentDirectional`, leading affordances at start, chevron at end), page identity renders in locale numerals bidi-isolated (FSI/PDI), the `loading` state shows a calm `surfaceContainerLow` skeleton, and a `Semantics` container announces "Revise today" with section roles. The View never sorts, caps, load-balances, calls the engine, or reads `DateTime.now()` — it renders only what the controller already produced. Per-locale (fa/ckb/ar) widget + golden coverage proves the geometry, numerals, and section order on the real bundled fonts. The all-done / silent-resume empty states (E12-T04), the budget-feedback line (E12-T04), and the catch-up banner (E12-T05) are *slots* this task lays out but does not author.

## Implementation notes

This is a **layout/composition** task, not correctness-critical engine math; there is no frozen-vector or `glados` work here. The test-first discipline applies to the **bidi/numeral mapping helper** (the one place a defect silently corrupts page identity) and to the **section-order assertion**, which are written before their production code (see Tests).

1. **Where it lives.** The populated list is a leaf under the Today feature: `packages/features/lib/src/today/widgets/daily_session_list.dart` (the `CustomScrollView`), with `packages/features/lib/src/today/widgets/session_section.dart` (one sliver = header + its rows) and `packages/features/lib/src/today/widgets/session_skeleton.dart` (the loading state). The dumb `today_screen.dart` (E12-T01) selects among `TodayState` cases and mounts `DailySessionList` for `populated`, `SessionSkeleton` for `loading`. One primary public type per file, `lower_snake_case` filenames (eng-add-feature-module).

2. **Render only the pre-built day.** `DailySessionList` takes the controller's already-grouped, already-capped, already-ordered `TodayDay` (E12-T02) — three ordered `List<DueItem>` sections (`far`, `near`, `new_`) — by constructor. The widget **never** sorts, caps, load-balances, filters, calls `engine.buildToday`/`loadBalance`, or reads `DateTime.now()`. The injected `CalendarDate` owns "today"; if the row needs a relative-day supporting hint, it comes from the `DueItem` the controller computed, not a clock read in the widget (ui-daily-session-list rule 10; eng-define-service-boundary). A code-review note + the test in §Tests assert no engine/`DateTime.now()` symbol appears in `today/widgets/`.

3. **The three sliver sections, in fixed order.** Build a `CustomScrollView` whose slivers are, in this exact order, Far (manzil) → Near (sabqi) → New (sabaq). Each `SessionSection` is a `SliverMainAxisGroup` (or header `SliverToBoxAdapter` + `SliverList`): a quiet header in `type.title`, `space.6` above each section, rows `space.2` apart (design-system §1; ui-daily-session-list rule 2). Iterate the sections from a single ordered `const [far, near, new]` enum list so the order is structurally impossible to permute; **never** expose a sort/reorder control. An **empty section renders nothing** (no empty header) — e.g. an active memorizer with no new sabaq today shows only Far + Near.

4. **Section headers are localized term-set strings.** Header text is `l10n.sectionFarManzil` / `l10n.sectionNearSabqi` / `l10n.sectionNewSabaq` resolved through the swappable term-set the user selected (PRD §13.4; the ARB/term-set pipeline is eng-add-localized-string, owned upstream). Never a hardcoded English literal, never the engine's `FAR`/`NEAR`/`NEW` enum name. ckb's longer transcreated labels must reflow within the same insets — verified in the ckb golden.

5. **Assemble the E10 page-card row.** For each `DueItem`, the feature layer maps domain → E10's domain-blind `PageCard`: `pageNumber`/`juzNumber` → the locale-numeral bidi-isolated headline; `track` → the localized track-chip label; `decayBand` (the engine's `R` rolled to solid/holding/needs-revision, computed upstream) → the decay indicator; the optional next-due/weak hint → the supporting line. `PageCard` is a primitive-taking leaf (it never sees `Card`/`DueItem`/`Grade`); the mapping is in this feature layer (eng-add-feature-module §2). The row is **one** ≥48dp tap; the chip and decay indicator are labels inside it, never second tap targets.

6. **The tap opens the recite route — wire it, don't build it.** The row's single tap navigates to the recite/grade route via the typed `GoRoute` `page/:pageId` under the `ShellRoute` (E12-T07 owns the destination; this task passes `item.pageId`). Navigation is `context.go`/`context.push` from the View, never from the controller (eng-add-feature-module §6). Do not bypass the onboarding/core-pack redirect guard with an imperative `Navigator.push`.

7. **Page identity: locale numerals, bidi-isolated.** The headline "Page N · Juz M" is built by a small pure helper `formatPageIdentity(pageNumber, juzNumber, locale)`: format each number through `numberFormatFor(locale)` (pinned `-u-nu-arabext` for fa/ckb, `-u-nu-arab` for ar — eng-rtl-and-bidi-layout rule 7), inject into the ARB placeholder (`l10n.pageJuzLabel(page, juz)`), and isolate the mixed run with the bidi helper (`isolate`/`isolateLtr`) so it never reorders to "30 of 7". **No raw ASCII digit** is ever concatenated into the string (grep-banned). This helper is the one correctness-sensitive piece — write its tests first (§Tests).

8. **RTL by geometry, not a mode.** Every inset/alignment is `EdgeInsetsDirectional`/`AlignmentDirectional`/`Positioned.directional` — no `EdgeInsets.only(left:/right:)`, no `Alignment.centerLeft/Right` (CI-grep-banned in `features/**`). Section headers and the row's leading cluster (chip + indicator) sit at the **start** (right); the row chevron sits at the **end** (left) and mirrors automatically via the auto-mirroring `Icons.arrow_*`/`Icons.chevron_*` family. Do **not** wrap this widget in a hardcoded `Directionality` — RTL is a consequence of the locale (eng-rtl-and-bidi-layout rules 1–4). No Quran glyph appears on the row, so nothing here ever mirrors a sacred glyph.

9. **The loading skeleton.** `SessionSkeleton` is a brief `surfaceContainerLow` placeholder with the three section shapes, **no spinner theatre**, no progress percentage. It honours reduce-motion (no shimmer when reduce-motion is on — degrade to a static block; E08). It is shown only while `TodayState == loading`; it is not a perpetual indicator (do not `pumpAndSettle` it in tests — pump explicit durations).

10. **The `Semantics` container + per-row merge.** Wrap the `CustomScrollView` in one `Semantics(container: true, label: l10n.todaySemanticTitle /* "Revise today" */)`; give each section a section-role `Semantics` node carrying its localized header so a screen-reader user hears three ordered groups in their locale (design-system §1; ui-daily-session-list rule 8). Each row is wrapped (in E10's `PageCard`) in `MergeSemantics` reading one localized phrase ("Page ۲۵۳, Juz ۱۳, far-revision, needs revision"); this task only ensures the container/section roles and reading order are correct under RTL.

11. **No gamification, no shame, anywhere on this surface.** No streak, badge, score, XP, completion %, count-up of completed items, confetti, or success haptic on the list or any row; decay is the calm indicator only; nothing is ever "safe to drop"/"mastered" (PRD R3, C6, §7.12; cross-checked by domain-adab-and-religious-integrity). The all-done celebration ban is enforced by E12-T04's state, but this list must not introduce a counter (e.g. "3 of 12 done") — it shows the day, it does not score it.

12. **Pitfalls to avoid.** (a) Sorting/capping/re-balancing in the widget instead of rendering the pre-built day. (b) Reordering or making the three sections user-reorderable. (c) Rendering an empty section's header. (d) Concatenating ASCII digits into the page string, or skipping bidi isolation so the run reorders. (e) `EdgeInsets.only(left:/right:)` / `Alignment.centerLeft` slipping in. (f) Making the track chip or decay indicator a second tap target. (g) Reading `DateTime.now()` for a "next due" hint instead of using the `DueItem`'s computed value. (h) An infinite/lazy-loading feed (`ListView.builder` with a page fetcher) instead of a finite, fully-materialised list. (i) Re-authoring the `PageCard` chip/indicator internals that belong to E10.

## Acceptance criteria

- [ ] `DailySessionList`, `SessionSection`, and `SessionSkeleton` exist under `packages/features/lib/src/today/widgets/`, one primary public type per file, `lower_snake_case`; `today_screen.dart` mounts them for the `populated` and `loading` states.
- [ ] The populated list is a single **finite, fully-materialised** `CustomScrollView` that visibly ends — no infinite scroll, no lazy page-fetcher, no count-up of completed items.
- [ ] Sections render in the fixed order **Far (manzil) → Near (sabqi) → New (sabaq)**, driven by a `const` ordered enum list; there is no sort or reorder affordance; a test asserts the rendered order matches `[FAR, NEAR, NEW]`.
- [ ] Each section header is a localized term-set string (`l10n.*`, fa/ckb/ar), `type.title`, `space.6` above the section, rows `space.2` apart; no hardcoded English; no engine enum name in the UI; an empty section renders **nothing** (no orphan header).
- [ ] Each row is an E10 `PageCard` assembled from one `DueItem` by the feature-layer mapping, is one ≥48dp tap into the recite route (`page/:pageId`), renders **no Quran glyph** and **no D/S/R / percentage / "safe to drop"**, and its chip + decay indicator are labels, not second tap targets.
- [ ] Page identity renders as locale numerals (Extended Arabic-Indic for fa/ckb, Arabic-Indic for ar) via `numberFormatFor(locale)`, injected into an ARB placeholder, and the mixed run is bidi-isolated (FSI/PDI) — no raw ASCII digit is concatenated; the run never reorders ("30 of 7" is impossible).
- [ ] Layout is RTL by geometry: only `EdgeInsetsDirectional`/`AlignmentDirectional`/`Positioned.directional` and `start`/`end`; no `EdgeInsets.only(left:/right:)`, `Alignment.centerLeft/Right`, or hardcoded `Directionality` survives the `features/**` grep; leading cluster at start, chevron at end (auto-mirrored).
- [ ] The `loading` state shows the calm `surfaceContainerLow` skeleton with the three section shapes, no spinner theatre, no percentage, reduce-motion respected; it is not a perpetual/indefinite indicator.
- [ ] A single `Semantics` container announces the list as "Revise today" with section roles, so a screen-reader user hears three ordered groups in the active locale; reading order is correct under RTL.
- [ ] The View is **dumb**: `today/widgets/` contains no `engine`/`buildToday`/`loadBalance` call and no `DateTime.now()`; a test/grep asserts this. It renders only the controller's pre-built `TodayDay`.
- [ ] No gamification/shame anywhere on the list: no streak/badge/score/XP/completion %/confetti/success-haptic, and no "N of M done" counter; decay is the calm indicator only.
- [ ] The catch-up / all-done / budget-feedback **slots** are laid out (mounted by `today_screen.dart` per `TodayState`) but their content is left to E12-T04 / E12-T05 — this task does not author that copy.

## Tests

Under `packages/features/test/today/` (`flutter test`, in-memory Riverpod fakes for the controller; the throwing `HttpOverrides` offline bootstrap installed). **Write the mapping/order tests first.**

- `page_identity_format_test.dart` (write first) — unit tests for `formatPageIdentity(page, juz, locale)`: for `fa`/`ckb` the digits are Extended Arabic-Indic (`۲۵۳`, `۱۳`), for `ar` Arabic-Indic (`٢٥٣`, `١٣`); no ASCII digit appears in any output; the output contains the FSI/PDI isolate controls around the numeric run; a known-direction run uses `isolateLtr`, not bare FSI, where the helper claims direction.
- `daily_session_list_order_test.dart` (write first) — a widget test that seeds a `TodayDay` with rows in all three sections and asserts the rendered headers/rows appear in exactly `Far (manzil) → Near (sabqi) → New (sabaq)` order; a second case with an empty `new_` section asserts **no** New header is rendered.
- `daily_session_list_tap_test.dart` — tapping a row pushes the `page/:pageId` route with the row's `pageId`; tapping the chip/decay-indicator region does **not** create a second navigation (one tap target per row); the tap target measures ≥48dp (`labeledTapTargetGuideline`).
- `daily_session_list_dumb_view_test.dart` — a guard test (source grep over `packages/features/lib/src/today/widgets/`) asserting no `buildToday`/`loadBalance`/`DateTime.now()` symbol is present; and a behavioural test that mutating the input `TodayDay` is the *only* thing that changes the rendered list (the widget recomputes nothing).
- `session_skeleton_test.dart` — the `loading` state renders the `surfaceContainerLow` skeleton (no `CircularProgressIndicator`, no percentage text); under reduce-motion no shimmer animation runs; pumped with explicit durations, never `pumpAndSettle`.
- `daily_session_list_a11y_test.dart` — the list exposes one `Semantics` container labelled "Revise today" with three section-role nodes carrying the localized headers; reading order under `Directionality.rtl` is Far → Near → New; runs the E08 per-screen accessibility audit (≥48dp targets, merged row phrase).
- `daily_session_list_golden_test.dart` (`@Tags(['golden'])`) — RTL/layout goldens of the populated list **and** the loading skeleton, one per locale `fa`/`ckb`/`ar`, loading the **real** bundled UI fonts via `FontLoader` (never Ahem) with pinned DPR/size/theme, so Sorani extra letters (پ چ ژ گ ڤ ڕ ڵ ۆ ێ ە) and Persian digits are actually exercised and ckb's longer headers prove they reflow without truncation; a deuteranope/grayscale pass confirms the decay band stays readable (the band is E10's, re-checked in context).
- Offline guard: the suite runs under the shared throwing-`HttpOverrides` bootstrap; this surface opens no socket by construction — a stray connection is a named failure.

## Definition of Done

- [ ] All acceptance criteria met; all tests above green in CI on every PR; the RTL/golden test runs in the pinned golden job on real fonts.
- [ ] **Offline / no-network:** this surface opens no socket; the throwing-`HttpOverrides` guard passes; the only input is a tap.
- [ ] **No AI / no audio / no microphone:** nothing on the list records, infers, recognizes, or fetches — it renders streamed state only.
- [ ] **Quran text fidelity (R1):** the list renders **no** Quran glyph and re-typesets no āyah; page identity is metadata (number + juz), never sacred text; the muṣḥaf appears only in the recite/reader surface.
- [ ] **Never "safe to drop" / no engine internals:** no row shows D/S/R, a percentage, "mastered", or "safe to drop"; decay is the calm indicator only (PRD §7.12, R3, C6).
- [ ] **The View is dumb:** the list renders the controller's pre-built, capped, ordered day; it never sorts/caps/load-balances, never calls the engine, never reads `DateTime.now()` — proven by the guard test; reads are `StreamProvider`s upstream (E12-T02), no mutation happens in this widget.
- [ ] **RTL + fa/ckb/ar:** every string ships through the ARB term-set pipeline in fa/ckb/ar; layout is RTL by geometry (`EdgeInsetsDirectional`/`AlignmentDirectional`, no left/right, no hardcoded `Directionality`); page/juz render in locale numerals with the mixed run bidi-isolated; ckb's longer headers reflow without truncation; no hardcoded user-facing string survives.
- [ ] **Accessibility (WCAG 2.2 AA):** one `Semantics` container announces "Revise today" with section roles; each row is one merged labelled phrase; every interactive target is ≥48dp; the loading skeleton honours reduce-motion; the per-screen accessibility audit passes.
- [ ] **Sect-neutral adab:** the section term-sets and any list copy have cleared the adab review — no fiqh ruling, no app-as-authority phrasing, calm and autonomy-supportive in each locale; no gamification/shame surface (no streak/score/counter/celebration) is introduced.
- [ ] **Deterministic tests:** widget/golden tests use in-memory fakes and pinned DPR/size/fonts; no wall clock and no network; goldens are regenerated only via a reviewed `--update-goldens` run, never blessed in CI.
