## Goal

The recite/grade flow from E12-T07 gains a single labelled `Switch.adaptive` ("Teacher present"), visibly distinct and pinned in the grade band's lower region, whose only job is to change the verdict's **source** — from `self` (`sourceConfidence ≈ 0.5`) to `teacher` (`1.0`) — for the *same* Again/Hard/Good/Easy verdict and the *same* stumble lines. No second grade band is drawn. When the verdict commits with the toggle on, the E12-T06 pipeline emits a `ReviewInput` with `source = Source.teacher`; that verdict is authoritative — it sets/clears weak-flags, can graduate/demote, and overrides prior self-rated and algorithmic state — and rides the existing single write path (persist transactionally **before** republishing) into the append-only `review_log` with `source = teacher` + an optional teacher label. The resulting card row and log entry carry a **teacher-sourced marker** (shape/glyph + accessible label, never color alone). The device-local halaqa path is profile-switch → recite → sign off → next student, each against that student's own `review_log`, with no server anywhere. All copy is autonomy-supportive "for your teacher to confirm" in fa/ckb/ar, no celebration, fully offline.

## Implementation notes

This task is **correctness-critical for R6** (teacher always overrides) and **adab-critical for the copy** → the source-switch, the override, and the no-celebration / servant-to-teacher wording are pinned **test-first** (widget + unit) before the visuals are polished. It introduces **no new write method and no new normalizer**: it flips the `source` the E12-T06 pipeline already reads and rides the existing `recordReview` single write path. There is no audio, no network, no `DateTime.now()` anywhere in this path.

1. **Files** (all inside the existing `today` feature module, `packages/features/lib/src/today/`, per eng-add-feature-module — no new package, no `pubspec.yaml` edit):
   - `widgets/teacher_present_toggle.dart` — a dumb `TeacherPresentToggle`: a labelled `Switch.adaptive` ("Teacher present") taking a `bool present` + an `onChanged` callback. No state of its own, no engine, no clock, no write. ≥48dp `touch.min` row, `EdgeInsetsDirectional`, `Semantics` label + state.
   - `widgets/teacher_sourced_marker.dart` — a dumb `TeacherSourcedMarker`: shape/glyph (the E10 leaf) + a localized accessible label; rendered on a teacher-sourced card row / log entry. Never color alone.
   - Extend `recite_grade_view_model.dart` (the E12-T07 recite view-model): add a view-local `bool teacherPresent` (default `false`) and a `setTeacherPresent(bool)` mutator over the flow's view-local state — this is `StatefulWidget`/`autoDispose`-scoped UI state, **not** an app-scope store (eng-create-riverpod-store §"View-local UI state"). On commit, the view-model maps `teacherPresent` → `source = teacherPresent ? Source.teacher : Source.self_` and hands it to the **E12-T06 normalizer**; it does not compute the confidence weight (that is `kSelfConfidence`/teacher, referenced by name in the engine).
   - Wire `TeacherPresentToggle` into the E12-T07 grade band's **lower region** (below Again/Hard/Good/Easy), visibly distinct; wire `TeacherSourcedMarker` into the E10 page-card row and the (E15-owned) log surface call site via a `source == Source.teacher` flag on the read model.
2. **The toggle changes only `source` — never the grade, never the band, never reveal-on-tap.** Flipping `teacherPresent` must not redraw a second Again/Hard/Good/Easy band, must not change which verdict is selected, and must not unhide/rehide the page. It re-attributes the *same* pending verdict + the *same* `errorLines`. A widget test pins: toggling off→on with a verdict already chosen leaves the selected verdict and the revealed/stumble state byte-identical; only the source the commit would emit changes.
3. **The commit rides the existing single write path — persist before republish, append-only.** On grade-commit, the view-model calls the **existing** E12-T06 `recordReview` repository method (one `db.transaction`: append the `review_log` row, write the card update from `onReview`, **commit before** the `await` returns and before the Today stream re-emits). This task **adds no new transaction and no new DAO call** — it passes a `ReviewInput` whose `source` is `teacher`. The `review_log` row carries `source = teacher` and the optional `teacher_label` (a free-text label the teacher may enter; bidi-isolated when rendered). Never mutate persisted state in the widget; never `UPDATE`/`DELETE` a prior `review_log` row (append a new one).
4. **The teacher verdict is authoritative — assert the override, don't re-implement it.** Setting/clearing weak-flags, graduate/demote, and "teacher beats prior self-rated state" are the **engine's** `onReview` (E04/E12-T06) behavior driven by `source = teacher` + `sourceConfidence = 1.0`; this task feeds the correct `source` and asserts the outcome. A widget/integration test: a page with a prior `self` Good, signed off `teacher` Again, ends in the demoted/weak-flagged state and the **teacher** row is the latest authoritative one — and a *later* self-grade does not silently erase the teacher's standing in the log (the log keeps both rows; the engine respects the teacher verdict per R6).
5. **The teacher-sourced marker is redundantly encoded.** `TeacherSourcedMarker` renders shape/glyph (E10 leaf) **plus** a localized `Semantics`/visible label ("Signed off by your teacher" / the teacher label if present) — **never** color alone (07-components §4/§8). It appears on the resulting card row and the log entry so self and teacher inputs are never conflated; a golden proves the marker renders and a self-sourced row does **not** show it.
6. **Halaqa = profile-switch → recite → sign off → next, device-local.** The loop is: the teacher taps the **E16-owned** `activeProfileProvider` switcher to student A → opens the recite route for A's due page → recites → flips "Teacher present" → commits → the verdict appends to **A's** `review_log` → switches to student B → repeat. Per-profile providers are `family`-keyed by `ProfileId` so no data leaks between students (eng-create-riverpod-store §"family/autoDispose"). There is **no server, no dashboard, no remote surveillance**; transfer to a teacher's own device is export/import only (E16/backup). This task wires the loop over the existing switcher; it does not build the switcher.
7. **No celebration on any verdict — a teacher Good/Easy is recorded quietly.** A teacher verdict fires the **same** calm path as a self verdict and as Again: a state change + `haptic.confirm` at most, **no** confetti, chime, streak bump, badge, or success haptic (07-components §5 "none celebratory"; R3/C6). A golden + a fake-haptic test pin this for the signed-off Good/Easy case.
8. **Copy is servant-to-teacher, autonomy-supportive, in fa/ckb/ar** (eng-add-localized-string): the toggle label ("Teacher present"), the supporting line ("for your teacher to confirm"), the teacher-sourced marker label ("Recorded with your teacher's sign-off"), and the optional teacher-label field hint are `l10n.*` keys in `app_ar.arb` (base) + transcreated `fa`/`ckb` (ckb flagged pending native + scholar review). The app credits the teacher, never itself; **no** "the app says you passed" / "Approved" / fiqh ruling / "safe to drop"; the softened-Arabic-imperative / warm-respectful-Persian register per voice §8. Any teacher label is free text rendered with FSI/PDI bidi isolation.
9. **RTL by geometry** (eng-rtl-and-bidi-layout): the toggle row is `EdgeInsetsDirectional` (label at start, switch at end), mirroring correctly across fa/ckb/ar; the teacher-sourced marker sits at the logical start of the card's supporting region; ckb's longer copy reflows within the same insets without truncating a load-bearing label.
10. **Accessibility** (E08 conventions): the toggle announces its label + state ("Teacher present, off") and toggling announces the new state; the teacher-sourced marker exposes its label (the shape/glyph is decorative-with-label, not label-by-color); the toggle is ≥48dp with a visible focus ring and M3 state layers; reduce-motion is honored (no celebratory transition exists to degrade).
11. **Pitfalls to avoid:** drawing a second Again/Hard/Good/Easy band or a separate "teacher screen" (the toggle is a source switch, not a second band); flipping the toggle to change the *grade* or to bypass reveal-on-tap; adding a new write method instead of reusing `recordReview`; mutating or overwriting a prior `review_log` row (it is append-only); letting a later self-grade or the algorithm silently override a teacher verdict (R6); marking the teacher source by **color alone**; requiring an account/network or building any remote dashboard; conflating self and teacher inputs on the card/log; any celebration (confetti/chime/streak/haptic fanfare) on a teacher Good/Easy; app-as-authority copy ("Approved"/"you passed") or a fiqh ruling; introducing any microphone/recording/ASR (C2/R5); reading `DateTime.now()` in the view-model (the injected `clockProvider` owns "today").

## Acceptance criteria

- [ ] `today/widgets/teacher_present_toggle.dart` and `today/widgets/teacher_sourced_marker.dart` exist in the `features` package; each is a dumb stateless widget taking a pre-built value/callback and contains **no** engine call, **no** `DateTime.now()`, **no** direct DAO/`review_log` write, **no** audio/microphone API (verifiable by grep over the files).
- [ ] The control is a single labelled `Switch.adaptive` ("Teacher present"), pinned in the grade band's **lower region**, visibly distinct from the self-grade — and there is **no** second Again/Hard/Good/Easy band and **no** separate teacher screen.
- [ ] Flipping the toggle changes **only** the verdict's `source` (`Source.self_` → `Source.teacher`); it does not change the selected grade, the revealed/stumble state, or the reveal-on-tap masking; the confidence weight is referenced by name (`kSelfConfidence`/teacher `1.0`), never inlined.
- [ ] On grade-commit with the toggle on, the **existing** E12-T06 `recordReview` single write path runs (one `db.transaction`, persist-then-republish), appending a `review_log` row with `source = teacher` and the optional `teacher_label`; **no** new write method, **no** `UPDATE`/`DELETE` of a prior row.
- [ ] A teacher verdict is authoritative: against a prior `self` Good, a `teacher` Again demotes/weak-flags the page per the engine, the teacher row is the latest authoritative input, and a later self-grade does not silently erase it (R6, §7.12) — pinned by a test, not just asserted in prose.
- [ ] The teacher-sourced marker renders on the resulting card row and log entry with shape/glyph **and** an accessible label — **never** color alone; a self-sourced row shows **no** marker.
- [ ] The halaqa path works device-local: switch student profile (via the E16 `activeProfileProvider`) → recite → sign off → next, each appending to **that student's** `review_log`; `family`-keyed by `ProfileId` with no cross-student leakage; **no** server, dashboard, account, or socket.
- [ ] No celebration on any verdict including a teacher Good/Easy: **no** confetti, chime, streak bump, badge, or success haptic — only a state change + `haptic.confirm` at most.
- [ ] Every user-facing string (toggle label, "for your teacher to confirm", marker label, teacher-label hint) is an `l10n.*` ARB key present in fa/ckb/ar (ckb flagged pending native + scholar review); no hardcoded user-facing text; the teacher label renders bidi-isolated.
- [ ] Layout is RTL by geometry (`EdgeInsetsDirectional`/`AlignmentDirectional`, no `left`/`right`); the toggle is ≥48dp with a `Semantics` label + state and a visible focus ring; ckb's longer copy reflows without truncating a load-bearing label.
- [ ] The never-ship banned-phrase lint passes over all three locales for every string this task adds (no app-as-authority "Approved"/"you passed", no fiqh ruling, no guilt/fear/loss, no "safe to drop"/"mastered"/"done", no `!`/emoji); the adab conscience pass is recorded.
