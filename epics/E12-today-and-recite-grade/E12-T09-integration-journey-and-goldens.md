## Goal

The whole daily loop is proven to work as one assembled system on the E07 spine, and every face of it is frozen per locale. A single `integration_test` journey drives the real Drift/SQLite + asset + render stack over one continuous flow — a cold-start-seeded profile → the engine's `buildToday`-selected due pages on Today → tap a page-card → recite on a hidden page → reveal line-by-line → mark stumble lines → grade → advance to the next page → simulate a gap by advancing the injected `CalendarDate` → assert the calm re-spread catch-up plan (never a red overdue pile). A per-locale (fa/ckb/ar) widget + golden suite captures every Today state (loading / populated / all-done / catch-up / budget-feedback) and every recite/grade stage (hidden / revealing / grading / signed-off) on the **real bundled KFGQPC + UI fonts**, under a throwing `HttpOverrides` offline guard. The journey additionally proves the appended grade survives a kill-and-relaunch and that an in-flow teacher verdict beats a prior self-grade (INV-5). This task asserts the assembled behaviour; it builds no new screen, copy, or engine rule.

## Implementation notes

This task is **assembly + verification only**: no new screen, no new copy/ARB key, no new engine rule, no new mutation. It writes one apex `integration_test` journey over the real stack and a per-locale widget+golden suite, both offline by construction. Per `eng-write-dart-test`, push nothing unit-able up into the journey — the engine arithmetic, the sacred-text cap, and INV-1…INV-6 are already pinned at the cheapest tier in E04/E12-T06; the journey proves the **seams** hold (cold-start seed → engine → Today → recite → grade → persist → re-spread) and that the persisted grade survives a relaunch.

1. **Journey file** (the apex, one of the four PRD journeys): `integration_test/journey_daily_loop_test.dart`, `void main()` calling `IntegrationTestWidgetsFlutterBinding.ensureInitialized()` first. It runs the real Drift/SQLite + asset + render stack — **not** in-memory fakes (those belong to the widget tier). Stitch J1→J2→J3→J4 as one continuous flow so the journey proves the *assembled* loop, not four disconnected slices. Pump the app via the E07 composition root with the **injected `clockProvider` (`CalendarDate`)** overridden to a literal seed day and the asset loader pointed at the mock-verified core pack — never the wall clock, never a real socket.

2. **J1 — cold start → buildToday.** Seed a partial/complete-ḥāfiẓ profile by invoking the **E11 cold-start helper** (held juz + per-juz Solid/Shaky/Rusty + cycle preset) — do not hand-write card rows; the journey must exercise the real seeding path so the engine's priors are real. Open Today; assert a finite, capped, Far→Near→New `TodayState.populated` day exists (`find.byType(RevisionItem)` / the page-card row finds widgets) and that the held juz are all scheduled once early (the calibration pass). Assert the View rendered the engine's pre-built day — no engine schedule call originates in a widget (the dumb-View covenant from E12-T03 carried through the assembled stack).

3. **J2 — tap → recite → reveal → mark → grade → next.** Tap a page-card to open the full-screen recite route (E12-T07) with the page **masked**; assert the four-grade band is **disabled until ≥1 reveal**. Reveal line-by-line via taps, tap a stumble line (≥48dp hit-area → a coordinate overlay, never a re-typeset), then grade. Assert a `review_log` row is **appended** (append-only — count grew by exactly one, no UPDATE/DELETE), the card's S/D updated through `onReview`, `due_at ≤ cycle ceiling`, and the flow **advanced to the next page** with no celebration (no confetti/streak/badge widget, no success haptic on the fake haptic boundary). Drive a **dropped/added/swapped-word** stumble in one grade leg and assert the persisted grade is capped at `Grade.hard` (the sacred-text guard from E12-T06 enforced through the real pipeline) — a property test pins the universal claim; here we prove the wired path honours it once.

4. **J3 — in-flow teacher sign-off beats self (INV-5).** With the device-local halaqa path (E12-T08), grade a page **self** = Good, then flip "Teacher present" and have the teacher grade the same page **Again**. Assert: a second `review_log` row with `source = teacher` is appended (the prior self row is **not** mutated — append-only), the teacher verdict is authoritative (the page is re-surfaced / weak-flag set, not left at the self-Good state), and the teacher-sourced marker renders on the resulting card/log. This is INV-5 proven through the assembled write path, complementing E04's `glados` property.

5. **J4 — simulate a gap → calm catch-up.** Advance the injected `CalendarDate` past a multi-day gap and reopen Today. Assert `TodayState.catchUp` renders the engine's **pre-built re-spread plan** (E12-T05 banner: empathy → fact → plan → choice, "you missed N days — here is an M-day plan that still completes your cycle"), that **FAR/manzil due items are all present** (never dropped, §7.9), and that **no red overdue pile / no shame string** appears (a banned-phrase assertion over the rendered tree: no "behind", no "you'll lose", no "welcome back, N days", no "safe to drop"). Then advance the clock with **nothing to catch up** and assert the **silent-resume** path greets nothing.

6. **Kill-and-relaunch survival.** After J2's grade is appended, **tear down and re-pump** the app against the *same* on-disk Drift database (do not reset the temp DB dir between the two pumps) and assert the appended `review_log` row and the updated card state are still present — the persist-before-republish single write path proven crash-safe end-to-end (mirrors E07's spine guarantee; do not graceful-shutdown between pumps). Keep this within the one journey (a second `testWidgets` re-pumping the seeded DB), not a fifth journey.

7. **Per-locale state goldens** — `packages/features/test/today/today_states_golden_test.dart`, `@Tags(['golden'])`, `flutter test --tags=golden` on the pinned job. Load the **real KFGQPC + UI fonts via `FontLoader`** in `setUpAll`; pin DPR/`physicalSize`/theme; disable animations. For each locale in `{ar, fa, ckb}`, pump the `TodayScreen` under `Directionality(textDirection: TextDirection.rtl)` with that locale's numerals/calendar, over **in-memory Riverpod fakes** (the controller's `TodayState` set to each variant — golden tier uses fakes, not the real stack), and `await expectLater(..., matchesGoldenFile('goldens/today/<state>_<locale>.png'))` for each of: `loading`, `populated`, `allDone`, `catchUp`, `budgetFeedback`. ckb's longer copy must reflow without truncating a load-bearing label — its golden is the proof.

8. **Per-locale recite/grade stage goldens** — `packages/features/test/today/recite_grade_stage_golden_test.dart`, same harness and tags. Over the **masked E13 page surface** (the immutable glyph layer composed from the real bundled page font — never re-typeset here), capture each of: `hidden` (band disabled, "reveal to grade"), `revealing` (≥1 line revealed, a stumble overlay rectangle drawn), `grading` (band enabled, four `FilledButton`s in RTL verb order), `signedOff` (teacher source switched, teacher-sourced marker). For each of ar/fa/ckb. A dropped/shifted glyph or a moved overlay must change pixels and fail — the text-fidelity covenant (R1) frozen.

9. **Offline guard / bootstrap** (eng-write-dart-test §8). Both golden suites and the journey install the throwing `HttpOverrides` via the shared `useOfflineTestPolicy()` bootstrap so any stray socket open is a loud named failure, not a silent 400. The journey's one-time core-pack install is **mock-verified** (the asset loader is the injected fake/mock at the composition root) — no real network. No file in this task imports `package:http` / `dart:io HttpClient` / `package:dio`; the CI banned-import gate has nothing to catch here.

10. **Determinism.** Seed every fixture explicitly: literal `CalendarDate` "today", a fixed cold-start seed (held juz + confidences), a fixed fake-haptic boundary, fuzzing OFF so `buildToday` is byte-identical across runs (INV-4 holds at the engine tier; the journey relies on it for a stable plan). Never `pumpAndSettle()` the loading skeleton or any intentionally-indefinite indicator — pump explicit `motion.duration.*` durations. Goldens are regenerated with `--update-goldens` **locally** and reviewed in the diff; CI never blesses.

11. **Pitfalls to avoid:** pushing a unit-able fact (engine arithmetic, the sacred-text cap rule, an INV property) up into the journey instead of citing E04/E12-T06 (inverts the pyramid); growing the journey suite past the four PRD flows (decision-log amendment required); rendering goldens with Ahem instead of real KFGQPC fonts (defeats the fidelity test); resetting the temp Drift dir between the two relaunch pumps (would hide a persistence regression); `pumpAndSettle()` on the loading skeleton (hangs); reading `DateTime.now()` anywhere in the harness instead of the injected `CalendarDate`; letting any test but the (absent here) downloader reach the network; capturing only one locale; asserting a streak/score/completion-% in any golden; a golden that asserts nothing (no `matchesGoldenFile`); re-typesetting an āyah to draw a stumble overlay in the stage goldens.
