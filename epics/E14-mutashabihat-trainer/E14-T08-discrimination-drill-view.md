## Goal

The `DiscriminationDrillView` exists as a leaf View inside the `mutashabihat` feature, reachable from the E14-T07 trainer via the deep-linkable child route `/mutashabihat/drill/:groupId`. Given one confusable group (its members + per-member `distinguishing_word_index_json`) from the E14-T06 read model, it iterates the **whole group A→B→…** back-to-back in **one session** — no spacing, interstitial, or unrelated page between siblings — each branch **hidden → reveal-on-tap → then anchor highlight**, composing E13's reader page surface over E05's immutable `MushafPageView` + `MushafOverlayPainter`. The drill never re-typesets, reshapes, reflows, or re-orders the sacred text: each member is drawn glyph-only in its dedicated KFGQPC page font, and the anchor lands in the next task (E14-T09) as a coordinate `Rect` overlay — T08 wires the choreography and the overlay-marker seam, frames the contrast calmly as a desirable difficulty (no points/badges/streaks/confetti, no "cured"/"safe to drop"), respects OS Reduce-Motion, and stays a dumb View reading one controller through the single read path with **no** swap-logging mutation, **no** `DateTime.now()`, and **no** network/microphone/model.

## Implementation notes

This task is **choreography + composition, not correctness-critical engine math** — there is no frozen oracle to assert (the dataset/graph/engine-wiring test-first work is E14-T01/T03/T04/T05; the anchor `Rect` math is E14-T09; the real-font golden is E14-T12). The bar is: the whole-group back-to-back sequence with no spacing between siblings, the hidden→reveal→anchor order, the immutable-page composition with zero re-typeset, and the calm no-gamification register. Write the View + its widget/behaviour tests together.

1. **Files** (eng-add-feature-module §2; project-structure §3.4): add under the E14-T07 module —
   ```
   packages/features/lib/src/mutashabihat/
   ├── discrimination_drill_screen.dart        # dumb ConsumerWidget entry View (reads exactly discriminationDrillControllerProvider)
   ├── discrimination_drill_view_model.dart     # 1:1 DiscriminationDrillController extends AsyncNotifier<DiscriminationDrillState>
   ├── widgets/
   │   ├── drill_branch_view.dart               # one member: hidden cover → revealed immutable page → anchor seam
   │   └── drill_progress_strip.dart            # calm "branch i of n" group position (locale numerals, no score)
   └── mutashabihat_providers.dart              # extend with discriminationDrillControllerProvider (family by groupId)
   ```
   One primary public type per file, `lower_snake_case`. This is inside the existing `features` umbrella package — no new top-level package, no `pubspec.yaml` edit.

2. **`DiscriminationDrillState` is an immutable UI-state value** (eng-add-feature-module §3): a small `copyWith` value type holding the ordered group members and the drill's position — the member list (each `(pageNumber, lineRefs, distinguishingWordIndices)` the View needs to compose a page + place the anchor seam), the `activeIndex` into that list, and a per-branch `BranchPhase { hidden, revealed, anchored }`. It carries **no rendered strings, no glyph codes, and no reconstructed verse text** — only the geometry/ref values the E05 page view and the overlay marker consume. It is a value type in `models`/`engine`, never a mutable `Card`/`ConfusionEdge`.

3. **`DiscriminationDrillController` (the 1:1 view-model)** (flutter §1.1, §1.3): `class DiscriminationDrillController extends AsyncNotifier<DiscriminationDrillState>` exposed `family`-keyed by `groupId` in `mutashabihat_providers.dart`. `build(groupId)` reads the **E14-T06** group read model, orders the members deterministically, and publishes the initial state with `activeIndex == 0` and every branch `hidden`. The controller exposes only **local UI transitions** — `reveal()` (`hidden → revealed` on the active branch), `showAnchor()` (`revealed → anchored`), `next()` (advance `activeIndex`, reset the next branch to `hidden`) — these are **ephemeral view state, not persisted mutations**. It holds **no** swap-logging command (the `confusion_edge` write is E14-T03 / the recite flow E12; the standalone drill logs nothing here), **never navigates**, and **never reads `DateTime.now()`** (no scheduling day is computed in the drill; any "today" enters only via the injected `clockProvider` and the drill needs none).

4. **`DiscriminationDrillScreen` (the dumb View)** (flutter §1.3): a `ConsumerWidget` reading exactly `discriminationDrillControllerProvider(groupId)` and rendering `state.when(...)`:
   - `loading:` → a calm `surfaceContainerLow` skeleton, never a bare spinner / "loading…" guilt text;
   - `error:` → `RetryView(onRetry: () => ref.invalidate(...))`, calm copy;
   - `data:` → the active `DrillBranchView` plus the `DrillProgressStrip`. No `try/catch` or business logic in `build()`; no engine/repository/Drift call from the widget; tokens (`color.*`/`type.*`/`space.*`/`motion.*`) by name only; **no** confetti/streak/badge/score affordance.

5. **`DrillBranchView` — hidden → reveal-on-tap → anchor** (ui-mutashabihat-drill rules 3–4; ui-mushaf-page-view): the branch composes E13/E05's `MushafPageView` for the member's `pageNumber` (its dedicated `qpcFontFamily`, glyph-only, RTL, empty fallback). Phase order is mandatory:
   - **hidden:** the page is occluded by a calm reveal cover (the ḥāfiẓ recites the continuation from memory); the cover carries a per-locale `Semantics` "reveal" affordance and is the only interactive target.
   - **reveal-on-tap:** tapping calls `controller.reveal()`; the cover fades over `motion.duration.short` with the standard easing — no bounce, no success choreography — exposing the **immutable** page (the page is never re-rendered or re-typeset by the reveal; only the cover's opacity animates). Under OS Reduce-Motion the cover toggles without animation.
   - **anchor:** a follow-up tap (or auto after reveal per the skill's "*then* show the anchor") calls `controller.showAnchor()`, which flips the branch to `anchored` and supplies an `OverlayMarker` for the member's `distinguishingWordIndices` to the page's `MushafOverlayPainter` seam. **T08 wires the seam and passes the marker; the marker's device-`Rect` computation from `distinguishing_word_index_json` + bundled word geometry is E14-T09.** Until T09 lands, the seam is exercised in tests with a fixed fake geometry resolver injected behind the same interface — never a hardcoded pixel rect in production, never a re-typeset highlight.

6. **Whole-group, back-to-back, no spacing** (science 05 §4, §5; ui-mutashabihat-drill rules 1–2): the drill iterates **all** members of the group; there is **no code path** that renders a single sibling as an isolated review when an unpracticed twin exists, and `next()` advances directly from member *i*'s `anchored` state to member *i+1*'s `hidden` state with **no** results interstitial, spacing screen, or unrelated page between them. The `DrillProgressStrip` shows the calm position ("branch i of n") in locale numerals — informational, never a score or a streak. The terminal state after the last member is a calm "group complete" line (no celebration, no confetti, no "cured"/"safe to drop"), with a quiet return to the trainer.

7. **RTL + bidi by construction** (eng-rtl-and-bidi-layout): all chrome positioning is logical `start`/`end` (`EdgeInsetsDirectional`/`AlignmentDirectional`/`Positioned.directional`) — no physical left/right; no hardcoded `Directionality` (RTL comes from the locale). The progress numerals route through `numberFormatFor(locale)` (`-u-nu-arabext` for fa/ckb, `-u-nu-arab` for ar); any mixed-script run is FSI/PDI-isolated. The muṣḥaf glyph layer is the E05 page view's own RTL pipeline — the chrome toolkit never reaches it.

8. **Localized scaffolding only** (eng-add-localized-string): every user-facing drill string (the reveal affordance label, the "branch i of n" template, the group-complete line, retry copy, the `Semantics` labels) is an ARB key in `app_ar.arb` (template/base content language) and at minimum stubbed for `fa`/`ckb`; the **full transcreation + adab conscience-check is E14-T11**. Copy stays calm, sect-neutral, an *aid to revision*: no guilt/fear/loss, no streak/score/badge/confetti, nothing "cured"/"resolved"/"safe to drop"/"safe to stop drilling", no tafsīr, no fiqh ruling, servant-to-the-teacher.

9. **Pitfalls to avoid:** rendering a single sibling alone, or inserting any spacing / interstitial / unrelated page between siblings (the exact discrimination break); showing the page (or the anchor) before a reveal tap — the retrieval-practice inversion; re-typesetting, reshaping, reflowing, normalizing, or splitting the glyph string to "highlight" the distinguishing word (must be a coordinate overlay only); hardcoding a pixel `Rect` for the anchor instead of going through the E14-T09 geometry seam; a `softWrap`/width-wrap/`TextPainter` line-break on Quran text, or grouping by verse; adding any `fontFamilyFallback` on the sacred line; a swap-logging or any persisted mutation in the controller; `DateTime.now()` anywhere; a `StateProvider`/legacy Riverpod/`get_it`/Bloc import; a celebratory reveal (bounce/confetti/haptic fanfare) or any score/streak/badge; a hardcoded UI literal, hardcoded hex/pt/ms, or a physical left/right inset; an imperative `Navigator.push` or a controller-triggered navigation; any microphone/audio/model/HTTP reference.
