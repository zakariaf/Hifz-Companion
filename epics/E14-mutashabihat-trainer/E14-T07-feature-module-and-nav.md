## Goal

The `mutashabihat` feature is a fully-formed feature module at the canonical anatomy — a dumb `MutashabihatTrainerScreen` View, its 1:1 `MutashabihatTrainerController` view-model, an empty `widgets/` folder, and a scoped `mutashabihat_providers.dart` — registered as a `GoRoute('/mutashabihat')` under the single E07 `ShellRoute` so the **متشابهات** tab appears in the persistent RTL bottom nav in order Today · Muṣḥaf · Mutashābihāt · Progress · Settings. This task fixes the module's *shape* and its place in the app: the trainer's landing surface reads the E14-T06 group/hotspots read-model providers and renders the calm `loading`/`error`/`data` `.when` shells, so the later E14 tasks (the discrimination drill, the anchor overlay, the hotspots view, the strings, the golden) land inside a clean, audited, downward-only folder. The View stays **dumb and downward-only** (engine/data/quran/l10n/profiles): no engine logic, no swap-logging mutation, no dataset access, no `DateTime.now()`. T07 ships the module + tab + the empty `data`-state shell only; the `DiscriminationDrillView` is E14-T08, the anchor overlay E14-T09, and the `ConfusionHotspotsView` body E14-T10.

## Implementation notes

This task is **scaffolding, not correctness-critical math** — no test-first oracle is required (the dataset/graph/engine-wiring test-first work is E14-T01/T03/T04/T05; the drill golden is E14-T12). The bar is the audited module shape, the dumb-View boundary, and the verified RTL tab. Write the module + the widget/golden shell tests together.

1. **Folder anatomy** (eng-add-feature-module §2; project-structure §3.4): add `packages/features/lib/src/mutashabihat/` in the canonical four-part shape:
   ```
   lib/src/mutashabihat/
   ├── mutashabihat_trainer_screen.dart      # dumb ConsumerWidget entry View (reads exactly mutashabihatTrainerControllerProvider)
   ├── mutashabihat_trainer_view_model.dart  # 1:1 MutashabihatTrainerController extends AsyncNotifier<MutashabihatTrainerState>
   ├── widgets/                               # leaf views (calm skeleton, RetryView, calm empty line) — one type per file
   └── mutashabihat_providers.dart            # SCOPED providers (mutashabihatTrainerControllerProvider; re-exports/wraps the E14-T06 read-model providers)
   ```
   One primary public type per file, `lower_snake_case` (project-structure §4). This is a feature **folder inside the existing `features` umbrella package** — never a new top-level package, never a `pubspec.yaml` edit (that boundary is `eng-create-package`).

2. **`MutashabihatTrainerState` is an immutable UI-state value** (eng-add-feature-module §3): a small `copyWith` value type the controller publishes, holding the calm landing shape — the browsable group summaries and the hotspot summaries the E14-T06 read models expose (counts/labels only, no Quran text). In T07 it may carry an empty/`isEmpty` discriminant (the rich drill/hotspots content is filled by E14-T08/T10). It is a value type in `models`/`engine`, never a mutable `Card`/`ConfusionEdge`, and never holds rendered strings, glyph codes, or any reconstructed verse text.

3. **`MutashabihatTrainerController` (the 1:1 view-model)** (flutter §1.1, §1.3): `class MutashabihatTrainerController extends AsyncNotifier<MutashabihatTrainerState>` exposed as `final mutashabihatTrainerControllerProvider = AsyncNotifierProvider<MutashabihatTrainerController, MutashabihatTrainerState>(MutashabihatTrainerController.new);` in `mutashabihat_providers.dart`. `build()` reads the active profile via `ref.watch(activeProfileProvider)` and composes the **E14-T06** group + hotspots read-model providers (`family`-keyed by `ProfileId`) into the landing state. The controller holds **no** swap-logging or any other mutation command (the swap write is E14-T03; T07 lands no mutation), **never navigates** (it publishes state; the View/`go_router` decide what is on screen), and **never reads `DateTime.now()`** — any "today" enters only through the injected `clockProvider` (`CalendarDate`), and only where a later surface needs it.

4. **`MutashabihatTrainerScreen` (the dumb View)** (flutter §1.3; eng-add-feature-module §3): a `ConsumerWidget` reading exactly `mutashabihatTrainerControllerProvider` and rendering `state.when(...)`:
   - `loading:` → a calm `surfaceContainerLow` skeleton leaf — **never** a bare spinner or "loading…" guilt text;
   - `error:` → `RetryView(onRetry: () => ref.invalidate(mutashabihatTrainerControllerProvider))` — calm copy, never a spinner-of-shame;
   - `data:` → for T07, a calm landing/empty line (the browsable group list + drill entry are E14-T08; the hotspots list is E14-T10).
   No `try/catch` or business logic in `build()`; no engine/repository/Drift call from the widget; design tokens (`color.*`/`type.*`/`space.*`/`motion.*`) referenced **by name** only, no inlined hex/pt/ms; **no** confetti/streak/badge/score affordance anywhere.

5. **Downward-only dependency audit** (eng-add-feature-module §4): the feature imports down on `engine`, `data`, `quran`, `l10n`, `profiles` (+ `flutter_riverpod`, `go_router`) and nothing else — **no** `package:http`/`dio`, **no** `drift`/DAO import (it reads only through the E14-T06 providers / `data` repositories), **no** import of another feature's `src/`. The engine is never reached from a widget. Confirm the CI banned-import / no-network gate has nothing to catch in the new files.

6. **Tab wiring — add the RTL destination** (eng-add-feature-module §9; flutter §6; E07-T03/T04): add a `GoRoute(path: '/mutashabihat', builder: (context, state) => const MutashabihatTrainerScreen())` as a child of the **single** `ShellRoute` that hosts the persistent bottom nav, in RTL nav order Today · Muṣḥaf · **Mutashābihāt** · Progress · Settings (Mutashābihāt is the third tab from the right). Register the matching `NavigationDestination` in `HomeShell` with a **localized** label (the متشابهات term-set key) and an auto-mirroring directional icon; the bar mirrors automatically under the app-wide locale-derived `Directionality`. Do **not** add a second `ShellRoute`, redefine the E07-T03 route table, bypass the onboarding/`appReady` redirect guard, navigate from the controller, or use an imperative `Navigator.push`. The drill is a deep-linkable child route (`/mutashabihat/drill/:groupId` with `int`/`String`-typed params) added by **E14-T08** — not this task.

7. **RTL + bidi by construction** (eng-rtl-and-bidi-layout §2, §5, §7): all positioning is logical `start`/`end` (`EdgeInsetsDirectional`/`AlignmentDirectional`/`Positioned.directional`) — no `EdgeInsets.only(left:/right:)`, no `Alignment.centerLeft/Right`, no `Positioned(left:/right:)` (grep-banned in `features/**`). Do **not** wrap the screen in a hardcoded `Directionality`; RTL comes from the locale (`supportedLocales: [ar, fa, ckb]` + `GlobalWidgetsLocalizations`, set in `app/`). Any mixed-script scaffolding run is routed through the bidi-isolation helper (FSI/PDI); any number formats via `numberFormatFor(locale)` (`-u-nu-arabext` for fa/ckb, `-u-nu-arab` for ar) — but **T07 renders no number**, so no numeral path is exercised yet (it lands with the group/hotspot rows in T08/T10). The localization toolkit stops at the chrome — nothing here reaches the muṣḥaf glyph layer.

8. **Localized scaffolding only** (eng-add-feature-module §12): every user-facing scaffolding string (the screen title, the calm landing/empty line, the retry copy, the متشابهات tab label) is a key in the E09 ARB pipeline — added to `app_ar.arb` (template/base content language) and transcreated to `fa`/`ckb` (the full transcreation + adab conscience-check pass is E14-T11), `nullable-getter: false` key coverage — never a hardcoded literal. Copy stays calm, sect-neutral, and an *aid to revision*: no guilt/fear/loss, no streak/score/badge/confetti, nothing "cured"/"resolved"/"safe to drop"; no fiqh ruling, no tafsīr; servant-to-the-teacher.

9. **Pitfalls to avoid:** a second top-level package or a `pubspec.yaml` edit (wrong skill — `eng-create-package`); two primary types in one file, or naming the View `mutashabihat_trainer_view.dart`; an engine/repository/Drift call or `try/catch` business logic in `build()`; importing `drift`/a DAO or another feature's `src/`; a `StateProvider`/`StateNotifierProvider`/legacy Riverpod import, or `get_it`/Bloc; `DateTime.now()` anywhere in the feature; an un-keyed "current profile" provider, or `autoDispose` on an app-scope singleton; a hardcoded `Directionality` or any physical left/right inset; a hardcoded UI literal or a fa/ckb-incomplete ARB key; an imperative `Navigator.push`, a controller-triggered navigation, a second `ShellRoute`, or a redefined route table; the tab placed out of RTL order (Mutashābihāt must be third from the right); rendering any Quran glyph or composing the drill/anchor/hotspots body here (those are T08/T09/T10); any swap-logging mutation (that is T03); any streak/badge/score/confetti, or any microphone/audio/model/HTTP reference.
