## Goal

Every user-facing string the Mutashābihāt trainer surfaces — the **متشابهات** tab label, the trainer screen title and calm landing/empty line, the discrimination-drill choreography copy (the hidden/reveal/anchor prompts, the "next sibling" affordance, the per-locale `Semantics` labels), and the confusion-hotspots row copy ("you keep swapping these two") — is a key in the `gen_l10n` ARB pipeline, authored first in `app_ar.arb` (the template/base content language) and **transcreated** (not literally translated) into `fa` and `ckb`, with `nullable-getter: false` key coverage so a missing/typo key is a compile error. Hardcoded literals that E14-T07/T08/T10 left as `// TODO l10n` placeholders are replaced with `l10n.*` lookups. Every one of those strings then passes the **adab conscience-check** as the first gate: reverent, calm, plain-and-warm, honest; sect/madhhab-neutral with zero tafsīr; no streak/score/badge/confetti; nothing "cured"/"resolved"/"safe to drop"/"safe to stop drilling"; the trainer says drilling *reduces* swaps and stays an *aid to revision* and a servant to the teacher; and — because the dataset and the Kurdish-Sorani register are not yet scholar-signed-off — the religious/term-set wording ships flagged `needs native + scholar review` and issues no fiqh ruling.

## Implementation notes

This task is **copy + localization correctness, not engine math** — no frozen golden vector is required. The bar is: every trainer string is an ARB key with a transcreated `fa`/`ckb`/`ar` value, the bidi/numeral/plural/`Semantics` paths are correct, and every string passes the adab conscience-check first. Author the ARB entries and the conscience-check record together; run the localization & accessibility gate (engineering 12 §8) before calling it done.

1. **Inventory the trainer's strings first.** Sweep the three sibling surfaces in `packages/features/lib/src/mutashabihat/` — `mutashabihat_trainer_screen.dart` + its `widgets/` (E14-T07), the `DiscriminationDrillView` + `widgets/` (E14-T08), the `ConfusionHotspotsView` + `widgets/` (E14-T10) — for every `Text(...)`, `tooltip:`, `semanticsLabel:`, and `Semantics(label:/hint:)` that a user sees or hears, plus the متشابهات `NavigationDestination` label in `HomeShell`. Each becomes (or already is) an `l10n.*` key. No user-facing literal may survive the `features/**` hardcoded-string grep.

2. **Author every key in `app_ar.arb` first** (eng-add-localized-string §1; engineering 12 §1): `lib/l10n/app_ar.arb` is the `l10n.yaml` `template-arb-file` and the base content language. Each key carries an `@`-metadata block with a `description` (translator + scholar context, e.g. "Drill prompt: invites the user to recall the next sibling's continuation from memory — aid to revision, never a command") and `placeholders` where interpolated. Suggested key namespace `mutashabihat*` / `drill*` / `hotspot*`, e.g. `mutashabihatTabLabel`, `mutashabihatTrainerTitle`, `drillRevealPrompt`, `drillAnchorHint`, `drillNextSiblingLabel`, `hotspotsEmptyLine`, `hotspotPairRowSemantics`.

3. **Transcreate `fa` and `ckb` against `ar`, never literal-translate** (eng-add-localized-string §10; voice §8): set register deliberately — **fa** respectful-warm (*šomā*/honorific, *taʿārof*-aware), **ar** number/gender-appropriate with **imperatives softened to statements of readiness** (a bare drill command verb reads curt — prefer "your next sibling is ready to recall" over "recite the next one"), **ckb** register *and* religious vocabulary by native reviewers, defaults provisional. `app_fa.arb`, `app_ckb.arb`, `app_ar.arb` all carry every key (`nullable-getter: false` makes a gap a compile error).

4. **`ckb` is canonical-encoded and flagged provisional** (eng-add-localized-string §9): author Sorani with U+06D5 (ە) for AE and U+06A9 (ک) for kaf; no stray U+200C (ZWNJ), no Teh-Marbuta-for-AE hack (a lint rejects these pre-build). Every `ckb` trainer value and the متشابهات term-set carry `needs native + scholar review` in the `@description` and stay an *aid to revision* until sign-off; the bundled UI font's Sorani coverage (پ چ ژ گ ڤ ڕ ڵ ۆ ێ ە) is CI-verified.

5. **Interpolation is one full ICU message with `{placeholder}` slots — never concatenate** (eng-add-localized-string §4): any drill/hotspot string that embeds a value (a page reference, a juz/ayah-page index in a `Semantics` label, a surah name beside RTL copy) is a single ICU message; the opposite-direction run stays a placeholder so the generated `intl` formatter can isolate it. No `"… " + n.toString()`, no glued localized fragments.

6. **Numerals via `numberFormatFor(locale)`; bidi via the one FSI/PDI helper** (eng-add-localized-string §6, §7; engineering 12 §4, §5): any number a trainer string carries (a page index in a hotspot row, a sibling position in a `Semantics` label) is formatted via `numberFormatFor(locale)` — Extended Arabic-Indic (`-u-nu-arabext`) for fa/ckb, Arabic-Indic (`-u-nu-arab`) for ar — then injected as a placeholder, never hand-formatted ASCII. Every mixed-script run goes through the `l10n/bidi.dart` helper (`isolate` for unknown direction, `isolateLtr`/`isolateRtl` for known). **Critically:** no ayah number, juz/sajda marker, or page *glyph* runs through this path — the muṣḥaf glyph layer is owned by `domain-mushaf-text-integrity`, and the anchor highlight is a coordinate overlay (engineering 08 §4), not a localized string.

7. **Any count-bearing string is an ICU `plural` with Arabic's six categories** (eng-add-localized-string §5; engineering 12 §6): if a hotspot or drill string counts anything user-facing (e.g. "N pairs to discriminate", "N siblings in this group"), it is an ICU `plural` with all six CLDR categories (`zero`/`one`/`two`/`few`/`many`/`other`) and the count is locale-numeral-formatted before placement — never a `count == 1 ? …` ternary, never an omitted `few`/`many`. If no trainer string is count-bearing, none is added (resist inventing a counter — that risks a scoreboard).

8. **Term-set strings are an ICU `select` / region-override, never an edited base string** (eng-add-localized-string §8): if a trainer surface shows a sabaq/sabqi/manzil track label or a traditional grade verb, it reads the active term-set via the region-override `select`, not a hard-coded "Manzil"/"Sabqi". The متشابهات tab label is the traditional term; do not invent a regional vocabulary — defer to the existing term-set keys.

9. **The adab conscience-check is the first gate, on every string** (domain-adab-and-religious-integrity; voice §2, §6, §9): before clarity or brevity, each `fa`/`ckb`/`ar` value passes the four voice attributes and the never-ship list. Specifically for this trainer: drill/hotspot copy says drilling **reduces** swaps; it never says a pair is "cured"/"resolved"/"mastered"/"done"/"safe to drop"/"safe to stop drilling"; it carries **no** points/badges/streaks/confetti/score and no exclamation mark or emoji; the hotspots line is calm actionable framing ("you keep swapping these two"), never "you keep failing" or a guilt grid; it **commands nothing** ("you must/should/don't" banned — phrase as invitation/readiness); it issues **no** fiqh ruling and bundles **no** tafsīr/translation to "explain" a difference (the drill shows wording divergence objectively); it credits the teacher and stays a servant to the *talaqqī*; and the dataset-adjacent register stays an *aid to revision* flagged pending scholarly sign-off.

10. **The engine stays locale-free** (eng-add-localized-string §3): no `AppLocalizations`/`intl` import enters `/engine` or the dataset/graph layer; the trainer's controller/View map the engine's enums/value-types to `l10n.*` at the feature boundary. T11 adds no string to `engine`/`data`/`quran`.

11. **Pitfalls to avoid:** a surviving hardcoded UI literal in any of the three surfaces; hand-editing `fa`/`ckb` first or literal-translating so register drifts; an omitted Arabic plural category; a `count == 1 ? …` ternary or a `"$n "` splice; ASCII digits, or the Arabic-Indic block for fa/ckb (or arabext for ar); a raw opposite-direction concatenation instead of the FSI/PDI helper; running an **ayah number** or **glyph** through `NumberFormat`/`bidi.dart` (sacred-text boundary breach — that path is `domain-mushaf-text-integrity`); a stray ZWNJ / Teh-Marbuta in `ckb`; locking `ckb`/term-set/dataset wording before native + scholar review; any "cured"/"resolved"/"safe to drop"/"mastered"/"done" phrasing; any streak/score/badge/confetti or guilt/fear/loss/command/transaction word; any tafsīr/translation/fiqh ruling; a CLAIMS id or citation invented inside a trainer string; a string added to `/engine`.

## Acceptance criteria

- [ ] Every user-facing string in the `mutashabihat` feature (T07 trainer screen + landing/empty/retry, T08 drill reveal/anchor/next-sibling prompts + `Semantics` labels, T10 hotspots rows + empty line, the متشابهات tab label) resolves through `AppLocalizations` (`l10n.*`); the `features/**` hardcoded-string grep finds **zero** literals in the feature.
- [ ] Every key exists in `app_ar.arb` (template, with `@description`) **and** is transcreated in `app_fa.arb` and `app_ckb.arb`; `nullable-getter: false` key coverage passes (a missing/typo key is a compile error); `fa`/`ckb`/`ar` values are transcreations diffed against `ar`, not literal translations.
- [ ] `fa` register is respectful-warm (*šomā*/honorific); `ar` imperatives are softened to statements of readiness; `ckb` is canonical-encoded (U+06D5 ە, U+06A9 ک; no ZWNJ, no Teh-Marbuta-for-AE) and every `ckb`/term-set value carries `needs native + scholar review` in its `@description`.
- [ ] Any interpolated string is one full ICU message with `{placeholder}` slots (no concatenation); any count-bearing string is an ICU `plural` with all six Arabic CLDR categories (`zero`/`one`/`two`/`few`/`many`/`other`); any numeral comes from `numberFormatFor(locale)` (arabext for fa/ckb, arab for ar) injected as a placeholder — no ASCII digits, no wrong digit block.
- [ ] Every mixed-script run passes through the `l10n/bidi.dart` FSI/PDI helper; **no** ayah number, juz/sajda marker, page glyph, or reconstructed verse text is run through `NumberFormat`/`bidi.dart` (the sacred-text boundary holds; the anchor is a coordinate overlay, not a string).
- [ ] Every trainer/drill/hotspot string passes the adab conscience-check: it is reverent/calm/plain-and-warm/honest; it says drilling *reduces* swaps; it contains **no** "cured"/"resolved"/"mastered"/"done"/"safe to drop"/"safe to stop drilling", **no** streak/score/badge/confetti, **no** guilt/fear/loss, **no** command ("must/should/don't") or transaction word, **no** exclamation mark or emoji, **no** tafsīr/translation, and **no** fiqh ruling; the hotspots line is calm actionable info; the teacher is credited / outranks the machine.
- [ ] T11 surfaces no on-screen number or methodology claim and invents no CLAIMS id or citation; the trainer's behaviour maps to the already-graded rows C-026/C-027/C-028/C-029/C-030/C-045 rendered on the science screen (E19), not inside a trainer string.
- [ ] No `AppLocalizations`/`intl` import enters `/engine`; no string is added to the engine/dataset/graph layer; the localization toolkit reaches no muṣḥaf glyph.
- [ ] Every changed Dart/ARB file passes the analyzer/lint config and `dart format`; touched Dart files carry the REUSE SPDX header (`GPL-3.0-or-later`) and `///` docs on public APIs; the religious/term-set wording is recorded as flagged-pending (or cleared) in the per-locale review record.

## Tests

`packages/features/test/mutashabihat/` and the localization gate steps — `flutter test` in the fast CI job, plus the ARB-completeness and grep gates from engineering 12 §8. The shared bootstrap installs the throwing `HttpOverrides` (the feature touches no socket); widget tests pump under `Locale('fa')`, `Locale.fromSubtags(languageCode: 'ckb')`, and `Locale('ar')` with the real bundled UI fonts (never `Ahem`).

- `mutashabihat_l10n_coverage_test.dart` (unit, deterministic): assert every `mutashabihat*`/`drill*`/`hotspot*` key present in `app_ar.arb` is also present in `app_fa.arb` and `app_ckb.arb` (no missing/typo key), and that each value is non-empty — pinning `nullable-getter: false` coverage at test time as well as compile time.
- `mutashabihat_l10n_plural_test.dart` (unit): for every count-bearing `mutashabihat*`/`drill*`/`hotspot*` ICU `plural` message in `app_ar.arb`, assert all six Arabic CLDR categories (`zero`/`one`/`two`/`few`/`many`/`other`) are defined — a missing category fails (the Arabic-plural completeness check). If no trainer string is count-bearing, the test asserts that and documents why (no counter = no scoreboard).
- `mutashabihat_l10n_widget_test.dart` (widget): pump the trainer screen, a one-group `DiscriminationDrillView`, and a populated `ConfusionHotspotsView` under fa / ckb / ar with read-model providers faked via `overrideWith`; assert the resolved drill-reveal prompt, anchor hint, next-sibling label, and the hotspot row + `Semantics` labels render the localized value (not a key string), exercise the locale-numeral path (no ASCII digit in any rendered number), and that no mixed-script run reorders (the bidi helper applied).
- `mutashabihat_l10n_goldens_test.dart` (golden): per-locale `matchesGoldenFile` of the drill prompt strip and the hotspots row in **fa, ckb, ar** on the real bundled UI fonts with a pinned runner — proving the Sorani extra letters / Persian digits path renders and that copy **wraps rather than truncates** (ckb's longer strings reflow). (The full whole-group reveal→anchor drill muṣḥaf golden is E14-T12; this golden covers only the localized chrome strip.)
- `mutashabihat_adab_lint_test.dart` (unit, deterministic): scan the resolved `fa`/`ckb`/`ar` values of every trainer/drill/hotspot key against the banned-phrase list — no "cured"/"resolved"/"mastered"/"done"/"safe to drop"/"safe to stop drilling", no streak/score/badge/confetti token, no guilt/fear/loss or command/transaction word, no exclamation mark or emoji, no tafsīr/translation/fiqh-ruling marker — in **all three** locales; a hit is a named failure.
- **Offline guard:** the suite runs under the throwing `HttpOverrides`; a stray connection attempt is a named failure; the no-network / banned-import / `features/**` hardcoded-string / physical-side / ASCII-digit / canonical-`ckb` gates stay green over the new ARB and Dart files.
