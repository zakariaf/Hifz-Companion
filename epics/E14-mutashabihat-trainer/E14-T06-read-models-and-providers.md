## Goal

The Mutashābihāt trainer reads two **reactive, read-only** projections of Drift through scoped Riverpod providers: a **group read model** (`MutashabihGroupView` — a group's `type` + `note_key` plus its members, each carrying its `ayah_id` and the validated `distinguishing_word_index_json`, assembled from the read-only `mutashabih_group`/`mutashabih_member` reference tables of E14-T01) and a **confusion-hotspots read model** (`ConfusionHotspot` rows — the active profile's `confusion_edge` pairs ranked by `weight` descending, then `last_confused_at`, exposing the `(ayah_a, ayah_b)` pair, its `weight`, and `last_confused_at` from E14-T02). Both are `StreamProvider`s over Drift queries — pure derivations that re-emit when (and only when) the committed write path changes the underlying rows; neither holds a second cache, recomputes anything the engine owns, or mutates persisted state. The hotspots stream is `family`-keyed by `ProfileId` (gated by the active-profile `Notifier`) and `autoDispose`d; the DAOs they read are injected as `Provider`s through the data layer, never reached as globals — and no shell code on this path reads `DateTime.now()`.

## Implementation notes

**TEST-FIRST:** write the read-model suite in `## Tests` below before the providers/view types. The group-assembly case, the hotspots ranking/ordering case, the per-profile isolation case, the "re-emits after a committed write" case, and the "no write/mutation surface and no clock read" assertion must exist and **fail** before the projection code is written.

1. **Immutable read-model value types live in `models`, not `data` or `features`.** Add small `@immutable` value types the View binds to — no Drift symbol crosses the boundary (`docs/engineering/04` §1.1; the DAO returns these, never a raw Drift `*Data` row to the shell):
   - `MutashabihMemberView { AyahId ayahId; List<int> distinguishingWordIndices; }` (the JSON parsed once into an in-range index list — T01 already validated range; this view only deserializes).
   - `MutashabihGroupView { MutashabihGroupId groupId; MutashabihType type; String noteKey; List<MutashabihMemberView> members; }` — `noteKey` is a **localizable resource key**, not prose; the view carries the key verbatim and **no reconstructed verse text, no glyph codes, no tafsīr/translation**.
   - `ConfusionHotspot { ProfileId profileId; AyahId ayahA; AyahId ayahB; double weight; CalendarDate lastConfusedAt; }` — reuses E14-T02's `ConfusionEdge` fields verbatim (canonical `ayahA < ayahB`); the View renders, it never re-orders. Each carries `copyWith`/value equality so the stream's distinct-emission and the goldens are stable.
2. **The group read model is a read-only `StreamProvider` over the reference DAO.** In the feature's scoped providers file (`packages/features/lib/src/mutashabihat/mutashabihat_providers.dart`), expose `mutashabihGroupsProvider` (all groups, for the browse list) and a `family`-keyed `mutashabihGroupProvider(MutashabihGroupId)` (one group's members, for the drill) as `StreamProvider`s that `watch` the **query-only** `MutashabihDao` (T01) and map each Drift row to the `models` view types. The reference tables are read-only and bundled-once, so this stream is effectively static after install — still expose it as a stream for one uniform read shape (and so the View never holds a snapshot the DB could outlive). Assemble `members` per group (a join/two-query map), preserving the dataset's stable member order so the drill iterates A→B→… deterministically.
3. **The hotspots read model is a `family`+`autoDispose` `StreamProvider` keyed by `ProfileId`.** `confusionHotspotsProvider(ProfileId)` watches the `ConfusionEdgeDao`'s `watchEdgesForProfile(profileId)` Drift query (T02) and maps each `ConfusionEdge` row to a `ConfusionHotspot`, **ranked `weight` DESC, then `last_confused_at` DESC** as a tiebreak (most-confused, most-recent first). Key it with `family` by the stable equatable `ProfileId` and `autoDispose` it on unmount; the active screen passes the id from the active-profile `Notifier` gate (`activeProfileProvider`) — there is **no** global mutable "current profile" and a profile switch yields the right rows with **zero leakage between students**. Ordering belongs in the read model (deterministic), not the View.
4. **The DAOs are injected, not reached as globals (service boundary).** The two DAOs reach the shell as Riverpod `Provider<MutashabihDao>` / `Provider<ConfusionEdgeDao>` over their interface, wired **once** in `main`'s `ProviderScope` overrides off the live `AppDatabase`; the un-overridden placeholder **throws** when read (a forgotten wiring is a loud startup failure, never silent empty data). Tests install a `NativeDatabase.memory()`-backed DAO (or a plain in-memory DAO double) via `overrideWith` — no mock framework. The `watch*` methods that return streams live in `data` (T01/T02 own the SQL); if a `watchEdgesForProfile`/`watchAllGroups` stream method does not yet exist on the DAO, add the **read-only** `Stream<...>` query there (no `insert`/`update`/`delete` is added on this path).
5. **No clock, no write, no engine math on this path.** This is a pure read derivation: it reads no `DateTime.now()`/`Calendar.current`/`TimeZone.current` (the hotspot's `last_confused_at` is the **stored** `CalendarDate` from T03's injected `today`, displayed verbatim — never compared against a live "now" here; any "X days ago" relative phrasing is the View/`l10n` layer's job in E14-T10/T11 and still reads the injected clock, not `DateTime.now()`). It calls no `onReview`/`expandMutashabihat`/FSRS symbol (a grep over this file finds none) and computes no `D`/`S`/`R`/juz-health. It exposes **no** mutation method — every write to either graph is the single write path elsewhere (T01 build-DB load; T03 `logSwap`).
6. **The provider is a thin wire; the View is dumb.** The providers map rows to immutable views and rank; they hold no business logic, no navigation, and no hard-coded user-facing string. The browse-list / drill / hotspots Views (E14-T07/T08/T10) `ref.watch` these providers and render — they do not query Drift, do not parse JSON, and do not sort. `note_key` and any row copy are resolved to transcreated fa/ckb/ar strings by `l10n` at the View layer (E14-T11), never embedded in the read model.
7. **Pitfalls to avoid:**
   - **Holding a second cache** (a `Notifier` mirror of the stream, a `List` field updated by hand) — the committed Drift stream is the single source of truth; the View rebuilds from a re-emit, there is no second place to update.
   - **A mutation method on this path** (an `insert`/`update`/a "mark resolved"/"clear hotspot" call) — read-only; both graphs are written only through the single write path elsewhere. A "safe to drop"/"cured"/"resolved" flag is forbidden outright (adab).
   - **`DateTime.now()` to rank or to compute "days ago"** in the read model — ranking is by stored `weight`/`last_confused_at`; relative time is the View+injected-clock layer.
   - **Re-ordering the canonical pair** — `(ayah_a, ayah_b)` arrive already `ayah_a < ayah_b` (T02's `CHECK`); the read model preserves them, the View decides display order of the two siblings without rewriting the row.
   - **Reconstructing verse text / a tafsīr gloss into the view type** — the group view carries `ayah_id` + word **indices** + a `note_key` only; the glyph page and anchor overlay are E05/E14-T08/T09, not this model.
   - **Keying the hotspots `family` on a mutable object** (a `Card`, a `Profile`) instead of the stable `ProfileId`, or forgetting `autoDispose` (a per-student memory leak on low-end Android), or `autoDispose`-ing the app-scope DAO/DB.
   - **Importing `drift` into `features`/`models`** — the DAO is reached only through its injected `Provider<Interface>`; no Drift symbol crosses out of `data`.

## Acceptance criteria

- [ ] `MutashabihGroupView`, `MutashabihMemberView`, and `ConfusionHotspot` are `@immutable` value types in `models` with value equality + `copyWith`; they carry only `ayah_id`/word-indices/`type`/`note_key` (group) and the `confusion_edge` fields (hotspot) — **no** reconstructed verse text, glyph codes, tafsīr/translation, or derived health, and **no** Drift symbol.
- [ ] `mutashabihGroupsProvider` and `mutashabihGroupProvider(groupId)` are `StreamProvider`s over the **query-only** reference DAO; each `MutashabihGroupView` carries its whole group's members in stable dataset order (group-not-node), and parses `distinguishing_word_index_json` into an in-range `List<int>`.
- [ ] `confusionHotspotsProvider(ProfileId)` is a `family`+`autoDispose` `StreamProvider` over `watchEdgesForProfile`, mapping rows to `ConfusionHotspot` **ranked `weight` DESC then `last_confused_at` DESC**; the active id comes from the active-profile `Notifier` gate.
- [ ] The two DAOs are injected as Riverpod `Provider<Interface>`s wired once in the composition root; the un-overridden placeholder **throws**; no provider/view on this path imports `drift`, opens a socket, or reaches a global.
- [ ] The whole read path contains **no** mutation/`insert`/`update`/`delete`/"mark resolved" method and **no** `DateTime.now()`/`Calendar.current`/`TimeZone.current`; a grep over the providers file finds no FSRS/`onReview`/`expandMutashabihat` symbol.
- [ ] A committed write to `confusion_edge` (T03) re-emits `confusionHotspotsProvider` with the new ranking; a committed reference-DB build (T01) is reflected in the group providers — the stream is the single source of truth (no second cache).
- [ ] Per-profile isolation: `confusionHotspotsProvider(A)` and `(B)` return disjoint, correctly-keyed rows; switching the active profile yields B's hotspots with zero leakage from A.
- [ ] Every public declaration carries a `///` doc comment; the files carry the REUSE SPDX header and pass `dart format`/analyzer; types are immutable, error/empty states are values the View renders calmly (no guilt copy at this layer).

## Tests

`packages/features/test/mutashabihat/mutashabihat_read_models_test.dart` (mirrors the providers' feature) with `ProviderContainer` + `overrideWith`, and `packages/data/test/...` stream-query coverage where a new `watch*` method is added — `package:test`/`flutter_test`, an in-memory `NativeDatabase.memory()` DB (or a plain in-memory DAO double) installed via `overrideWith`, a `FixedClock`/literal `CalendarDate` everywhere (no host wall clock), the seeded fixtures from T01 (a few `surah`/`ayah` + `mutashabih_*` rows) and T02 (`confusion_edge` rows for two distinct `ProfileId`s). The shared throwing-`HttpOverrides` offline bootstrap stays installed (this path opens no socket). Required cases, written **FIRST**:

- **Group assembly:** a seeded group with ≥ 2 members yields one `MutashabihGroupView` carrying the whole group's members in stable dataset order, with `type`/`note_key` verbatim and each `distinguishingWordIndices` parsed to the exact in-range `List<int>` stored by T01.
- **Hotspots ranking:** seeded `confusion_edge` rows with mixed `weight`/`last_confused_at` emit `ConfusionHotspot`s ordered `weight` DESC then `last_confused_at` DESC; the `(ayah_a, ayah_b)` canonical ordering is preserved (never re-ordered), `weight` and `last_confused_at` round-trip verbatim.
- **Per-profile isolation (`family` keying):** `confusionHotspotsProvider(A)` returns only A's edges and `(B)` only B's; the streams are keyed by `ProfileId` with no cross-profile leakage; the active-profile `Notifier` switch from A→B re-emits with B's rows.
- **Re-emits after a committed write (single source of truth):** with the providers watched, committing a `logSwap` (T03) through the write path re-emits `confusionHotspotsProvider` with the strengthened pair re-ranked — proving the stream, not a second cache, drives the View; no manual republish exists.
- **Empty / first-run:** a profile with no logged swaps emits an empty hotspot list (a calm value the View renders as the empty state, E14-T10) — not an error, not null.
- **No mutation / no clock surface:** a compile-time/grep assertion in the test doc + an `expect` that the providers expose no write method and the providers file contains no `DateTime.now()`/`Calendar.current`/`onReview`/`expandMutashabihat`/`drift` symbol.
- **`autoDispose`:** the hotspots `family` provider disposes when its last listener unmounts (no retained per-student state); the app-scope DAO/DB providers are **not** `autoDispose`d.
- **Offline guard:** the throwing-`HttpOverrides` guard fails the suite on any network attempt from the read path.

All cases run under `dart test`/`flutter test` in CI on every PR; the offline guard fails the build on any network attempt.

## Definition of Done

- [ ] All acceptance criteria met; the test-first read-model suite is green locally and in CI on every PR.
- [ ] **Offline / no-network:** the read path opens no socket and imports no networking symbol; it reads bundled/locally-persisted Drift rows only; the throwing-`HttpOverrides` guard and E01's banned-import/no-network gates stay green.
- [ ] **No AI / no microphone / no inference:** both projections are plain bookkeeping reads — the group set is the bundled reviewed dataset (T01) and the hotspots are the user's own logged swaps (T03); nothing infers "similar verses", ranks by a model, trains on data, or captures audio (PRD C2, R5; science 05 §7).
- [ ] **Quran text fidelity (existential):** the group read model carries `ayah_id` + word **indices** + `type` + `note_key` only — **never** reconstructed/normalized verse text, glyph codes, tafsīr, or translation; the immutable glyph page and anchor overlay are E05/E14-T08/T09, not this model; the reference rows stay read-only (PRD R1, §11.2; science 05 §6).
- [ ] **Single write path / read-only derivation:** the read models are `StreamProvider`s over Drift that re-emit after a committed write; this task adds **no** mutation surface and **no** second cache; every write to either graph is the single write path elsewhere (T01 load, T03 `logSwap`).
- [ ] **Deterministic / no clock in shell:** no `DateTime.now()`/`Calendar.current`/`TimeZone.current` anywhere on this path; `last_confused_at` is the stored `CalendarDate` displayed verbatim; ranking is by stored `weight`/`last_confused_at`; identical rows → identical ordered output; the engine is never reached.
- [ ] **RTL + fa/ckb/ar strings / accessibility:** N/A by construction at the read-model layer — the providers hold value types and render no user-facing string; `note_key` and any hotspot/row copy are resolved to transcreated, sect-neutral, calm fa/ckb/ar strings (locale numerals, bidi-isolated) by `l10n` at the View layer (E14-T10/T11), never hard-coded here.
- [ ] **Sect-neutral adab / nothing safe to drop:** the hotspots model surfaces actionable info ranked to drill back-to-back — it carries **no** "cured"/"resolved"/"safe to drop" flag, **no** score/streak/badge/leaderboard field, and ranks to help, never to shame (PRD R3, R6; science 05 §8; CLAIMS C-029, C-045).
- [ ] **No unsourced number:** the read models surface no user-facing number or claim of their own (`weight` is an internal ranking key the View labels via E14-T10/T11, sourced to C-029/C-045); **no** CLAIMS id is invented.
- [ ] Every Dart file carries the REUSE SPDX header and `///` docs on public APIs; modern Riverpod only (no legacy/`get_it`/Bloc/`provider`); `family`+`autoDispose` per-profile keying; immutable value types; no Drift symbol crosses out of `data`; passes the analyzer/lint config and `dart format`.
