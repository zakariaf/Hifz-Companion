## Goal

`ConfusionHotspotsView` is the personal hotspots body inside the `mutashabihat` feature module (E14-T07): a calm, scrollable list of the active profile's own most-confused pairs — "you keep swapping these two" — read **read-only** from `confusionHotspotsProvider(ProfileId)` (E14-T06), each row naming its pair `(ayah_a, ayah_b)` by page + juz in locale numerals and tapping into that pair's discrimination drill (`/mutashabihat/drill/:groupId`, E14-T08). Each `ConfusionHotspot` row is a page-card-shaped, label-only tile (no glyphs, no `weight` number, no decay gauge) wrapped in `MergeSemantics`; the list shows a calm empty state when the profile has logged no swaps. It is **actionable information, never a scoreboard**: no points, streaks, leaderboard, "fix your weaknesses" grid, or "cured"/"resolved"/"safe to drop" label, and copy frames drilling as *reducing* swaps, a servant to the teacher. The View is dumb and downward-only — it watches the read model and renders; it mutates nothing, queries no Drift, infers nothing, and reads no `DateTime.now()`.

## Implementation notes

This task is **a calm read-only list View, not correctness-critical engine math** — no test-first oracle table is required (the dataset/graph/engine-wiring test-first work is E14-T01/T03/T04/T05; the ranking is already proven in E14-T06). The bar is the audited dumb-View boundary, the page-card-shaped row, the no-scoreboard adab, and the per-locale RTL goldens. Write the View + its widget/golden tests together.

1. **File placement** (eng-add-feature-module anatomy; ui-mutashabihat-drill rule 10): the body lives at `packages/features/lib/src/mutashabihat/widgets/confusion_hotspots_view.dart` (a leaf `ConsumerWidget` under the T07 module's `widgets/` folder), and a private row leaf `_ConfusionHotspotRow` either in the same file or a sibling `widgets/confusion_hotspot_row.dart` (one primary public type per file, `lower_snake_case`). `MutashabihatTrainerScreen` (T07) composes this View into its `data:` state. **No** new top-level package, **no** `pubspec.yaml` edit.

2. **Read the pre-ranked read model — never sort or query here** (ui-mutashabihat-drill rule 10; E14-T06): the View `ref.watch(confusionHotspotsProvider(profileId))` where `profileId` comes from `ref.watch(activeProfileProvider)` (the single active-profile gate; never a global mutable "current profile"). It renders the already-ranked `List<ConfusionHotspot>` **as given** — it does **not** re-sort, re-rank, parse JSON, or open a Drift query. Surface the provider's `loading`/`error`/`data` via `.when` (or read the `AsyncValue` the T07 controller already merged): `loading` → a calm `surfaceContainerLow` skeleton, never a bare spinner; `error` → a calm `RetryView`, never a spinner-of-shame.

3. **Empty state is calm, never guilt** (ui-empty-state register; science 05 §8): a profile with **no** logged swaps yields an empty `List<ConfusionHotspot>` from T06 — render a quiet, welcoming line (e.g. a transcreated "no confusion pairs logged yet — they appear here as you revise"), **never** "you haven't logged anything", a sad-face, an alarm, or a "0 hotspots" score. An empty list is the *good* state (nothing is being swapped), and the copy must read that way; it is not an error and not null. The literal is an ARB key (E14-T11), not hard-coded.

4. **The row is a page-card-shaped, label-only pair tile** (ui-page-card rules 1–3, 9, 10; deviations noted): build `_ConfusionHotspotRow` as a flat `ListTile`/`Card` at elevation Level 0–1, **no decorative shadow, no brand tint**. Unlike a Today page card it names a **pair**: the headline is two page identities — `"Page ۲۵۳ · Juz ۱۳ ↔ Page ۲۸۷ · Juz ۱۵"` (or a two-line stacked layout), each "Page N · Juz M" run in `type.body`, locale numerals, **bidi-isolated (FSI/PDI)** so the mixed Latin-glyph/numeral run never reorders. The leading slot carries a small neutral, **non-alarm** mutashābihāt/contrast affordance (a glyph + nothing else) — **no track chip, no decay indicator, no `R`/`weight`/percentage**. The trailing slot is a directional chevron into the drill. The row renders **no Quran glyph** (the muṣḥaf lives only in the reader/drill).

5. **The whole row is one tap into the pair's drill — group-not-node** (ui-mutashabihat-drill rules 6, 1; ui-page-card rule 9): the entire row is a single ≥`touch.min` (≥48dp) tap target that navigates `context.go('/mutashabihat/drill/:groupId')` for the **group** this confusion pair belongs to (E14-T08 owns that route). The tap opens the **whole-group** discrimination drill (the contrasting pair / full group), **never** a lone-sibling review — resolve the `groupId` from the pair via the E14-T06 group read model (the `ConfusionHotspot` carries `(ayahA, ayahB)`; the pair maps to its `MutashabihGroupView`). No sub-element of the row (no glyph, no chevron) is a second tap target. The View **never** `Navigator.push`es imperatively and the controller never navigates — the View calls `go_router`.

6. **`weight` and `last_confused_at` are ranking keys, not user-facing numbers** (PRD §10.2; ui-page-card rule 7; CLAIMS — no unsourced number): the rows arrive ranked by `weight`; the View **never prints `weight`**, a percentage, a count, or a "confusion score". If a relative-recency line is shown at all (optional, calm — "last mixed up ~۳ days ago"), it is a `type.caption` / `color.text.secondary` supporting line formatted from the **stored** `last_confused_at` `CalendarDate` against the **injected** `clockProvider`/`CalendarDate` (never `DateTime.now()`/`Calendar.current`), via the locale-numeral/calendar presenter — and it is framed neutrally, never as overdue/alarm. Default to **no** number at all if in doubt; the row's value is the pair identity + the tap, not a figure.

7. **No scoreboard, no gamification, honest copy** (science 05 §8; PRD C6, R3; ui-mutashabihat-drill rules 5–8): **forbidden outright** anywhere in this View — points, XP, badges, streaks, confetti, a leaderboard, a "weaknesses" alarm grid, a progress bar to "100% resolved", and any "cured"/"resolved"/"mastered"/"safe to drop"/"safe to stop drilling" label or per-row "done" check. Copy frames the list as an aid to revision — drilling *reduces* swaps — and defers to the teacher (a teacher-pinned pair is honored; the app records, never arbitrates). No celebratory motion exists to add; any reveal/skeleton uses `motion.duration.short` with OS Reduce-Motion respected.

8. **RTL + bidi + locale numerals by construction** (eng-rtl-and-bidi-layout; ui-page-card rule 3; ui-mutashabihat-drill rule 9): all positioning is logical `start`/`end` (`EdgeInsetsDirectional`/`AlignmentDirectional`/`Positioned.directional`) — no `EdgeInsets.only(left:/right:)`, no `Alignment.centerLeft/Right` (grep-banned in `features/**`); no hard-coded `Directionality` (RTL comes from the locale). Every "Page N · Juz M" numeral renders via `numberFormatFor(locale)` (`-u-nu-arabext` for fa/ckb, `-u-nu-arab` for ar) and each mixed run is FSI/PDI-isolated; the pair separator/icon is direction-aware (the trailing chevron mirrors via the auto-mirroring `Icons.arrow_*`/`chevron` family). ckb's longer copy **wraps, never truncates** sacred-adjacent vocabulary.

9. **Accessibility** (ui-page-card rule 9; eng-rtl-and-bidi-layout): wrap each row in `MergeSemantics` so a screen reader hears **one** localized phrase naming the pair and the action ("you keep swapping page ۲۵۳ and page ۲۸۷ — tap to drill"), with the correct `TextDirection` and locale; the leading glyph is announced as a word or `ExcludeSemantics`-ed if decorative; the row is a labelled ≥48dp target with a visible focus ring (`color.outline`). The mutashābihāt affordance is **not encoded by colour alone** — glyph + label.

10. **Localized strings only** (eng-add-localized-string; E14-T11): every literal here (the empty-state line, any "Page N · Juz M" template, the optional recency phrasing, the `Semantics` label) is an ARB key in `app_ar.arb` (template/base content language) and transcreated `fa`/`ckb`, `nullable-getter: false` coverage — **no** hard-coded literal. The full transcreation + adab conscience-check is E14-T11; this task adds the keys and wires them.

11. **Pitfalls to avoid:** rendering `weight` / a percentage / a "confusion count" / a "resolved" progress bar; any "cured"/"safe to drop"/"mastered"/per-row done-check; a streak/score/badge/leaderboard/confetti; a sad-face or "you haven't logged" empty state; re-sorting or re-ranking the rows in the View (T06 already ranked them); opening a Drift query or parsing `distinguishing_word_index_json` here; a lone-sibling drill from a row (must be the whole group); a `DateTime.now()`/`Calendar.current` for "days ago" (use the injected clock + presenter); rendering a Quran glyph or re-typesetting an āyah in the row; a second tap target inside the row (chip/glyph/chevron); a hard-coded `Directionality` or any physical `left`/`right` inset; an imperative `Navigator.push` or controller-driven navigation; a hard-coded UI literal or an `fa`/`ckb`-incomplete ARB key; any microphone/audio/model/HTTP reference.

## Tests

`packages/features/test/mutashabihat/confusion_hotspots_view_test.dart` (mirrors the View) — `flutter_test` widget tests + per-locale goldens, run with `flutter test` in the fast CI job. The shared bootstrap installs the throwing `HttpOverrides` (the View touches no socket); `confusionHotspotsProvider` and the group read model are faked via `ProviderContainer.test()` + `overrideWith` (never the `Notifier`); a `FixedClock`/literal `CalendarDate` is injected everywhere (no host wall clock); fixtures are a few seeded `ConfusionHotspot`s for two distinct `ProfileId`s plus a matching `MutashabihGroupView` so a row can resolve its `groupId`. Required cases:

- **Renders ranked rows verbatim:** with three faked `ConfusionHotspot`s (pre-ranked by T06), the View lists exactly three rows in the **given order**, each naming both pages of its pair in locale numerals; the View performs no re-sort (swap the fake's order → on-screen order follows the fake, proving no local sort).
- **No number / no scoreboard leaks:** a `find`-based assertion that **no** `weight` value, percentage, count, "score", streak, badge, "100%", "cured", "resolved", "mastered", or "safe to drop" string is present in the rendered tree (a grep over the View source for those tokens also passes).
- **Tap opens the whole-group drill:** tapping a row calls `context.go('/mutashabihat/drill/:groupId')` with the **group** id resolved for that pair (verified via a fake router/observer); no lone-sibling route, no imperative `Navigator.push`; the whole row (not the chevron/glyph) is the single tap target.
- **Empty state is calm:** an empty `List<ConfusionHotspot>` renders the welcoming empty line (the ARB key), not an error/null/spinner and not a guilt or "0 hotspots" message.
- **Per-profile isolation:** overriding the active profile A→B re-emits B's hotspots only, with zero rows leaking from A (the `family` keying from T06 is honored by the View).
- **Loading / error shells:** the provider in `loading` renders the calm skeleton (no bare spinner-of-shame); in `error` renders the `RetryView` whose action invalidates the provider.
- **Per-locale RTL goldens:** `matchesGoldenFile` for the populated list **and** the empty state in **fa, ckb, ar** on the **real bundled UI fonts** with a pinned runner — proving RTL geometry (start/end), locale numerals (Extended Arabic-Indic for fa/ckb, Arabic-Indic for ar) in the page identities, the mirrored trailing chevron, ckb long-copy wrapping, and a grayscale/deuteranope pass that the leading affordance is readable without colour (never `Ahem`/a placeholder font); the row renders **no Quran glyph**.
- **Accessibility:** a `MergeSemantics` assertion that each row reads as one localized phrase naming the pair + action in the active locale with the correct `TextDirection`; `labeledTapTargetGuideline`/`androidTapTargetGuideline` pass (≥48dp, focus ring present).
- **Offline guard:** the suite runs under the throwing `HttpOverrides`; any stray connection attempt is a named failure; the no-network / `DateTime.now()` / banned-import / physical-side and ASCII-digit grep gates stay green over the new files.

(The `confusion_edge` ranking/`family`-keying correctness itself is proven in E14-T06; the drill behaviour + reveal→anchor goldens in E14-T08/T09; the full transcreation + adab conscience-check in E14-T11; the drill muṣḥaf golden in E14-T12 — this task does not duplicate them.)
