## Goal

The personal confusion graph has its persistence floor: an immutable `ConfusionEdge` value type in the pure `models` package (one unordered pair, a `weight`, and `lastConfusedAt` as a **`CalendarDate` serial day**), and the read-write `confusion_edge` Drift table + `ConfusionEdgeDao` confined to the `data` package — `STRICT`, `PRIMARY KEY (profile_id, ayah_a, ayah_b)`, `CHECK (ayah_a < ayah_b)` (one row per unordered pair), `weight REAL NOT NULL DEFAULT 0`, FKs to `profile` and the read-only `ayah` reference table, append-grow-only (no `DELETE`/clear DAO). It ships with its **guided Drift migration** — a `schemaVersion` bump, a committed JSON snapshot, the first real `stepByStep` `fromNToM` callback that `createTable`s `confusion_edge`, and the release-blocking `v(n−1)→v(n)` `integrity_check` fixture case plugged into E03-T09's harness. No Drift symbol crosses the `data` boundary; the DAO hands the rest of the app the plain `ConfusionEdge` value type only. This task ships the model + table + DAO read/upsert surface + migration; the `logSwap` repository write method that *strengthens* an edge is E14-T03 and the `D` bump that *reads* it is E14-T04.

## Implementation notes

**TEST-FIRST (correctness-critical for the migration and the canonical-ordering invariant).** The value→row→value round-trip (including the `CalendarDate`-serial identity), the canonical `ayah_a < ayah_b` ordering, and the `v(n−1)→v(n)` migration `integrity_check` fixture are authored **before** the table/DAO body and must be red, then green. The plain value type is a data carrier (round-trip + `copyWith` only); the migration and the ordering invariant carry the strong discipline.

1. **Reconcile the `ConfusionEdge` value type in `models` to a `CalendarDate` day — do not fork it.** E03-T01 placed `ConfusionEdge` in `packages/models/lib/src/records/confusion_edge.dart` with `lastConfusedAtInstant: DateTime`. This epic's design (EPIC.md; E14-T03) stamps `last_confused_at` from the **injected `today`** as a `CalendarDate` *scheduling day*, not a wall-clock instant — a swap belongs to the calendar day it happened on. **Align the field to `CalendarDate lastConfusedAt`** (the type the schema column will hold), surfacing the change as an explicit, reviewed edit to the existing record (it is in the same `models` package), not a second parallel type. The reconciled record:
   - `ProfileId profileId`
   - `AyahId ayahA`, `AyahId ayahB` — canonical `ayahA < ayahB` asserted in the `const` constructor (the schema `CHECK` is the storage backstop; the assert documents intent)
   - `double weight` (≥ 0)
   - `CalendarDate? lastConfusedAt` — the day of the most recent logged swap (nullable only to model a not-yet-stamped edge; in practice every persisted row is stamped on first swap by E14-T03)
   - `@immutable`, `final` fields, `const` constructor, hand-written `copyWith`, `dart:core`/`package:meta`/`CalendarDate` imports only — **no** `package:drift`. Re-export from the `models` barrel. If reconciling E03-T01's field is out of scope for this PR's review, **file the `DateTime → CalendarDate` field decision against E03/E14 rather than duplicating the type** — the one thing forbidden is two `ConfusionEdge`s.
2. **The `AyahId` identity + stable ordering is defined here for the edge.** `ayahA`/`ayahB` are `AyahId`s — the `"s:a"` (surah:ayah) stable identity keyed into the read-only `ayah` reference table (E14-T01/E05). Define (or reuse from `models`, if a typed-id scaffold already landed in E03-T01) the canonical ordering used to put a pair into `(ayahA < ayahB)` form, and make it the **single** ordering E14-T03's `logSwap` compares on. Keep `ayahA`/`ayahB` as their reference-table key type (the `ayah_id` the `ayah` table uses) — do not introduce a parallel id scheme.
3. **The Drift table class lives in `data` only, invariants in the schema.** Add `ConfusionEdges` to the `@DriftDatabase` table list in `packages/data` (e.g. `packages/data/lib/src/tables/confusion_edges.dart`):
   - `STRICT`
   - `profileId TEXT NOT NULL REFERENCES profile(profile_id) ON DELETE CASCADE` (a profile owns its edges; erasing a profile cascades them away)
   - `ayahA TEXT NOT NULL REFERENCES ayah(ayah_id)`, `ayahB TEXT NOT NULL REFERENCES ayah(ayah_id)` (FKs into the **read-only** reference table — this table references it, never writes it)
   - `weight REAL NOT NULL DEFAULT 0` (`CHECK (weight >= 0)`)
   - `lastConfusedAt INTEGER` — the **`CalendarDate` serial-day integer**, *not* a `DateTime` instant text column; serialize exactly as E03-T01's `CalendarDate` columns do (one canonical serialization, reused)
   - `PRIMARY KEY (profile_id, ayah_a, ayah_b)` and `CHECK (ayah_a < ayah_b)` — **one row per unordered pair**, so a swap logged in either direction lands on the same row (the ordering is computed by E14-T03 before the upsert; the `CHECK` is the backstop)
   The `buildToday`-class read index is not needed; the per-profile hotspots read (E14-T06) scans `WHERE profile_id = ?`, covered by the PK's leading column.
4. **The DAO maps rows to `ConfusionEdge` value types and is confined to `data`.** Author `ConfusionEdgeDao` (`packages/data/lib/src/daos/confusion_edge_dao.dart`) with exactly the read/upsert surface this task owns:
   - `Future<ConfusionEdge?> edgeFor({required ProfileId profileId, required AyahId ayahA, required AyahId ayahB})` — fetch the single canonical-ordered edge (or `null`); maps the row to the plain value type
   - `Stream<List<ConfusionEdge>> watchEdgesForProfile(ProfileId profileId)` — the reactive read backing the hotspots read model (E14-T06), ordered descending by `weight` so the calmest "you keep swapping these two" ordering is data-supplied, not view-computed
   - `Future<void> upsertEdge(ConfusionEdge edge)` — `insertOnConflictUpdate` for the create-or-strengthen write (called **only** from E14-T03's `logSwap` repository method inside its `db.transaction`; this task supplies the DAO primitive, not the transaction body or the weight rule)
   No Drift `Companion`/`TableInfo`/row class escapes `data`; the engine, features, quran, and profiles packages see only `ConfusionEdge`. This task adds **no** `delete`/`clear`/`update` DAO method (the graph is append-grow-only; the only bulk touch is export/erase — E17).
5. **The guided migration — this is the first real `stepByStep` step (correctness-critical).** E03-T09 stood up the harness with `schemaVersion = 1` and an empty-but-typed `onUpgrade: stepByStep()`. This task is the first to bump it:
   - bump `schemaVersion` to the next integer (e.g. `2`) on the `@DriftDatabase` class
   - `dart run drift_dev schema dump packages/data/drift_schemas/` → commit the new `drift_schema_v2.json` snapshot; `dart run drift_dev make-migrations` → commit the regenerated `test/migration/schema/` fixtures
   - add one typed callback: `onUpgrade: stepByStep(from1To2: (m, schema) async { await m.createTable(schema.confusionEdges); })` — `createTable` only; **never** `select`/`update` a not-yet-existing column in an older step
   - extend E03-T09's parameterized fixture with a `v1 → v2` case: `startAt(1)` → seed a profile + cards + a `review_log` row + a `cycle_config` → `migrateAndValidate(db, 2)` → assert every seeded row survived and `PRAGMA integrity_check == 'ok'`, and that the new `confusion_edge` table exists empty
   The shipped migration is append-only — a future fix is a new higher version, never a back-edit. `eraseDatabaseOnSchemaChange` stays DEBUG-only (E03-T09).
6. **No write method, no engine math, no weight rule here.** This task ships the table, the DAO `edgeFor`/`watchEdgesForProfile`/`upsertEdge` primitives, and the migration. It does **not** implement `logSwap`, the `kInitialConfusionWeight`/`nextConfusionWeight` rule, the `today` stamping, the controller seam (all E14-T03), the `(11−D)` `D` bump (E14-T04), or sibling massing (E14-T05). A grep over this task's files should find no `onReview`/`nextDifficulty`/`expandMutashabihat` reference and no `DateTime.now()`.
7. **Pitfalls to avoid:**
   - **Two `ConfusionEdge` types** — reconcile E03-T01's record in place (or file the field decision), never duplicate it.
   - **`last_confused_at` as a `DateTime` instant** — it is a `CalendarDate` serial-day integer (the swap belongs to a *day*); a `DateTime`-typed day is a review-blocking defect.
   - **A mirror-duplicate edge** because the `CHECK (ayah_a < ayah_b)` or the PK is missing — both must be in the schema so a reversed pair cannot create a second row.
   - **A `delete`/`clear`/`update` DAO on the graph** — append-grow-only; the only bulk touch is export/erase (E17).
   - **Writing the `ayah` reference table** — the FKs *reference* it; no DAO here exposes a write to `ayah`/`mutashabih_*`.
   - **Editing a shipped migration / skipping the committed snapshot** — bump forward, commit the `drift_dev schema dump`, never back-edit.
   - **A Drift symbol crossing the `data` boundary** — the DAO returns `ConfusionEdge`, never a `Companion`/row class; `models` imports no `package:drift`.
   - **Persisting a derived health/weight roll-up elsewhere** — `weight` is the only stored authority; no second source of truth.

## Tests

Authored test-first where correctness-critical (the migration and the canonical-ordering round-trip). The value type is a pure `package:test` unit; the DAO/round-trip and migration use the real Drift/SQLite stack at the unit tier. Every file carries the REUSE SPDX `GPL-3.0-or-later` header, full-word/unit-bearing names, and typed `catch`; the shared throwing-`HttpOverrides` offline bootstrap stays installed (this path opens no socket).

- `packages/models/test/records/confusion_edge_test.dart` — `copyWith` identity (a no-arg `copyWith` is field-equal; a `copyWith(weight: …)` changes only `weight`); type pins: `ConfusionEdge(...).lastConfusedAt` is a `CalendarDate?` and `weight` is a `double` (a static-type assertion so a future `DateTime lastConfusedAt` fails to compile); the canonical `ayahA < ayahB` invariant (constructing the reverse pair is rejected or normalized — pin whichever the implementation chooses); no streak/score/health field on the public surface.
- `packages/data/test/daos/confusion_edge_dao_test.dart` (`drift`'s in-memory `NativeDatabase.memory()`, seeded with a profile and FK-valid `ayah` rows):
  - **Round-trip identity:** `upsertEdge(edge)` then `edgeFor(...)` returns a `ConfusionEdge` value-equal to the input, including the `CalendarDate lastConfusedAt` serial-day round-trip and the `weight` `double` — `value → row → value` is identity.
  - **Canonical ordering / no mirror duplicate:** an edge upserted as `(A, B)` and one upserted as `(B, A)` (each pre-ordered by the canonical comparison) land on **one** row ordered `ayah_a < ayah_b`; a direct reversed-order insert is rejected by the `CHECK`.
  - **`upsertEdge` strengthens, never duplicates:** a second `upsertEdge` for the same canonical pair updates the single row (count stays one), proving `insertOnConflictUpdate` is wired (the *weight rule itself* is E14-T03; here only the upsert primitive is exercised).
  - **`watchEdgesForProfile` ordering + scope:** the stream emits the profile's edges ordered by descending `weight` and excludes another profile's edges (per-profile scope honored).
  - **FK + cascade:** an edge whose `ayah_a`/`ayah_b` is not a seeded `ayah` row is rejected (FK enforced under `foreign_keys=ON`); deleting the owning `profile` cascades its edges away (`ON DELETE CASCADE`); the `ayah`/`mutashabih_*` reference tables are never written by any DAO call here.
  - **Append-grow-only:** the `ConfusionEdgeDao` public surface exposes no `delete`/`clear`/`update` method (asserted by review + the documented surface).
- `packages/data/test/migration/migration_test.dart` (extends E03-T09's `SchemaVerifier` harness, `flutter test`) — **`v1 → v2` (written first, red against the not-yet-wired step):** `startAt(1)` → seed a `profile`, `card` rows (incl. the memorized-`due_at`-non-null invariant and an `UNMEMORIZED` card), **at least one `review_log` row** (the *sanad* trail), a `cycle_config` → `migrateAndValidate(db, 2)` → assert every seeded row survived byte-intact, the new `confusion_edge` table exists (and is empty), and `PRAGMA integrity_check == 'ok'`. The seeded `review_log` survival is the load-bearing assertion.

No golden, widget, or `integration_test` is in scope — this is a `models`/`data`-tier model+table+DAO+migration unit. The drill muṣḥaf golden is E14-T12; the hotspots view goldens are E14-T10.
