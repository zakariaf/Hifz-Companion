## Goal

A small, reusable `Semantics` convention layer exists under `packages/features/lib/src/a11y/` (`semantics.dart`) and is applied to the E07 shell: a documented label/value/role helper pattern (`label`/`hint`/`button`/`header`), a `MergeSemantics` wrapper so a multi-part item (a page-card / placeholder card) is read as **one** localized phrase rather than three fragments, an `ExcludeSemantics` wrapper for pure decoration, and a single `announce(...)` path over `SemanticsService.announce(message, TextDirection.rtl)` for background state changes the reader would otherwise miss — "catch-up plan ready," "page graded," "sign-off recorded." Every label/message resolves through the `AppLocalizations` (ARB) set, so the reader speaks the active locale (fa/ckb/ar), and each run carries its `locale`. The helpers are wired into the E07 `HomeShell` nav items and the four inert placeholder cards. Widget tests assert intelligible, **merged**, **localized** labels per locale, and that the announce path emits the right localized message in `TextDirection.rtl`. The muṣḥaf glyph layer is never exposed as text to the reader: where a page is referenced, the announced string is its **reference** (surah / ayah range / juz), never the QPC glyph codepoints.

## Implementation notes

**TEST-FIRST (correctness-critical for the announce path):** the announce path is a side-effecting seam that must speak the right localized message in the right direction. Write `features/test/a11y/announce_test.dart` (a fake/recording `SemanticsService` channel; assert message == the `l10n` string and `textDirection == TextDirection.rtl` per locale) **before** the announce code. The merged-label widget assertions below are also written before the helper bodies.

1. **Files (new `a11y` chrome module).** Under `packages/features/lib/src/a11y/` (a non-navigable shared chrome folder, downward-only deps: l10n + design-system tokens only — no engine, no drift, no http): `semantics.dart` holds the wrapper helpers; `announce.dart` holds the announce path. One primary public type/function per file is not required for tiny stateless wrappers, but keep the announce function and any `SemanticsRole`-style enum in their own file. These are widget-layer helpers, not a package — they live in `features`, consumed by the shell and (later) E10's component library.

2. **The label/value/role wrapper.** A thin convention over Flutter's `Semantics`, not a re-abstraction of it. Provide a small helper (e.g. `labeled({required String label, String? hint, bool button, bool header, Widget child})`) that returns `Semantics(label: label, hint: hint, button: button, header: header, child: child)`. The **label and hint are always passed in already-localized** (`l10n.*`) — the helper never looks up a key itself (keeps it l10n-package-free and testable). Document the role mapping in a `///` doc comment: a tappable control → `button: true`; a section/title → `header: true`; a static value carrier → `label` (+ `value:` only where Flutter's `value` semantics genuinely apply, e.g. a slider-like control — not used for plain text). Per ds-09 §7 / eng-12 §7, every icon-only control (the mirrored back/next nav glyphs, future track chips, the teacher sign-off toggle) gets a label.

3. **`MergeSemantics` wrapper — one phrase, not fragments.** Provide `mergedItem({required Widget child})` (or a documented direct-`MergeSemantics` convention) so a multi-part row/card collapses into **one** spoken node. Apply it to each of the four inert placeholder cards (E07-T04) so the title + body line are read as one phrase in the active locale. The page-card (E07-T08) already uses `MergeSemantics`; refactor that call site to the shared wrapper without changing its content. The label of the merged node is the composed localized run (e.g. title + " · " + body), with any number/page-reference run formatted via `numberFormatFor(locale)` and FSI/PDI-isolated through `l10n/bidi.dart` so the reader voices it in order (eng-rtl §5–§7).

4. **`ExcludeSemantics` wrapper — decoration is silent.** Provide `decoration({required Widget child}) => ExcludeSemantics(child: child)` and apply it to purely decorative dividers/ornament in the shell and placeholders so the reader does not read them out (ds-09 §7). Do not exclude anything that carries meaning — the wrapper is for ornament only.

5. **The RTL announce path.** `announce.dart` exposes a single function, e.g. `Future<void> announceState(BuildContext context, String message)` that calls `SemanticsService.announce(message, Directionality.of(context))`. Direction is read from context (`Directionality.of(context)`), which resolves to `TextDirection.rtl` for the active locale — **never** a hardcoded `TextDirection.rtl` constant in logic (eng-rtl §2). The `message` is always an already-localized `l10n.*` string ("catch-up plan ready" → `l10n.a11yAnnounceCatchUpReady`, "page graded" → `l10n.a11yAnnouncePageGraded`, "sign-off recorded" → `l10n.a11yAnnounceSignOffRecorded`). These three ARB keys are authored in `app_ar.arb` first (template), transcreated fa/ckb (ckb canonical-encoded, flagged provisional); their copy is calm/adab (no guilt/loss, no "safe to drop"). **No `value`-spliced number** unless it traces to a CLAIM (C-016) and goes through the numeral + isolate path. (Post-Flutter-3.35 `SemanticsService.announce` is superseded by `sendAnnouncement`; keep the call behind this one function so the migration is a single-site change — note it in the doc comment.)

6. **Wire the helpers into the E07 shell.** The five `HomeShell` nav destinations carry `labeled(button: true, label: l10n.navToday…)`; the four placeholder cards are wrapped in `mergedItem` with their composed localized label; decorative dividers use `decoration(...)`. Where the shell has a real background state change in E07's slice (the grade command's stream re-emit on Today, E07-T08), fire `announceState(context, l10n.a11yAnnouncePageGraded)` from the View after the controller's `grade(...)` future completes — calm, once, no repetition; "catch-up plan ready" and "sign-off recorded" announce keys are authored here and consumed by E11/E12 (left un-fired on surfaces that don't exist yet, but the keys + the path exist).

7. **The muṣḥaf reference, never the glyph (sacred-text boundary).** No `Semantics` label or announce message in this task is derived from a QPC glyph codepoint or a re-typeset āyah string. Where a page is named, the label is the page **reference** — surah / ayah range / juz — from the bundled structure data (the inert placeholders carry no page reference yet; the page-card names "Page N · Juz M" only, never glyphs). The helpers live in `a11y/` (chrome) and never import the muṣḥaf/glyph layer.

8. **Pitfalls to avoid.** A hardcoded `TextDirection.rtl` constant in the announce path (must be `Directionality.of(context)`); a label/hint looked up inside the helper (it must receive already-localized strings, keeping the wrapper l10n-free and pure); an English label surviving in an fa/ckb/ar build (the `features/**` hardcoded-string grep + per-locale assertions catch it); reading a page-card as three fragments (must be one `MergeSemantics` phrase); excluding a meaningful node with `ExcludeSemantics` (decoration only); announcing the **same** state repeatedly or chattily (calm, once); splicing a raw ASCII number into a label/announce message (route through `numberFormatFor(locale)` + `isolate*`); exposing any muṣḥaf glyph codepoint or re-derived āyah as "text" to the reader (announce the reference); importing engine/drift/http into the `a11y/` module; introducing any streak/score/celebratory phrasing into an announce message.

## Acceptance criteria

- [ ] `packages/features/lib/src/a11y/semantics.dart` exists with the `labeled(...)` (label/hint/button/header), `mergedItem(...)` (MergeSemantics), and `decoration(...)` (ExcludeSemantics) wrappers; each carries a `///` doc comment stating its role mapping; the module imports only l10n + design-system tokens (no engine/drift/http/glyph layer — verifiable by grep).
- [ ] `labeled(...)` passes through already-localized `label`/`hint` and the `button`/`header` flags to `Semantics`; it never performs an ARB lookup itself.
- [ ] `packages/features/lib/src/a11y/announce.dart` exposes one announce function that calls `SemanticsService.announce(message, Directionality.of(context))` — direction read from context, **no** hardcoded `TextDirection.rtl` constant in logic; the message is an already-localized `l10n.*` string.
- [ ] Three announce ARB keys (`a11yAnnounceCatchUpReady`, `a11yAnnouncePageGraded`, `a11yAnnounceSignOffRecorded`) exist in `app_ar.arb` (template) with fa/ckb transcreations (ckb provisional); copy is calm/adab — no guilt/loss, no "safe to drop," no celebration.
- [ ] The five `HomeShell` nav items are wrapped via `labeled(button: true, label: l10n.nav…)`; the four inert placeholder cards are wrapped via `mergedItem` and read as **one** localized phrase each; decorative dividers/ornament use `decoration(...)`.
- [ ] A screen reader (asserted via `SemanticsTester`/the semantics tree) describes every nav item and placeholder card with an intelligible, **localized** label under each of ar/fa/ckb; a placeholder card surfaces as one merged node, not three fragments.
- [ ] The announce path, exercised in a test, emits the correct localized message with `TextDirection.rtl` for each of ar/fa/ckb; calling it twice for the same state does not spam (called once per real state change in the shell wiring).
- [ ] No `Semantics` label or announce message is derived from a muṣḥaf glyph codepoint or a re-typeset āyah; where a page is named it is the page reference (surah/ayah/juz / "Page N · Juz M"), never glyphs.
- [ ] Any number/page-reference run inside a label/announce message goes through `numberFormatFor(locale)` and is FSI/PDI-isolated via `l10n/bidi.dart`; the ASCII-digit grep and the physical-side grep stay green; no hardcoded UI literal survives the `features/**` grep.
- [ ] No `package:http`/`dio`/`dart:io HttpClient`, no `drift`/DAO import, no microphone/ASR/model, and no `DateTime.now()` appears anywhere reachable from the `a11y/` module.

## Tests

Dart tests run in CI on every PR (`flutter test`); goldens load the **real** bundled UI fonts (never `Ahem`) and pin the runner. **Write the announce and merged-label tests first.**

- `features/test/a11y/announce_test.dart` — **unit/widget (write first)**: install a recording fake over the `SemanticsService` announce channel (capture `(message, textDirection)`); pump a minimal `MaterialApp` under each of `Locale('ar')`/`Locale('fa')`/`Locale('ckb')`; call `announceState(context, l10n.a11yAnnouncePageGraded)` and assert the captured message equals the locale's ARB value and `textDirection == TextDirection.rtl`; assert no hardcoded direction constant is used (the captured direction follows the ambient `Directionality`). Repeat for the catch-up-ready and sign-off-recorded keys.
- `features/test/a11y/semantics_helpers_test.dart` — **widget (write first)**: pump `labeled(...)`, `mergedItem(...)`, and `decoration(...)` and assert via the semantics tree (`SemanticsTester`/`tester.getSemantics`) that `labeled` exposes `label`/`hint` + the `button`/`header` flags; `mergedItem` collapses a two-`Text` child into **one** node whose label is the composed phrase; `decoration` produces a node with no readable label (excluded).
- `features/test/a11y/shell_semantics_test.dart` — **widget**: pump `HomeShell` (with a placeholder child) and the four placeholder cards under each of ar/fa/ckb; assert every nav item has a localized `Semantics` label and `button` role; each placeholder card is one merged, intelligible localized phrase; no English label leaks in a non-`ar` build; `meetsGuideline(labeledTapTargetGuideline)` passes over the shell (the audit gate proper is wired PR-blocking in E08-T07).
- `features/test/a11y/shell_semantics_rtl_golden_test.dart` — **golden** (`@Tags(['golden'])`): `matchesGoldenFile` of `HomeShell` + a placeholder card under fa, ckb, and ar on the **real** bundled UI fonts, so the labeled/merged chrome the screen-reader traverses is pinned per locale (the full RTL traversal proof is E08-T08).
- **Offline / localization gates**: a throwing `HttpOverrides` is installed in these widget tests; a banned-import grep over `features/a11y` finds no `http`/`dio`/`HttpClient`/`drift`/glyph-layer symbol; `app_ar.arb` key-coverage compiles with `nullable-getter: false`; the `features/**` hardcoded-string grep, the physical-side grep, and the ASCII-digit grep stay green; fa/ckb/ar values present for the three new announce keys (ckb flagged provisional). No test asserts a coverage percentage.

## Definition of Done

- [ ] All acceptance criteria met; the announce + merged-label unit/widget tests (written first), the shell-semantics widget + RTL golden tests, and the localization + grep gates are green in CI on every PR.
- [ ] **Offline / no-network** (non-negotiable): the `a11y/` module imports no `package:http`/`dio`/`dart:io HttpClient` and opens no socket; the announce path is local `SemanticsService` only; tests install a throwing `HttpOverrides`; E01's banned-import + allow-list gates stay green (C-048).
- [ ] **No AI / no microphone** (non-negotiable): nothing in the semantics helpers or the announce path uses ASR, ML, a model, or audio; the screen-reader integration is one-way output to the OS, records nothing, and adds no microphone permission (C-048).
- [ ] **Quran text fidelity** (non-negotiable): no `Semantics` label or announce message is derived from a QPC glyph codepoint or a re-typeset āyah; the reader is fed the page **reference** (surah/ayah/juz), never the sacred glyph layer; the `a11y/` module never imports or reaches the muṣḥaf glyph layer (ds-09 §7).
- [ ] **RTL + fa/ckb/ar localization** (non-negotiable): every `Semantics` label and every announce message resolves through `AppLocalizations` (ar template + fa/ckb transcreations; no hardcoded literal); each run sets its `locale`; the announce path passes `Directionality.of(context)` (RTL by construction), never a hardcoded constant; any embedded number is locale-numeral + FSI/PDI-isolated.
- [ ] **Accessibility** (this epic's subject): every interactive shell element carries a localized label and role; multi-part cards merge into one phrase; decoration is excluded; background state changes announce in reading order — the §7 (name-role-value, 4.1.2) conventions the feature epics inherit are in place and applied to the real running shell.
- [ ] **Sect-neutral adab / no gamification** (non-negotiable): no announce message or label introduces a streak, score, badge, celebratory phrasing, or guilt/fear/loss copy; "catch-up plan ready" is strengthening, never shame (C-009); copy implies no madhhab/sect ruling.
- [ ] **Nothing safe to drop / no unsourced number**: no label or announce message marks a page optional; any interpolated due/catch-up value is the trust-clamped engine value (C-016), surfaced honestly; no claim/CLAIMS id is invented.
- [ ] **Determinism**: no `DateTime.now()`/`Calendar.current` reachable from the `a11y/` module; the announce path is fired by an upstream state change (the View after the controller future), not a timer or clock read; the merged-label and announce tests are deterministic.
- [ ] **Standards**: the `a11y/` wrappers are thin, documented (`///` on every public helper), and downward-only (l10n + tokens only); every new Dart file carries the REUSE SPDX header (`GPL-3.0-or-later`) and passes the analyzer/lint config; tokens referenced by name only.
