## Implementation notes

TEST-FIRST is inverted here on purpose: the deliverable is *proving the existing gates go red, then keeping the tree green*. There is **no new production code** and **no committed failing test** â€” a permanently-red test would itself break the four-jobs-green DoD. Each violation is demonstrated as a local, throwaway probe (in a branch or a scratch run), captured as a documented manual step, then reverted.

1. **The deliberate-violation procedure (the documented artifact).** Author `packages/features/test/a11y/README.md` (or extend the harness doc E08-T07 created) with a four-row **"How to prove this gate still checks something"** table â€” one row per violation, each giving: the exact one-line edit that breaks it, the guideline/audit that must go red, and the revert. REUSE SPDX header (`GPL-3.0-or-later`) on the file. The rows:
   - **Unlabeled control (A7).** Remove the `label:` from one `HomeShell` nav item's `labeled(...)` helper (E08-T02) â†’ `meetsGuideline(labeledTapTargetGuideline)` and the explicit label-presence walk go red â†’ revert.
   - **Sub-48 dp target (A6).** Wrap one nav item in a `SizedBox(width: 40, height: 40)` (below the 48 dp floor) â†’ `meetsGuideline(androidTapTargetGuideline)` (and `iOSTapTargetGuideline` at 44 pt) go red â†’ revert.
   - **Below-floor contrast pair (A1).** Override one chrome `color.text.*`/`color.bg.*` token pair to a sub-4.5:1 combination in the contrast golden's pumped theme â†’ `meetsGuideline(textContrastGuideline)` goes red in the `golden` lane â†’ revert. (The token *values* live in E06; here we only force a bad pair to confirm the gate measures it â€” never a committed token change.)
   - **Color-only state (A3/Â§4).** Strip the shape + label channels from one state chip so only hue distinguishes it (E08-T06) â†’ the never-color-alone widget-audit fails â†’ revert.

2. **Demonstrate red-before-revert locally; assert the discriminator behaviour in a tiny self-test.** Optionally add one **passing** widget test, `packages/features/test/a11y/gate_discriminates_test.dart`, that builds a deliberately-broken *throwaway* stub widget (not the real shell) â€” e.g. an icon button with no label and a 40Ã—40 hit box â€” and asserts that `meetsGuideline(labeledTapTargetGuideline)` / `androidTapTargetGuideline` **does not** hold for it (`expectLater(tester, isNot(meetsGuideline(...)))`), proving the matcher discriminates a violation from a pass. This is a *green* test (it asserts the matcher correctly *rejects* a bad widget), so it satisfies four-jobs-green while still proving the gate is not vacuous. 3. **The DoD sweep checklist (the second artifact).** Add the epic-level DoD sweep as a checklist in the **final PR's description** (and mirror it in the harness README) â€” one line per epic non-negotiable, each pointing at the concrete proof already in the tree:
   - **Text fidelity:** the E08-T04 scaling-exclusion seam test asserts the layout data is unchanged under text scale; no audit derives from a QPC glyph codepoint; the reader announces the page *reference* (E08-T02).    - **Offline / no-AI / no-mic:** the throwing `HttpOverrides` is installed in every a11y test; the banned-import grep over `features/test/a11y` finds no `http`/`dio`/`dart:io HttpClient`/`drift`; no ASR/ML/model/microphone permission anywhere; E01's no-network + dependency-allow-list gates stay green (C-048).
   - **RTL `fa`/`ckb`/`ar` labels:** every audited label resolves through `AppLocalizations` (no English literal in a non-`ar` build); the contrast/traversal goldens load the **real** bundled UI fonts (never `Ahem`); the A8 traversal test (E08-T08) is green in all three locales.
