# E08-T06 — Never-color-alone redundant-encoding convention + a widget-audit that fails a color-only state

## Goal

A reusable redundant-encoding convention exists under `packages/features/lib/src/a11y/` (`redundant_encoding.dart`) that gives every state chip — `track` (sabaq/sabqi/manzil), `due`, `weak`, `signOff`, `decay` — its meaning in **three** channels at once: a design-system color token, a non-color shape/pattern (icon/hatch glyph), and a localized label or number. Each state is also paired with the `Semantics` label from E08-T02 so the non-color channel carries through to the screen reader. A test-first **widget-audit** (`assertStateChipRedundancy`) inspects a chip's rendered subtree and **fails** any state conveyed by hue alone — proving SC 1.4.1 is enforced, not assumed. The convention is applied to the E07 shell's placeholder state chips and documented as the rule E15's heat-map cells and E12's recite/grade-flow cards inherit; the heat-map's own cells and the page-card's own track/decay encodings are not built here.

## Implementation notes

**TEST-FIRST (correctness-critical — this is an enforcement gate):** the widget-audit is the deliverable. Write `features/test/a11y/redundant_encoding_audit_test.dart` first: a **positive** fixture (a chip with color + shape + label) passes, and a **negative** fixture (a deliberate color-only chip — a `Container` with only a `color`, no shape/icon, no label) **fails** the audit. The audit and the negative fixture must exist and the negative case must fail before the convention helper is finalized — otherwise the gate asserts nothing.

1. **Files (extend the `a11y` chrome module from E08-T02).** Under `packages/features/lib/src/a11y/` (the non-navigable shared chrome folder; downward-only deps: l10n + design-system tokens only — no engine, no drift, no http, no glyph layer): add `redundant_encoding.dart`. It holds (a) a small `StateChip` enum or a `ChipState { color, shape, label }` value carrier describing the **three required channels** for the five states, and (b) a `stateChip({required ChipState state, ...})` widget helper that composes the color token, the shape/pattern glyph, and the label into one chip — wrapping the label channel through E08-T02's `labeled(...)`/`mergedItem(...)` so the chip is read as one localized phrase. The audit function `assertStateChipRedundancy(WidgetTester tester, Finder chip)` lives in a test-support file (`features/test/a11y/redundant_encoding_audit.dart`) so production code never depends on a test util.

2. **The five states, three channels each.** Enumerate exactly the epic's five: `track` (sabaq/sabqi/manzil — three sub-values), `due`, `weak`, `signOff`, `decay`. For each, the convention fixes: a design-system **color** token (named, ≥4.5:1 vs background per E06; never red↔green for weak/strong — green *receding to neutral* for decay per ds-09 §4); a **shape/pattern** — an icon or a hatch/dot fill that survives grayscale, *distinct per state* (the audit checks shape varies across states, not just color); and a **label/number** — the localized word (track name, "due", "weak"/"needs revision", "signed off") and/or a locale-numeral number (decay %, due-day count). The shape glyph is a fixed-convention state mark, **never** a mirrored directional arrow.

3. **The widget-audit `assertStateChipRedundancy`.** Given a chip finder, it asserts, over the rendered subtree, that **at least two non-color channels are present** for the state: (a) a non-color visual differentiator — an `Icon`/`CustomPaint`/pattern node, or a text run; **and** (b) a `Semantics` `label` reachable on the subtree (`tester.getSemantics`) that is non-empty and not derived from color. It must **fail** a chip whose only differentiator across states is a `Color` (a bare `Container(color: …)` / `DecoratedBox(BoxDecoration(color: …))` with no icon, pattern, text, or label). The audit is a pure widget-tree inspection — no golden, no pixel diff, no network. Keep it deterministic: no clock, no randomness.

4. **Pair each state with its E08-T02 `Semantics` label.** The label channel is **not** re-authored here — it is the localized `l10n.*` string already passed through `labeled(...)`/`mergedItem(...)` (E08-T02). This task wires each of the five states to its label so the non-color word reaches the screen reader: `track` → the localized track name (the regional sabaq/sabqi/manzil term-set value), `due` → `l10n.stateDue`, `weak` → `l10n.stateWeak`, `signOff` → `l10n.stateSignedOff`, `decay` → `l10n.pageHealth(percentLocalized, …)`. Any new ARB keys for these state words are authored in `app_ar.arb` first (template), transcreated fa/ckb (ckb canonical-encoded, flagged provisional), calm/adab copy (no guilt/loss, no "safe to drop").

5. **Apply to the shell's placeholder state chips.** E07's inert placeholder cards carry placeholder state chips; replace any color-only placeholder chip with `stateChip(state: …)` so the running shell is itself audited green. Do **not** build the heat-map cells (E15), the page-card's real track/decay chips (E12/page-card skill), or any feature surface — only the shell's placeholders, so the spine is proven and the feature epics extend a green convention.

6. **RTL + numeral correctness of the label/number channel.** A chip's number (decay %, due-day count) is formatted via `numberFormatFor(locale)` and FSI/PDI-isolated via `l10n/bidi.dart` (eng-rtl §5–§7); the chip lays out with `EdgeInsetsDirectional` / logical `start`/`end` only; the state-shape glyph is never mirrored. The encoding is identical across fa/ckb/ar — only the digit block and the transcreated word change.

7. **Document the inheritance contract.** A `///` doc comment on `stateChip`/`ChipState` states the rule plainly: *every state chip in this app carries color + shape + label/number; a color-only state is a SC 1.4.1 failure and is caught by `assertStateChipRedundancy`; E15's heat-map cells and E12's cards must call this audit in their own tests.* This is the documented rule the feature epics inherit.

8. **Pitfalls to avoid.** A chip that differs across states by **color only** (the exact failure the audit must catch); using a red↔green weak/strong ramp (ds-09 §4 anti-pattern) instead of a lightness ramp / green-receding-to-neutral; reusing the **same** shape glyph for two states (then shape stops being a differentiator — the audit checks shape varies); deriving the label from the color token instead of the localized state word; an English state word surviving in an fa/ckb/ar build (the `features/**` hardcoded-string grep + per-locale assertions catch it); splicing a raw ASCII number into a chip label (route through `numberFormatFor(locale)` + `isolate*`); mirroring the state-shape glyph (it is fixed-convention, never directional); a "decay" chip rendered as an alarming red scoreboard or any "safe to drop" implication (C-019/C-016); importing engine/drift/http/glyph-layer into `a11y/`; making the audit a golden/pixel diff instead of a tree inspection; letting production code depend on the test-only audit util; any streak/score/badge/celebratory framing on a chip (C-041/C-044).

## Acceptance criteria

- [ ] `packages/features/lib/src/a11y/redundant_encoding.dart` exists with the `ChipState`/`StateChip` carrier (color + shape + label/number) and the `stateChip(...)` helper; each carries a `///` doc comment stating the color + shape + label/number rule and naming SC 1.4.1; the module imports only l10n + design-system tokens (no engine/drift/http/glyph layer — verifiable by grep).
- [ ] All five states — `track` (sabaq/sabqi/manzil), `due`, `weak`, `signOff`, `decay` — are defined with a named color token, a *distinct* non-color shape/pattern, and a localized label/number; no two states are distinguished by color alone (shape varies across states).
- [ ] `assertStateChipRedundancy(tester, finder)` exists as a test-support helper; it **passes** a chip with ≥2 non-color channels and **fails** a chip differentiated by `Color` alone (bare `Container(color:)`/`DecoratedBox` with no icon/pattern/text/label).
- [ ] Each state is paired with its E08-T02 `Semantics` label; the audit reads a non-empty, non-color-derived label back from the semantics tree for every state.
- [ ] The decay/weak chips' label/number never implies "safe to drop"; the decay chip is green-receding-to-neutral (not red↔green); any number is locale-numeral, FSI/PDI-isolated, and the trust-clamped value (C-016).
- [ ] The E07 shell's placeholder state chips use `stateChip(...)`; the running shell passes `assertStateChipRedundancy` for every placeholder chip under ar/fa/ckb.
- [ ] A `///`-documented inheritance contract states that E15's heat-map cells and E12's cards must each carry color + shape + label/number and call this audit.
- [ ] Any number in a chip goes through `numberFormatFor(locale)` and is FSI/PDI-isolated; layout uses `EdgeInsetsDirectional`/logical `start`/`end`; the state-shape glyph is never mirrored; the ASCII-digit grep and physical-side grep stay green; no hardcoded UI literal survives the `features/**` grep.
- [ ] No `package:http`/`dio`/`dart:io HttpClient`, no `drift`/DAO import, no microphone/ASR/model, and no `DateTime.now()` appears anywhere reachable from `redundant_encoding.dart`.

## Tests

Dart tests run in CI on every PR (`flutter test`); RTL/label goldens load the **real** bundled UI fonts (never `Ahem`) and pin the runner. **Write the audit + the negative fixture first.**

- `features/test/a11y/redundant_encoding_audit_test.dart` — **widget (write first)**: a **positive** fixture (`stateChip` with color + icon + label) **passes** `assertStateChipRedundancy`; a **negative** fixture (a deliberate color-only chip — `Container(color: …)` only, no shape/label) **fails** it with a clear message; assert the audit fails for *each* of the five states when its shape/label channel is removed (parameterize: strip the icon → fails; strip the label → fails; both present → passes).
- `features/test/a11y/redundant_encoding_states_test.dart` — **widget**: pump each of the five `stateChip` states under ar/fa/ckb; assert each renders a named color token **and** a distinct non-color shape **and** a localized label (read from the semantics tree via `tester.getSemantics`); assert no two states share the same shape glyph; assert the decay chip's number is the locale-numeral, FSI/PDI-isolated value and carries no "safe to drop"/streak/score copy.
- `features/test/a11y/shell_state_chips_test.dart` — **widget**: pump the E07 shell with its placeholder cards under ar/fa/ckb; assert every placeholder state chip passes `assertStateChipRedundancy` and carries a localized label; no English state word leaks in a non-`ar` build.
- `features/test/a11y/state_chips_rtl_golden_test.dart` — **golden** (`@Tags(['golden'])`): `matchesGoldenFile` of the five chips (and a placeholder card carrying them) under fa, ckb, and ar on the **real** bundled UI fonts, so the shape + localized label/number channels are pinned per locale and a grayscale-surviving shape is visible.
- **Offline / localization gates**: a throwing `HttpOverrides` is installed in these widget tests; a banned-import grep over `features/a11y` finds no `http`/`dio`/`HttpClient`/`drift`/glyph-layer symbol; `app_ar.arb` key-coverage compiles with `nullable-getter: false`; the `features/**` hardcoded-string grep, the physical-side grep, and the ASCII-digit grep stay green; fa/ckb/ar values present for any new state-word keys (ckb flagged provisional). No test asserts a coverage percentage.
