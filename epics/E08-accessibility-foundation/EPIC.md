# E08 — Accessibility Foundation

Make accessibility a Definition-of-Done discipline the feature epics inherit, not a retrofit. This epic lands the cross-cutting contract — dynamic text scaling that never touches the muṣḥaf glyph layer, the `Semantics` label/value/role conventions and merge/exclude rules, right-to-left screen-reader traversal verified in `fa`/`ckb`/`ar`, reduce-motion substitutions, a never-color-alone enforcement helper, and a PR-blocking accessibility audit harness — wired into the real running shell (E07) so every later screen rides a known-good a11y spine. The conformance bar is **WCAG 2.2 Level AA**, claimed explicitly and made measurable, because the work is *lillāh* and *iḥsān* is the standard.

## Why this epic exists

The users recite daily, often quickly, in three RTL scripts, and many are older huffaz with presbyopia or reduced color vision ([design-system 09 §intro](../../docs/design-system/09-accessibility-and-inclusivity.md); [PRD §3, §18](../../docs/PRD.md)) — so accessibility here is *adab* toward the user and toward the muṣḥaf, and the design-system file makes it precise and citable rather than aspirational ([design-system 09 §1](../../docs/design-system/09-accessibility-and-inclusivity.md)). The category's documented failure is treating accessibility as a post-launch backlog; this app refuses that — the §10 gates are release-blocking *alongside* the muṣḥaf-integrity gates ([design-system 09 §1, §10](../../docs/design-system/09-accessibility-and-inclusivity.md); [PRD §20](../../docs/PRD.md)). The reason a foundation epic must precede the feature epics is the same reason E07 stitches a walking skeleton: the RTL/multilingual 20% of accessibility bugs only surface with a screen reader running *in-language*, and they live in the seams between layout, semantics, text scale, and direction ([design-system 09 §10](../../docs/design-system/09-accessibility-and-inclusivity.md); [engineering 12 §8](../../docs/engineering/12-localization-rtl-accessibility-impl.md)) — so the conventions and the audit harness must exist while the surface is small enough that a violation is a one-screen fix, not an archaeology project across twelve screens.

The non-negotiables this epic structurally serves, each grounded in the docs:

- **The sacred text is never put at risk by an a11y feature.** Two scaling systems are kept deliberately separate: OS text-scale governs the chrome to 200%+; the muṣḥaf is rendered from fixed KFGQPC per-page glyph fonts and is **never** OS-scaled, re-typeset, or reflowed — a low-vision ḥāfiẓ enlarges the page through a dedicated zoom transform of the rendered layer, never by scaling a glyph ([design-system 09 §5](../../docs/design-system/09-accessibility-and-inclusivity.md); [engineering 12 §7](../../docs/engineering/12-localization-rtl-accessibility-impl.md); [PRD R1, §11.2, §18](../../docs/PRD.md)). The screen reader is fed the page's *reference* (surah / ayah range / juz) from the bundled structure data, never a reconstruction of the QPC glyph codepoints ([design-system 09 §7](../../docs/design-system/09-accessibility-and-inclusivity.md)).
- **Nothing decays silently; the heat-map signal survives the loss of color.** Color is never the only channel: the retention signal lives in a lightness ramp plus a number plus the text weakest-pages list, so a deuteranope or a screen-reader user reads the decaying juz exactly as well ([design-system 09 §4](../../docs/design-system/09-accessibility-and-inclusivity.md); [engineering 12 §7 spec](../../docs/engineering/12-localization-rtl-accessibility-impl.md); [PRD §18](../../docs/PRD.md)). This epic ships the *enforcement* of never-color-alone (the lint/widget-audit and the redundant-encoding convention); the heat-map values themselves live in E15.
- **Offline / no-AI / no-microphone are banked as accessibility wins, not re-opened.** No account removes the 3.3.7/3.3.8 cognitive-function barriers by construction; no microphone removes any voice-input dependency; the calm, no-motion, non-gamified design avoids the seizure/flash and anxiety triggers ([design-system 09 §2, §9](../../docs/design-system/09-accessibility-and-inclusivity.md); [PRD C1, C2, C6, R3, R5, §17](../../docs/PRD.md)). This epic states those wins in the conformance claim and refuses any feature that would re-introduce a removed barrier.
- **The bar is identical across `fa`/`ckb`/`ar`.** Nothing about Arabic-script density or RTL relaxes any criterion — RTL only changes *how* a criterion is satisfied (focus/reading order = visual order), never *whether* ([design-system 09 §1, §8](../../docs/design-system/09-accessibility-and-inclusivity.md)). The audit therefore runs per-locale on the real bundled UI fonts, never a placeholder font ([engineering 12 §8](../../docs/engineering/12-localization-rtl-accessibility-impl.md)).

## Scope

### In scope

- **The conformance claim, stated and made measurable.** WCAG 2.2 AA recorded in the README as the explicit bar, with the two architecturally-satisfied criteria (3.3.7 Redundant Entry, 3.3.8 Accessible Authentication) noted as held *by construction* — no account, no login, no re-entry ([design-system 09 §1, §2](../../docs/design-system/09-accessibility-and-inclusivity.md)).
- **The `Semantics` convention layer:** a documented label/value/role pattern (`label`/`hint`/`button`/`header`), `MergeSemantics` for multi-part items (a page-card read as one phrase, not three fragments), `ExcludeSemantics` for decoration, and the `SemanticsService` announce path (with `TextDirection.rtl`) for background state changes ("catch-up plan ready", "page graded", "sign-off recorded") — labels resolve through the ARB set so the reader speaks the active locale ([design-system 09 §7](../../docs/design-system/09-accessibility-and-inclusivity.md); [engineering 12 §7](../../docs/engineering/12-localization-rtl-accessibility-impl.md)).
- **Dynamic text scaling via `TextScaler`:** chrome flows through Flutter's automatic `MediaQuery.textScalerOf` path; `MediaQuery.withClampedTextScaling` used only where a dense row would clip, as a documented ceiling — never a fixed pixel font, never `TextScaler.noScaling` on user-facing text; verified legible/usable to 200%+ (and the iOS AX sizes) without clipping or horizontal scroll ([design-system 09 §5](../../docs/design-system/09-accessibility-and-inclusivity.md); [engineering 12 §7](../../docs/engineering/12-localization-rtl-accessibility-impl.md)).
- **The muṣḥaf scaling exclusion seam:** a documented, tested boundary asserting the glyph layer is never OS-scaled or reflowed; reader enlargement is the dedicated zoom transform only — the contract the E13 reader is built against ([design-system 09 §5](../../docs/design-system/09-accessibility-and-inclusivity.md); [PRD R1, §11.2](../../docs/PRD.md)).
- **A reduce-motion substitution convention:** honor the OS Reduce Motion preference (`MediaQuery.disableAnimations`), substituting an instant/cross-fade state change for any non-essential motion — the calm-receipt recite confirmation, banner reveals — with no celebratory or flashing motion anywhere by policy ([design-system 09 §9](../../docs/design-system/09-accessibility-and-inclusivity.md); [PRD R3, C6](../../docs/PRD.md)).
- **The never-color-alone enforcement convention:** a reusable redundant-encoding pattern (color + shape/pattern + label/number) for every state chip (track, due, weak, sign-off, decay) plus the widget-audit/lint that fails a color-only state, applied to the shell's placeholder states and documented as the rule E15's heat-map and E12's cards inherit ([design-system 09 §4](../../docs/design-system/09-accessibility-and-inclusivity.md); [engineering 12 §7](../../docs/engineering/12-localization-rtl-accessibility-impl.md)).
- **The PR-blocking accessibility audit harness:** widget-test wiring for the automated Flutter guidelines — `textContrastGuideline`, `androidTapTargetGuideline` (48×48 dp), `iOSTapTargetGuideline` (44×44 pt), `labeledTapTargetGuideline` — plus `SemanticsTester`-style label assertions, run on every PR as a CI gate that extends E01's lanes; and the per-locale RTL screen-reader-traversal golden/test that proves focus and reading order run right-to-left, top-to-bottom in `fa`/`ckb`/`ar` ([design-system 09 §10](../../docs/design-system/09-accessibility-and-inclusivity.md); [engineering 12 §8](../../docs/engineering/12-localization-rtl-accessibility-impl.md); [PRD §20](../../docs/PRD.md)).
- **The recorded manual-pass procedure (A9):** a documented, repeatable TalkBack + VoiceOver in-language checklist over the cold-start → first day → review → catch-up path — the human pass that catches the RTL/multilingual 20% the automated gates cannot, marked release-blocking and owned at E20 release readiness ([design-system 09 §10](../../docs/design-system/09-accessibility-and-inclusivity.md)).

### Out of scope

- **The contrast token *values* and the theme palettes** (Light / Sepia / Dark) that must clear the ≥4.5:1 / ≥3:1 floors — authored where the tokens live → **E06** (Mihrab design-system foundation) and re-audited per appearance there; this epic owns only the floor as a gated check, not the color values ([design-system 09 §3](../../docs/design-system/09-accessibility-and-inclusivity.md)).
- **The 48 dp touch-target *implementation* in components** (the grade band, stumble-line hit areas) → **E10** (Mihrab component library) and the consuming feature epics; this epic ships the gate that proves the floor, not the sized widgets ([design-system 09 §6](../../docs/design-system/09-accessibility-and-inclusivity.md)).
- **The ARB / `gen_l10n` string pipeline, the `ckb` localizations delegate, per-locale numerals, ICU plural/select, bidi-isolation helper, and the physical-side / ASCII-digit grep gates** → **E09** (localization & RTL foundation); E08 *consumes* localized labels and the shared per-locale RTL goldens, and owns only the screen-reader/semantics half of the shared gate ([engineering 12 §1–§6, §8](../../docs/engineering/12-localization-rtl-accessibility-impl.md)).
- **The retention heat-map, its cells, and its per-cell labels** → **E15** (progress & heat-map); the muṣḥaf reader and its zoom/theme transform → **E13**; the recite/grade flow's reveal-on-tap and grade-band semantics → **E12** — each *applies* this epic's conventions; none is built here.
- **The final release trust-pack and the executed A9 manual pass on real devices in all three languages** → **E20** (release readiness); E08 authors the procedure and wires the automated gates, E20 runs the human pass as a release gate.
- **Reduce-motion's underlying motion/haptics token budget** (durations, the no-celebratory-motion rule) is owned by the Mihrab motion foundation → **E06**; this epic only depends on it and adds the substitution convention.

### Enables

- **E09 localization-rtl-foundation** — the screen-reader/semantics half of the shared localization & accessibility gate (the RTL focus/reading-order invariant is co-owned); E09 supplies the localized labels these `Semantics` annotations speak.
- **E10 mihrab-component-library** — every component is authored against the label/value/role, merge/exclude, touch-floor, and never-color-alone conventions this epic fixes, with the audit harness already green.
- **E11–E16 feature epics** (onboarding/cold-start, today & recite-grade, muṣḥaf reader, mutashābihāt, progress & heat-map, settings/profiles) — each rides the a11y spine: the muṣḥaf scaling-exclusion seam (E13), the never-color-alone heat-map (E15), the announce path for catch-up/grade/sign-off state (E12), and the per-screen audit gate that blocks their PRs.
- **E20 release-readiness** — the recorded A9 manual TalkBack/VoiceOver in-language procedure becomes the release-blocking human pass over the cold-start → review → catch-up journey.

## Definition of Done

- [ ] The audit harness is a real PR gate: a deliberately unlabeled control, a sub-48 dp target, a below-floor contrast pair, and a color-only state each fail their `meetsGuideline`/audit check (non-zero) and block the PR; revert leaves the gate green — proving the gate checks something, not nothing.
- [ ] **Text fidelity preserved:** the muṣḥaf glyph layer is never OS-scaled, re-typeset, or reflowed for any a11y goal; the scaling-exclusion seam test asserts the layout data is unchanged under text scale; the screen reader announces the page *reference* (surah/ayah/juz), never the QPC glyph codepoints ([design-system 09 §5, §7](../../docs/design-system/09-accessibility-and-inclusivity.md); [PRD R1, §11.2](../../docs/PRD.md)).
- [ ] **Offline / no-network by construction:** every accessibility helper is local layout — nothing here touches the network or any model; the throwing-`HttpOverrides` offline guard stays installed; the no-network and banned-import gates stay green ([PRD C1, §17](../../docs/PRD.md)).
- [ ] **No AI / no microphone:** no a11y feature adds an ML/ASR/audio dependency or requests a microphone; voice-input independence is preserved and noted as the banked win it is ([PRD C2, R5](../../docs/PRD.md)).
- [ ] **RTL + fa/ckb/ar localization:** every `Semantics` label resolves through the ARB set (no English label in an fa/ckb/ar build); each text run carries its `locale` so the reader speaks the right voice; focus/reading order = visual order is a tested invariant in all three locales; the goldens load the real bundled UI fonts, never `Ahem` ([design-system 09 §7, §8](../../docs/design-system/09-accessibility-and-inclusivity.md); [engineering 12 §8](../../docs/engineering/12-localization-rtl-accessibility-impl.md)).
- [ ] **Accessibility (this epic's subject):** dynamic text scaling to 200%+ without clipping; reduce-motion honored with calm substitutions; never-color-alone enforced; every interactive element labeled and ≥48 dp/44 pt; the conformance claim is honestly WCAG 2.2 AA, not "accessible" unqualified ([design-system 09 §1, §4, §5, §9, §10](../../docs/design-system/09-accessibility-and-inclusivity.md)).
- [ ] **Sect-neutral adab:** no streak, score, scoreboard, celebratory or flashing motion, or guilt/loss surface is introduced by any a11y affordance; reduce-motion substitutions keep the calm receipt calm, never a removed delight that re-appears as a celebration ([PRD R3, C6](../../docs/PRD.md); domain-adab-and-religious-integrity).
- [ ] **Tests:** the audit gate runs in CI on every PR at the cheapest tier (widget `meetsGuideline` + label assertions); the per-locale RTL traversal test is green; the A9 manual procedure is recorded and dated; coverage is published, never gated; no test asserts a coverage percentage ([engineering 12 §8](../../docs/engineering/12-localization-rtl-accessibility-impl.md); eng-write-dart-test).
- [ ] All four CI jobs green on the epic's final PR; the public logs show the accessibility gate results, so the auditable-by-strangers trust story holds for accessibility as it does for muṣḥaf integrity.

## Risks

- **Accessibility deferred to a "polish" phase.** The classic failure is filing a11y against a future release; by then twelve screens carry the same violation. *Mitigation:* the audit harness (E08-T07) is a PR gate from this epic forward, the conventions land before E10–E16, and the DoD's deliberate-violation proof (E08-T10) keeps the gate honest ([design-system 09 §1, §10](../../docs/design-system/09-accessibility-and-inclusivity.md)).
- **Automated checks create false confidence; the RTL/multilingual 20% slips through.** `meetsGuideline` catches the easy 80% but cannot judge intelligibility or RTL reading order in `fa`/`ckb`/`ar`. *Mitigation:* the per-locale traversal test (E08-T08) and the recorded human A9 pass (E08-T09, executed at E20) are mandatory and release-blocking, never replaced by the automated gate ([design-system 09 §10](../../docs/design-system/09-accessibility-and-inclusivity.md)).
- **A scaling helper leaks into the muṣḥaf.** A well-meant "make the page bigger with the OS setting" change would OS-scale or reflow the glyph layer and end the project. *Mitigation:* the scaling-exclusion seam (E08-T04) is a documented, tested boundary; the muṣḥaf enlarges only through E13's zoom transform; the reader is fed the page reference, never glyph codepoints ([design-system 09 §5, §7](../../docs/design-system/09-accessibility-and-inclusivity.md); [PRD R1](../../docs/PRD.md)).
- **A clamp becomes a disable.** Reaching for `TextScaler.noScaling` or a hard pixel size to "protect" a dense row is itself an accessibility failure. *Mitigation:* only `withClampedTextScaling` with a documented ceiling is permitted, the audit asserts legibility at 200%, and `ckb` (often the longest string) is the binding reflow case ([design-system 09 §5](../../docs/design-system/09-accessibility-and-inclusivity.md); [engineering 12 §7](../../docs/engineering/12-localization-rtl-accessibility-impl.md)).
- **Boundary blur with E09.** Semantics labels are *authored* as ARB strings (E09's pipeline) but *applied* as `Semantics` annotations (this epic); the RTL focus-order invariant is co-owned. *Mitigation:* E08 owns the screen-reader/semantics half of the shared gate and consumes E09's labels and per-locale RTL goldens; the split is recorded in both epics so neither re-implements the other ([engineering 12 §7, §8](../../docs/engineering/12-localization-rtl-accessibility-impl.md)).
- **A reduce-motion substitution quietly re-introduces a celebration.** Replacing a removed animation with a "nicer" fallback could smuggle in a streak/confetti delight. *Mitigation:* substitutions are instant/cross-fade only and the no-celebratory-motion policy holds; the adab conscience-check (domain-adab-and-religious-integrity) reviews any motion copy/affordance ([PRD R3, C6](../../docs/PRD.md)).
- **Auditing a mock instead of the real shell.** A harness wired against a toy widget proves nothing about the running app. *Mitigation:* this epic depends on E07; the conventions and gates are wired into the real `ShellRoute` shell and its first screens, so E10–E16 extend a spine already proven against a running app ([design-system 09 §10](../../docs/design-system/09-accessibility-and-inclusivity.md)).
