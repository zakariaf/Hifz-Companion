## Goal

The persistence handle is reachable through a single injectable seam, not a global. A small framework-free Dart interface — `PersistenceHandle` — names the persistence operations the rest of the app calls; the live implementation in `packages/data/` wraps the Drift `HifzDatabase` (E03-T03/T04). The interface is exposed as a Riverpod `Provider<PersistenceHandle>` (`persistenceProvider`) whose default body **throws** when read un-overridden, so a forgotten wiring is a loud startup failure rather than silent null data. The one place a live database is constructed is `app/`'s `main`, which opens `HifzDatabase` and supplies it via `ProviderScope(overrides: [...])`. For deterministic tests and previews, the boundary ships two hand-written doubles in `packages/data/`: an in-memory `PersistenceHandle` backed by `NativeDatabase.memory()` (a real Drift store with no file, no network, no migration of an on-disk DB), and a `FixedClock` implementing the injected `Clock` so every stored `CalendarDate` is a literal the test chose, never a value that drifts with the host clock or timezone. No `get_it`, no second DI container, no global singleton; the pure engine takes no injection and reads no clock. This task ships the boundary *shape* and *wiring* and the doubles — not the DAOs (E03-T06), not the `commitReview` transaction body (E03-T07).

## Implementation notes

TEST-FIRST for the two correctness-critical properties of a boundary (the rest is plain wiring): (a) the un-overridden `persistenceProvider` **throws** when read; (b) the `FixedClock` returns exactly the `CalendarDate` it was constructed with and never reads the host clock. Write `persistence_provider_test.dart`'s throwing-placeholder case and `fixed_clock_test.dart`'s literal-day case before the provider and the clock double exist.

1. **The interface — `PersistenceHandle` — lives below the boundary, framework-free.** Place `packages/data/lib/src/persistence_handle.dart` declaring `abstract interface class PersistenceHandle`. Its method signatures reference value types and DAOs/repositories only — **no `package:drift`/`package:sqlite3` symbol appears in the interface's public surface, and no `package:flutter`/`dart:io`**. Keep the surface minimal for this task: a handle that exposes the repositories the app reaches (e.g. `CardRepository get cards;`, `ReviewLogRepository get reviewLog;`, `ProfileRepository get profiles;`) and a `Future<void> close()`. Do not put query bodies or transaction logic here — those are E03-T06/T07. (If the repository *classes* are not yet authored, declare the handle against the interfaces/typedefs E03-T06 will fill and leave a `// E03-T06` marker; the handle is the seam, not the implementation.)

2. **The live implementation wraps `HifzDatabase`, in `data` only.** `packages/data/lib/src/live_persistence_handle.dart` declares `final class LivePersistenceHandle implements PersistenceHandle`, constructed over a `HifzDatabase` (E03-T03/T04). This is the one file that holds a Drift handle behind the interface; its repositories are built over the `HifzDatabase` instance. `LivePersistenceHandle` and `HifzDatabase` are the only places `package:drift`/`package:sqlite3` appear in the change — the banned-import gate (E01) keeps them out of `engine`/`features`/`quran`/`models`.

3. **`openAppDatabase()` is the live constructor, called only from `main`.** Expose `Future<HifzDatabase> openAppDatabase()` (or `Future<PersistenceHandle> openLivePersistence()`) from `data` — it builds the `LazyDatabase`/`NativeDatabase` with the E03-T04 pragmas and returns the live handle. It is **not** called at import time, in a provider body, or anywhere but the composition root, so no IO opens silently on a stray read.

4. **Expose the boundary as `persistenceProvider`, with a throwing placeholder.** In the shell DI module (`packages/app/lib/src/providers/` or alongside the other composition-root providers), declare:
   ```dart
   final persistenceProvider = Provider<PersistenceHandle>((ref) {
     throw StateError(
       'persistenceProvider was read without an override. '
       'Wire the live handle in main()\'s ProviderScope, or override it with an '
       'in-memory fake in tests.',
     );
   });
   ```
   The provider body is a thin wire holding **no business logic** — it only declares the seam and fails loudly if un-wired. Consumers (repositories, controllers) `ref.watch(persistenceProvider)`; nothing reaches the handle through a global.

5. **Wire the live handle exactly once in `main`.** In `packages/app/lib/main.dart`'s composition root: `WidgetsFlutterBinding.ensureInitialized();` → `final handle = await openLivePersistence();` → `runApp(ProviderScope(overrides: [persistenceProvider.overrideWithValue(handle)], child: const HifzApp()));`. This mirrors `04 §1.2` exactly (`appDatabaseProvider.overrideWithValue(db)`). No live database/handle is constructed anywhere else; no global mutable "current database" exists.

6. **The injected clock is the sibling boundary; this task ships its `FixedClock` double.** "Today" enters the system only through the injected `Clock` (E02), declared as `clockProvider` and read as `clock.today()` returning a `CalendarDate` — never `DateTime.now()`. This task does not redefine the `Clock` interface (E02 owns it) but **does** ship `FixedClock` in `packages/data/lib/src/testing/fixed_clock.dart` (or the agreed test-double location): `final class FixedClock implements Clock { const FixedClock(this._today); final CalendarDate _today; @override CalendarDate today() => _today; }`. It reads no host clock and no timezone. Every test/preview that touches persistence overrides `clockProvider` with a `FixedClock` so a stored `due_at`/`last_review_at` is a literal serial day the test chose.

7. **The in-memory fake is a real Drift store on `NativeDatabase.memory()`.** Ship `inMemoryPersistenceHandle()` (or `LivePersistenceHandle.inMemory()`) in `packages/data/lib/src/testing/` that builds a `HifzDatabase` over `NativeDatabase.memory()` and wraps it in the same `LivePersistenceHandle` — a genuine SQLite store with the v1 schema (`createAll` via `onCreate`), no file, no migration of an on-disk DB, no socket. Tests install it with `persistenceProvider.overrideWithValue(inMemoryPersistenceHandle())`. This is the fake the round-trip tests (E03-T06) and the transaction test (E03-T07) run against — fast, headless, deterministic, and identical in behaviour to the live store for SQL semantics. **No mock framework, no codegen-mocked DAO.**

8. **The doubles live in `data`, reachable by tests, never by the live app.** Put `FixedClock` and the in-memory handle factory under a clearly test-facing path (e.g. `packages/data/lib/src/testing/`) exported via a `package:hifz_data/testing.dart` barrel, so a feature/test imports the double explicitly and the production barrel (`package:hifz_data/data.dart`) does not pull `NativeDatabase.memory()` into a release binary. Do not gate them behind `kDebugMode` logic in production code paths; they are simply not referenced by the live composition root.

9. **The boundary moves bytes and returns values — it decides nothing.** `PersistenceHandle` never computes a schedule (that is the engine, reached only through a repository), never renders Quran text (that is `quran`), and never phones home. A mutating call routed through this handle persists in one Drift transaction and commits before any republish (the single write path, E03-T07); a teacher sign-off — a *sanad* act — is never acknowledged before its durable commit. This task does not implement that transaction; it provides the seam the transaction is built on and must not foreclose it (the handle exposes the repositories, not raw DAOs, to callers).

10. **Offline is structural here.** The persistence boundary imports **no** networking package — only `assets/` may open a socket. `grep` over the change finds no `package:http`/`package:dio`/`dart:io HttpClient` in `data`'s persistence files. Tests install the throwing `HttpOverrides` offline guard (E01 bootstrap) so any stray network call from a boundary fails loudly.

11. **Pitfalls to avoid.** A live `HifzDatabase` constructed in a provider body or at import (IO opens on a stray read — construct only in `main`); a `persistenceProvider` that defaults to a live database instead of throwing (silent IO, not a loud miss-wire); a Drift type (`HifzDatabase`, a `Table`, a `Selectable`) leaking through `PersistenceHandle`'s public surface or into the provider's type (re-couples the shell to Drift, fails the banned-import gate); `get_it`/`provider`/a second DI container, or a legacy Riverpod provider (`StateProvider`/`StateNotifierProvider` / `legacy.dart` import); a global mutable "current database" singleton; `DateTime.now()` anywhere reachable from the boundary or a test (use `FixedClock`); a unit test that opens a real on-disk file or hits the network instead of `NativeDatabase.memory()`; business logic (a query, a transaction, a schedule computation) inside the provider; a hardcoded user-facing failure string inside the handle (calm retry copy lives in `l10n` at the feature layer).

## Acceptance criteria

- [ ] `PersistenceHandle` exists as an `abstract interface class` in `packages/data/lib/src/persistence_handle.dart`; its public surface references value types / repositories only — `grep` finds no `package:drift`, `package:sqlite3`, `package:flutter`, or `dart:io` symbol in the interface file.
- [ ] `LivePersistenceHandle implements PersistenceHandle` wraps a `HifzDatabase` and lives in `packages/data/`; `LivePersistenceHandle` and `HifzDatabase` are the only places `package:drift`/`package:sqlite3` appear in the change; the banned-import gate is green (no Drift symbol in `engine`/`models`/`features`/`quran`).
- [ ] `persistenceProvider` is a `Provider<PersistenceHandle>` whose un-overridden body **throws** a descriptive `StateError` when read; it holds no business logic and no live IO.
- [ ] `main` (composition root) is the only place a live handle is constructed: `openLivePersistence()`/`openAppDatabase()` is called once, and `persistenceProvider.overrideWithValue(handle)` is the only override that wires it; no live database is constructed in a provider body, at import, or in any other file.
- [ ] A `FixedClock implements Clock` double exists, returns exactly the `CalendarDate` it was constructed with, and reads no host clock/timezone; every persistence test/preview overrides `clockProvider` with a `FixedClock`.
- [ ] An in-memory `PersistenceHandle` factory backed by `NativeDatabase.memory()` exists, creates the v1 schema (`createAll`), opens no file and no socket, and is installable via `persistenceProvider.overrideWithValue(...)`; the doubles live under a test-facing path/barrel and are not referenced by the live composition root.
- [ ] No `get_it`, no `provider` package, no Bloc, no legacy Riverpod provider import (`flutter_riverpod/legacy.dart`), and no global mutable "current database" anywhere in the change.
- [ ] The persistence boundary imports no networking package; `grep` over `data`'s persistence files finds no `package:http`/`package:dio`/`dart:io HttpClient`.
- [ ] Every `public` declaration on `PersistenceHandle`, `LivePersistenceHandle`, `FixedClock`, and the in-memory factory carries a `///` doc comment; `dart analyze --fatal-infos` and `dart format` are clean.

## Tests

All tests are pure `package:test` / `flutter_test` units under `packages/data/test/` and `packages/app/test/` as noted, run with `dart test`/`flutter test`; each file carries the REUSE SPDX `GPL-3.0-or-later` header, uses full-word names and typed `catch`, and installs the throwing-`HttpOverrides` offline bootstrap (E01) so any stray network call fails loudly. No test opens a real on-disk database or a socket — the in-memory double is the stack. Required files and cases:

- `packages/app/test/persistence_provider_test.dart` — **(written first)** Throwing placeholder: reading `persistenceProvider` from a bare `ProviderContainer()` (no overrides) throws a `StateError` whose message names the missing override (`expect(() => container.read(persistenceProvider), throwsA(isA<StateError>()))`). Override path: a `ProviderContainer(overrides: [persistenceProvider.overrideWithValue(inMemoryPersistenceHandle())])` yields the fake handle from `container.read(persistenceProvider)` with no throw.
- `packages/data/test/testing/fixed_clock_test.dart` — **(written first)** `FixedClock(CalendarDate(...)).today()` returns exactly the constructed `CalendarDate` (value-equal), and two reads return the same value; the suite reads no host clock (`DateTime.now()` appears nowhere). A type pin asserts `today()` is a `CalendarDate`, not a `DateTime`.
- `packages/data/test/testing/in_memory_persistence_handle_test.dart` — `inMemoryPersistenceHandle()` opens over `NativeDatabase.memory()`, exposes the v1 schema (a trivial insert→select round-trip through one repository surface, or `customSelect` of `sqlite_master` proving the tables exist), opens no file and no socket, and `close()` completes cleanly. (Full value→row→value round-trips are E03-T06; this asserts the double is a live, schema-bearing store.)
- `packages/data/test/persistence_handle_boundary_test.dart` — A type/grep guard: `PersistenceHandle`'s public members expose only value types/repositories (no Drift `GeneratedDatabase`/`Table`/`Selectable` in the signatures), proving the seam does not leak a Drift symbol. Pair with the CI banned-import assertion that `package:drift`/`package:sqlite3` appear only in `data`.
- Offline guard (all suites): the throwing `HttpOverrides` from the shared bootstrap is installed; a control case (or the bootstrap itself) confirms a stray `HttpClient` call throws — no persistence-boundary test reaches the network.
