## Goal

The six user-record value types — `Profile`, `Card`, `ReviewLog`, `ConfusionEdge`, `CycleConfig`, `LineBlock` — exist in the pure `models` package (`packages/models/lib/src/records/`), each as an immutable `@immutable` class with `final` fields, a `const` constructor, and a hand-written `copyWith`, importing `dart:core` / `package:meta` only — no `package:drift`, no `package:flutter`, no `dart:io`. Closed value sets (`ProfileRole`, `ReviewTrack`, `ReviewGrade`, `GradeSource`, `ProfileLocale`) are `enum`s so invalid states are unrepresentable. Scheduling days (`dueAt`, `lastReviewedDay`) are typed `CalendarDate` (E02); true event instants (`reviewedAtInstant`, `createdAtInstant`, `lastConfusedAtInstant`) are typed `DateTime`; every quantity carries its unit in the name (`stabilityDays`, `cycleCeilingDays`, `dailyBudgetMinutes`, `nearWindowJuz`). The set models exactly PRD §10.2 and the engineering 05 §2 user-table column set — no record encodes a streak, score, badge, or any Quran/factual claim, and no derived health roll-up is a field. These are the plain value types the `data` DAOs (E03-T06) map rows to and from; this task ships the types and their construction/round-trip units only, no Drift symbol.

## Implementation notes

TEST-FIRST is **not** required for the whole task — these are plain data carriers, not correctness-critical arithmetic. But the two correctness-critical *boundaries* a record can get wrong are pinned with a unit written first: (a) a scheduling day is `CalendarDate`, never `DateTime` (the DST off-by-one class the epic exists to remove); (b) `copyWith` preserves every field it is not asked to change. Write `card_test.dart`'s `copyWith`-identity and `dueAt is CalendarDate` cases before the `Card` body.

1. **Files**: one primary type per file under `packages/models/lib/src/records/` — `profile.dart` → `Profile`, `card.dart` → `Card`, `review_log.dart` → `ReviewLog`, `confusion_edge.dart` → `ConfusionEdge`, `cycle_config.dart` → `CycleConfig`, `line_block.dart` → `LineBlock`. Enums live beside the record that owns them (e.g. `ReviewTrack`/`ReviewGrade`/`GradeSource` in or beside `card.dart`/`review_log.dart`), or in a small `records/enums.dart` if shared — no `utils/`/`core/` junk-drawer. Re-export each from the package barrel `packages/models/lib/models.dart` (`export 'src/records/card.dart' show Card, ReviewTrack;` …).
2. **Imports**: `package:meta/meta.dart` only (for `@immutable`), plus the import that brings in `CalendarDate` from E02. **No** `package:drift`, **no** `package:flutter`, **no** `dart:io`, **no** `package:intl`/`hijri`/`shamsi_date`. The banned-import gate (E01) enforces `models` purity; this file set is part of the audit evidence.
3. **`CalendarDate` dependency (resolve, don't invent)**: the records' day fields must reference `CalendarDate`, which E02-T01 placed in the `engine` package's `dates/` tree, while `engine` depends on `models` (not the reverse) — so `models` cannot import from `engine` as-is. The arch doc Layer-0 list (01 §2, §3.1) names `CalendarDate` a Layer-0 value type, i.e. it belongs at the `models` level. Treat the resolution as owned upstream: either `CalendarDate` is sourced from `models` (with `engine` re-exporting it) or both depend on a shared pure base — file the import-direction decision against E01/E02 rather than duplicating the type. Whichever lands, this task imports `CalendarDate` by its single canonical name and never re-implements epoch-day math here.
4. **Immutability shape**: each record is `@immutable`, all fields `final`, a single `const` constructor with `required`/defaulted named params, and a hand-written `copyWith` returning a new instance (every field threaded `field ?? this.field`). No `freezed` is required (the standard permits it; hand-written is simpler for six small records and keeps the package dependency-free). `///` on the class and every public field — summary first, then units/edge behavior; document why `dueAt` is nullable (an `UNMEMORIZED` card has no due day).
5. **Closed sets as enums (mirror the schema `CHECK`s exactly)**: `ProfileRole { self, student, child }`; `ProfileLocale { ar, fa, ckb }`; `ReviewTrack { newPage, near, far, unmemorized }` (wire-values `NEW`/`NEAR`/`FAR`/`UNMEMORIZED` — the Dart name avoids the `new` keyword; the row mapping is the DAO's job, E03-T06); `ReviewGrade { again, hard, good, easy }`; `GradeSource { self, teacher }`. Each enum's members must match the `CHECK (... IN (...))` set in 05 §2 one-for-one so E03-T03's table and this task never drift.
6. **`Card`** mirrors `card` (05 §2 / PRD §10.2): `ProfileId profileId`, `int pageId`, `ReviewTrack track`, `double difficulty` (the FSRS `D`, 1–10 — the unit/role is in the name, not bare `d`), `double stabilityDays` (the FSRS `S` in days, ≥ 0), `CalendarDate? lastReviewedDay`, `CalendarDate? dueAt` (nullable — null exactly when `track == unmemorized`), `int reps`, `int lapses`, `bool isWeak`, `int signoffs`, `bool hasManualLock`, `bool isPrayerCritical`, `bool isEnabled`. Do **not** add a stored `retrievability`/health field — `R` and juz/page health are computed from `card` on read by the engine/feature layer, never persisted (05 §2; PRD §10.3).
7. **`ReviewLog`** mirrors `review_log`: `LogId logId`, `ProfileId profileId`, `int pageId`, `DateTime reviewedAtInstant` (UTC), `ReviewTrack trackAtReview`, `ReviewGrade grade`, `List<int>? errorLineIndices` (the decoded `error_lines_json` — line indices only, never Quran text), `int elapsedDays` (the `CalendarDate`-serial delta fed to the curve, an `int`, not a `Duration`), `double? predictedRetrievability`, `double? stabilityDaysBefore`, `double? stabilityDaysAfter`, `double? difficultyBefore`, `double? difficultyAfter`, `GradeSource source`, `String? teacherLabel`. The value type carries **no** mutation method beyond `copyWith` for construction; append-only is enforced by the DAO's absence of update/delete (E03-T06), not by this type.
8. **`Profile`/`ConfusionEdge`/`CycleConfig`/`LineBlock`**: `Profile` — `ProfileId profileId`, `String displayName` (the only PII, PRD §17), `ProfileRole role`, `ProfileLocale locale`, `String mushafId`, `DateTime createdAtInstant` (UTC), `Map<String, Object?>? settings` (the decoded `settings_json`, schema-shaped, never health/Quran). `ConfusionEdge` — `ProfileId profileId`, `String ayahA`, `String ayahB` (canonical `ayahA < ayahB` — assert in the constructor; the schema `CHECK` is E03-T03), `double weight`, `DateTime? lastConfusedAtInstant` (UTC). `CycleConfig` — `ProfileId profileId`, `String cycleType`, `int newLinesPerDay`, `int nearWindowJuz`, `int farTargetPerDay`, `int cycleCeilingDays` (`far_cycle_days`, the cycle ceiling, PRD §7.6 — named for what it *is*, not its column), `int dailyBudgetMinutes`, `bool isPureCycleMode`, `String termLabelSet`, `String? regionPreset`. `LineBlock` — `BlockId blockId`, `ProfileId profileId`, `int pageId`, `int lineStart`, `int lineEnd` (both 1–15), `int errorCount`.
9. **Identifier types**: prefer thin typed-id wrappers (`ProfileId`, `LogId`, `BlockId` over UUID `String`s) so a `profileId` is never confused with a `mushafId` at a call site — small `@immutable` `extension type` or value classes wrapping `String`. If a typed-id scaffold already lands in E01/E03-T02, reuse it; otherwise define them here beside the records. Keep `pageId`/`ayahA`/`ayahB` as their schema types (`int`/`String`) since they key into the read-only reference tables E03-T02 owns.
10. **`app_meta` is not a record here**: the `app_meta(key, value)` table is a string key/value singleton store, not a per-profile domain record; it needs no `models` value type and is handled directly by its DAO (E03-T03/T06). Do not author an `AppMeta` value class in this task.
11. **Pitfalls to avoid**: a `DateTime` field for a scheduling day, or a `CalendarDate` for a true instant (both are review-blocking — name *and* type must agree); a bare `d`/`s`/`budget`/`target` without its unit; a `String track`/`grade` instead of an enum; a mutable field or a missing `copyWith` field (the `copyWith`-identity test catches the latter); a stored `retrievability`/health/streak/score field (no derived authority, no gamification — PRD §10.3, R3/C6); reconstructing Quran text into `errorLineIndices`/`settings`; a `throw` from a record (value types are total); a `package:drift` or `package:flutter` import sneaking into `models`; a hardcoded user-facing label (role/locale/term-set/verdict strings live in `l10n`, not in a record).

## Acceptance criteria

- [ ] Six record types exist under `packages/models/lib/src/records/` (`Profile`, `Card`, `ReviewLog`, `ConfusionEdge`, `CycleConfig`, `LineBlock`), each `@immutable` with `final` fields, a `const` constructor, and a `copyWith`; each re-exported from `packages/models/lib/models.dart`.
- [ ] The `models` package imports `dart:core`/`package:meta` and the `CalendarDate` dependency only — `grep` over `packages/models/` finds no `package:drift`, `package:flutter`, `dart:io`, `package:intl`, `hijri`, or `shamsi_date` import; the banned-import gate is green.
- [ ] Closed sets are enums — `ProfileRole`, `ProfileLocale`, `ReviewTrack`, `ReviewGrade`, `GradeSource` — and each enum's members match the corresponding `CHECK (... IN (...))` set in 05 §2 one-for-one (no free `String` for `track`/`grade`/`source`/`role`/`locale`).
- [ ] Scheduling-day fields (`Card.dueAt`, `Card.lastReviewedDay`) are `CalendarDate` (nullable where the schema allows); true instants (`ReviewLog.reviewedAtInstant`, `Profile.createdAtInstant`, `ConfusionEdge.lastConfusedAtInstant`) are `DateTime` named as instants; `ReviewLog.elapsedDays` is an `int`, not a `Duration` — verifiable by a unit asserting the static types.
- [ ] Every quantity carries its unit/role in the name (`difficulty`, `stabilityDays`, `cycleCeilingDays`, `dailyBudgetMinutes`, `nearWindowJuz`, `newLinesPerDay`); no bare `d`/`s`/`budget`/`target`/`cycle`. Booleans read as assertions (`isWeak`, `hasManualLock`, `isPrayerCritical`, `isEnabled`, `isPureCycleMode`).
- [ ] No record has a stored `retrievability`/health/streak/badge/score field; no record encodes a Quran/factual claim; `Profile.displayName` is the only PII (PRD §17). `Card.dueAt` is nullable and documented as null exactly for an `UNMEMORIZED` card.
- [ ] `ConfusionEdge` asserts (or documents and round-trip-proves) the canonical `ayahA < ayahB` ordering; `LineBlock.lineStart`/`lineEnd` and `Card.difficulty`/`stabilityDays` ranges are documented (the enforcing `CHECK`s are E03-T03's, referenced).
- [ ] `///` doc on every public class and field passes `dart analyze --fatal-infos` (`public_member_api_docs`); `dart format` clean; no `throw` from any record; no `!`/`late`/`dynamic` on a record field.

## Tests

All tests are pure `package:test` units under `packages/models/test/records/`, run with `dart test` (no `flutter_test`, no widget binding — these are Layer-0 value types). Each file carries the REUSE SPDX `GPL-3.0-or-later` header and uses full-word names. The throwing-`HttpOverrides` offline bootstrap (E01-T06) is not needed for pure value-type units but the no-network posture holds by construction (no IO import). Required files and cases:

- `card_test.dart` — **(written first)** `copyWith` identity: a `copyWith()` with no args is field-equal to the original; a `copyWith(dueAt: …)` changes only `dueAt` and preserves every other field. Type pins: `Card(...).dueAt` is a `CalendarDate?` and `Card(...).stabilityDays` is a `double` (a `static`-type assertion, e.g. via a typed local, so a future `DateTime` `dueAt` fails to compile/assert). Construct an `UNMEMORIZED` card with `dueAt: null` and a `FAR` card with a concrete `CalendarDate dueAt`.
- `review_log_test.dart` — construct a self-graded and a teacher-graded `ReviewLog`; assert `reviewedAtInstant` is a `DateTime` (UTC by contract) and `elapsedDays` is an `int`; `copyWith` identity; `errorLineIndices` holds line indices only (a `List<int>?`), never text. Assert the value type exposes no `delete`/`update`/setter — only `copyWith` (append-only is the DAO's enforcement, but the record offers no mutation path either).
- `profile_test.dart` — `ProfileRole`/`ProfileLocale` round-trip through their wire strings (helper or inline map) match the 05 §2 `CHECK` sets exactly; `copyWith` identity; `settings` is a `Map<String, Object?>?` with no health/Quran shape.
- `confusion_edge_test.dart` — the `ayahA < ayahB` canonical-ordering invariant holds (constructing the reverse pair is rejected or normalized — pin whichever the implementation chooses); `copyWith` identity; `lastConfusedAtInstant` is a nullable `DateTime`.
- `cycle_config_test.dart` / `line_block_test.dart` — `copyWith` identity for every field; `cycleCeilingDays`/`dailyBudgetMinutes`/`nearWindowJuz` are unit-named `int`s; `LineBlock.lineStart`/`lineEnd` documented 1–15. Assert no streak/score/health field exists on either (a reflective or explicit field-name check, or simply that the public surface matches the documented field list).
- `enums_test.dart` — every enum's member set equals the `CHECK (... IN (...))` literal set from 05 §2 (a frozen list compared one-for-one), so a future schema/enum drift fails this unit.

No golden, widget, or `integration_test` is in scope — these are value types; rendering/RTL/numeral goldens belong to the feature epics that display them. The value→row→value round-trip (proving `CalendarDate`-serial identity through SQLite) is E03-T06, layered on these types.

## Definition of Done

- [ ] All acceptance criteria met; `dart test` green for `packages/models/`; `dart analyze --fatal-infos` and `dart format --set-exit-if-changed` clean; CI green.
- [ ] **Offline / no-network**: `models` imports no networking package and no `dart:io`; the records hold no telemetry/account/off-device field; no per-user data path is introduced (verifiable by grep + the banned-import gate).
- [ ] **No-AI / no-microphone**: no record references audio, a microphone, a recognizer, or any ML/AI artifact; `grade`/`source` carry only the `self`/`teacher` self-rating-or-teacher-sign-off signal (a `GradeSource`, never an inferred score).
- [ ] **Quran text fidelity**: no record stores or reconstructs Quran text — `errorLineIndices` is line indices, `settings`/`error_lines_json` decode to small schema-shaped non-Quran data only; the reference structure these records key into (`pageId`, `ayahA`/`ayahB`) is read-only and owned by E03-T02 (R1).
- [ ] **RTL + fa/ckb/ar strings**: no user-facing string is hardcoded in any record — role/locale/term-set/verdict labels live in `l10n` (`ar` template, `fa`/`ckb`), rendered RTL via `Directionality` at the feature layer; the records carry only enum/value data (§1.1 rule 4).
- [ ] **Accessibility**: N/A by construction (no widget, no rendered surface) — the records are data carriers; a11y labels for any field's display are the owning feature's responsibility, documented at the call site, not here.
- [ ] **Sect-neutral adab**: no record encodes a streak/badge/score/confetti, a "safe to drop" flag, a fiqh ruling, a tafsīr/translation, or any sect/madhhab marker; `mushafId`/`riwayah` is a neutral descriptor; `signoffs` is a *sanad* count, never a gamified tally (no record presents it as a reward).
- [ ] **Deterministic tests**: every test is pure `dart test`, constructs dates as literal `CalendarDate`s and instants as explicit UTC `DateTime`s (no `DateTime.now()`, no wall clock), asserts behaviour with full-word names and typed `catch`, and carries the REUSE SPDX header; the suite is reproducible across machines and timezones.
