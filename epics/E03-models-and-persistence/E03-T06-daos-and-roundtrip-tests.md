## Goal

The DAOs that map Drift rows ↔ the pure `models` value types live in `packages/data/lib/src/db/daos/`, one per user record (`profile`, `card`, `line_block`, `review_log`, `confusion_edge`, `cycle_config`) plus the `app_meta` key/value DAO and the read-only reference reads — and **no Drift symbol (`Companion`, `TableInfo`, a generated row class) ever crosses the `data` boundary**: every public method takes and returns `models` value types only. The `review_log` DAO exposes `insert` (append) and reads **only** — there is no `update`/`delete`/setter method on it, so the append-only audit trail is enforced by *absence*, reviewed in CI. Value→row→value round-trip tests under plain `dart test` assert byte-for-byte identity through SQLite — including the `CalendarDate`-serial day (`due_at`/`last_review_at`) and every enum (`track`/`grade`/`source`/`role`/`locale`) mapping — and a guard test asserts the `review_log` DAO surface has no mutation path. This is the row-mapping floor the single write path (E03-T07) and the cold-start seed (E03-T08) sit on; this task ships the mappers, the per-record read/append DAOs, and the round-trip + append-only tests only — the one-transaction `commitReview`/`seedColdStart` bodies are E03-T07/T08.

## Implementation notes

TEST-FIRST (correctness-critical): the two properties this task exists to hold are correctness-critical and their tests are written **first**, failing, before the mappers: (a) value→row→value is identity **including the `CalendarDate`-serial day and every enum** (a `DateTime`-instant masquerading as a scheduling day, or a silently coerced enum string, is the exact defect class the epic exists to remove); (b) the `review_log` DAO surface exposes **no** mutation path (append-only by absence). Write `card_round_trip_test.dart`'s serial-day-and-enum identity cases and `review_log_append_only_test.dart`'s no-mutation-surface assertion before authoring the mappers.

1. **Files** — one DAO per user record under `packages/data/lib/src/db/daos/`: `profile_dao.dart` → `ProfileDao`, `card_dao.dart` → `CardDao`, `line_block_dao.dart` → `LineBlockDao`, `review_log_dao.dart` → `ReviewLogDao`, `confusion_edge_dao.dart` → `ConfusionEdgeDao`, `cycle_config_dao.dart` → `CycleConfigDao`, and `app_meta_dao.dart` → `AppMetaDao` (the `(key,value)` singleton store — no `models` value type, maps to `String`/`String`). The read-only reference **reads** (used later by E05/E14) live in `reference_read_dao.dart` → `ReferenceReadDao` (select-only over `Pages`/`Lines`/`Ayat`/`Surahs`/`Mushafs`/`MutashabihGroups`/`MutashabihMembers`, mapping to the T02 reference DTOs) — **no insert/update/delete**. Use Drift's `DatabaseAccessor`/`@DriftAccessor` so each DAO is generated against `HifzDatabase` (E03-T03's `database.dart`). Do **not** export any DAO's Drift companions/row classes from the `data.dart` barrel — export the DAO types (and the repository interface from T05) only.

2. **The row↔value mappers are the whole job; keep them in `data`, beside the DAO.** Each DAO holds two private mappers: `_toModel(GeneratedRow row) → ModelValueType` and `_toCompanion(ModelValueType value) → TableCompanion` (a `Value(...)`-wrapped companion for insert/upsert). These are the only place a Drift symbol appears; the public method signatures take/return `models` types only. A free `String`/`int` column maps to its enum/typed-id via an explicit, total mapper — never a `dynamic` cast, never `.toString()` round-tripping.

3. **Enum mapping — wire-value tables, total both ways, mirror the §2 `CHECK` sets exactly.** Map each enum to/from its SQL wire string with a small total bidirectional table: `ReviewTrack.far ↔ 'FAR'`, `…newPage ↔ 'NEW'`, `…near ↔ 'NEAR'`, `…unmemorized ↔ 'UNMEMORIZED'`; `ReviewGrade {again,hard,good,easy} ↔ {'again','hard','good','easy'}`; `GradeSource {self,teacher} ↔ {'self','teacher'}`; `ProfileRole {self,student,child}`; `ProfileLocale {ar,fa,ckb}`. The wire set must equal the `CHECK (col IN (...))` literal set in 05 §2 one-for-one (the `Card.track` Dart name avoids the `new` keyword but maps to `'NEW'`). On decode, an unknown wire string is a typed mapping error, never a silent default — but a column reaching the mapper has already passed its `CHECK`, so this surfaces only schema/enum drift (which `enums_round_trip_test.dart` catches statically).

4. **`CalendarDate`-serial day mapping (the load-bearing correctness boundary).** `card.due_at`/`card.last_review_at` are `INTEGER` serial-day columns (nullable); map them with E02's `CalendarDate` ↔ serial-day-`int` conversion (`CalendarDate.fromSerial(int)` / `value.serialDay` — use the canonical names E02-T01 ships, do not reinvent epoch-day math). `review_log.elapsed_days` maps to a plain `int` (a `CalendarDate`-serial delta, not a `Duration`). **Never** read a scheduling-day column into a `DateTime` or build a `CalendarDate` from a wall-clock instant. True instants (`profile.created_at`, `review_log.reviewed_at`, `confusion_edge.last_confused_at`) map to UTC `DateTime` via ISO-8601 `String` (assert/normalize `.isUtc` on encode). The round-trip test pins that a `CalendarDate` written and read back is identical and that the column physically holds the serial `int`, not an instant.

5. **`review_log` DAO — append + read only; the append-only property is structural.** `ReviewLogDao` exposes exactly: `Future<void> insert(ReviewLog)` (the append; called inside E03-T07's transaction), `Future<List<ReviewLog>> forCard(ProfileId, int pageId)`, and any other **read** the audit/heat-map needs (`forProfile`, `latestForCard`) — and **nothing else**. There is **no** `update`, `delete`, `replace`, `deleteWhere`, `clear`, or exposed companion-update method, and none is generated by leaving them un-declared (do not use a Drift mixin that auto-generates an `update`/`delete` surface for this table). The only sanctioned bulk touch of `review_log` is export / one-tap erase, owned by E17 — not a DAO method here. The same enforced-by-absence rule that protects the reference tables (T02) protects this audit trail. A `// no update/delete — append-only` comment is documentation; the *absence of the method* is the enforcement.

6. **`float` columns map without coercion; no derived state crosses the mapper.** `card.d`/`card.s` and `review_log.r_predicted`/`s_before`/`s_after`/`d_before`/`d_after` are `REAL`; map them to `double` directly (the table is `STRICT`, so no silent `TEXT` coercion). No DAO computes, returns, or writes a `retrievability`/juz/page/ḥizb health roll-up — `R` and health are computed from `card` on read by the engine/feature layer (05 §2; PRD §10.3); a DAO that returns a pre-aggregated health number is the wrong layer. `*_json` columns (`settings_json`, `error_lines_json`) decode to the small schema-shaped value-type field (`Map<String,Object?>?`, `List<int>?`) and never to Quran text or a health fact.

7. **Reference reads map to T02's reference DTOs, read-only.** `ReferenceReadDao` holds `select`-only methods (`pageByNumber`, `linesForPage`, `ayatForPage`, `surahById`, `mushafById`, `mutashabihGroupsFor`, …) returning `Page`/`Line`/`Ayah`/`Surah`/`Mushaf`/`MutashabihGroup`/`MutashabihMember`. `Line.textGlyphRef` maps as an **opaque** `String` — never `.trim()`/`.split()`/`.toLowerCase()`/`.contains()`/logged (domain-mushaf-text-integrity). No insert/update/delete method exists on this DAO; the asset loader (E05) is the only writer, behind the checksum verifier.

8. **Errors are typed and confined to this boundary; nothing swallowed.** A decode that cannot map a row (an out-of-set enum that somehow bypassed its `CHECK`, a malformed `*_json`) throws the `data` layer's one sealed persistence error type (introduced with T05/T07; reuse it, do not invent a parallel one), surfaced to the feature layer to handle exhaustively. Type every `catch` (`on FormatException catch (e)`, never bare `catch (_)`); never swallow a mapping error, never `print`/log user data, no `!`/`late` on a persistence value. The mappers themselves are total in the normal case (a row that passed its `CHECK`s maps cleanly); throwing is the schema-drift escape hatch only.

9. **The DAOs are reached through T05's boundary; this task does not wire transactions or providers.** The repository interface and its Riverpod `Provider` (T05), the one-`db.transaction` `commitReview`/`seedColdStart` bodies (T07/T08), and any `Notifier`/`StreamProvider` controller (E07) are **not** this task. This task ships the per-record read/append DAO methods and their mappers, callable from those transaction bodies; a DAO method here performs a single statement, not a multi-row transaction.

10. **Pitfalls to avoid:** a Drift `Companion`/`TableInfo`/generated row class escaping `data` via a public method signature or the barrel (the determinism boundary break); an `update`/`delete`/`replace`/`clear` method on `ReviewLogDao` (the *sanad* breach — append-only by absence); reading a `due_at`/`last_review_at` serial-day column into a `DateTime`, or a `CalendarDate` built from `DateTime.now()` (the DST off-by-one); `==` on a `double` field in the round-trip test (use `closeTo(_, 1e-6)`); a `dynamic`/`.toString()` enum round-trip instead of a total wire-value table; a DAO returning a persisted health/juz roll-up (no derived authority — PRD §10.3); string-processing `text_glyph_ref`; a write DAO on any reference table; a swallowed/ bare-`catch` mapping error or a `print` of a row; a hardcoded user-facing label in a mapper (role/locale/term-set/verdict strings live in `l10n`).

## Acceptance criteria

- [ ] One DAO per user record exists under `packages/data/lib/src/db/daos/` (`ProfileDao`, `CardDao`, `LineBlockDao`, `ReviewLogDao`, `ConfusionEdgeDao`, `CycleConfigDao`, `AppMetaDao`) plus a select-only `ReferenceReadDao`; each is a Drift `DatabaseAccessor` over `HifzDatabase`; `dart run drift_dev` generates with no errors.
- [ ] **No Drift symbol crosses the `data` boundary**: every public DAO method takes and returns `models` value types (or `String`/`int` for `AppMeta`) — no `Companion`/`TableInfo`/generated row class appears in any public signature or is exported from `packages/data/lib/data.dart`; verifiable by grep + the banned-import gate (`models`/`engine`/`features`/`quran` import no `package:drift`/`package:sqlite3`).
- [ ] Each record's `_toModel`/`_toCompanion` mappers are total and column-faithful: every §2 column maps to exactly one field and back; enums map via a wire-value table whose set equals the §2 `CHECK (col IN (...))` literal set one-for-one (`ReviewTrack`/`ReviewGrade`/`GradeSource`/`ProfileRole`/`ProfileLocale`).
- [ ] Scheduling-day columns (`card.due_at`/`last_review_at`) map to/from `CalendarDate` via E02's serial-day conversion (nullable preserved); `review_log.elapsed_days` maps to `int`; true instants (`created_at`/`reviewed_at`/`last_confused_at`) map to UTC `DateTime` — proven by a round-trip unit asserting the column physically holds the serial `int`, not an instant.
- [ ] **`ReviewLogDao` exposes `insert` (append) + reads only — no `update`/`delete`/`replace`/`clear`/`deleteWhere`/companion-update method exists or is generated**; the append-only property is enforced by absence (reviewed in CI) and asserted by the no-mutation-surface guard test.
- [ ] `ReferenceReadDao` is select-only — no insert/update/delete against any reference table; `Line.textGlyphRef` is mapped opaque and never string-processed (R1).
- [ ] No DAO computes, returns, or writes a persisted health/juz/ḥizb/`R` roll-up; `*_json` columns decode to small schema-shaped value-type fields only, never Quran text or a health fact (PRD §10.3).
- [ ] Mapping errors throw the `data` layer's one sealed persistence error type, typed `catch` only, nothing swallowed; no `print`/log of user data; no `!`/`late` on a persistence value; `dart analyze --fatal-infos` and `dart format --set-exit-if-changed` clean; `///` on every public DAO method.

## Tests

All tests are pure `dart test` under `packages/data/test/daos/` — no `flutter_test`, no widget binding — over an in-memory `NativeDatabase.memory()` `HifzDatabase` (`createAll()` to materialize the schema, reference rows seeded via a raw `customStatement`/`batch` confined to the test, never a shipped write DAO). The throwing-`HttpOverrides` offline guard (E01) is installed in the `data` test bootstrap so a stray socket fails loudly. Days are literal `CalendarDate`/`SerialDay`; instants are explicit UTC `DateTime`s; no `DateTime.now()`, no wall clock. Each file carries the REUSE SPDX `GPL-3.0-or-later` header and full-word names. Required files and cases:

- `card_round_trip_test.dart` — **(written first)** value→row→value identity for `Card`: insert a `FAR` card with a concrete `CalendarDate dueAt`/`lastReviewedDay`, read it back, assert field-equality with the original; `d`/`s` asserted `closeTo(_, 1e-6)` (never `==`). **Serial-day pins**: assert the read-back `dueAt` is the *same* `CalendarDate` and that the raw column (`SELECT due_at`) holds the serial `int`, not an instant; an `UNMEMORIZED` card round-trips `dueAt: null`. **Enum pin**: every `ReviewTrack` value round-trips through its wire string.
- `review_log_round_trip_test.dart` — value→row→value identity for a self-graded and a teacher-graded `ReviewLog`: `reviewedAtInstant` round-trips as UTC `DateTime`, `elapsedDays` as `int`, `grade`/`trackAtReview`/`source` through their wire strings, `errorLineIndices` as a `List<int>?` (line indices only, never text), the `r_predicted`/`s_*`/`d_*` floats `closeTo(_, 1e-6)`.
- `review_log_append_only_test.dart` — **(written first)** the correctness-critical guard: assert `ReviewLogDao`'s public surface exposes **no** mutation path — a source-grep/reflection check that the type declares no `update`/`delete`/`replace`/`clear`/`deleteWhere`/companion-update method, and that no such method is reachable through the `data.dart` barrel; the intent — "the audit trail is append-only; nothing decays silently" — is named in the test. `insert` then `forCard` returns the appended row; there is no API to mutate or remove it.
- `profile_confusion_cycle_round_trip_test.dart` — round-trip identity for `Profile` (`role`/`locale` enums, `createdAtInstant` UTC, `settings` map), `ConfusionEdge` (the `ayahA < ayahB` canonical ordering survives, `weight` `closeTo`, `lastConfusedAtInstant` nullable UTC), `CycleConfig` (`cycleCeilingDays`/`dailyBudgetMinutes`/`nearWindowJuz` unit-named `int`s, `isPureCycleMode` bool), and `LineBlock` (`lineStart`/`lineEnd`, `errorCount`).
- `app_meta_dao_test.dart` — `(key,value)` set/get round-trips a `String`/`String` pair; reading a missing key returns null (not a throw); no per-profile/domain shape leaks in.
- `reference_read_dao_test.dart` — seed reference rows via raw `customStatement`, then `ReferenceReadDao` reads map to the T02 DTOs (`Page`/`Line`/`Ayah`/`Surah`/`Mushaf`/`MutashabihGroup`/`MutashabihMember`); assert `Line.textGlyphRef` is preserved verbatim (opaque-glyph: equality, not normalization) and that `ReferenceReadDao` declares no insert/update/delete.
- `enums_round_trip_test.dart` — every enum's wire-value table maps both ways for every member and equals the §2 `CHECK (... IN (...))` literal set one-for-one, so a future schema/enum drift fails this unit statically.

No golden, widget, or `integration_test` is in scope — these are row↔value mappers, not a rendered surface; RTL/numeral/muṣḥaf goldens belong to the feature epics that display the mapped data. The one-transaction `commitReview` rollback unit is E03-T07; the `seedColdStart` all-or-nothing unit is E03-T08; the migration `integrity_check` fixture is E03-T09.
