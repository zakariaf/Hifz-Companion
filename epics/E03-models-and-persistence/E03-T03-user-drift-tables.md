## Goal

The seven read-write user tables exist as `STRICT` Drift table classes in `packages/data/lib/src/db/tables/user/` — `Profiles`, `Cards`, `LineBlocks`, `ReviewLog`, `ConfusionEdges`, `CycleConfigs`, `AppMeta` — each generating **exactly** the v1 DDL of `05-persistence-and-encryption.md` §2 / PRD §10.2: every enum column constrained by `CHECK (... IN (...))` (`role`, `track`, `grade`, `source`), every range constrained by `CHECK` (`d BETWEEN 1 AND 10`, `s >= 0`, the counters `>= 0`, the boolean flags `IN (0,1)`), every per-profile child carrying a foreign key with `ON DELETE CASCADE`, the load-bearing engine invariant `CHECK (track = 'UNMEMORIZED' OR due_at IS NOT NULL)` on `card`, the canonical `CHECK (ayah_a < ayah_b)` on `confusion_edge`, and the four indices `card_due`, `review_log_by_card`, `line_block_by_card` (+ the reference `ayah_by_page`/`line_by_page` already owned by T02). `due_at`/`last_review_at` are `INTEGER` `CalendarDate` serial days; `reviewed_at`/`created_at`/`last_confused_at` are UTC `TEXT` instants. All seven are wired into the `@DriftDatabase(tables: [...])` list on `HifzDatabase` alongside T02's reference tables. This task owns only the schema (tables, columns, constraints, indices, registration) — no DAO, no transaction body, no pragmas.

## Implementation notes

This task is schema authoring. The `CHECK`/FK invariants are correctness-critical and **defended by test alongside the schema** (see Tests): every documented `CHECK` must be proven to reject the out-of-range value, and the FK cascade proven to delete children — write those tests with the table classes, not after.

1. **User table classes in `data`** — `packages/data/lib/src/db/tables/user/` (one file per table: `profiles.dart`, `cards.dart`, `line_blocks.dart`, `review_log.dart`, `confusion_edges.dart`, `cycle_configs.dart`, `app_meta.dart`), matching T02's per-table file layout under `tables/reference/`. Each is a Drift table class generating **exactly** the §2 DDL. Use `@DataClassName(...)` and column-name overrides (`.named('snake_case')` or `@JsonKey`-style mapping) so the generated SQLite columns are byte-identical to the documented snake_case DDL, and mark each table `STRICT` (Drift `bool get isStrict => true;`). Transcribe every constraint into the table definition — do **not** validate in Dart.

2. **`Profiles`** — `profileId` TEXT PK (UUID); `displayName` TEXT NOT NULL (the only typed "PII", PRD §17); `role TEXT NOT NULL CHECK (role IN ('self','student','child'))`; `locale TEXT NOT NULL CHECK (locale IN ('ar','fa','ckb'))`; `mushafId TEXT NOT NULL REFERENCES mushaf(mushaf_id)` (the FK targets T02's `Mushafs`); `createdAt TEXT NOT NULL` (UTC ISO-8601 instant — a `String`, *not* a `DateTime` column, and *not* a `CalendarDate`); `settingsJson TEXT` nullable (decode-validated; never health/Quran facts). `STRICT`.

3. **`Cards`** — composite `PRIMARY KEY (profile_id, page_id)`; `profileId TEXT NOT NULL REFERENCES profile(profile_id) ON DELETE CASCADE`; `pageId INTEGER NOT NULL REFERENCES page(page_id)` (T02's `Pages` — **no** cascade: a profile owns its cards, the immutable page does not); `track TEXT NOT NULL CHECK (track IN ('NEW','NEAR','FAR','UNMEMORIZED'))`; `d REAL NOT NULL CHECK (d BETWEEN 1 AND 10)`; `s REAL NOT NULL CHECK (s >= 0)`; `lastReviewAt INTEGER` nullable (**`CalendarDate` serial day**, NULL if never reviewed); `dueAt INTEGER` nullable (**`CalendarDate` serial day**); the counters `reps`/`lapses`/`signoffs` `INTEGER NOT NULL DEFAULT 0 CHECK (… >= 0)`; the flags `weakFlag`/`manualLock`/`prayerCritical`/`enabled` `INTEGER NOT NULL DEFAULT … CHECK (… IN (0,1))` (`enabled` defaults `1`, the rest `0`); and the load-bearing table-level `CHECK (track = 'UNMEMORIZED' OR due_at IS NOT NULL)`. `STRICT`. Index `card_due ON card(profile_id, track, due_at)` — the column order `buildToday()` scans to find due FAR/NEAR cards (do not reorder).

4. **`LineBlocks`** — `blockId` TEXT PK (UUID); `profileId TEXT NOT NULL REFERENCES profile(profile_id) ON DELETE CASCADE`; `pageId INTEGER NOT NULL REFERENCES page(page_id)`; `lineStart INTEGER NOT NULL CHECK (line_start BETWEEN 1 AND 15)`; `lineEnd INTEGER NOT NULL CHECK (line_end BETWEEN line_start AND 15)` (the cross-column range — `line_end` references `line_start` in its own `CHECK`); `errorCount INTEGER NOT NULL DEFAULT 0 CHECK (error_count >= 0)`. `STRICT`. Index `line_block_by_card ON line_block(profile_id, page_id)`.

5. **`ReviewLog`** — the append-only audit trail. `logId` TEXT PK (UUID); `profileId TEXT NOT NULL REFERENCES profile(profile_id) ON DELETE CASCADE`; `pageId INTEGER NOT NULL REFERENCES page(page_id)`; `reviewedAt TEXT NOT NULL` (**UTC instant — the event wall-clock time, a `String`, never a scheduling day**); `trackAtReview TEXT NOT NULL CHECK (track_at_review IN ('NEW','NEAR','FAR','UNMEMORIZED'))`; `grade TEXT NOT NULL CHECK (grade IN ('again','hard','good','easy'))`; `errorLinesJson TEXT` nullable (stumble line indices — small structural list, never Quran text); `elapsedDays INTEGER NOT NULL` (a `CalendarDate`-serial delta fed to the curve); the audit doubles `rPredicted`/`sBefore`/`sAfter`/`dBefore`/`dAfter` `REAL` nullable; `source TEXT NOT NULL CHECK (source IN ('self','teacher'))`; `teacherLabel TEXT` nullable (the local *sanad* audit hint). `STRICT`. Index `review_log_by_card ON review_log(profile_id, page_id, reviewed_at)`. **Author no `UPDATE`/`DELETE` over this table** (DAO is T06; the append-only property is the absence of those methods) — the schema task simply must not invite one (no mutable-status column, no `is_deleted` flag).

6. **`ConfusionEdges`** — composite `PRIMARY KEY (profile_id, ayah_a, ayah_b)`; `profileId TEXT NOT NULL REFERENCES profile(profile_id) ON DELETE CASCADE`; `ayahA TEXT NOT NULL REFERENCES ayah(ayah_id)` and `ayahB TEXT NOT NULL REFERENCES ayah(ayah_id)` (both target T02's `Ayat`); `weight REAL NOT NULL DEFAULT 0`; `lastConfusedAt TEXT` nullable (**UTC instant**); and the canonical table-level `CHECK (ayah_a < ayah_b)` — one edge per unordered pair, so `(2:1, 2:2)` is storable but `(2:2, 2:1)` is rejected, and a self-loop `(x, x)` is impossible. `STRICT`. (No extra index — the composite PK covers the per-profile lookups; do not add one speculatively.)

7. **`CycleConfigs`** — `profileId TEXT PRIMARY KEY REFERENCES profile(profile_id) ON DELETE CASCADE` (PK *and* FK — one config row per profile); `cycleType TEXT NOT NULL` (`'7_manzil' | '1_juz_day' | 'half_juz_day' | 'custom' | …` — a free `TEXT` here, the named-preset closed set is validated at the feature/engine layer, not pinned by a schema `CHECK`, since presets evolve); `newLinesPerDay INTEGER NOT NULL DEFAULT 0 CHECK (new_lines_per_day >= 0)`; `nearWindowJuz INTEGER NOT NULL CHECK (near_window_juz >= 0)`; `farTargetPerDay INTEGER NOT NULL CHECK (far_target_per_day >= 0)`; `farCycleDays INTEGER NOT NULL CHECK (far_cycle_days > 0)` (the cycle ceiling, PRD §7.6 — strictly positive); `dailyBudgetMinutes INTEGER NOT NULL CHECK (daily_budget_minutes > 0)`; `pureCycleMode INTEGER NOT NULL DEFAULT 0 CHECK (pure_cycle_mode IN (0,1))`; `termLabelSet TEXT NOT NULL`; `regionPreset TEXT` nullable. `STRICT`.

8. **`AppMeta`** — `key TEXT PRIMARY KEY`; `value TEXT NOT NULL`. The app-level singleton key/value store (`'schema_version'`, `'text_checksum_verified_at'`, `'active_profile'`, `'encryption_enabled'`, …). **Not** per profile — no `profile_id`, no FK. `STRICT`. Do not model the keys as an enum column here; it is a deliberately open key/value table whose keys are documented in the doc comment.

9. **Register all seven on `HifzDatabase`** — extend the `@DriftDatabase(tables: [...])` list in `packages/data/lib/src/db/database.dart` so the user (read-write) block reads `Profiles, Cards, LineBlocks, ReviewLog, ConfusionEdges, CycleConfigs, AppMeta` and the reference (read-only) block from T02 follows it (`Pages, Lines, Ayat, Surahs, Mushafs, MutashabihGroups, MutashabihMembers`), matching §1's listing exactly. Run `dart run build_runner build` so `_$HifzDatabase` regenerates with `createAll()` materializing all fourteen tables; `schemaVersion` stays `1`. Do **not** add the `setup`/`beforeOpen` pragmas or the `MigrationStrategy.beforeOpen` FK assertion in this task — that is E03-T04. Do **not** add a `stepByStep` callback — that is E03-T09.

10. **Pitfalls to avoid:**
   - A `DateTime`/Drift `DateTimeColumn` for `due_at`/`last_review_at`/`elapsed_days` — these are `INTEGER` `CalendarDate` serial days; a `DateTimeColumn` reintroduces the DST off-by-one wrong "next due" (the explicit §2 refusal). Conversely, `reviewed_at`/`created_at`/`last_confused_at` are `TEXT` UTC instants, **not** `DateTimeColumn` either (stored as ISO-8601 strings per §2) — do not let Drift's `DateTimeColumn` Unix-seconds default sneak in.
   - Forgetting `ON DELETE CASCADE` on a per-profile child (`card`, `line_block`, `review_log`, `confusion_edge`, `cycle_config`) — an orphaned hifz row after a profile delete is a leak; the page FK on `card`/`line_block`/`review_log` deliberately has **no** cascade (deleting a page is impossible; the muṣḥaf is immutable).
   - Dropping or weakening the `CHECK (track = 'UNMEMORIZED' OR due_at IS NOT NULL)` — it is the storage-layer defense of the engine invariant; a memorized card with a NULL `due_at` is "decay happening silently."
   - Reversing or omitting `CHECK (ayah_a < ayah_b)`, or writing it as `<=` (which would admit a self-loop) — it is strict `<`, the canonical-ordering guard that keeps one row per unordered confusable pair.
   - Reordering the `card_due` index columns (`profile_id, track, due_at`) — the order matches the `buildToday()` scan; a different order silently de-optimizes the hot path.
   - A non-`STRICT` table (silent `REAL`/`TEXT` coercion), or validating a range in Dart instead of a `CHECK`.
   - Authoring a DAO, a `commitReview`, or any `into(...).insert`/`update`/`delete` in this task — schema only; the write surface is T06/T07. Especially: no `UPDATE`/`DELETE` invited against `review_log`.
   - Persisting a derived health roll-up / `R` as a column, or widening `settings_json`/`error_lines_json` into health or Quran facts — refused by §2.
   - A streak/badge/score/confetti column, a tafsīr/translation column, or any sect/madhhab-specific field — the schema is sect-/madhhab-neutral and gamification-free.

## Tests

All pure `dart test` in `packages/data/test/user/` — no widget binding, no Flutter SDK; offline by construction (no networking import in `data`; a throwing `HttpOverrides` is installed in the `data` test entrypoint so a stray socket fails loudly). Each test opens an in-memory `NativeDatabase.memory()` `HifzDatabase`, runs `createAll()`, and (because the FK-on pragma is E03-T04's job and not yet wired into the connection) issues `PRAGMA foreign_keys = ON` in the test setup so the FK/cascade assertions are meaningful; reference rows the user FKs need are seeded with a confined raw `customStatement`/`batch` in the test file (never a shipped DAO — there is none).

- `packages/data/test/user/user_schema_test.dart` — assert all seven user tables and the three indices (`card_due`, `review_log_by_card`, `line_block_by_card`) exist (`sqlite_master` / `PRAGMA index_list`); each table is `STRICT` (`PRAGMA table_list` reports `strict=1`); the primary keys match §2 (single, composite, and the `cycle_config` PK-is-also-FK). Confirm the generated DDL is byte-equivalent to the committed `drift_schemas/` snapshot so a drift between §2 and the table classes fails the build.
- `packages/data/test/user/user_check_constraints_test.dart` — the correctness-critical guard: each documented `CHECK` rejects the out-of-range insert via a raw `customStatement`, named by intent. Cover at least: `role = 'imam'` throws; `locale = 'en'` throws; `track = 'GOLD'` throws and `grade = 'perfect'` throws and `source = 'app'` throws; `d = 0`/`d = 11`/`s = -1` throw; `line_start = 0`/`line_end < line_start` (e.g. start 5, end 3) throw; a negative `reps`/`error_count` throws; `far_cycle_days = 0`/`daily_budget_minutes = 0` throw; a flag set to `2` throws. **The load-bearing card invariant**: inserting `track='NEW'` with `due_at = NULL` throws, while `track='UNMEMORIZED'` with `due_at = NULL` succeeds. **The canonical confusion-edge guard**: `(ayah_a='2:2', ayah_b='2:1')` throws, `(ayah_a='2:1', ayah_b='2:2')` succeeds, and a self-loop `(x, x)` throws.
- `packages/data/test/user/user_foreign_keys_test.dart` — with `foreign_keys = ON`: a `card`/`line_block`/`review_log`/`confusion_edge`/`cycle_config` referencing a missing `profile_id` is rejected; a `card` referencing a missing `page_id`, a `confusion_edge` referencing a missing `ayah_id`, and a `profile` referencing a missing `mushaf_id` are rejected. **The cascade**: after inserting a profile with a card, a line_block, a review_log row, a confusion_edge, and a cycle_config, deleting the profile deletes all five children (count → 0); deleting a profile leaves the referenced reference rows (`page`/`ayah`/`mushaf`) untouched.
- `packages/data/test/user/calendar_date_columns_test.dart` — assert (via `PRAGMA table_info`) that `card.due_at`, `card.last_review_at`, and `review_log.elapsed_days` are declared `INTEGER` (the `CalendarDate` serial form) and that `review_log.reviewed_at`, `profile.created_at`, `confusion_edge.last_confused_at` are declared `TEXT`; round-trip a known serial day through `due_at` and a known ISO-8601 instant through `reviewed_at` and read back identical values (the `CalendarDate`-serial / UTC-string boundary, the E03-T01 value-type round-trip is T06's).

CI: these run in the `fast` unit job (`docs/engineering/11-testing-strategy.md`); the schema-equivalence check is gated against the committed `drift_schemas/` snapshot so a drift between the §2 DDL and the table classes fails the build.

## Definition of Done

- [ ] All acceptance criteria met; the four test files green locally and in CI's `fast` job.
- [ ] **Crash-safe / nothing-decays-silently (non-negotiable)**: the load-bearing `CHECK (track='UNMEMORIZED' OR due_at IS NOT NULL)` is present and proven to reject a memorized card with no due date; `review_log` carries no mutable-status / `is_deleted` column and invites no `UPDATE`/`DELETE`; the per-profile FKs cascade so no orphaned hifz row survives a profile delete.
- [ ] **Quran text fidelity (non-negotiable, R1)**: the user tables reference the read-only `page`/`ayah`/`mushaf` tables but never write them; `error_lines_json` carries only structural line indices, never reconstructed Quran text; no `*_json` column is widened into Quran or health facts.
- [ ] **Append-only `review_log` (sanad)**: no `UPDATE`/`DELETE` is authored or invited against `review_log` in this schema task; the only future mutation is `INSERT` (T07) and the only bulk touch is export/erase (E17).
- [ ] **CalendarDate boundary**: `due_at`/`last_review_at`/`elapsed_days` are `INTEGER` serial days, `reviewed_at`/`created_at`/`last_confused_at` are `TEXT` UTC instants — no `DateTimeColumn` on any scheduling-day or instant column (the DST off-by-one is foreclosed at the schema).
- [ ] **Offline / no-network (C1)**: `data` imports no networking package; the `data` user tests install a throwing `HttpOverrides`; no telemetry, no account, nothing leaves the device.
- [ ] **No AI / no microphone**: nothing in this task touches audio, a microphone, or any model/inference — these are pure structural tables.
- [ ] **Layer boundary**: no `package:drift`/`package:sqlite3` symbol crosses into `models`/`engine`/`features`/`quran`; the table classes live only in `data` (banned-import gate green).
- [ ] **No gamification / sect-neutral adab**: no streak/badge/score/confetti column; no derived health persisted (computed on read from `card`); no tafsīr/translation/commentary column and no sect/madhhab-specific field; `role`/`locale`/`track`/`grade`/`source` are neutral closed sets.
- [ ] **RTL + fa/ckb/ar / accessibility**: N/A by construction — these tables hold structural data and locale *codes* (`'ar'`/`'fa'`/`'ckb'`), not user-facing strings; `display_name` is user-typed; any label a consumer derives is localized at the feature/`l10n` layer, never hardcoded in a record.
- [ ] **Deterministic tests**: all pure `dart test`, full-word/unit-bearing names, typed `catch`, no `print`/`!`/`late` on persistence values, REUSE SPDX header on every new file; assert behaviour (constraints reject, cascades delete, columns are the right SQLite type), not line counts.
