# E17 — Backup & Restore

The local, offline "never trapped" promise in bytes: the pure-Dart `backup/` package that serializes a ḥāfiẓ's review history into a versioned, integrity-checked `.hifzbackup` file the user saves anywhere — **no cloud, no account, no server** — and the restore path that reads it back. It ships the self-describing container (magic + format version + mode + body SHA-256), the truth-only versioned JSON payload, the **replace-vs-merge** restore (a content-addressed set-union over the append-only `review_log` for teacher↔student transfer), the optional Argon2id→ChaCha20-Poly1305 encryption envelope, the one-tap erase, the calm Settings backup card, and the honest "lose this phone *and* the file and your record is gone" tradeoff copy. The app never transmits the file; the OS share sheet moves it.

## Why this epic exists

The whole product exists to stop a ḥāfiẓ from *silently* losing the Quran ([PRD §2](../../docs/PRD.md)), and the backup file is where the "never trapped" half of that covenant becomes a physical artifact the user controls. The PRD requires a **documented, versioned** export so "data is never trapped," written to the user's own storage/share-sheet with the app performing **no** network transfer, plus a restore with clear merge/replace semantics and a one-action erase that is right-to-be-forgotten by construction ([PRD §16, §17](../../docs/PRD.md)). This epic is the only sanctioned bulk path that touches the append-only `review_log` — and because a teacher sign-off is a *sanad* act recorded there ([PRD §10.3](../../docs/PRD.md)), a merge that silently dropped, reordered, or duplicated a sign-off would be a breach of trust, not a bug; so merge is defined as a deduplicating **set union** by stable `logId`, never an overwrite ([engineering 10 §7](../../docs/engineering/10-backup-format.md)).

Three documented constraints shape every decision here. First, **offline by construction (C1)**: export and import perform zero network I/O — the OS share sheet moves the file and the pure-Dart `backup/` package literally cannot open a socket, which the CI banned-import gate proves structurally ([engineering 10 §1](../../docs/engineering/10-backup-format.md); [PRD §19.3](../../docs/PRD.md)). Second, **text fidelity (R1)**: the Quran is *never* in the file — a backup references the muṣḥaf by `{id, riwayah, checksumSha256}` only and never embeds glyphs, fonts, layout, or the mutashābihāt dataset, both because they balloon the file and because shipping an *unverified* copy of the sacred text through a side channel is a direct R1 hazard ([engineering 10 §4](../../docs/engineering/10-backup-format.md)). Third, **honest adab**: the backup is a *plain documented artifact the user holds*, not a cloud safety net dressed up with a sync spinner — so the card states both halves of ownership plainly (you own this; therefore you are responsible for it), never implies a recoverable server copy, and never weaponizes the no-recovery tradeoff into fear or a guilt nudge ([design 10 §8](../../docs/design-system/10-privacy-and-trust-ux.md); [design 11 §4](../../docs/design-system/11-voice-and-tone.md)). Integrity is mandatory; confidentiality is the user's choice; transmission is never the app's.

## Scope

### In scope

- The pure-Dart `backup/` package over value types and bytes only — never Drift, `sqlite3`, or any networking import — exposing `HifzBackup.export(BackupSnapshot, {passphrase}) → Uint8List` and `HifzBackup.import(Uint8List, {passphrase}) → BackupSnapshot`, plus the `BackupSnapshot` value model and the distinct typed `BackupError` enum.
- The self-describing, self-validating **container**: big-endian magic `HIFZBK` + `0x1F` separator + 1-byte format version (`0x01`) + 1-byte mode (`0x01` plaintext JSON · `0x02` encrypted envelope) + UInt32 body length + 2 reserved zero bytes + 32-byte body SHA-256 + body (min file 49 bytes), with the normative restore-side parse order emitting a distinct `BackupError` at each stage.
- The **truth-only versioned JSON payload**: a canonical (sorted-key) value-typed export of `profile`, `cycleConfig`, `cards`, `lineBlocks`, the append-only `reviewLog`, and `confusionEdges` keyed by stable UUIDs — scheduling days as floating `"YYYY-MM-DD"` strings, true instants as UTC ISO-8601 — carrying the `mushaf {id, riwayah, name, checksumSha256}` for compatibility and **no** derived state and **no** Quran bytes; with `schemaVersion` (stamped from Drift) and per-version forward-migration on decode.
- The mandatory **SHA-256 body integrity** check in both modes, fail-closed before any decode/write (corruption detection, stated plainly as not tamper resistance).
- The optional **encryption envelope** (mode `0x02`): NFC-normalized passphrase → Argon2id (RFC 9106, params clamped to range *before* derivation) → ChaCha20-Poly1305 (RFC 8439, AAD = the binary header), run off the UI isolate; AEAD failure surfaces the single `wrongPasswordOrDamaged`; the file key is independent of the device DB key; no recovery.
- The shell-side **restore**: **replace** (validate-in-memory-first → one transaction wipes the in-scope rows and inserts verbatim → engine re-clamps every imported `dueAt`) and **merge** (set-union by `logId` over the append-only `review_log` → rebuild each touched card from the merged log → re-clamp), each separately confirmed, each one Drift transaction with all-or-nothing rollback, and a cross-`mushaf_id`/checksum refusal.
- The shell-side **export & erase**: build the snapshot through `/data` → serialize/encrypt off-UI isolate → atomic temp file → OS share sheet (no plaintext left for an encrypted export); erase = one confirmed irreversible action deleting `.sqlite` + `-wal` + `-shm` (+ the secure-storage key if encryption was on).
- The **backup card** in Settings: Export / Import rows, the plain "Last backup: <localized date>" / "No backup yet" status line, the optional-encryption toggle, the calm stale-backup nudge, the no-recovery tradeoff and plaintext-is-readable honesty copy, and the replace-vs-merge confirmation — no cloud iconography, no account, no "sync".
- The **erase confirmation gate**: concrete-consequence copy, Cancel-is-primary, a second deliberate step sized to the blast radius, placed out of easy thumb reach.
- The pure-Dart `backup/` **test harness**: round-trip identity, encrypted round-trip, committed golden files with header-offset asserts, integrity-rejection, truncation, wrong-passphrase, Argon2 param-clamp, newer-format/schema, timezone-shift invariance, merge idempotence + superset, trust-clamp-after-import, cross-muṣḥaf refusal, crash-mid-import, and erase-completeness — under at least two `TZ` values; plus per-locale widget/golden coverage of the card and the no-cloud/no-sync assertion.

### Out of scope

- The live Drift/SQLite store the snapshot is read from and written back to, the one-`db.transaction` write path, WAL crash-safety, the opt-in DB `sqlite3mc` cipher, and the `flutter_secure_storage` DB key the backup passphrase is deliberately independent of → **E03 models-and-persistence** (this epic reads/writes through `/data` and adds no schema).
- The pure engine recompute that rebuilds D/S/`dueAt` from the merged `review_log` and re-applies the trust clamp after import → **E04 scheduling-engine** (this epic calls it; the arithmetic and the §7.12 invariant are the engine's).
- What a stored `dueAt`/`lastReviewAt` serial-day *means*, the `CalendarDate` value type, and the Hijri/Jalālī/Gregorian display of the status date → **E02 calendar-and-date-core** (this epic only *encodes* dates as floating `"YYYY-MM-DD"` and *renders* the status date via the presenter).
- The read-only Quran reference tables, KFGQPC fonts, layout, the mutashābihāt dataset, and their checksum governance — **never** in a backup → **E05 quran-data-and-rendering** (the backup records only the muṣḥaf id + checksum).
- The leaf component primitives (button rows, list tiles, switch, dialog scaffold) the card and the confirm gate are assembled from → **E10 mihrab-component-library** (this epic composes them; their anatomy/goldens are E10's).
- The grouped Settings surface the backup card and erase entry point sit inside, and the profile switcher / export scope (this profile vs all, the halaqa transfer path) → **E16 settings-profiles-teacher**.
- The single permitted network socket — the one-time GitHub-Releases asset-pack download and its SHA-256 verifier → **E05 quran-data-and-rendering** (the backup path opens *no* socket).
- The `pubspec.yaml`, `resolution: workspace` membership of `backup/`, and the banned-import / no-network CI gates that wall Drift and sockets out of the package → **E01 repo-scaffold-and-ci** (this epic lives under those gates).

## Definition of Done

- [ ] The portability loop works end-to-end: export a profile → move the file by any external means → import (replace *or* merge) on another install → the engine recomputes and the cards/log are present; a teacher's merged sign-offs are *added*, never duplicated, and survive a kill-and-relaunch.
- [ ] **Offline / no-network**: the `backup/` package imports no networking package and opens no socket; only the shell's share-sheet/file-pick step touches `dart:io`; an `HttpOverrides` offline guard test passes and the CI banned-import/no-network gate over `backup/` stays green (C1, PRD §17).
- [ ] **No AI / no audio / no microphone**: nothing in the export/import/erase path records, infers, scores, or runs a model — the file is a deterministic serialization of stored truth (C1, C2).
- [ ] **Text fidelity (R1)**: no Quran glyphs/text/fonts/layout or mutashābihāt data is ever in the file; only `mushaf {id, riwayah, name, checksumSha256}` is recorded, and a cross-`mushaf_id`/checksum import is refused with a clear message, never coerced.
- [ ] **The append-only `review_log` is honored**: merge is a content-addressed set-union by `logId` (idempotent; no row updated or deleted; no teacher sign-off duplicated); each touched card is rebuilt from the merged log; the idempotence and superset properties hold.
- [ ] **Trust clamp survives transfer (§7.6)**: after restore or merge, every imported card's `dueAt ≤` the **receiving** device's cycle ceiling via the engine recompute + re-applied trust clamp; an imported page is never due past the local cycle — proven by a test.
- [ ] **Integrity is mandatory, fail-closed**: the body SHA-256 is verified in both modes before any decode/write; a single flipped byte ⇒ `integrityFailed`; every truncation boundary is rejected with the right error; no MD5/SHA-1; the UI distinguishes corruption from a wrong key.
- [ ] **Confidentiality is the user's choice, stated honestly**: default is plaintext versioned JSON; the encryption toggle is opt-in/protective-default-on and one honest line; the card states plainly an unencrypted file is readable by anyone; the passphrase is never stored/logged and a forgotten one is unrecoverable; AEAD failure surfaces only `wrongPasswordOrDamaged`; Argon2 params are clamped before derivation (a KDF spy asserts zero derivation on an out-of-range header).
- [ ] **All-or-nothing restore**: replace and merge are separately confirmed, each in one Drift transaction; a crash mid-import leaves the live store byte-identical (Drift auto-rollback), proven by the crash-mid-import test.
- [ ] **Erase actually erases**: one confirmed, irreversible action deletes the `.sqlite` + `-wal` + `-shm` siblings (+ the secure-storage key if encryption was on); no soft-delete or hidden "deleted" flag; the `review_log` rows are gone; after erase nothing about the user persists on-device.
- [ ] **No cloud / no account / no fake sync**: the card shows no cloud glyph, no "Sign in", no "Upload", no "syncing…" theater; export progress reflects only real CPU/crypto work; the no-recovery tradeoff is a checkable fact stated with empathy, never "your data is safe" or a guilt nudge; the stale nudge is a calm offer, never a red alarm.
- [ ] **No gamification / no shame (R3, C6)**: backup status is a plain status line, never a streak/score/"you're overdue" scold; erase completion fires no celebratory motion/haptic and no alarming flourish; nothing here gamifies worship.
- [ ] **The View is dumb / single write path**: export/import/erase and the encryption toggle route through one controller method that persists transactionally *before* republishing the card's status; no widget mutates persisted state, opens a socket, or calls `DateTime.now()`; the status date comes from the injected `CalendarDate` clock.
- [ ] **RTL + fa/ckb/ar localization**: every string (status, tradeoff, encryption, stale nudge, replace/merge, erase) ships through the ARB pipeline in fa/ckb/ar (transcreated, reviewed per locale for the sensitive copy); layout is RTL by `EdgeInsetsDirectional`; the backup file name (mixed Latin/number run) is FSI/PDI-isolated; the status date renders in the locale numeral set and the user's calendar; the file format itself leaks no locale.
- [ ] **Accessibility (WCAG 2.2 AA)**: every action (export/import/erase/dismiss-nudge) is one un-obstructed labelled control; the erase gate announces the full consequence and which button is destructive with the focus ring on the safe action; reduce-motion degrades every transition; targets are ≥48dp; the per-screen accessibility audit passes.
- [ ] **Sect-neutral adab**: the tradeoff, encryption, stale-nudge, and erase copy have cleared the adab review — no guilt/fear/loss leverage, no cloud-safety reassurance, no app-as-authority phrasing, autonomy-supportive in each locale.
- [ ] **Tests**: the format harness is written test-first and green (round-trip, golden, integrity, truncation, wrong-passphrase, param-clamp, newer-format, timezone invariance, merge idempotence + superset, trust-clamp, cross-muṣḥaf, crash-mid-import, erase-completeness) under ≥2 `TZ`; the no-cloud/no-sync widget test and per-locale card goldens run in CI on the real bundled fonts with an offline guard; all gates stay green.

## Risks

- **A merge drops, reorders, or duplicates a teacher sign-off.** The `review_log` is a *sanad* audit trail; a silent loss or duplicate here is a breach of trust, not a cosmetic bug. *Mitigation:* merge is a set-union keyed by device-stable `logId`, never an overwrite; merge-idempotence (`merge(merge(∅,f),f) == merge(∅,f)`) and merge-superset (student ∪ teacher's superset = teacher's superset, no duplicate) are property-tested; the `review_log` DAO exposes no `UPDATE`/`DELETE` so the union cannot mutate a row.
- **The Quran leaks into the file.** A "while I'm here" convenience that embedded page text/fonts/layout would balloon the file and ship an unverified copy of the sacred text through a side channel (R1). *Mitigation:* the payload schema carries only `mushaf {id, riwayah, checksumSha256}`; a test asserts no glyph/text/font/layout field exists; a cross-`mushaf_id`/checksum import is refused, never coerced.
- **An imported `dueAt` outlives the receiving device's cycle.** Copying a card's `dueAt` blindly across cycle configs would let an imported page be due past the local cycle ceiling, breaking the §7.6 trust-clamp invariant. *Mitigation:* the card is treated as a cache of the log — after restore/merge the engine rebuilds D/S and re-applies the trust clamp under the *receiving* device's config; a test asserts every imported `dueAt ≤` the local ceiling.
- **A pre-auth Argon2 header DoS.** A hostile encrypted file could demand minutes/hours of memory-hard work before authentication. *Mitigation:* Argon2 memory/iterations are clamped to the documented ranges *before* any key derivation; a KDF spy asserts zero derivation calls on an out-of-range header.
- **The card drifts into a cloud safety net.** A sync spinner, a cloud glyph, or "your data is safe" copy would turn an honest local file into a false promise. *Mitigation:* the card has no cloud/account/sync chrome (a widget test asserts no such glyph/string in the tree); the no-recovery tradeoff is stated as a checkable fact with empathy; the adab review and the §11 dark-pattern release gate vet every line.
- **A `backup/` import accidentally pulls in Drift or a socket.** A stray dependency would break the offline-by-construction and store-blind guarantees. *Mitigation:* `backup/` is pure Dart over value types + `crypto` only; the CI banned-import/no-network gate that walls Drift into `/data` walls it out of `backup/`; the package receives a `BackupSnapshot` and never queries the DB, so a scoping mistake cannot leak another profile's rows.
- **A partial import corrupts the live store.** A crash mid-restore could leave half the snapshot applied. *Mitigation:* replace and merge each run inside one Drift transaction with automatic rollback on throw; the crash-mid-import test asserts the live store is byte-identical after a simulated failure; replace validates the whole file in memory *before* touching the store.
