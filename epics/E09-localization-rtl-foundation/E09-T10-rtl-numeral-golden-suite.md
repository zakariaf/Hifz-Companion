## Implementation notes

**TEST-FIRST is structural here, not a separate suite.** The goldens *are* the test ‚Äî there is no production code to write after them; the work is (a) authoring per-surface, per-locale golden cases that fail until the real masters are captured, and (b) wiring the gate so each grep/lint fails on a planted violation. Capture masters only with a reviewed local `--update-goldens`; CI never blesses. A golden that renders with `Ahem` or a missing font asserts nothing and is the bug the gate exists to prevent.

1. **Golden suite location & shape.** `packages/<features-or-l10n-goldens>/test/goldens/rtl_numeral_golden_test.dart` (place in the package that can depend on the feature surfaces being screenshotted; if the surfaces span features, a dedicated `app`/`golden_harness` test target). `flutter_test`, `@Tags(['golden'])` so it runs only in the pinned Linux golden job (eng-write-dart-test ¬ß5; eng-add-ci-check ¬ß6), REUSE SPDX header. In `setUpAll`: load the **real** E06 bundled UI font(s) via `FontLoader('Vazirmatn')` (and the Sorani-covering family) from the bundled asset bytes ‚Äî never `Ahem`, never `google_fonts` (eng-write-dart-test ¬ß5; engineering 12 ¬ß8). Pin `tester.view.devicePixelRatio`/`physicalSize`, a fixed theme, and disable animations.
2. **The surfaces to screenshot (chrome only ‚Äî never a mu·π£·∏•af glyph).** Render the epic's representative localized surfaces so every E09 mechanism is exercised in one frame set: a **Today**-style strip (the "Juz N ¬∑ Page M" mixed run for the isolation + numeral proof, a "pages due" plural line, a "next due" date for the calendar-display proof), a **bottom-nav** row (Today rightmost ‚Äî the mirror/logical-order proof from T04), a **page card** with its track chip + grade verb (the term-set proof from T09), and a **progress**-style numeric line (retention "P%" digit block). Each is built from the same widgets the feature epics will use, fed fixtures (no clock, no DB, no network) ‚Äî Riverpod overrides supply any state. **No surface renders the mu·π£·∏•af glyph page**: the goldens stop at the chrome boundary (design 12 ¬ß8); the immutable page is E05's and is never mirrored, re-typeset, or screenshotted here.
3. **Pump each surface once per locale, under locale-derived RTL.** Loop `[Locale('ar'), Locale('fa'), Locale.fromSubtags(languageCode: 'ckb')]`; wrap each in the real `localizationsDelegates` + that `locale` (so `GlobalWidgetsLocalizations` yields RTL ‚Äî do **not** hardcode `Directionality.rtl` at the app root; the point is to prove direction *follows the locale*, per eng-rtl-and-bidi-layout rule 1). For a tightly-scoped widget golden that cannot stand up the full `MaterialApp`, `Directionality(textDirection: TextDirection.rtl)` is acceptable per eng-write-dart-test ¬ß7 ‚Äî but the locale must still drive numerals/term-set/calendar so the three locales differ. `await expectLater(find.byType(<surface>), matchesGoldenFile('goldens/<surface>_<locale>.png'))`. The `fa`/`ckb` masters show `€∞€±€≤‚Ä¶` (U+06F0..) and Sorani letters; the `ar` masters show `ŸÝŸ°Ÿ¢‚Ä¶` (U+0660..) ‚Äî a digit-block, mirror, or isolation regression changes pixels and fails the build.
4. **What each golden proves (name it in the test, not just the pixels).** (a) **Mirroring** ‚Äî Today is the rightmost nav destination, chevrons/progress fills run start‚Üíend (T04). (b) **Bidi isolation** ‚Äî "Juz N ¬∑ Page M" reads start‚Üíend as one line, never "M of N" (T05; the ¬ß3 failure mode). (c) **Numeral block** ‚Äî the exact Extended/Arabic-Indic digits per locale on **both** the number and the date (T06/T08; the ¬ß4 date-vs-number consistency point). (d) **Calendar render-only** ‚Äî the date is the locale's rendered (not converted) form with CLDR week-start where shown (T08). (e) **Term-set** ‚Äî the track chip/grade verb is the active set's word, `ckb` clearly provisional (T09). Keep the assertion behavioural ‚Äî the golden is the artifact; do not duplicate T06's codepoint unit asserts here (they live in `numerals_test.dart`).
5. **Wire the full gate into CI green (eng-add-ci-check).** In the pinned-toolchain workflow (E01 scaffold; `subosito/flutter-action@v2` pinned to the README `flutter-version`, `channel: stable`, on **every** job): the **golden** layer runs as `flutter test --tags golden` on one pinned **Linux** runner (never macOS/Windows ‚Äî cross-OS golden drift is real); the **compile/grep/lint** layers run in the fast/restraint job ‚Äî key coverage is the codegen build itself (`nullable-getter: false`), and the gate script (`tool/check_l10n_complete.*`, filled from the E01 skeleton) runs the hardcoded-string grep, the physical-side grep, the ASCII-digit grep, the raw-bidi-concat grep, the Arabic six-category plural completeness check, the canonical-`ckb` encoding lint, and the banned-phrase (adab) lint over `features/**` + `lib/l10n/*.arb`. Map every layer to the **gate‚Üíjob table** as PRD gate 5 (eng-add-ci-check ¬ß1). **No layer is `continue-on-error`**; the release job (`on: tags: v*.*.*`) re-runs the suite and blocks publish on any red (eng-add-ci-check ¬ß9).
6. **Every grep/lint is proven against a deliberate violation (eng-add-ci-check ¬ß6, ¬ß10).** A gate that matches nothing passes green and protects nothing. Confirm each layer's proven-violation fixture (authored in its sibling task ‚Äî physical-side/ASCII-digit/hardcoded-string in T02/T04/T06, plural in T07, canonical-`ckb` in T03, adab in T02/voice) makes the gate exit non-zero, and exits zero over the compliant tree. Where a sibling has not yet supplied a fixture, add a small throwaway fixture (or an inline test asserting the gate's exit code) and keep the **pattern** as the single source of truth in the authoring task ‚Äî do not fork patterns into this aggregation.
7. **Gate-script + workflow standards.** Any gate script touched here carries the REUSE SPDX header (`GPL-3.0-or-later`), uses typed errors, is `dart format` clean, prints no user data, and stays locale/madhhab-blind ‚Äî it greps/lints bytes, it does not interpret scripture (eng-add-ci-check ¬ß10). The workflow pins the SDK on every job (a floating `flutter-action` silently re-renders goldens and breaks reproducibility ‚Äî refuse it; eng-add-ci-check ¬ß2).
8. **Offline / no-network / no-engine-import.** The goldens load fonts from **bundled assets** via `FontLoader` (no `google_fonts`, no runtime fetch) and run under the shared throwing-`HttpOverrides` bootstrap (E01) so a stray socket is a loud, named failure; no `DateTime.now()` (fixtures use injected `CalendarDate`/`SerialDay` literals and a fixed converted date); no surface imports `/engine`. The gate scripts run on source/assets only and transmit nothing off the runner.
9. **Pitfalls to avoid**: rendering a golden with `Ahem` or a missing font (every glyph a solid square ‚Äî the check would assert nothing; this is the named risk in the EPIC); running mu·π£·∏•af-adjacent surfaces (there are none ‚Äî chrome only); screenshotting on macOS/Windows in CI (golden drift); floating the Flutter SDK (re-renders masters); `continue-on-error: true` on any gate layer (makes a release-blocker advisory); marking a grep green that matches nothing (no proven-violation fixture); hardcoding `Directionality.rtl` at the app root in the harness (defeats the "direction follows the locale" proof ‚Äî let the locale drive it, reserve manual `Directionality` for a scoped widget golden only); blessing masters with `--update-goldens` in CI; duplicating T06's codepoint unit assertions in the golden test (keep layers separate); letting the term-set/calendar/numeral surfaces render Latin digits or a Monday week-start (the very regressions the suite must catch).
