## Goal

The `gen_l10n`/ARB pipeline is locked as the single accessor for every chrome string. The repo-root `l10n.yaml` is finalized to `template-arb-file: app_ar.arb` (Arabic the base content language), `nullable-getter: false` (a missing/typo key is a **compile error**, never a silent empty widget), `use-escaping: true`, and `synthetic-package: false`; the generated `AppLocalizations` is emitted into `packages/l10n/lib/src/generated/` and **committed** so every reviewer reads it in-tree. `packages/l10n/lib/src/arb/app_ar.arb` is populated with the foundation key set — each key carrying `@description` translator/scholar context — and `app_fa.arb`/`app_ckb.arb` are **transcreated** against it (never literally translated), with `ckb` canonically encoded and flagged "needs native + scholar review". After this task every chrome string reads only through `l10n.*`, no user-facing literal lives in `features/**`, and no user-facing string lives in `/engine`.

## Implementation notes

TEST-FIRST is **not** the primary mode here — the load-bearing correctness gate is the **compiler** (`nullable-getter: false` makes a missing/typo key a build error) plus codegen, so the proof is "the workspace builds, `AppLocalizations` exposes every foundation key, and a per-locale resolution test reads them". Write that resolution test (Tests below) alongside the ARB authoring, not after.

1. **Finalize `l10n.yaml`** (repo root, already present from E01-T02). Add the two fields E01-T02 deferred to this epic, leaving the rest verbatim per engineering 12 §1 / project-structure §6:
   ```yaml
   # l10n.yaml — Arabic is the template/base content language.
   arb-dir: packages/l10n/lib/src/arb
   template-arb-file: app_ar.arb          # ar is the source of truth (PRD §13.1)
   output-dir: packages/l10n/lib/src/generated
   output-localization-file: app_localizations.dart
   output-class: AppLocalizations
   nullable-getter: false                 # ADDED: missing/typo key => compile error
   use-escaping: true                     # ADDED: literal braces/apostrophes in fa/ar/ckb copy
   synthetic-package: false               # emit into source so it is reviewable in-repo
   ```
   Do **not** add a second `l10n.yaml`, `flutter_gen`, or `google_fonts` anywhere (E01-T04 pitfall). `pubspec` already carries `flutter: generate: true` + `flutter_localizations`/`intl` from E01.
2. **Regenerate and commit `AppLocalizations`.** Run `flutter gen-l10n` (or `flutter pub get`) so `packages/l10n/lib/src/generated/app_localizations.dart` (+ the per-locale parts) regenerate against the finalized config, and **commit them in-tree** (`synthetic-package: false`). The generated file is a build artifact reviewers read; never `.gitignore` it. Because `nullable-getter: false`, the generated getters are non-nullable (`l10n.todayTitle`, not `l10n.todayTitle ?? ''`) and `AppLocalizations.of(context)` returns a non-null instance — call sites use it directly.
3. **Populate `app_ar.arb` — the base content language — with the foundation key set only.** This is the *cross-feature chrome* every later epic reuses, NOT any feature's screen copy (Today/Onboarding/Settings strings belong to E11–E19 *using* this pipeline; the EPIC's out-of-scope list is binding). Author each key with `@`-metadata carrying a `description` (translator/scholar context) and `placeholders` where interpolated. The foundation set:
   - **App identity / chrome**: `appTitle` (used by `onGenerateTitle`); the five bottom-nav labels `navToday`/`navMushaf`/`navMutashabihat`/`navProgress`/`navSettings` (logical order; RTL renders them right-first by construction — engineering 12 §2).
   - **Universal action verbs** reused everywhere: `actionSave`, `actionCancel`, `actionConfirm`, `actionUndo`, `actionRetry`, `actionClose`, `actionBack`, `actionNext`.
   - **The riwāyah identity string** (chrome, design 12 §8): `mushafRiwayahLabel` → Arabic naming "Ḥafṣ ʿan ʿĀṣim — Madani muṣḥaf"; its `@description` states it names the *chrome* edition, never the Quran in the absolute, and that the page glyphs are never localized.
   - **One interpolated exemplar** proving the placeholder discipline: `juzLabel` with a `{juz}` placeholder (type `String` — it receives an *already locale-numeral-formatted, bidi-isolated* token from `numberFormatFor`/`bidi.dart`, never a raw `int`), so the foundation demonstrates "format → isolate → inject", not concatenation.
   - **One ICU `plural` exemplar** proving the six-category contract: `pagesDue` with `{count, plural, zero{…} one{…} two{…} few{…} many{…} other{…}}` and `placeholders: { count: { type: int } }`. Author all six Arabic categories in `app_ar.arb` (the §1/§6 template is the shape). The completeness *check* is E09-T07; authoring the six here is what that check guards.
   - **One needs-review term-set seed** demonstrating the swap mechanism without owning the vocabulary (E09-T09 owns it): `trackFar` as `{region, select, levant{…} subcontinent{…} other{…}}` with `@description` "Far-revision (manzil/dhor) track label; varies by region. NEEDS scholar review." Keep this minimal — one entry to lock the `select`/region-override shape.
4. **Transcreate `app_fa.arb` and `app_ckb.arb` against `app_ar.arb` — never literal-translate** (design 11 §8). Every key present in `app_ar.arb` gets a transcreated value in both:
   - **`fa`**: respectful-warm register — *šomā*/honorific stance, never cold over-formality nor the over-familiar *to*; `pagesDue` uses Persian's CLDR categories (`one`/`other`); numerals are Extended Arabic-Indic downstream (E09-T06).
   - **`ckb`**: authored with **canonical Unicode** — U+06D5 (ە) for AE, U+06A9 (ک) for kaf; **never** the heh+ZWNJ (U+200C) AE hack or Teh-Marbuta (ة) for AE. Every `ckb` `@description` (or a file-level note) carries **"needs native + scholar review"**; the values ship clearly provisional and state no ruling. The canonical-encoding *lint* is E09-T03; this task simply authors them correctly so that lint passes.
5. **The `l10n.dart` barrel** (E01-T04 created it) re-exports `AppLocalizations` as the single public accessor; confirm it still exports `AppLocalizations` (and the `numerals`/`bidi`/`ckb` helpers) after regeneration, and that `///` doc on the barrel names it the single chrome-string accessor (eng-write-to-coding-standards §4). No feature reads the generated `app_localizations.dart` directly — only through the barrel.
6. **Refuse, by construction:** (a) any user-facing `Text('…')`/`tooltip:`/`semanticsLabel:`/`label:` literal in `features/**` — every such string resolves through `l10n.*` (the grep *gate* is E09-T02; the *discipline* is enforced here by leaving the keys to read); (b) any `AppLocalizations`/`intl` import in `/engine` — the engine returns enums/values, the feature layer localizes them; (c) raw ASCII-digit or fragment concatenation into an ARB value — interpolation is always a `{placeholder}` fed a pre-formatted, pre-isolated token.
7. **Pitfalls to avoid**: leaving `nullable-getter` unset/`true` (re-introduces the silent-empty-widget failure the §1 pitfall forbids); `.gitignore`-ing the generated `AppLocalizations` (it must be committed, `synthetic-package: false`); hand-editing `fa`/`ckb` first or literal-translating so register/tone drifts (design 11 §8); authoring an *incomplete* Arabic `plural` (a missing `few`/`many` is a silent grammatical error — author all six); splicing a raw `int`/Latin token into a value instead of a placeholder; over-scoping `app_ar.arb` with feature screen copy that belongs to E11–E19; shipping `ckb` with the ZWNJ/Teh-Marbuta hack or without the provisional flag; adding `google_fonts`/a second `l10n.yaml`; locking the `trackFar` term-set vocabulary as final before scholar review.

- [ ] `packages/l10n/lib/src/generated/app_localizations.dart` (+ per-locale parts) is regenerated against the finalized config and **committed** in-tree; the generated getters are **non-nullable** (a referenced-but-undefined key fails the build, not returns `null`).
- [ ] `app_ar.arb` contains the foundation key set (app identity + five nav labels + the universal action verbs + `mushafRiwayahLabel` + `juzLabel` + `pagesDue` + `trackFar`), each with an `@description`; every interpolated key declares its `placeholders` types.
- [ ] `pagesDue` defines all six Arabic CLDR plural categories (`zero`/`one`/`two`/`few`/`many`/`other`) in `app_ar.arb`; `juzLabel` takes a `String` placeholder (a pre-formatted, pre-isolated token), not a raw `int`.
- [ ] `trackFar` is an ICU `select` over `region` with a "NEEDS scholar review" `@description`; no track/grade/cycle vocabulary is hard-coded in any widget.
- [ ] `app_fa.arb` and `app_ckb.arb` carry a **transcreated** value for **every** `app_ar.arb` key (zero missing keys, zero English fallback); `fa` uses the respectful-warm register; `ckb` values are canonically encoded (U+06D5 ە, U+06A9 ک; no stray U+200C, no Teh-Marbuta-for-AE).
- [ ] Every `ckb` value (or a file-level metadata note) is flagged **"needs native + scholar review"** and states no fiqh ruling; the `mushafRiwayahLabel` names the chrome edition, never the Quran in the absolute.
- [ ] No user-facing string literal in `features/**` and no `AppLocalizations`/`intl` import in `/engine` (the keys exist for every chrome string the shell currently shows; full feature copy is added by E11–E19 through this pipeline).
- [ ] The `l10n.dart` barrel exports `AppLocalizations` as the single chrome-string accessor with a `///` doc; no feature imports the generated `app_localizations.dart` directly.

## Tests

`packages/l10n/test/l10n_foundation_test.dart` (mirrors the package, `flutter_test`), runs under the fast lane in CI on every PR. Required cases:

- **Key coverage is a build invariant (compile proof).** With `nullable-getter: false`, the test simply *references* each foundation getter (`l10n.appTitle`, `l10n.navToday`, …, `l10n.actionSave`, `l10n.mushafRiwayahLabel`) under `AppLocalizations` for `ar` — the file would not compile if a key were missing or mistyped. The presence of this test file is the standing guard that the foundation set stays defined.
- **Per-locale resolution, all three locales.** For each of `Locale('ar')`, `Locale('fa')`, `Locale.fromSubtags(languageCode: 'ckb')`, build `AppLocalizations` and assert every foundation key resolves to a **non-empty, non-English-fallback** value distinct per locale where the script differs — proving zero missing keys and that `fa`/`ckb` are not falling back to `ar`/English.
- **ICU plural exercises all six Arabic categories.** Assert `pagesDue` under `ar` produces distinct, correctly-inflected strings for representative counts that hit `zero`/`one`/`two`/`few`/`many`/`other` (e.g. 0, 1, 2, 3, 11, 100). (The ARB-completeness *static* check is E09-T07; this is the runtime companion.)
- **Interpolation is placeholder-shaped, not concatenated.** Assert `juzLabel(token)` injects the passed token into one message (the token round-trips intact), proving the ARB value is `"…{juz}…"`, not a spliced substring.
- **`select` term-set switches whole vocabulary.** Assert `trackFar` returns the `other`-branch value for an unknown region key and the matching branch for a known one — locking the region-override shape E09-T09 builds on.
- **Canonical `ckb` encoding spot-check.** Assert representative `ckb` values contain U+06D5/U+06A9 and contain **no** U+200C and **no** Teh-Marbuta (U+0629) standing for AE. (The full lint over all keys is E09-T03; this is the unit-level smoke.)

Offline / no-network guard: the shared throwing-`HttpOverrides` bootstrap (E01-T06) covers `packages/l10n`; ARB loading and `intl` formatting touch no socket — any stray connection attempt is a loud, named failure. No font is fetched at runtime (no `google_fonts`); the generated source is read in-tree, not produced at test time.

## Definition of Done

- [ ] All acceptance criteria met; `l10n_foundation_test.dart` green locally and in CI on every PR; the workspace builds with the committed regenerated `AppLocalizations`.
- [ ] **Offline / no-network preserved:** nothing here opens a socket or fetches a font/locale at runtime; ARB is bundled, `intl` is local, the generated source is committed (PRD C1, §13.5, §19.3).
- [ ] **No AI / no microphone:** no ML/translation-service/ASR dependency introduced; the `fa`/`ckb` values are human transcreations, not machine-translated; no audio or recognition path touched (PRD C2; design 11 §8).
- [ ] **Quran text fidelity / sacred boundary held:** the pipeline touches only chrome — no key, `@description`, numeral, or bidi control reaches a muṣḥaf glyph or an ayah number; `mushafRiwayahLabel` names "Ḥafṣ ʿan ʿĀṣim — Madani muṣḥaf" as the chrome edition, never the Quran in the absolute; no in-app translation/transliteration of the Quran (PRD R1, R2, §11.2, §13.1; design 12 §8).
- [ ] **RTL + fa/ckb/ar strings structural:** `app_ar.arb` is the template; `app_fa.arb`/`app_ckb.arb` carry a transcreated value for every key (zero missing keys, zero hardcoded user-facing strings, no English runtime fallback); the ARB values carry no physical-side or hardcoded-RTL assumption — direction follows the locale from E07's `MaterialApp` (PRD C4, §13, §20 gate 5).
- [ ] **Accessibility seam respected:** the keys this pipeline stores are the same `Semantics` labels E08 audits and the same strings E09-T10's per-locale goldens render on the **real** bundled fonts (never `Ahem`); this task introduces no streak/score/shame surface.
- [ ] **Sect-neutral adab:** every foundation string passes the adab gate first and the four voice attributes (reverent/calm/plain-and-warm/honest); no never-ship phrase (guilt/fear/loss, "safe to drop"/"mastered", "you must/should/don't", exclamation marks, emoji, "upgrade/premium"); Arabic imperatives softened to statements of readiness; no string issues a fiqh ruling or speaks for the Quran (PRD R3, R6, §7.12; design 11 §3–§6, §9).
- [ ] **Scholarly review flagged where pending:** every `ckb` value and the `trackFar` term-set carry "needs native + scholar review" and ship clearly provisional until a native speaker (register) and a scholar (terminology) clear them; the architecture lets a cleared term-set/`ckb` value replace them as a one-file data change (PRD §13.4, §21.1; design 12 §6).
- [ ] **Deterministic tests:** `l10n_foundation_test.dart` is deterministic (fixed locales, fixed counts, no clock, no network); no `DateTime.now()` in any helper or test; the generated source under test is the committed artifact, not regenerated mid-test.
- [ ] **No CLAIMS attach by construction:** this task originates no user-facing factual number, scheduling rule, or methodology claim; it stores and key-coverages already-cleared wording only — any future claim is registered by the feature epic that authors it (domain-claims-register-and-science-screen).
