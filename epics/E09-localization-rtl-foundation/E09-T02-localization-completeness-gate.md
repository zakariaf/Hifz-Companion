## Goal

The dormant `tool/check_l10n_complete.sh` skeleton from E01-T05 becomes the real, layered, mostly compile-time/grep **localization-completeness gate** that implements PRD §20 gate 5's structural half — the five non-golden layers from engineering 12 §8. Concretely: (1) compile-time **key coverage** rides E09-T01's `nullable-getter: false` codegen (a missing/typo ARB key is a build error, not a runtime null); (2) a **`features/**` hardcoded-string grep** bans string literals in `Text(...)`, `tooltip:`, `semanticsLabel:`, and `label:`; (3) a **physical-side grep** bans `EdgeInsets.only(left:/right:)`, `Alignment.center{Left,Right}`, and `Positioned(left:/right:)`; (4) an **ASCII-digit-interpolation grep** bans raw `$int` splices into `Text`/ARB; (5) a never-ship **banned-phrase (adab) lint** run per locale over every ARB value. The grep checks live in `tool/` shell, run with no arguments, exit non-zero on the first violation, and are wired into the `fast` CI job. The verifiable artifact is the eng-add-ci-check deliberate-violation proof: **every ban is demonstrated to flip non-zero against a seeded violation and back to 0 on revert**, so no layer is a hollow gate. (The per-locale RTL + numeral *golden* layer of gate 5 is E09-T06/E09-T10, not this task; the Arabic six-category plural-completeness check is E09-T07, which consumes this gate's harness.)

## Implementation notes

TEST-FIRST is satisfied here by the **deliberate-violation proof** (eng-add-ci-check §1; E01-T05 note 1), not a Dart unit test: a grep with a bad path glob exits green forever while real bugs slip through (the EPIC's "a gate that passes because it checks nothing" risk). Before declaring any layer done, run it against a *seeded* violation, confirm it exits non-zero with a named message citing the rule, then revert and confirm it exits 0. The seeded-violation matrix is in **Tests** and is re-run by E09-T10's full-gate sweep.

1. **Extend `tool/check_l10n_complete.sh`** (the present-but-dormant E01-T05 script) into the real gate rather than adding new top-level scripts — engineering 02 §7 keeps one file per gate, run identically locally and in CI. Keep the REUSE SPDX header (`GPL-3.0-or-later`), `set -euo pipefail`, no arguments, repo-root cwd, and one named `echo "::error::…"` to stderr + `exit 1` per violation citing the PRD/doc ref. **No suppression path** (no `--allow`, no skip env var) — a gate you can quietly turn off is not a gate (coding-standards §7.2). Mind `set -e` + `grep` semantics: `grep` returns 1 on *no match* (the clean-tree success case), so wrap every ban as `! grep -rnE …` or guard deliberately — a naive `grep && exit 1` aborts the script on a clean tree.

2. **Layer A — key coverage (compile-time, not grep).** This rides E09-T01's `nullable-getter: false` codegen: the `fast` job runs `flutter pub get` (triggering `gen_l10n`) then `flutter analyze --fatal-infos`, so a referenced-but-undefined `l10n.foo` getter is a *compile error*. The shell script keeps the cheap pre-codegen superset guard from E01-T05 (every key in `packages/l10n/lib/src/arb/app_ar.arb` must exist in `app_fa.arb` **and** `app_ckb.arb`, and vice-versa) so a *missing translation value* (a key present in the template but absent in a locale, which codegen tolerates by falling back) still fails. Pure shell/grep + `python3 -c`/`jq`-free key extraction if already used by E01-T05; do not introduce a new ARB-parser dependency. Cite PRD §20 gate 5, engineering 12 §1/§8.

3. **Layer B — `features/**` hardcoded-string grep.** Over `app/lib` and `packages/features/*/lib`, ban string-literal arguments to `Text('…')`/`Text("…")`, `tooltip: '…'`, `semanticsLabel: '…'`, and `label: '…'` that are **not** resolved through `l10n.*` (allow `Text(l10n.…)`, `Semantics(label: l10n.…)`, `tooltip: l10n.…`). The literal user-facing string is the violation (engineering 12 §1; design 12 §7). This widens the E01-T05 `Text('literal')` check to the four call sites engineering 12 §1 names verbatim. Keep the allow-list narrow and explicit — a const key like `Text(key: ...)` or a debug-only `Text('')` empty placeholder should be considered: prefer banning *any* non-`l10n` literal and fixing call sites, but document the one tolerated case (e.g. an `isolate(...)`-wrapped already-localized run) if E09-T05's bidi helper output passes through `Text`. Cite PRD §20 gate 5.

4. **Layer C — physical-side grep.** Over `app/lib` and `packages/features/*/lib`, ban `EdgeInsets.only(` with a `left:`/`right:` argument, `Alignment.centerLeft`/`centerRight`/`topLeft`/`topRight`/`bottomLeft`/`bottomRight`, and `Positioned(` with `left:`/`right:` (the exact list, engineering 12 §2/§8; design 12 §1). Only `EdgeInsetsDirectional`, `AlignmentDirectional`, `Positioned.directional`, and `start`/`end` pass. Because the app is RTL in every locale, this is the always-on layout gate, not a latent-locale check. **Scope**: `features/**` + `app/lib` only — the muṣḥaf/glyph layer (E05 `packages/quran`) and any genuinely physical hardware-anchored element are out of scope; do not grep `packages/quran`. Cite engineering 12 §2, PRD §13.2.

5. **Layer D — ASCII-digit-interpolation grep.** Over `app/lib`, `packages/features/*/lib`, **and** the ARB files, flag raw integer interpolation into a user-facing string: a `'…$count…'` / `'… $n …'`-style splice of a bare identifier (heuristically, `\$[a-zA-Z_]` or `\$\{` inside a `Text('…')`/ARB value) and `+ n.toString()` concatenation into `Text`. The fix is `numberFormatFor(locale)` (E09-T06) then an ICU placeholder, never a splice. The grep is a *heuristic* (it cannot fully parse Dart) — tune it to catch the documented `'Page ' + n.toString()` and `'$count pages'` shapes engineering 12 §5 names, and accept that a `{placeholder}` ICU value (no `$`) and a pre-formatted, `isolate`-wrapped run pass. Cite PRD §13.3, engineering 12 §5.

6. **Layer E — banned-phrase (adab) lint, per locale.** A new check over **every value** in `app_ar.arb`, `app_fa.arb`, `app_ckb.arb` (and any region-override ARB) for the design 11 §9 never-ship patterns. Because the patterns are multilingual and span the exclamation/emoji classes, author this layer as a small Dart entrypoint `tool/check_adab_lint.dart` (REUSE SPDX header, a `sealed AdabViolation` typed error, full-word names like `bannedPattern`/`offendingKey`/`localeTag`, `///` on `main`, no `print` of user data beyond the offending **ARB key + matched pattern + locale**, `dart format`/`dart analyze --fatal-infos` clean — eng-write-to-coding-standards) and call it from `check_l10n_complete.sh` (or wire both in the `fast` job). The pattern set, with a comment tracing each to its §9 row:
   - **Exclamation / emoji**: any `!` in a product-copy value (allow none — design 11 §2/§9); any emoji codepoint (Unicode `Emoji` property ranges).
   - **Mandate words** (per locale, native-reviewed list): English "you must / have to / should" / "don't"; Persian and Arabic equivalents supplied by the native-review pass — **the lint ships with the language-specific lists as data**, flagged "extend per native review", never an English-only check that a transcreated curt imperative slips past (design 11 §6, §8).
   - **Guilt/fear/loss + streak**: "lose your hifz", "falling behind", "haven't opened", "streak" / "don't break", and locale equivalents.
   - **Forbidden verdicts**: "safe to drop", "mastered", "done with" (the engine-invariant copy mirror, design 11 §5; PRD §7.12).
   - **Commercial-transaction**: "upgrade", "premium", "unlock", paywall/SKU words.
   The lint is a *necessary-not-sufficient* machine layer; the human native + scholar review (design 11 §9) still runs per locale and is recorded as a sign-off artifact, exactly as eng-add-ci-check §1 treats gates 7–8. Cite PRD §20 gate 5, design 11 §9.

7. **Wire into CI.** In `.github/workflows/ci.yml`, replace the `# TODO: ARB-key check for gate 5` placeholder in the **`fast`** job with `tool/check_l10n_complete.sh` (now covering layers A-pre-codegen, B, C, D and invoking the adab lint), keeping `flutter analyze --fatal-infos` as layer A's compile-time half. Update the `# PRD §20 gate → job mapping` comment so gate 5's structural half reads "fast (key coverage + hardcoded/physical/ASCII greps + adab lint)" alongside the existing "golden (RTL goldens)". **No `continue-on-error`** on the step (eng-add-ci-check §9). The greps and the adab lint make **no** network call — they read the local tree only (offline by construction).

8. **Pitfalls to avoid.** (a) A bad path glob that prunes the wrong directory → the hollow gate; the seeded-violation matrix is the guard. (b) Greping `packages/quran` (E05) or `packages/engine` for hardcoded strings/physical sides — the engine is locale-free and the muṣḥaf is the immutable glyph layer; scope to `features/**` + `app/lib`. (c) Folding the *canonical-ckb encoding* lint (ZWNJ/Teh-Marbuta) into this task — that is E09-T03; this gate is locale-coverage + phrasing, not Sorani codepoint validation. (d) An English-only adab lint that a transcreated mandate slips past — ship the per-locale pattern lists as data with a native-review hook. (e) Re-implementing a full Dart ARB key-diff parser when codegen + `nullable-getter: false` already gives compile-time key coverage — keep the shell layer the cheap superset guard. (f) Letting the ASCII-digit heuristic flag a legitimate `{placeholder}` ICU message — tune to the `$`/`+ toString()` splice shapes only.

## Acceptance criteria

- [ ] `tool/check_l10n_complete.sh` (extended from E01-T05) carries the REUSE SPDX `GPL-3.0-or-later` header, `set -euo pipefail`, takes no arguments, runs from repo root, exits 0 on the clean tree, and exits non-zero with a named `::error::` message (citing the PRD/doc rule) on the first violation of any layer it owns.
- [ ] **Key coverage**: with E09-T01's `nullable-getter: false`, a referenced-but-undefined `l10n.*` getter fails `flutter analyze --fatal-infos`; the shell superset check additionally fails on any key present in `app_ar.arb` but missing a value in `app_fa.arb`/`app_ckb.arb` (or vice-versa).
- [ ] **Hardcoded-string grep**: fails on a string literal passed to `Text(...)`, `tooltip:`, `semanticsLabel:`, or `label:` anywhere in `app/lib` or `packages/features/*/lib` that is not resolved through `l10n.*`; passes when the same call sites read `l10n.*`.
- [ ] **Physical-side grep**: fails on `EdgeInsets.only(left:/right:)`, `Alignment.center{Left,Right}` (and the other physical `Alignment.*` corners), or `Positioned(left:/right:)` in `features/**`/`app/lib`; passes on the `*Directional`/`start`/`end` equivalents; does not scan `packages/quran` or `packages/engine`.
- [ ] **ASCII-digit grep**: fails on a raw `$int`/`${…}` interpolation or `+ n.toString()` splice into a `Text(...)` or ARB value; passes on an ICU `{placeholder}` message and a pre-formatted `isolate`-wrapped run.
- [ ] **Banned-phrase (adab) lint** (`tool/check_adab_lint.dart`) runs per locale over every value in `app_ar.arb`/`app_fa.arb`/`app_ckb.arb`, fails on any never-ship pattern (exclamation/emoji, mandate words, guilt/fear/loss + streak, "safe to drop"/"mastered"/"done", commercial-transaction words) with the offending key + pattern + locale, and ships per-locale pattern lists as data with a documented native-review extension hook; it is `dart format`/`dart analyze --fatal-infos` clean with a `sealed` typed error and prints no user data beyond key/pattern/locale.
- [ ] The `fast` CI job runs the extended `check_l10n_complete.sh` + the adab lint (replacing the gate-5 `# TODO`), with `flutter analyze --fatal-infos` providing the compile-time key-coverage half; the gate→job mapping comment is updated; no step is `continue-on-error`.
- [ ] **Every ban has been proven against a seeded violation and a revert** (the Tests matrix), observed to flip non-zero ↔ 0; no layer is a hollow gate, and no script has a CI-only branch or a suppression flag.

## Tests

This task's verifiable artifact is the **deliberate-violation proof** (eng-add-ci-check §1, the documented gate-validation procedure), not Dart unit tests for the shell layers; `tool/check_adab_lint.dart` additionally carries a small `package:test` suite asserting each pattern class matches a crafted offending value and passes a clean one. The seeded-violation matrix, run locally per layer and re-run by E09-T10's full-gate sweep:

| Layer | Seeded violation (then revert) | Expected |
|---|---|---|
| Key coverage (compile) | reference `l10n.doesNotExist` in a feature widget | `flutter analyze --fatal-infos` fails (compile error); clean after revert |
| Key coverage (shell superset) | delete one key's value from `app_ckb.arb` | non-zero, names PRD §20 gate 5; 0 after revert |
| Hardcoded strings | add `Text('Today')` (not `Text(l10n.todayTitle)`) to a feature; separately `tooltip: 'help'` | non-zero on each, names gate 5; 0 after revert |
| Physical sides | add `EdgeInsets.only(left: 16)`; separately `Alignment.centerLeft`; separately `Positioned(left: 0)` | non-zero on each, names engineering 12 §2; 0 after revert |
| ASCII digits | add `Text('$count pages')`; separately `Text('Page ' + n.toString())` | non-zero on each, names PRD §13.3; 0 after revert |
| Banned-phrase (adab) | set an `app_fa.arb` value to a string containing `!`; separately a value with an emoji; separately the English "you must"; separately a Persian mandate from the native list; separately "safe to drop"; separately "upgrade" | non-zero on each, names design 11 §9 + the offending key/pattern/locale; 0 after revert |

`tool/check_adab_lint.dart` unit suite (`tool/test/check_adab_lint_test.dart`, `package:test`): one case per pattern class (exclamation, emoji, each mandate-word list, guilt/fear/loss, forbidden-verdict, commercial) asserting a crafted offending value is flagged with the correct key/locale and a clean value passes; an empty-ARB run exits 0.

**Offline / no-network guard**: the shell greps and the adab lint read only the local tree and ARB files — they open no socket, fetch no font or locale, and call no service; a fresh clone runs them identically. The `fast` job's `flutter pub get` is the only network step and is the standard pinned-toolchain setup, not part of these checks.
