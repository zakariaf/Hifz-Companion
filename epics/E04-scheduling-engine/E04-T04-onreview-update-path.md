## Goal

`SchedulingEngine.onReview(Card, ReviewInput, SerialDay)` exists in the pure-Dart `engine/` package as the **one deterministic update path**, exactly as engineering 06 §4 / PRD §7.7 specify, running its steps in a fixed order: the **sacred-text guard** (a `missedOrAlteredWord` review is capped at `Grade.hard` *before any arithmetic*, R1) → **source-confidence scaling** (a self-rating's `kSelfConfidence ≈ 0.5` scales only the *applied stability gain*, a teacher's `1.0` does not) → the **lapse branch** (`postLapseStability` clamped `≤ S`, `D += kLapseDifficultyBump`, `weakFlag` set) **vs** the **success branch** (`stabilityOnSuccess` with the `w[15]`/`w[16]` Hard/Easy multipliers) → **`nextDifficulty`** mean-reverting toward the Good anchor → the **weak-line `(11−D)` difficulty channel** (`kWeakLineFactor · weakLineCount`, applied at full strength regardless of source). It returns a new `Card` via `copyWith`; graduation and the trust clamp are *call sites* this task wires to **stubs** that E04-T05 and E04-T07 replace — `onReview` returns a non-null `dueAt` from the stub so the type stays honest, but the real ceiling math is not this task. The branches are frozen by golden vectors (both stability branches + Hard + Easy + the sacred-text cap) computed from an independent oracle.

## Implementation notes

**TEST-FIRST.** This path is the single most consequential arithmetic in the product — an inverted branch lets a forgotten page earn a longer interval (silent decay) or lets a dropped word read "Good" (a text-fidelity breach). Write the golden-vector table and the sacred-text-cap unit case **first**, watch them fail against an unimplemented `onReview`, then implement until green. The vectors are oracle rows (see Tests), not values read back from the engine.

1. **Files** (in the package from E04-T01, types from E04-T02, curve from E04-T03):
   - `packages/engine/lib/src/review_update.dart` — the free functions `stabilityOnSuccess`, `postLapseStability`, `nextDifficulty`, `initialDifficulty`, and the `onReview` body (or a method on the `SchedulingEngine` façade if T01 chose a class — match T01's shape). Re-export from the `packages/engine/lib/engine.dart` barrel. REUSE SPDX header (`GPL-3.0-or-later`).
   - The named weight vector `w` and the constants (`kSelfConfidence`, `kLapseDifficultyBump`, `kWeakLineFactor`, `kMinStability`) are **E04-T10's** file — reference them by name; if T10 has not yet landed, declare them as `// TODO(E04-T10)` named `const`s in one place and never inline a literal at a call site.

2. **Order is the contract — implement it exactly as engineering 06 §4** and never reorder:
   1. `elapsed = card.lastReview == null ? 0 : today.value − card.lastReview!.value`; `r = elapsed == 0 ? 1.0 : retrievability(elapsed, card.s)` (T03).
   2. **Sacred-text guard, before any math:** `var grade = rv.missedOrAlteredWord && rv.grade.index > Grade.hard.index ? Grade.hard : rv.grade;`. This is R1 in code — the cap happens *first* so no downstream branch can ever see a `Good`/`Easy` with a dropped word.
   3. `final conf = rv.source == Source.teacher ? 1.0 : kSelfConfidence;` — referenced by name, never `0.5`/`1.0` inlined.
   4. `double d = nextDifficulty(card.d, grade);` — mean-reverting; computed from the *capped* grade.
   5. **Branch on `grade == Grade.again`:**
      - **Lapse:** `lapses += 1`; `d = (d + kLapseDifficultyBump).clamp(1.0, 10.0)`; `s = postLapseStability(card.d, card.s, r)` (which internally clamps `≤ card.s` and `≥ kMinStability` — a lapse **never** lengthens the interval); `weak = true`.
      - **Success:** `hard = grade == Grade.hard ? w[15] : 1.0`; `easy = grade == Grade.easy ? w[16] : 1.0`; `raw = stabilityOnSuccess(card.d, card.s, r, hard, easy)`; `gain = (raw − card.s) * conf` (only the *gain* is confidence-scaled — a noisy self-rating moves S less); `s = card.s + gain`; clear `weak` only when `(grade == Grade.good || grade == Grade.easy) && rv.errorLines.isEmpty`.
   6. **Weak-line `(11−D)` channel:** `d = (d + kWeakLineFactor * weakLineCount).clamp(1.0, 10.0)` — applied at **full strength regardless of source** (an interference/weak-line bump is graph truth, not a confidence-scaled magnitude). Higher `D` → smaller `(11−D)` in `stabilityOnSuccess` next time → shorter interval automatically, with **no parallel scheduler**.
   7. `var next = card.copyWith(d: d, s: max(s, kMinStability), lastReview: today, reps: card.reps + 1, lapses: lapses, weakFlag: weak);`
   8. `next = updateGraduation(next, grade, rv.source);` — **stub** in this task (returns `next` unchanged, `// TODO(E04-T05)`); T05 makes it sign-off-gated and predictable.
   9. `return next.copyWith(dueAt: trustClamp(next, today));` — `trustClamp` is a **stub** here (`// TODO(E04-T07)`) that returns a finite non-null `SerialDay` (e.g. `today + interval(next.s, kStubTargetR)` clamped `[1, kMaxInterval]`, *no ceiling yet*) so the non-null-`dueAt` invariant holds; T07 replaces it with the real `min(ideal, ceiling)`. Document at the stub that the **clamp direction (`min`, never `max`) and the ceiling guarantee are T07's**, not asserted here.

3. **`weakLineCount` is an injected pure input, not an I/O read.** PRD §7.7 / engineering 06 §4 write `weakLineCount(card.pageId)`, but the per-page weak-line tally is owned by the `data` layer's lazy `line_block` rows (E03), which the pure engine **must not** read. Pass the count in as an `int weakLineCount` parameter to `onReview` (or carry it on `ReviewInput` if T02's shape allows) so the engine stays pure and the function stays total and deterministic. Document the boundary: the engine *consumes* the count, the caller (E12 recite flow) *supplies* it from `data`. Never `import 'package:data/...'` or reach a clock/DB from here.

4. **`nextDifficulty` and `initialDifficulty`** per engineering 06 §4: `deltaD = -w[6] * (g.index + 1 - 3)` (Good is the anchor, `G = 3`), `dPrime = d + deltaD * (10 - d) / 9`, `reverted = w[7] * initialDifficulty(Grade.good) + (1 - w[7]) * dPrime`, `.clamp(1.0, 10.0)`. `initialDifficulty` is the FSRS `D0(G)` seed form — if E04-T06 (cold start) owns it, import it; otherwise declare it here with a `// TODO(E04-T06)` note and the FSRS form, referenced by name. The grade fed in is the **capped** grade from step 2.

5. **Totality.** The engine is total — `onReview` and its helpers **never throw**. Use `assert` (not `throw`) for the debug-only invariants (`assert(s.isFinite)`, `assert(d >= 1 && d <= 10)` after the clamps, `assert(weakLineCount >= 0)`); the production path always returns a `Card`. No `print`/`debugPrint`; no logging of the grade (the append-only `review_log` write is the `data` layer's job, not the engine's — PRD §7.7's `log(...)` line is E03/E12, not this function).

6. **Pitfalls to avoid** (each is a documented engine rule, not a style nit):
   - **Reordering the guard after the math** — the sacred-text cap MUST precede `nextDifficulty`/the branch so a dropped word can never produce a Good/Easy outcome (R1; rule 7).
   - **Scaling the wrong quantity** — `conf` scales the *gain* `(raw − card.s)`, **not** `raw`, not `d`, not the lapse stability, and **not** `errorLines`/weak-line bumps. Scaling `raw` would shrink even an unchanged page's S; scaling the weak-line bump would down-weight real interference data (rule 10).
   - **Growing S on a lapse** — never skip `postLapseStability`'s `min(sf, s)` clamp; a `max` there, or scaling the lapse S by `conf`, lets a forgotten page earn a longer interval (rule 8; INV-3).
   - **`max` where `min` belongs at the clamp call site** — even though the real clamp is T07, the *stub* must still return the **earlier** date and never lengthen to infinity; leave a why-comment naming the covenant so T07 cannot silently invert it.
   - **Letting self-rating reach the top tier** — this task does not implement tiers (T05), but it must not special-case `Source.self_` to *un*-scale; the `kSelfConfidence` gate is the mechanism that, with T05's tier rule, keeps a self-rating off the prayer-critical floor (rule 9; C-021).
   - **Reaching into `data`/clock/RNG** — no `weakLineCount(pageId)` DB read inside the engine, no `DateTime.now()`, no `Random`, no interval fuzzing; `today` is the injected `SerialDay`, elapsed is integer subtraction (rule 2).
   - **Inlining a weight or constant** — `w[15]`/`w[16]`/`w[6]`/`w[7]` come from the named vector (T10); `kSelfConfidence`/`kLapseDifficultyBump`/`kWeakLineFactor`/`kMinStability` are named `const`s — never `0.5`/`1.0`/`0.2346` at a call site (rule 4; eng-write-engine-golden-vector rule 10).

## Acceptance criteria

- [ ] `review_update.dart` exists under `packages/engine/lib/src/`, is re-exported from the `engine.dart` barrel, and carries the REUSE SPDX header; it imports only within `engine/` (curve from T03, types from T02, constants from T10) — no `flutter`, `dart:io`, `drift`, `DateTime`, or runtime `fsrs`, verifiable by grep and the E04-T01 purity gate.
- [ ] `onReview(Card, ReviewInput, SerialDay, {required int weakLineCount})` runs the engineering 06 §4 / PRD §7.7 steps in exact order: sacred-text guard → source-confidence → lapse/success split → `nextDifficulty` → weak-line channel → `updateGraduation` (stub) → `dueAt` via the `trustClamp` (stub); it returns a new `Card` via `copyWith` and mutates nothing.
- [ ] **Sacred-text guard:** a `ReviewInput` with `missedOrAlteredWord == true` and `grade` Good or Easy is capped to `Grade.hard` *before* any arithmetic; the resulting `S`/`D` equal those for an actual `Hard` review; an `Again` with the flag stays `Again` (the cap only lowers).
- [ ] **Lapse branch:** `grade == Again` ⇒ `S' = postLapseStability ≤ card.s`, `S' ≥ kMinStability`, `D' = clamp(nextDifficulty + kLapseDifficultyBump, 1, 10)`, `lapses' = lapses + 1`, `weakFlag' = true` — a forgotten page never earns a longer interval than it had.
- [ ] **Success branch:** `grade ∈ {Hard, Good, Easy}` ⇒ `S' = card.s + (stabilityOnSuccess(..., hard, easy) − card.s) · conf`, with `hard = w[15]` only on Hard and `easy = w[16]` only on Easy; `weakFlag` clears only on a clean `Good`/`Easy` with empty `errorLines`.
- [ ] **Source confidence:** `Source.teacher` applies `conf = 1.0`, `Source.self_` applies `conf = kSelfConfidence`; the scaling touches **only** the success-branch stability gain — `errorLines` recording and the weak-line `D` bump are identical for both sources (full strength regardless of source).
- [ ] **Weak-line channel:** `D` increases by `kWeakLineFactor · weakLineCount` (clamped `[1,10]`) at full strength; a higher resulting `D` yields a measurably shorter next interval through `stabilityOnSuccess`'s `(11−D)` factor (no parallel scheduler, no separate due-date path).
- [ ] **Difficulty:** `nextDifficulty` mean-reverts toward `initialDifficulty(Grade.good)` and stays in `[1,10]`; it is computed from the **capped** grade.
- [ ] `updateGraduation` and `trustClamp` are interim **stubs** with `// TODO(E04-T05)` / `// TODO(E04-T07)` and a why-comment naming the covenant each later task fulfils; the stub `dueAt` is a finite, non-null `SerialDay` so no returned memorized card has a null/infinite `dueAt`.
- [ ] `onReview` and every helper are **total** (never throw); debug-only `assert`s guard the post-clamp invariants; no `print`/`debugPrint`, no `review_log` write here.
- [ ] Golden vectors (success-S-grows, lapse, Hard, Easy, sacred-text-cap) pass `closeTo(_, 1e-6)`; every constant is referenced by name; `dart format` and `dart analyze --fatal-infos` are clean; `///` on every public declaration.

## Tests

`packages/engine/test/review_update_test.dart` and the frozen table `packages/engine/test/vectors/review_update_vectors.dart`, `package:test` (pure Dart — **no** `flutter_test`, no widget binding, no font, no `HttpOverrides`; the package has no Flutter dep), `package:glados` for the local properties. `today`/`lastReview` are explicit `SerialDay` integer literals (`day(130)`); elapsed is integer subtraction; **no clock, no RNG**. Written **FIRST** where they pin a covenant:

- **Golden-vector table** (`vectors/review_update_vectors.dart`, oracle rows, written first): each `FsrsVector(d_in, s_in, grade, elapsed, d_out, s_out, notes)` computed **once** from `dart-fsrs` (`enableFuzzing: false`, `kDefaultWeights45`) or the FSRS closed form — never from `onReview` — and asserted with `closeTo(expected, 1e-6)`, never `==`. Required rows: (a) **on-time success, S grows** (`Good`, `elapsed ≈ S`, `S' > S`); (b) **lapse** (`Again`: `S' = postLapseStability ≤ S`, `D' = D + kLapseDifficultyBump`, `weakFlag` set); (c) **Hard multiplier** (`w[15]` applied, `S' < the Good row`); (d) **Easy multiplier** (`w[16]` applied, `S' > the Good row`); (e) **post-lapse clamp** (`postLapseStability(d,s,r) ≤ s` for several `(d,s,r)`).
- **Sacred-text guard (R1), written first** — a `ReviewInput(grade: Grade.easy, missedOrAlteredWord: true, ...)` yields the **exact** `(D, S)` of a `Grade.hard` review with the same other inputs; the same for `Grade.good`; an `Again`-with-flag is unchanged. Restate `// PRD §7.7 / R1: a dropped/altered word is NEVER Good` at the assertion. This case must exist and fail before `onReview` is implemented.
- **Source confidence (C-021)** — for an identical success review, `Source.self_` produces a strictly smaller `S` gain than `Source.teacher` (`gain_self = kSelfConfidence · gain_teacher`); the `D` change, `errorLines` handling, and the weak-line `D` bump are **identical** across the two sources (only the S-gain differs).
- **Weak-line `(11−D)` channel** — increasing `weakLineCount` raises `D'` by `kWeakLineFactor · count` (clamped) and, fed back through `stabilityOnSuccess`, yields a shorter next interval; the bump is the **same** for `self_` and `teacher` (full strength regardless of source).
- **Lapse demotes (INV-3 seed, also covered exhaustively in T11)** — a local `glados` property over a generated `(Card, elapsed)`: `Again ⇒ S' ≤ S`, never grows; `lapses` increments; `weakFlag` set. Rely on shrinking; no fixed lucky seed.
- **Determinism** — calling `onReview` twice on the same `(Card, ReviewInput, today, weakLineCount)` returns byte-equal `Card`s (`==`); no path reads a clock or RNG.
- **Purity / offline guard** (complements the E04-T01 banned-import grep gate) — `review_update.dart` and the vectors file import no `dart:io`/`flutter`/`drift`, reference no `DateTime`, and open no socket; airplane-mode-safe by construction.
- **Stub boundary** — `updateGraduation` returns its input unchanged (until T05) and `trustClamp` returns a finite non-null `SerialDay` (until T07); a test asserts the returned `dueAt` is non-null and finite for a memorized result, and a `// TODO` marks that the *ceiling* assertion lands in T07/T11, not here.

(No widget/golden-image or integration test — `engine/` renders nothing. The §7.12 invariant *properties* INV-3/INV-5 over full histories live in E04-T11; this task ships the branch-level vectors + the local lapse/determinism properties they build on.)
