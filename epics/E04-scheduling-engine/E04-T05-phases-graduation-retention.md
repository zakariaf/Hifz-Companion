## Goal

The three traditional tracks become **three lifecycle phases of one page card** in the pure-Dart `engine/` package — never three algorithms. `phaseOf(card)` derives the track from stability (`S < kNearMinS ~9` → New, `< kFarMinS ~60` → Near, else Far), with a `manualLock` teacher pin overriding the math. `updateGraduation(card, grade, source)` makes every transition predictable and sign-off gated: New → Near requires *N* sign-offs, Near → Far requires crossing `kFarMinS` **and** leaving the recent-juz window, a lapse demotes naturally as `S` shrinks, and a page graduates on **fluency (Easy)**, not bare correctness. `targetR(card)` returns the stakes-tiered retention target — `0.90 / 0.94 / 0.95`, escalating to `0.97+` for `prayerCritical` / `weakFlag` / previously-lapsed pages — never a global 0.99, never a user-facing slider. The `onReview` path from E04-T04 already calls `updateGraduation` (before the trust clamp) and `interval(...)` already reads `targetR`; this task fills in those two functions plus `phaseOf`, with the graduation transitions and the retention tiers pinned by frozen golden vectors and the first slice of the §7.12 invariants.

## Implementation notes

TEST-FIRST: graduation and retention tiering are correctness-critical (they decide when a page is treated as "owned" and how often a prayer-critical page returns — getting either wrong silently under-revises sacred text). Write the golden-vector retention rows and the graduation/lapse-demotes property below **before** the function bodies; watch the wrong-direction graduation case and the "global 0.99" guard fail before the code exists.

1. **Files** (in the package scaffolded by E04-T01, alongside T03's curve and T04's `onReview`): put `phaseOf`, `updateGraduation`, and `targetR` in `packages/engine/lib/src/phases.dart`, re-exported from the `packages/engine/lib/engine.dart` barrel. Every file carries the REUSE SPDX header (`GPL-3.0-or-later`). These are **free top-level pure functions** (or static methods on the façade) — no class state, no clock, no RNG, no I/O. `today` and `config` are passed in by the caller (`onReview`), never read here.

2. **`phaseOf(Card c)` → `Track`** — verbatim engineering 06 §5, derived **only** from `S` and state, with no stored `phase` field (rule 11):
   ```dart
   Track phaseOf(Card c) {
     if (c.track == Track.unmemorized) return Track.unmemorized;
     if (c.manualLock) return c.track;            // teacher pin wins over the math
     if (c.s < kNearMinS) return Track.newLesson; // still solidifying
     if (c.s < kFarMinS) return Track.near;
     return Track.far;
   }
   ```
   `kNearMinS` (≈9.0) and `kFarMinS` (≈60.0) are **E04-T10 named constants**, never inlined `9`/`60`. The `manualLock` short-circuit returns the card's *stored* `track` (the pin), so a pinned page's phase is whatever the teacher fixed — even if `S` would say otherwise.

3. **`targetR(Card c)` → `double`** — the stakes-tiered switch, exactly the §5 / PRD §7.5 table:
   ```dart
   double targetR(Card c) {
     switch (phaseOf(c)) {
       case Track.newLesson:   return config.newTargetR;  // 0.90 default
       case Track.near:        return config.nearTargetR; // 0.94 default
       case Track.far:
         return (c.prayerCritical || c.weakFlag || c.lapses > 0)
             ? config.criticalTargetR  // 0.97+ default
             : config.farTargetR;      // 0.95 default
       case Track.unmemorized: return config.farTargetR;  // unreachable; defensive default
     }
   }
   ```
   The tier **defaults** (0.90 / 0.94 / 0.95 / 0.97) live on `EngineConfig` (E04-T10), tunable but not user-facing — there is **no retention slider** anywhere (rule 13; PRD §7.5). If E04-T10's config plumbing is not yet wired in this PR, reference the named constants `kNewTargetR`/`kNearTargetR`/`kFarTargetR`/`kCriticalTargetR` and leave a `// TODO(E04-T10): read from EngineConfig` — never inline `0.90`/`0.97`.

4. **`updateGraduation(Card c, Grade grade, Source source)` → `Card`** — the predictable, sign-off-gated, fluency-gated transition. It is called by `onReview` (T04) **after** the S/D update and **before** `trustClamp`, on a card whose `s` already reflects this review. The transitions:
   - **Count the sign-off first.** A `source == Source.teacher` review increments `signoffs` (a self review does not, or counts toward a self-only sub-threshold per PRD §7.4 "*N* teacher/self sign-offs" — keep whichever the constant `kGraduationSignoffs` encodes; document the choice). This is the only place `signoffs` grows; `manualLock` is set elsewhere (a teacher action, not graduation).
   - **New → Near (promotion) requires fluency AND sign-offs.** Promote only when the post-update `phaseOf` would already be Near *and* the page has reached `kGraduationSignoffs` *and* this review was fluent (`grade == Grade.easy`, the automaticity signal — science 06 §4 / C-024). A correct-but-effortful `Hard` must **not** promote even if `S` crossed the band (rule 14). Because phase is derived from `S`, "promotion" here means *the gate is satisfied so the strengthened `S` is allowed to stand* — never force `track` ahead of what `S`/sign-offs warrant.
   - **Near → Far (promotion) requires crossing `kFarMinS` AND leaving the recent-juz window AND fluency.** A page only joins the manzil bulk when its `S ≥ kFarMinS`, it is no longer in the recent-juz window (the `inRecentWindow(c, config)` predicate E04-T08 owns; reference it, do not duplicate its definition here), and the recitation was fluent. Until all three hold, the page stays Near.
   - **A lapse demotes naturally — no explicit demote branch.** E04-T04 already shrank `S` (post-lapse `S ≤ S`) and set `weakFlag` on `Grade.again`; because `phaseOf` reads `S`, a far page that lapses re-derives as Near/New automatically — "a forgotten manzil page rejoins active revision." `updateGraduation` must **not** add a competing demotion that could disagree with `phaseOf`; it only governs *promotion* gates. (INV-3: `Again ⇒ S' ≤ S ∧ track' ≤ track`.)
   - **`manualLock` freezes graduation.** If `c.manualLock`, `updateGraduation` performs no promotion — the teacher's pin stands (PRD §7.4); only sign-off counting (if any) still applies.
   - Return a new `Card` via `copyWith` (signoffs and, where a stored `track` is updated for a satisfied gate, `track`); **mutate nothing**, **throw nothing** (asserts only — the engine is total, eng-write-to-coding-standards §6).

5. **Keep one source of truth.** `phaseOf` is the read model; `updateGraduation` is the write gate. They must never disagree: do not store a redundant `phase` field, and do not let `updateGraduation` set a `track` that `phaseOf` would contradict given the card's `S` (except the deliberate `manualLock` pin). The cleanest shape keeps `track` tracking `phaseOf` exactly **except** when a gate (sign-offs, window, fluency) deliberately *holds a page back* from a band its `S` reached — document that asymmetry at the function.

6. **Pitfalls to avoid:** inlining `9`/`60`/`0.90`/`0.94`/`0.95`/`0.97` as literals (all are E04-T10 named constants — `kNearMinS`, `kFarMinS`, `kNewTargetR`…; the magic-number ban is rule 13 / golden-vector skill rule 10); a **global 0.99** or any path that returns the same `targetR` for all phases (rule 13); exposing a retention **slider** or any `targetR` setter reachable from UI (PRD §7.5 — high R is reserved, not dialed); graduating on `Grade.good`/`Grade.hard` (bare correctness) instead of `Grade.easy` (fluency — rule 14 / science 06 §4); a **hidden jump** that promotes Near → Far on `S` alone without the recent-juz-window check (rule 12); duplicating `inRecentWindow` here instead of referencing E04-T08's predicate; adding an explicit "demote" branch that double-counts the natural `S`-driven demotion and can disagree with `phaseOf` (rule 11/INV-3); building a **second scheduler** keyed on `track` instead of deriving everything from one card (rule 11); reading a clock, `Random`, or any I/O (the purity boundary); letting **self-rating alone** push a page into the prayer-critical tier — `prayerCritical` is a card flag set upstream, and the top tier is also reached via `weakFlag`/`lapses`, but the *S growth* that gets there was already confidence-scaled in T04 (rule 9); a `throw` instead of `assert`.

## Acceptance criteria

- [ ] `phases.dart` exists under `packages/engine/lib/src/` and is re-exported from the `engine.dart` barrel; it carries the REUSE SPDX header and imports only what the engine boundary allows (`meta` (+ `models`) — no `drift`/`flutter`/`dart:io`, no `DateTime`, no `Random`).
- [ ] `phaseOf(Card)` returns `unmemorized` for an unmemorized card, the **stored `track`** when `manualLock` is set, and otherwise the `S`-band phase (`< kNearMinS` → newLesson, `< kFarMinS` → near, else far) — using the E04-T10 named thresholds, never inlined `9`/`60`.
- [ ] `targetR(Card)` returns `0.90 / 0.94 / 0.95` for New / Near / ordinary-Far and `0.97+` when `prayerCritical || weakFlag || lapses > 0`; the four values come from named constants / `EngineConfig`, never inlined; there is **no global single target** and **no public `targetR` setter or retention slider**.
- [ ] `updateGraduation(Card, Grade, Source)` promotes **New → Near** only when the sign-off count has reached `kGraduationSignoffs` **and** the review was fluent (`Grade.easy`); a correct-but-effortful `Hard` never promotes.
- [ ] `updateGraduation` promotes **Near → Far** only when `S ≥ kFarMinS` **and** the page is outside the recent-juz window (via the E04-T08 `inRecentWindow` predicate, referenced not duplicated) **and** the review was fluent.
- [ ] A `Grade.again` (lapse) never promotes; the resulting card's derived `phaseOf` is **≤** its prior phase (demotion is the natural consequence of the shrunk `S` from E04-T04, not a competing branch) — `Again ⇒ track' ≤ track`.
- [ ] A `manualLock` card is never auto-promoted or auto-demoted by `updateGraduation`; the teacher's pinned `track` stands.
- [ ] `phaseOf`, `targetR`, and `updateGraduation` are pure (`today`/`config` injected, no clock/RNG/I-O), total (assert, never throw), and return a new `Card` (`updateGraduation` mutates nothing).
- [ ] Every public declaration carries a `///` summary-first doc comment; the fluency-gate and the no-global-0.99 rules are restated as why-comments at their enforcement points; `dart format` and `dart analyze --fatal-infos` are clean.

## Tests

`packages/engine/test/phases_test.dart` (unit + table) and golden rows in `packages/engine/test/vectors/retention_tiers_vectors.dart`, plus graduation/lapse properties in `packages/engine/test/phases_property_test.dart`. All `package:test` (+ `package:glados` for properties) — **no `flutter_test`, no widget binding, no font, no `HttpOverrides`, no network**: the engine is pure Dart and so are its tests. Every `today` is a constructed `SerialDay` integer literal; elapsed days is integer subtraction; no `DateTime.now()`. Written FIRST where they pin a covenant:

- **`phaseOf` band table**: a row per boundary — `S` just below `kNearMinS` → newLesson, exactly `kNearMinS` → near, just below `kFarMinS` → near, exactly `kFarMinS` → far; an `unmemorized` card → unmemorized regardless of `S`; a `manualLock` card returns its stored `track` even when `S` would say otherwise. Boundary values exercised, not just mid-band.
- **`targetR` tier table (golden, `closeTo(_, 1e-6)`)**: New → 0.90, Near → 0.94, ordinary Far → 0.95, and Far with each of `prayerCritical` / `weakFlag` / `lapses > 0` (and combinations) → 0.97. Assert the **interval consequence** too, from an independent oracle: `interval(100, targetR(newCard)) ≈ interval(100, 0.90)` … `interval(100, 0.97) < interval(100, 0.95) < interval(100, 0.94) < interval(100, 0.90)` — a higher tier is a shorter interval. Constants referenced by name; a guard asserts no `targetR` ever returns ≥ 0.99 (the "never a global 0.99" covenant, named in a comment).
- **Graduation New → Near**: an `Easy` review at/above `kGraduationSignoffs` promotes; the *same* state with one fewer sign-off does **not**; the same state with a `Hard`/`Good` review (correct but not fluent) does **not** (the fluency gate, C-024 / science 06 §4 named in a comment); a `self`-source review's effect on the sign-off count matches `kGraduationSignoffs`' documented teacher/self rule.
- **Graduation Near → Far**: promotes only when `S ≥ kFarMinS` AND outside the recent-juz window AND `Easy`; each of the three conditions removed in turn blocks promotion (three negative cases). The `inRecentWindow` predicate is fed deterministically (an injected `config`/window fixture), never a clock.
- **Lapse demotes (property, INV-3)**: a `glados` property over generated `(Card, grade-sequence, today)` histories — for any history ending in `Grade.again`, `phaseOf(after) ≤ phaseOf(before)` and `targetR(after) ≥ targetR(before)` (a demoted/weakened page is revised *more* often, never less). `glados` shrinks to the minimal counterexample; no fixed lucky seed. The covenant ("a forgotten page rejoins active revision; SR may only make it MORE frequent") is named at the assertion.
- **`manualLock` freezes graduation (property)**: for any `(Card with manualLock, grade-sequence)`, `updateGraduation`'s returned `track` equals the pinned `track` — the teacher's authority is never overridden by the math.
- **Determinism / purity guard**: two `updateGraduation` runs on identical inputs are byte-identical (`==`); a lightweight grep/import check (complementing the E04-T01 banned-import gate) asserts `phases.dart` references no `DateTime`/`Random`/`dart:io`/`flutter`.

(Golden *vectors* here pin only the retention-tier intervals; the FSRS curve/stability anchors are E04-T03, the cold-start seed rows E04-T06. The full six-property §7.12 register lands in E04-T11, which reuses the lapse-demotes and manualLock-freeze properties seeded here.)
