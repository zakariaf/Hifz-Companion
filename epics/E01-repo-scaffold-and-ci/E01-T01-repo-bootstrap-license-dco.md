## Goal

The repository's **first commit** carries the trust-pack subset that closes the GPL App-Store / forked-derivative failure mode *before any second copyright holder exists*: a `LICENSE` containing the unmodified GPL-3.0 text immediately followed by the verbatim GPLv3 §7 App Store additional permission, a `LICENSING.md` recording the code-vs-assets split, a `CONTRIBUTING.md` reproducing the DCO 1.1 text verbatim with `git commit --signoff` instructions and the ≤~400 LOC / one-concern PR rule, the `.github/PULL_REQUEST_TEMPLATE.md` checklist from engineering 03 §8.1, `LICENSES/GPL-3.0-or-later.txt` plus a `.reuse/dep5` covering binaries/fonts, the per-file REUSE SPDX header convention applied to every committed file, and a `README.md` stub with the one-sentence promise. The end state: `reuse lint` is green, the §7 exception text is present verbatim, and DCO + PR-template files are in place — with **no CLA** anywhere. These are documents and metadata only; this task ships no Dart, no `pubspec.yaml`, and no CI workflow (those are T02–T07).

## Implementation notes

This task writes **prose, license text, and metadata** — there is no correctness-critical Dart, so the test-first discipline does not apply; the verification is `reuse lint` green + a byte-exact diff of the §7 block + a manual presence check (see Tests). Touch only the files below; do not create `pubspec.yaml`, any `.dart`, or any CI workflow here.

1. **`LICENSE`** (repo root). The **unmodified** GPL-3.0 text, then a blank line, then the additional-permission grant **copied verbatim** from engineering 13 §1's fenced block. The preamble sentence ("This program is free software … any later version."), the `--- Additional permission under GNU GPL version 3 section 7 ---` divider, the App-Store-conveyance paragraph naming Hifz Companion, clause **(1)** (Corresponding Source remains available under this License through a non-restrictive channel), clause **(2)** (you do not remove or alter this additional permission), and the closing "If you modify this program …" sentence must all be present, character-for-character. Do **not** paraphrase, reflow, or "tidy" the block — a single altered word weakens the legal grant.

2. **`LICENSING.md`** (repo root). The four-row code-vs-assets split table from engineering 13 §1 (`App code` → this repo, **GPL-3.0-or-later + §7 App Store exception**, "copyleft = waqf in law"; `Quran text` → data repo, Tanzil verbatim + attribution, never relicensed; `Page glyph fonts` → data repo, KFGQPC font terms, NonFreeAssets risk if restrictive; `Mutashābihāt dataset` → data repo, CC-BY scholar-reviewed). One short intro sentence: the app's GPL covers **code only**, never the muṣḥaf bytes. Link the asset rows to `docs/engineering/09-asset-packs-and-offline-integrity.md`.

3. **`CONTRIBUTING.md`** (repo root). Four sections: (a) **License & the App Store exception** — inbound code is GPL-3.0-or-later *with the §7 exception*; a contributor who cannot license that way cannot contribute that code; **no CLA, no copyright assignment**. (b) **DCO 1.1, verbatim** — reproduce the four clauses (a)/(b)/(c)/(d) of the Developer Certificate of Origin **character-for-character**, then the sign-off instruction (`git commit --signoff -m "engine: clamp due_at to the cycle ceiling"`) and the resulting `Signed-off-by: Name <email>` trailer; note that a missing sign-off is rebased to add it, never waved through (the CI check that enforces this is T07). (c) **PR rules** — PR-based even solo; ≤~400 changed LOC; one concern per PR; self-annotate non-obvious hunks; 100% review on `engine`/`data`/`quran`/the asset downloader (engineering 03 §8). (d) A pointer to `.github/PULL_REQUEST_TEMPLATE.md`.

4. **`.github/PULL_REQUEST_TEMPLATE.md`**. The engineering 03 §8.1 checklist, **verbatim**, preserving every section and its bracketed citations: **Every PR** (intent stated; ≤~400 LOC + self-annotation; format/analyze/fix clean + tests green; no new dependency / lock change explained — no analytics/ads/backend/crash SDK; `///` docs + comments-still-true); **Sacred text & rendering** (Quran bytes unchanged; pinned SHA-256 matches; no runtime line-breaking; markers stay coordinate overlays); **Scheduling correctness** (engine imports no Flutter/Riverpod/Drift/`dart:io`, "today" injected; trust clamp holds, manzil never silently dropped, nothing implies "safe to drop"; golden/property coverage + sacred-text guard intact); **Dates & calendars** (no `DateTime` below the boundary; `CalendarDate` integer-day math); **Persistence & privacy** (single write path, persist-before-republish; no bare `catch`/swallowed write errors/`!`/`late`; no `print`/`debugPrint`/logging of user data, no networking outside the downloader); **Localization & RTL** (no hardcoded user-facing strings; new keys in all three ARB locales; numerals/calendar per locale); **Trust-critical modules** footer (full checklist on 100% of changed lines; AI reviewer's written findings attached). Keep the citation links as written in the doc.

5. **`LICENSES/GPL-3.0-or-later.txt`**. The full GPL-3.0 license text, filed under its exact SPDX id — required by REUSE so every `SPDX-License-Identifier: GPL-3.0-or-later` resolves to a vendored text. (`LICENSES/CC-BY-4.0.txt` is **not** added here — no CC-BY doc/data is vendored into the code repo yet; it lands only when one is, per engineering 13 §2.)

6. **`.reuse/dep5`** (Debian `copyright`-format). Bulk SPDX coverage for files that **cannot** carry an inline header — at this stage the trust-pack subset is text-only, so seed `dep5` with the obvious future-proof entries (e.g. `*.png`, `*.ttf`, `*.otf`, `*.ico` glob stanzas mapping to the copyright holder + `GPL-3.0-or-later`, with the muṣḥaf-asset license carve-out documented as a comment pointing to the data repo). Every stanza names `Copyright:` and `License:`. This is the file T02–T04 extend as real binaries/fonts arrive; it must already make `reuse lint` green on the seed set.

7. **REUSE SPDX header on every committed file.** Apply the two-line convention from engineering 13 §2 / eng-write-to-coding-standards §4: the copyright holder string is **`2026 Zakaria Fatahi and Hifz Companion contributors`** (single canonical form, used identically everywhere); the identifier is **`GPL-3.0-or-later`**. Comment syntax follows the file type — `//` for Dart (none in this task), `#` for YAML, an HTML comment (`<!-- … -->`) at the top of a Markdown file, or `.reuse/dep5` bulk coverage for plain-text license files and any header-less file. The `LICENSES/*.txt` upstream license texts are covered by `dep5`, not edited.

8. **`README.md` stub** (repo root). One short paragraph carrying the **one-sentence promise** drawn from PRD §1 / §17 — plainly factual: *a fully offline Flutter app, built free as ṣadaqah jāriyah, that helps a ḥāfiẓ never silently lose the Quran; no accounts, no telemetry, no microphone, no ads.* No screenshots, no badges, no number, no superlative. A two-line "License" section pointing at `LICENSE`/`LICENSING.md` and a "Contributing" line pointing at `CONTRIBUTING.md` (DCO required). Keep it a stub — the full README is later.

**Pitfalls to avoid.** (a) Paraphrasing, reflowing, or "improving" the §7 block or the DCO clauses — both must be **verbatim**; the legal grant is the point of the task. (b) Adding a CLA, a copyright-assignment line, or "contributions licensed under MIT for convenience" — all three are explicit refusals (engineering 13 §1, §3). (c) Letting the copyright-holder string or year drift between files — `reuse lint` tolerates it but the audit story wants one canonical form. (d) Putting the license "in the README only" with no `LICENSES/` text and no per-file headers — the exact ambiguity REUSE exists to kill (engineering 13 §2 pitfall). (e) Wiring the DCO/`reuse lint` **CI** here — that is T07; this task ships the human-readable contract and the lint must pass *locally*. (f) Drafting `SECURITY.md`/`PRIVACY.md`/`verify-yourself.md`/`fastlane/metadata/` — those are E20. (g) Any marketing claim or on-screen number in the README that would need a CLAIMS row — keep the prose plainly factual.

## Tests

This task ships no Dart, so there are **no unit / widget / golden / integration tests** — by construction the deliverable is license text and metadata, and the verification is tool-driven and manual:

- **`reuse lint` (the machine gate).** Run `reuse lint` (the `fsfe/reuse-action` runs the same binary in T07's CI). It must report **every** file either has a header or is covered by `.reuse/dep5`, with **0** files unaccounted-for and **0** bad licenses — exit code 0. This is the dormant-but-active equivalent of the project's other fail-closed gates: it checks a real, non-empty file set from commit one.
- **§7 exception byte-diff (manual, recorded).** Diff the additional-permission block in `LICENSE` against the fenced block in engineering 13 §1; the diff must be empty modulo the surrounding GPL text. Record the check in the PR description (the self-annotation the §8.1 checklist requires) so a reviewer re-runs it.
- **DCO-clauses presence check (manual).** Confirm the four DCO 1.1 clauses (a)/(b)/(c)/(d) appear verbatim in `CONTRIBUTING.md` and the `git commit --signoff` instruction is present.
- **PR-template section check (manual).** Confirm all six §8.1 sections plus the Trust-critical-modules footer are present in `.github/PULL_REQUEST_TEMPLATE.md`.
- **Offline / no-network guard:** N/A by construction — no code, no network path, no dependency is introduced; nothing here can reach a socket. (The throwing-`HttpOverrides` test bootstrap is T06.)
