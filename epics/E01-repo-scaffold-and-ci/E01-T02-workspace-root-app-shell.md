## Goal

The pub-workspace root and the composition-root-only `app/` shell exist and resolve as one graph. The root `pubspec.yaml` (`publish_to: none`, `sdk: ^3.6.0`, a `workspace:` listing all nine members, **no application code**) plus a single committed root `pubspec.lock` make the whole dependency graph a one-file audit; `analysis_options.yaml` carries `flutter_lints` + `--fatal-infos` + `page_width: 80` + the path-scoped `avoid-banned-imports` layering bans at `severity: error`; `l10n.yaml` points `gen_l10n` at the `ar` template with `synthetic-package: false`. The `app/` target wires, gates, and declares only: `main.dart` runs `ProviderScope(child: HifzApp())` with no logic; `app.dart` builds a `MaterialApp.router` whose `supportedLocales` are `ar`/`fa`/`ckb` and whose locale forces `Directionality.rtl` by construction; the placeholder screen renders one string resolved through `AppLocalizations`, never a hardcoded literal. `dart pub get` resolves with one lock and no `dependency_overrides`; the app builds and launches to that placeholder on an Android emulator and an iOS simulator; the analyzer is clean; `ios/Runner/Info.plist` carries **no** `NSMicrophoneUsageDescription`.

## Implementation notes

This task is composition and configuration, not correctness-critical engine math, so there is **no frozen-vector / property test**. The one verifiable behaviour — *launches to a placeholder screen, RTL by construction, string from `l10n`* — is pinned by a widget test (see Tests), which is reasonable to write alongside the screen.

1. **Workspace root `pubspec.yaml`** (no source): `name: hifz_companion_workspace`, `publish_to: none`, `environment: { sdk: ^3.6.0 }`, and a `workspace:` list of all **nine** members in the 02 §1.1 / 01 §3.1 order: `app`, `packages/models`, `packages/engine`, `packages/profiles`, `packages/data`, `packages/quran`, `packages/l10n`, `packages/features`, `packages/assets`. The `models` package is the bottom-of-graph value-type member 01 §2/§3.1 splits out below `engine`; the count is nine members (`app` + eight packages), not the eight shown in the 02 §1 tree. No `dependencies:` block beyond the SDK constraint — the root resolves, it does not implement.
2. **Committed root `pubspec.lock`**: after `dart pub get` resolves the whole workspace cleanly, commit exactly **one** `pubspec.lock` at the repo root. No per-package lockfile is committed (`.gitignore` need not list it, but verify none exist). The lock is the §20-gate-6 audit map; it is regenerated only by an intentional `dart pub get`, never hand-edited.
3. **`analysis_options.yaml`** (repo root, the single analysis context for the whole workspace): copy the 03 §7.1 body verbatim — `include: package:flutter_lints/flutter.yaml`; `formatter: page_width: 80`; `analyzer.language.strict-casts`/`strict-raw-types: true`; the `errors:` promotions (`avoid_print`, `avoid_dynamic_calls`, `avoid_catches_without_on_clauses`, `cancel_subscriptions`, `close_sinks`, `cast_nullable_to_non_nullable`, `public_member_api_docs`, `dangling_library_doc_comments` all `error`; `todo: ignore`); the `exclude:` of `**/*.g.dart` / `**/*.freezed.dart` / `**/*.drift.dart`; the `linter.rules` additions (`prefer_const_constructors`, `prefer_const_constructors_in_immutables`, `prefer_const_declarations`, `prefer_final_locals`, `prefer_final_in_for_each`, `avoid_redundant_argument_values`, `require_trailing_commas`, `unnecessary_lambdas`, `use_super_parameters`, `prefer_is_empty`, `prefer_is_not_empty`, `avoid_positional_boolean_parameters`, `unnecessary_null_checks`, `null_check_on_nullable_type_parameter`). Then add the path-scoped `avoid-banned-imports` block from 02 §5 / 03 §7.2 at `severity: error`: (1) engine purity — `paths: ['packages/engine/.*\.dart']` deny `package:flutter/.*`, `dart:io`, `dart:ui`; (2) no-network — `paths: ['(?!packages/assets/).*\.dart']` deny `package:http/.*`, `package:dio/.*`, `dart:io.*HttpClient`; (3) quran isolation — `paths: ['(?!packages/quran/).*\.dart']` deny `package:quran/src/glyph_page.dart`; (4) features-depend-down — the per-feature self-import ban; plus the project-wide legacy-Riverpod ban (`package:flutter_riverpod/legacy.dart`, `package:riverpod/legacy.dart`). Each entry carries its `message:`. The bans are config only here — their grep twins (E01-T05) and deliberate-violation proofs (E01-T08) follow.
4. **`l10n.yaml`** (repo root): verbatim 02 §6 — `arb-dir: packages/l10n/lib/src/arb`, `template-arb-file: app_ar.arb`, `output-dir: packages/l10n/lib/src/generated`, `output-localization-file: app_localizations.dart`, `output-class: AppLocalizations`, `synthetic-package: false`. The `l10n` package, the three ARB files, and the generated `AppLocalizations` are E01-T04/E09's full content; this task lands the config and consumes whatever placeholder key the shell needs (coordinate with T04 so `app_ar.arb` carries at least the one placeholder key + its `fa`/`ckb` values, and the generated source is committed `synthetic-package: false`).
5. **`.gitignore`**: ignore `.dart_tool/` and `build/` (02 §1). Do **not** ignore `pubspec.lock` or the generated `app_localizations.dart` — both are committed audit artifacts.
6. **`app/pubspec.yaml`** (the composition root — the one manifest allowed full fan-in): `name: app` (or `hifz_app`), `publish_to: none`, `resolution: workspace`, `environment: { sdk: ^3.6.0, flutter: ">=3.38.0" }`; `dependencies:` `flutter: { sdk: flutter }`, `flutter_localizations: { sdk: flutter }`, `flutter_riverpod: ^3.0.0`, `go_router: ^16.0.0`, and a `path:`/workspace edge to **every** package (`models`, `engine`, `profiles`, `data`, `quran`, `l10n`, `features`, `assets`) — this fan-in is expected here and only here. **No `google_fonts`.** Declare the bundled UI fonts under `flutter: fonts:` pointing at `app/assets/fonts/` (Vazirmatn/Estedad for fa/ar, a Sorani-covering font for ckb — PRD §13.5); ship the actual `.ttf` files or a documented placeholder so the build succeeds. `dev_dependencies: flutter_test: { sdk: flutter }` (+ `integration_test: { sdk: flutter }` reserved for T06).
7. **`app/lib/main.dart`** — exactly `void main() => runApp(const ProviderScope(child: HifzApp()));`, no other logic. One `///` is not required on `main`, but the REUSE header is.
8. **`app/lib/app.dart`** — `HifzApp` is a `ConsumerWidget` (or `StatelessWidget`) building `MaterialApp.router`. Wire: `localizationsDelegates: AppLocalizations.localizationsDelegates` (including the custom `ckb` Material delegate from the `l10n` package), `supportedLocales: AppLocalizations.supportedLocales` resolving to `[Locale('ar'), Locale('fa'), Locale('ckb')]`. RTL is **structural, not per-widget**: because all three supported locales are RTL, do not set a per-widget `Directionality`; instead force it at the app root — wrap the router's content in a `Builder` that returns `Directionality(textDirection: TextDirection.rtl, child: …)`, or rely on `MaterialApp`'s locale-derived directionality and assert it. The chosen mechanism must make "RTL by construction from the locale" provable by the widget test in Tests. The `routerConfig` comes from `composition/router.dart` (step 10).
9. **`app/lib/composition/providers.dart`** — an **empty** placeholder: a documented `final` list `<Override>[]` (or a commented stub) that `main.dart`'s `ProviderScope` will later receive. No live Drift handle, no asset loader, no injected `today` clock yet — those bind here in E07; this task only reserves the file as "the one place real implementations are bound."
10. **`app/lib/composition/router.dart`** — a minimal `GoRouter` with exactly one route `'/'` → the placeholder screen. RTL bottom-nav ordering (PRD §13.2) is reserved for E07; one route is enough to satisfy `MaterialApp.router`.
11. **`app/lib/bootstrap/first_run.dart`** — a **stub**: a documented function/class signature (`/// Triggers the one-time core-pack download, verifies SHA-256, builds the local DB, then routes to onboarding.`) that does nothing yet and is not invoked. It exists so E07/E05 have a named home; it must compile and import nothing from `assets`/`data` it cannot resolve.
12. **The placeholder screen** — a small `StatelessWidget` (in `app/lib/` or a `placeholder/` folder, **not** `utils/`/`common/`) rendering a `Scaffold` whose single `Text` reads `AppLocalizations.of(context)!.<placeholderKey>` — **never** a string literal. This is the load-bearing rule of the task: the one visible string proves the `l10n.*`-only discipline from the first screen.
13. **`app/android/`** — generated Flutter Android glue; `AndroidManifest.xml` carries no microphone permission (`RECORD_AUDIO`) and no telemetry service; the only network capability is the asset fetch (which `assets` owns, not the shell). Keep `INTERNET` only as Flutter's default; no extra permission.
14. **`app/ios/`** — generated Flutter iOS glue; **`ios/Runner/Info.plist` MUST NOT contain `NSMicrophoneUsageDescription`** and no microphone capability is requested (PRD R5, C2). Its absence is the structural privacy guarantee — verify by grep.
15. **Pitfalls to avoid:** putting any business logic in `app/lib/` (a review-blocking bug — the shell computes nothing); reaching for `dependency_overrides` to fix a resolution clash (fix the clash at `dart pub get` time instead); committing a per-package `pubspec.lock`; setting `synthetic-package: true` or omitting it (removed by Flutter — must be `false`); adding `google_fonts` or any analytics/ads/crash/backend SDK; hardcoding the placeholder string instead of resolving it via `l10n`; setting `Directionality` per-widget instead of by-construction-from-locale; leaving an `NSMicrophoneUsageDescription` or a `RECORD_AUDIO` permission in the platform glue; a `// ignore:` on any §7.2 architecture ban (forbidden).

## Tests

The shell's launch journey and the throwing-`HttpOverrides` offline bootstrap are E01-T06's deliverables (`integration_test/` + the global test setup); this task ships only the widget test that pins the placeholder behaviour, plus the static checks the acceptance criteria above already enumerate. Files:

`app/test/app_test.dart` (`flutter_test`, file name mirrors `app.dart`, runs in the fast CI lane):

- **Launches to the placeholder:** `pumpWidget(const ProviderScope(child: HifzApp()))` and `pumpAndSettle()` finds the placeholder `Scaffold` and the placeholder `Text`. Proves `main`/`app`/`router` wire to one screen.
- **String comes from `l10n`, not a literal:** the test loads `AppLocalizations` for a chosen locale and asserts the rendered `Text` equals `l10n.<placeholderKey>` (not a hardcoded string) — so a future hardcoded literal breaks the test.
- **RTL by construction:** pumped under each of `Locale('ar')`, `Locale('fa')`, `Locale('ckb')`, the test asserts the effective `Directionality.of(context)` is `TextDirection.rtl` for all three — proving direction is derived from the supported (all-RTL) locale set, not hand-set per widget.
- **Supported locales are exactly `ar`/`fa`/`ckb`:** assert `AppLocalizations.supportedLocales` (and the `MaterialApp`'s `supportedLocales`) is exactly that set, in RTL-first product order.

Offline/no-network guard (lands with the harness in E01-T06, exercised on this shell): the global test bootstrap installs the throwing `HttpOverrides` (11 §7) so the placeholder launch making any network call is a hard, named failure; the shell makes none. No engine golden vectors apply — this task touches no engine math.

CI lanes affected: the **fast** job runs `app/test/app_test.dart`, `dart analyze --fatal-infos`, and `dart format --set-exit-if-changed`; the **journeys** job (once T06/T07 land) runs the emulator/simulator launch of this shell. No new gate logic is added here beyond the `analysis_options.yaml` config.
