## Goal

The three host-testable pure-Dart packages exist under `packages/` — `engine`, `models`, `profiles` — each a `resolution: workspace` member whose `pubspec.yaml` declares the narrowest possible dependency set (`engine` → `meta` + `models`; `models` → `meta`; `profiles` → `meta`), carries **no `flutter:` line, no `dart:io`, no `dart:ui`, no clock**, exposes exactly one public `lib/<package>.dart` barrel over a private `lib/src/`, and ships one passing placeholder `dart test`. The `engine` manifest *is* the purity audit: an auditor reads one dependency line and knows the scheduler imports no Flutter and no I/O before any scheduling math exists. The `engine` placeholder test is a vector-style case shaped to receive E04's frozen `(state, grade, elapsed) → (D, S, due)` golden vectors. No scheduling arithmetic, no value-type logic — empty-but-compiling skeletons that make the fast `dart test` CI lane green from commit one.

## Implementation notes

TEST-FIRST (engine): write the engine placeholder vector case (Tests below) **before** the empty `engine` API, so the `dart test engine/` lane is green and the vector-shaped slot exists for E04. `models`/`profiles` placeholder cases may be written alongside their barrels.

1. **`packages/models/` — bottom of the graph, write first.** `pubspec.yaml`: `name: models`, `publish_to: none`, `resolution: workspace`, `environment: sdk: ^3.6.0` (no `flutter:` line), `dependencies: meta: ^1.15.0`, `dev_dependencies: test: ^1.25.0`. Barrel `lib/models.dart` (`library;`, REUSE header, summary doc, `export 'src/...' show ...` only — empty export list is fine for the stub). One private `lib/src/` placeholder type, e.g. a `@immutable` value type with `final` fields and a `const` constructor (no domain fields yet — the real `Card`/`Grade`/`CycleConfig`/`ReviewResult` land in E03). Add `- packages/models` to the root `workspace:` list.
2. **`packages/engine/` — copy 02 §3.3 verbatim.** `pubspec.yaml`: `name: engine`, `description: Pure-Dart Hifz scheduling engine — no Flutter, no I/O, "today" injected.`, `publish_to: none`, `resolution: workspace`, `environment: sdk: ^3.6.0`, `dependencies: meta: ^1.15.0` **and** `models: { path: ../models }`, `dev_dependencies: test: ^1.25.0` + `glados: ^1.1.0`. Keep the "Deliberately absent: flutter, dart:io, any clock" comment — it is audit evidence. Barrel `lib/engine.dart` over `lib/src/` (mirror the §3.2 tree skeleton: empty `models/` subfolder reserved; do **not** create `curve.dart`/`trust_clamp.dart` content — that is E04). The stub's one public function (if any) is total: returns a value for every input, never throws; "today" is the last argument as a `CalendarDate` placeholder type **only if** a `CalendarDate` stub already exists from E02 — otherwise the placeholder API stays a no-arg constant and the vector test asserts a frozen identity, leaving the `(state, grade, elapsed)` signature for E04. Add `- packages/engine` to the root `workspace:` list.
3. **`packages/profiles/` — `meta` only.** `pubspec.yaml`: `name: profiles`, `publish_to: none`, `resolution: workspace`, `environment: sdk: ^3.6.0` (no `flutter:` line), `dependencies: meta: ^1.15.0`, `dev_dependencies: test: ^1.25.0`. Barrel `lib/profiles.dart` over `lib/src/profile.dart` holding an `@immutable Profile` value-type stub (`final` fields placeholder, `const` ctor) — the real `Profile` (display name, role, locale, mushaf_id) is fleshed out by the profiles feature epic, not here. Add `- packages/profiles` to the root `workspace:` list.
4. **Purity is structural, not commented.** None of the three manifests declares `flutter:` in `environment` or `flutter: { sdk: flutter }` in `dependencies` — the *absence* is the audit proof (01 §3.3, 02 §3.3). `engine` depends ONLY on `meta` + `models`; it must not pull any package that transitively drags `flutter`/`dart:io`. No `import 'dart:io'`, no `import 'dart:ui'`, no `import 'package:flutter/...'`, no `DateTime.now()`/`DateTime.timestamp()` anywhere in any of the three trees.
5. **One barrel, private `lib/src/`.** Each `lib/<package>.dart` exports only the stable surface; no other package may import another's `lib/src/`. `engine.dart` never re-exports a future `trust_clamp.dart` internal; the barrel can start with zero exports for a true stub.
6. **REUSE + standards on every file.** Every `.dart` file (barrel, src unit, test) carries the two-line REUSE header (`SPDX-FileCopyrightText: 2026 Zakaria Fatahi and Hifz Companion contributors` / `SPDX-License-Identifier: GPL-3.0-or-later`). Public declarations get a `///` summary-first doc. `dart format` (page width 80, trailing commas) clean; full-word names, no abbreviations.
7. **Do not check in per-package lockfiles.** The workspace has exactly one committed `pubspec.lock` at the root (T02). After adding the three members, `dart pub get` at the root resolves the whole graph in one context with no `dependency_overrides` and no clash.
8. **Pitfalls to avoid:** adding a `flutter:` line "to borrow a matcher" (use `package:test`, never `flutter_test`, in a pure package); adding `glados` to `models`/`profiles` (it belongs to `engine` for the §7.12 invariants — `models`/`profiles` need only `test`); creating a `utils/`/`helpers/`/`core/` folder; writing any real scheduling math, `DateTime.now()`, or curve constant in `engine` (E04); writing the real domain fields of `Card`/`Profile` (E03 / profiles epic); a per-package `pubspec.lock`; forgetting to add the member to the root `workspace:` list (the package then will not resolve).

## Tests

All three suites use `package:test`, run with `dart test`, need no widget binding, and read no wall clock (mirroring 11 §2). File names mirror their source (`<source>_test.dart` under `<package>/test/`). The shared throwing-`HttpOverrides` offline bootstrap is wired by E01-T06; these cases are pure function/value assertions that touch no network by construction.

- **`packages/engine/test/engine_placeholder_test.dart`** — the **vector-style** placeholder, written test-first, shaped to receive E04's frozen vectors. A `group('engine golden vectors', …)` with one asserting case over the stub's public surface, using `closeTo(expected, 1e-6)` if the stub returns a `double`, or an exact `expect(actual, expected)` on a frozen identity if the stub is value-only. The case name and structure are the slot E04's `(state, grade, elapsed) → (D, S, due offset)` rows drop into — so when E04 lands the curve/interval/trust-clamp/cold-start vectors, the test file shape and the `dart test engine/` lane already exist (11 §2, §3). No `DateTime.now()`; any `today` is a constructed literal.
- **`packages/models/test/models_placeholder_test.dart`** — one asserting case over the placeholder value type: construct it via its `const` constructor and `expect` it equals an identical instance (proving immutability/equality wiring compiles and the barrel exports resolve). Value-only; no clock, no I/O.
- **`packages/profiles/test/profile_placeholder_test.dart`** — one asserting case constructing the `Profile` stub via its `const` constructor and asserting a placeholder field/equality. Value-only; no clock, no I/O.

No widget, golden, or `integration_test` here — these three are pure-Dart leaf packages; goldens and journeys live in the Flutter stubs (T04) and the test scaffolding (T06). The deliberate-violation proofs (a `package:http` / `DateTime.now()` / `import 'package:flutter/…'` planted in `engine` failing `check_engine_purity.sh` and the analyzer ban) are owned by E01-T05/T08 and exercise the harness this task makes auditable.
