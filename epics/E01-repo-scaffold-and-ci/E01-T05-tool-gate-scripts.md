## Goal

The five symbol-level grep gates live in `tool/` — `check_no_network.sh`, `check_engine_purity.sh`, `check_quran_isolation.sh`, `check_dep_allowlist.sh`, `check_l10n_complete.sh` — each runnable locally with **no arguments** and exiting non-zero on the first violation, plus the `tool/verify_asset_hashes.dart` SHA-256 harness wired and **dormant** on the present-but-empty pinned manifest. These scripts are the belt-and-suspenders symbol-level half of the boundary the analyzer's `avoid-banned-imports` rule (E01-T02's `analysis_options.yaml`) cannot reach: a bare `DateTime.now()` in `engine`, a `package:http` outside `assets`, a QPC glyph reference outside `quran`, an undeclared/banned external dependency, a missing ARB key. CI runs the *same files* (wired in E01-T07) — there is no CI-only logic to drift. On the clean tree from E01-T03/T04 every script exits 0 and the asset-hash harness runs green on the empty manifest.

## Implementation notes

TEST-FIRST is satisfied here by the **deliberate-violation proof**, not a unit test: a gate that passes because it checks nothing is the epic's first risk. Before declaring a script done, run it against a *seeded* violation (a `package:http` line in `engine`, a `DateTime.now()` in `engine`, a `package:flutter` import in `engine`, a QPC-glyph reference outside `quran`, an undeclared external dep, a deleted ARB key) and confirm it exits non-zero with a named message; then revert and confirm it exits 0. E01-T08 formalizes this sweep across all gates — these notes require it locally per-script as you write each one.

1. **All five shell scripts** live directly in `tool/` with `lower_snake_case` names matching engineering 02 §1 exactly. Each opens with the REUSE SPDX header (`# SPDX-FileCopyrightText: …` / `# SPDX-License-Identifier: GPL-3.0-or-later`) and `set -euo pipefail`. Each takes **no arguments**, runs from the repo root, and on a violation prints one named `echo` to stderr citing the rule (PRD/doc ref) and `exit 1`. No script echoes file *contents* — only the offending path/line from `grep -n`. No suppression path (no `--allow`, no skip env var): a gate you can quietly turn off is not a gate (03 §7.2).

2. **`check_no_network.sh`** — ban `package:http`, `package:dio`, and `dart:io` `HttpClient`/`Socket`/`SecureSocket` everywhere **except** `packages/assets/`. Use a path-excluding `grep -rnE` over `app/lib packages/*/lib packages/*/test` with `packages/assets` pruned (e.g. `--exclude-dir` or a `find … -not -path 'packages/assets/*'` feed). Bare `dart:io` is *not* banned wholesale (non-UI code may legitimately import it for `File`) — the symbol-level catch is `HttpClient`/`Socket` specifically (11 §7; the analyzer ban is import-path-level, this is the symbol-level half it misses). Message cites PRD C1, §17.

3. **`check_engine_purity.sh`** — copy the engineering 02 §5 shape verbatim in intent: `! grep -rnE "package:flutter|dart:io|dart:ui|DateTime\.now\(|DateTime\.timestamp\(" packages/engine/lib packages/engine/test`. The two `DateTime` symbols are the load-bearing addition an import-path lint cannot see — the engine takes `today` as an injected `CalendarDate`/`SerialDay`, so any wall-clock read is the bug (01 §5; 03 §6). Message cites PRD §19.1.

4. **`check_quran_isolation.sh`** — QPC-font registration and glyph-code-string handling are allowed **only** in `packages/quran/`. Grep `app/lib packages/*/lib` with `packages/quran` pruned for the markers that betray glyph handling outside `quran`: `package:quran/src/`, the QPC font-family token(s) used by `FontLoader`, and any glyph-code-string accessor name the `quran` stub reserves. Keep the marker list small and exact (a sloppy glob is the hollow-gate risk); the import-path half is the analyzer's job, this script catches the *symbol* (a stray `FontLoader('QCF_…')` in a feature). Message cites PRD R1.

5. **`check_dep_allowlist.sh`** — maintain an explicit in-script allow-list array of every sanctioned external package (the union of the engineering 02 §3.1 matrix: `meta`, `drift`, `sqlite3`, `path`, `http`, `crypto`, `flutter`, `intl`, `flutter_riverpod`, `go_router`, plus the pinned dev deps `test`, `glados`, `flutter_test`, `flutter_lints`). Walk every `*/pubspec.yaml` under `app/` and `packages/`, extract each declared dependency name, and **fail if any name is absent from the allow-list** (the audit map: a new dep is a decision-log event, never a quiet add). Independently fail on a hard banned-set match — the verbatim F-Droid regex `firebase_|crashlytics|sentry|google_analytics|facebook_|appsflyer|amplitude|mixpanel` **and** `google_fonts` (all UI fonts are bundled — PRD §13.5) — so a banned SDK fails with its own explicit message even before the allow-list check. This is the F-Droid entry ticket verbatim (13 §5) and PRD §20 gate 6. Message cites PRD C1, §20 gate 6.

6. **`check_l10n_complete.sh`** — two checks. (a) Zero missing ARB keys across `ar`/`fa`/`ckb`: read the key sets of `packages/l10n/lib/src/arb/app_ar.arb` (the template), `app_fa.arb`, `app_ckb.arb` and fail if any template key is absent from `fa` or `ckb` (or vice-versa). On the E01 placeholder key set this passes by being a strict superset check, not a hardcoded count. (b) No hardcoded UI strings: grep `app/lib packages/features/lib` for bare string-literal `Text('…')`/`Text("…")` constructions that are not resolved through `l10n.*` (allow `Text(l10n.…)`, `Semantics(label: l10n.…)`); a literal user-facing string is the violation (03 §4; strings live in the `l10n` ARB files only). Message cites PRD §20 gate 5. Keep this pure shell/grep — no Dart ARB parser dependency.

7. **`verify_asset_hashes.dart`** lives in `tool/` (a plain `dart run tool/verify_asset_hashes.dart` entrypoint, not a package). Follow engineering 11 §9 and the eng-add-ci-check §4 contract: load the **pinned manifest baked into the app binary** (in E01 this is the present-but-empty manifest the `assets` package exposes — never a sidecar `.sha256`), iterate `manifest.files`, re-compute `sha256.convert(await File(entry.path).readAsBytes()).toString()` via `package:crypto`, and `exit(1)` on any mismatch; compare `manifest.textSha256` against the authoritative Tanzil hash constant. **SHA-256 only** — never MD5/SHA-1 (FIPS 180-4). In E01 the manifest is empty (`files: []`, the Tanzil constant a documented placeholder), so the loop runs zero iterations and the harness exits 0 — **dormant, fail-closed, not auto-blessed**. The real per-file hashes, the 604 KFGQPC fonts, and the authoritative Tanzil hash arrive in E05 (the harness's first real consumer); an empty manifest is an explicit, reviewed state, recorded in the gate→job hand-off. Code it to the standards: REUSE SPDX header, a `sealed class AssetIntegrityFailure` with `final` subclasses (`ChecksumMismatch`, `ManifestUnreadable`) surfaced from a typed `on … catch`, full-word names (`expectedSha256`, `actualSha256`, `Sha256Manifest`), a `///` doc on `main`, **no `print` of user data** (CI-grade `stderr.writeln('::error::…')` of the offending *path* + expected/got hash only), and `dart format` clean. The harness is **locale/madhhab-blind**: it hashes bytes, it never interprets them (eng-add-ci-check §10).

8. **Pitfalls to avoid.** A bad path glob that prunes the wrong directory makes a script exit 0 forever — the hollow gate; this is why each script ships with a seeded-violation check (note 1) and E01-T08's revert sweep. `grep` exit-code semantics under `set -e`: `grep` returns 1 on *no match* (the success case here), so wrap the ban in `! grep …` or guard with `|| true` deliberately — a naive `grep && exit 1` aborts the script on the clean tree. Do **not** add a CI-only flag or env-gated branch to any script (engineering 02 §7: CI runs the same file). Do **not** point the asset-hash harness at a sidecar checksum, hash a subset, or use MD5/SHA-1. Do **not** auto-write/bless the manifest in the harness — it only *verifies*. Do **not** let any script `// ignore:`-style suppress or weaken on green; a gate that can be turned off is the bug (03 §7.2).

- [ ] `check_no_network.sh` exits non-zero on a `package:http`/`package:dio`/`dart:io HttpClient`/`Socket` reference anywhere outside `packages/assets/`, and 0 when those symbols appear inside `packages/assets/` (its sanctioned home).
- [ ] `check_engine_purity.sh` exits non-zero on `package:flutter`, `dart:io`, `dart:ui`, `DateTime.now()`, or `DateTime.timestamp()` anywhere under `packages/engine/lib` or `packages/engine/test` — the symbol-level bans (`DateTime.now()`/`.timestamp()`) the analyzer import rule cannot see.
- [ ] `check_quran_isolation.sh` exits non-zero on QPC-font registration or glyph-code-string handling outside `packages/quran/`, and 0 for the same symbols inside `packages/quran/`.
- [ ] `check_dep_allowlist.sh` fails on (a) any external dependency in any `pubspec.yaml` absent from the in-script allow-list and (b) any banned package matching `firebase_|crashlytics|sentry|google_analytics|facebook_|appsflyer|amplitude|mixpanel` or `google_fonts` — each with its own named message; it passes on the clean nine-package matrix where only `assets` declares `http`/`crypto`.
- [ ] `check_l10n_complete.sh` fails on any ARB key present in `app_ar.arb` but missing from `app_fa.arb`/`app_ckb.arb` (or vice-versa) and on any hardcoded `Text('literal')` not resolved via `l10n.*` in `app/lib`/`packages/features/lib`; it passes on the E01 placeholder key set as a strict superset check (no hardcoded count).
- [ ] `tool/verify_asset_hashes.dart` runs via `dart run tool/verify_asset_hashes.dart`, reads the binary-baked (present-but-empty) manifest, uses `package:crypto` SHA-256 only, exits 0 dormant on the empty manifest, and would `exit(1)` fail-closed on any file mismatch or a manifest-vs-Tanzil text-hash drift once E05 supplies real hashes — never a sidecar `.sha256`, never MD5/SHA-1, never an auto-bless.
- [ ] `verify_asset_hashes.dart` defines one `sealed` `AssetIntegrityFailure` error type with typed `on … catch`, full-word names, a `///`-documented `main`, prints no user data (CI `::error::` of path + expected/got hash only), and is `dart format`/`dart analyze --fatal-infos` clean.
- [ ] Each of the six gates has been run against a seeded violation and a revert (note 1) and observed to flip non-zero ↔ 0; no script has a CI-only branch or a suppression flag.

## Tests

This task's verifiable artifact is the **deliberate-violation proof**, the documented gate-validation procedure eng-add-ci-check mandates for every gate (a gate that passes because it checks nothing is the failure mode). There are no Dart unit tests for the shell scripts; `verify_asset_hashes.dart` is exercised by running it on the empty manifest. The seeded-violation matrix run locally per script (and formalized in E01-T08):

| Gate script | Seeded violation (then revert) | Expected |
|---|---|---|
| `check_no_network.sh` | add `import 'package:http/http.dart';` to a file in `packages/engine` | non-zero, names PRD C1/§17; 0 after revert |
| `check_engine_purity.sh` | add `final t = DateTime.now();` to `packages/engine/lib` | non-zero, names PRD §19.1; 0 after revert |
| `check_engine_purity.sh` | add `import 'package:flutter/widgets.dart';` to `packages/engine/lib` | non-zero; 0 after revert |
| `check_quran_isolation.sh` | reference a QPC font family / `package:quran/src/` in `packages/features/lib` | non-zero, names PRD R1; 0 after revert |
| `check_dep_allowlist.sh` | add `crashlytics:` (banned) and separately `some_undeclared_pkg:` to a `pubspec.yaml` | non-zero on each with distinct messages; 0 after revert |
| `check_l10n_complete.sh` | delete a key from `app_ckb.arb`; separately add `Text('hello')` to a feature file | non-zero on each; 0 after revert |
| `verify_asset_hashes.dart` | (E05) flip one byte of a hashed asset | non-zero `::error::` with path + expected/got; out of scope in E01 (empty manifest) |

Offline / no-network guard: these scripts and the harness make **no** network call (they read the local tree and the baked manifest); the harness fetches nothing — the binary ships lean and assets are never fetched at gate-run time (eng-add-ci-check §9). The throwing `HttpOverrides` airplane-mode bootstrap (the third no-network layer) is E01-T06, not this task.

## Definition of Done

- [ ] All acceptance criteria met; all six gates exit 0 on the clean E01-T03/T04 tree locally with no arguments, before any CI run (CI runs the same files in E01-T07 — no CI-only logic).
- [ ] **Offline / no-network is a build invariant (script half):** `check_no_network.sh` confines `package:http`/`package:dio`/`dart:io HttpClient`/`Socket` to `packages/assets/` at the symbol level; `check_dep_allowlist.sh` fails on any analytics/ads/backend/crash SDK or `google_fonts` and on any undeclared external dep; the gates make no network call themselves.
- [ ] **No AI / no microphone by construction:** no gate introduces an ML/ASR/audio dependency or a microphone path; the dep allow-list would reject one (the allow-list contains no audio-recognition package), and no gate reads a microphone.
- [ ] **Quran text fidelity:** `check_quran_isolation.sh` holds QPC glyph/font handling to `packages/quran/`; `verify_asset_hashes.dart` is wired and fail-closed (dormant on the empty manifest) so E05's per-file SHA-256 + the authoritative Tanzil hash plug into a live, byte-blind harness — SHA-256 only, binary-baked manifest, never a sidecar, never auto-blessed.
- [ ] **RTL + fa/ckb/ar strings:** `check_l10n_complete.sh` enforces zero missing keys across `ar`/`fa`/`ckb` and no hardcoded UI literals, so every later user-facing string resolves through `l10n.*` (full transcreation: E09); the gate is the structural guarantee, not a string itself.
- [ ] **Accessibility:** N/A by construction — this task ships gate scripts, renders no UI and no `Semantics`; the l10n gate keeps strings routable so a11y labels land via `l10n.*` later.
- [ ] **Sect-neutral adab:** N/A as authored copy, and structurally honored — `verify_asset_hashes.dart` stays **locale/madhhab-blind** (it hashes bytes, it never interprets them); no gate encodes a madhhab assumption or a fiqh ruling.
- [ ] **Deterministic / no clock:** every gate is a pure function of the checked-out tree (and the baked manifest); no gate reads a wall clock, a network resource, or any host state, so a fresh clone runs them identically.
- [ ] Each gate script carries the REUSE SPDX `GPL-3.0-or-later` header; `verify_asset_hashes.dart` uses a `sealed` typed error, full-word names, a `///` doc, prints no user data, and is `dart format`/`dart analyze --fatal-infos` clean; the deliberate-violation proof (note 1) was run per gate. No gate is `continue-on-error`-able by design (no suppression path); E01-T07 wires them release-blocking and E01-T08 runs the full revert sweep.
