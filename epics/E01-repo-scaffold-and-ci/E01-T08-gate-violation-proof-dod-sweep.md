## Goal

Every gate this epic erected is *proven to check something* — a gate that passes because it checks nothing is the epic's first risk and the actual bug. This task runs and records a **deliberate-violation drill** for each ban (a `package:http` import in `engine`, a `DateTime.now()` in `engine`, an `import 'package:flutter/…'` in `engine`, a QPC-glyph reference outside `quran`, a banned/undeclared external dependency, and a stray network call in a test): each violation must fail its `tool/` script (non-zero exit) **and** its matching CI job, and a revert must leave all four jobs green again. It then runs the **fresh-clone DoD sweep**: a clean clone runs `dart pub get` (one lock, no `dependency_overrides`, no clash), builds the app on emulator + simulator, runs all tests, and passes every `tool/` gate locally with no arguments. The drill is committed as the **documented manual procedure** that every future gate change must repeat (per eng-add-ci-check), and the final PR shows all four CI jobs (`fast` / `golden` / `restraint` / `journeys`) green in public logs — so "the engine is pure Dart," "networking lives in exactly one module," and "the muṣḥaf bytes cannot drift" are facts a stranger verifies, not promises.

## Implementation notes

TEST-FIRST is satisfied here, as in E01-T05, by the **deliberate-violation proof itself**, not a new unit test: the drill *is* the test of the gates. For each ban, the procedure is identical and must be recorded: **(1)** seed one minimal violation, **(2)** observe the local `tool/` script exit non-zero with its named message, **(3)** observe the matching CI job go red on a throwaway PR (or a local act-equivalent run), **(4)** revert, **(5)** observe the script exit 0 and all four jobs green. A gate that does not flip red on its seeded violation is the bug — fix the gate (back in T05/T07), not the drill.

1. **The drill is a committed repo artifact, not ad-hoc keystrokes.** Add `docs/gate-validation.md` (or `tool/README.md` if the repo keeps tool docs there — match whatever E01-T02/T05 established) containing the seeded-violation matrix below, the exact one-line edit that triggers each gate, the expected non-zero message, the matching CI job, and the revert step. It carries the REUSE SPDX `GPL-3.0-or-later` header. eng-add-ci-check §1 makes this the procedure every *future* gate change repeats; the doc states that obligation in one sentence. Do **not** automate-and-forget: the value is a human re-running it when a gate changes, exactly because a hollow gate passes silently.

2. **The six violations, each minimal and reverted (never merged).** Seed each in its own working copy / throwaway branch, never on the trunk that ships:
   - `package:http` in `engine` — add `import 'package:http/http.dart';` to a file under `packages/engine/lib`. Must fail `tool/check_no_network.sh` (symbol-level, names PRD C1/§17) **and** the analyzer banned-import scope in the `fast` job. Revert → green.
   - `DateTime.now()` in `engine` — add `final t = DateTime.now();` under `packages/engine/lib`. Must fail `tool/check_engine_purity.sh` (names PRD §19.1) — the symbol-level catch the import-path lint cannot see (the engine takes `today` as an injected `CalendarDate`/`SerialDay`). Revert → green.
   - `import 'package:flutter/…'` in `engine` — add `import 'package:flutter/widgets.dart';` under `packages/engine/lib`. Must fail `tool/check_engine_purity.sh` **and** the analyzer engine-purity ban in the `fast` job (and would fail `dart pub get`/resolution since `engine`'s manifest declares no `flutter`). Revert → green.
   - QPC-glyph reference outside `quran` — reference a QPC font family / `package:quran/src/` glyph accessor from `packages/features/lib`. Must fail `tool/check_quran_isolation.sh` (names PRD R1). Revert → green.
   - banned/undeclared external dependency — add (a) `crashlytics:` (banned-set) and, separately, (b) `some_undeclared_pkg:` (absent from the allow-list) to a `pubspec.yaml`. Each must fail `tool/check_dep_allowlist.sh` with its **own distinct message** (the banned-SDK regex hit vs. the allow-list-miss) and the `restraint` job's allow-list step. Revert → green.
   - stray network call in a test — add a real connection attempt (e.g. an `HttpClient().getUrl(...)` or a `package:http get(...)`) inside a placeholder test. Must throw the named `StateError` from E01-T06's `_ThrowingHttpOverrides` (a *loud named failure*, not a 400), failing the test in the `fast`/`journeys` job. Revert → green.

3. **Each violation must fail the matching CI job, not only the local script.** This is the load-bearing step E01-T05's per-script check did not cover: T05 proved the *scripts* flip; T08 proves the *wired jobs* flip. Run each seeded violation through CI on a throwaway PR (the cheapest honest evidence — public logs showing the red job) or an `act`-equivalent local job run if a real PR is undesirable; record which job went red against the PRD-gate→job table (11 §8). A violation whose script reddens but whose CI job stays green is a wiring bug in E01-T07 — file it back there.

4. **The fresh-clone DoD sweep — the "stranger from a cold clone" path (13 §8).** From a directory with no `.dart_tool/` and no prior `pub get`:
   - `git clone` the repo fresh (or `git worktree add` a pristine checkout); confirm **no** `dependency_overrides` in any manifest and exactly **one** committed root `pubspec.lock` with **no per-package lock**.
   - `dart pub get` (or `flutter pub get`) at the workspace root resolves with that one lock, no clash, no override (engineering 02 §1.1 — a clash is fixed at resolution time, the point of the workspace).
   - Build the `app/` shell to its placeholder screen on **both** an Android emulator and an iOS simulator (the §2 shell from E01-T02).
   - Run the full suite: `dart test engine/` (placeholder vectors), `flutter test` (widget + the per-package placeholders), `flutter test --tags golden` (dormant masters), `flutter test integration_test/` (the one trivially-green launch journey).
   - Run **every** `tool/` gate locally with **no arguments** and confirm each exits 0 on the clean tree: `check_no_network.sh`, `check_engine_purity.sh`, `check_quran_isolation.sh`, `check_dep_allowlist.sh`, `check_l10n_complete.sh`, and `dart run tool/verify_asset_hashes.dart` (dormant on the empty manifest). CI runs the same files — there is no CI-only logic to drift.

5. **Confirm the structural manifest/Plist/string/lock/provenance audit holds (the E01 DoD, re-verified end-to-end).** This task is where the epic's Definition-of-Done sentences are *checked as facts*, not asserted: (a) **manifest audit** — `engine` → `models` + `meta` only; dependencies point downward (`features → engine/data/quran/l10n/profiles`); **no leaf package imports `features`**; `quran` has no local package dep; only `assets` declares `http`/`crypto`; (b) the **no-mic `Info.plist`** — `ios/Runner/Info.plist` carries **no** `NSMicrophoneUsageDescription` and no microphone permission is requested on either platform (PRD C2, R5); (c) the **strings-from-`l10n`-only** placeholder — the placeholder screen resolves its one string via `l10n.*`, not a hardcoded literal, and `check_l10n_complete.sh` is green; (d) **one committed root `pubspec.lock`, no per-package lock**; (e) **DCO `Signed-off-by`** on every commit of the final PR; (f) **`reuse lint` green** (every file carries a `GPL-3.0-or-later` SPDX header).

6. **Pitfalls to avoid.** Do **not** merge any seeded violation — every one is reverted before the branch is dropped; the final PR contains only the validation doc (and any wiring fix the drill surfaced). Do **not** treat a green local script as proof the *CI job* fires — prove the job red (note 3); a script that reddens while its job stays green is the exact hollow-gate failure this task exists to catch. Do **not** reach for `dependency_overrides` to make the fresh-clone resolve — a clash is a real layering bug to fix at the manifest, not paper over (engineering 02 §1.1, §5). Do **not** auto-bless anything: `verify_asset_hashes.dart` stays dormant/fail-closed on the empty manifest, the golden masters are never `--update-goldens`ed in CI, the vectors are never `--update-vectors`ed in CI. Do **not** mark any gate `continue-on-error` to "get the PR green" — a gate you can turn off is not a gate (eng-add-ci-check §9; engineering 03 §7.2). Do **not** let the drill doc rot — eng-add-ci-check §1 binds every future gate change to re-run it.

## Tests

This task's verifiable artifact is the **deliberate-violation drill + the fresh-clone DoD sweep** — the documented gate-validation procedure eng-add-ci-check mandates — not a new Dart unit test. The six seeded-violation runs (each followed by a revert) are the test of the gates; the sweep is the test of the cold-clone path. The matrix, run and recorded:

| Gate (CI job) | Seeded violation (then revert) | Expected |
|---|---|---|
| `check_no_network.sh` + analyzer (`fast`) | `import 'package:http/http.dart';` in `packages/engine/lib` | non-zero, names PRD C1/§17; `fast` reddens; 0 + green after revert |
| `check_engine_purity.sh` (`fast`) | `final t = DateTime.now();` in `packages/engine/lib` | non-zero, names PRD §19.1; 0 after revert |
| `check_engine_purity.sh` + analyzer (`fast`) | `import 'package:flutter/widgets.dart';` in `packages/engine/lib` | non-zero; `fast` reddens; 0 + green after revert |
| `check_quran_isolation.sh` (`fast`) | QPC font family / `package:quran/src/` ref in `packages/features/lib` | non-zero, names PRD R1; 0 after revert |
| `check_dep_allowlist.sh` (`restraint`) | `crashlytics:` (banned) and `some_undeclared_pkg:` (undeclared) in a `pubspec.yaml` | non-zero on each with distinct messages; `restraint` reddens; 0 + green after revert |
| throwing `HttpOverrides` (`fast`/`journeys`) | a real connection attempt inside a placeholder test | throws named `StateError` (loud, not a 400); job reddens; passes after revert |

Offline / no-network guard: the drill and the sweep make **no** network call beyond the single `git clone` of the public repo and the standard `pub get` package fetch on the cold clone (the asset-pack download is mock-verified, never live — assets are not fetched at build/gate time, eng-add-ci-check §9). The throwing `HttpOverrides` airplane-mode bootstrap (E01-T06) stays installed throughout; the stray-network-call proof above is the very test that the bootstrap is live. Engine/golden/vector runs in the sweep are deterministic (`today` injected, fuzz OFF) — no wall clock anywhere.
