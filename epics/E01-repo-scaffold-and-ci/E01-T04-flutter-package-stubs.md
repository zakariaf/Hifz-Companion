## Goal

The five Flutter/IO packages exist under `packages/` — `data`, `quran`, `l10n`, `features`, `assets` — each a `resolution: workspace` member whose `pubspec.yaml` declares the narrowest possible **downward-only** dependency set from the engineering 02 §3.1 matrix, exposes exactly one public `lib/<package>.dart` barrel over a private `lib/src/`, and ships one passing placeholder `flutter_test`. The manifests *are* the boundary audit: `assets` is the **only** package in the whole repo that declares `http`/`crypto` (the single network socket); `data` declares `drift`/`sqlite3`/`path` and the downward edges `engine`/`profiles` but **no `http`, no `features`**; `quran` declares `flutter` and **no local package dep** (it receives page geometry as value types so it golden-tests in isolation); `l10n` holds the `app_ar.arb` template + empty `fa`/`ckb` ARBs with the committed generated `AppLocalizations`; `features` is the umbrella that depends down on `engine`/`data`/`quran`/`l10n`/`profiles` + `flutter_riverpod`/`go_router` and carries one placeholder feature library in the View+1:1 ViewModel anatomy. No repository logic, no Drift schema, no QPC glyph render, no downloader internals — empty-but-compiling skeletons that make the golden/restraint/journeys CI lanes green from commit one and make "networking lives in exactly one module" provable by reading manifests.

## Implementation notes

TEST-FIRST is light here — these are empty-but-compiling stubs, not correctness-critical logic, so the placeholder cases (Tests below) are written alongside each barrel rather than strictly before; the genuine test-first rigor (frozen vectors, fidelity goldens, invariant properties) belongs to E04/E05 where the logic lands. Write each placeholder case so it asserts behaviour (a real `expect`) and exercises its CI lane, not as a no-op.

1. **`packages/data/` — Drift host, downward edges only.** `pubspec.yaml`: `name: data`, `publish_to: none`, `resolution: workspace`, `environment: sdk: ^3.6.0` + `flutter: ">=3.38.0"`, `dependencies: flutter: { sdk: flutter }`, `drift: ^2.x`, `sqlite3: ^2.x`, `path: ^1.x`, `engine: { path: ../engine }`, `profiles: { path: ../profiles }`. **No `http`, no `features`, no `dynamic`.** Keep a "Deliberately NO http (no user-data repository reaches the network — PRD C1), NO features" comment. Barrel `lib/data.dart` exports **repositories + DTOs only, never the raw DAOs** (the §3.2 rule); reserve the `lib/src/db/`/`lib/src/daos/`/`lib/src/repositories/` folder skeleton empty — the `@DriftDatabase`, WAL + `synchronous=FULL`, migrations, DAOs, and repositories are E03. The stub may expose one placeholder repository interface returning a value type, or start with a zero-export barrel. Add `- packages/data` to the root `workspace:` list. WAL/at-rest-encryption wiring is deferred to E03 — the stub compiles but wires no encryption.
2. **`packages/quran/` — Flutter only, NO local dep.** `pubspec.yaml`: `name: quran`, `publish_to: none`, `resolution: workspace`, `environment: sdk: ^3.6.0` + `flutter: ">=3.38.0"`, `dependencies: flutter: { sdk: flutter }` — **and no local package dependency at all** (it receives page geometry as plain value types so it golden-tests in isolation — §3.2). Barrel `lib/quran.dart` over `lib/src/`; reserve `glyph_page.dart`/`overlay_painter.dart`/`page_geometry.dart` empty — QPC-glyph-font selection and glyph-code drawing (never the OS shaper, never parsing the glyph string as Arabic text) is the **only** thing that lives here and is authored in E05. The placeholder may be a trivial `StatelessWidget` taking a value-type geometry parameter, or a zero-content barrel. Add `- packages/quran` to the root `workspace:` list. `quran` must not gain `http`/`crypto` or any local dep — `check_quran_isolation.sh` (T05) greps that QPC handling stays here.
3. **`packages/l10n/` — strings, numerals, bidi, ckb delegate.** `pubspec.yaml`: `name: l10n`, `publish_to: none`, `resolution: workspace`, `environment: sdk: ^3.6.0` + `flutter: ">=3.38.0"`, `dependencies: flutter: { sdk: flutter }`, `intl: ^0.19.x`. **No second `l10n.yaml`** (the workspace-root one from T02 points `arb-dir` here); **no `google_fonts`**. Create `lib/src/arb/app_ar.arb` (template, with at least one placeholder key + its `@`-metadata so the screen resolves one string via `l10n.*`, not a literal), `app_fa.arb` and `app_ckb.arb` (present, value entries empty/placeholder for that one key). Run `gen_l10n` (via T02's `l10n.yaml`, `synthetic-package: false`) and **commit** `lib/src/generated/app_localizations.dart` + the per-locale outputs. Add `numerals.dart` (per-locale `NumberFormat` stub: Extended Arabic-Indic ۰۱۲ for fa/ckb, Arabic-Indic ٠١٢ for ar), `bidi.dart` (one FSI…PDI isolation helper stub), and `ckb_material_localizations.dart` (vendored Material-delegate stub for the custom ckb locale — bodies filled at E09). Barrel `lib/l10n.dart` exports `AppLocalizations` + the numeral/bidi/calendar helpers. Add `- packages/l10n` to the root `workspace:` list.
4. **`packages/features/` — umbrella, View+1:1 ViewModel anatomy.** `pubspec.yaml`: copy the §3.3 excerpt verbatim — `name: features`, `publish_to: none`, `resolution: workspace`, `environment: sdk: ^3.6.0` + `flutter: ">=3.38.0"`, `dependencies: flutter: { sdk: flutter }`, `flutter_riverpod: ^3.0.0`, `go_router: ^16.0.0`, and the five downward `path:` edges (`engine`, `data`, `quran`, `l10n`, `profiles`). Keep the "Deliberately NO http/dio (no feature fetches anything), NO drift (features touch the DB only through data's repositories)" comment. **No `provider`/`get_it`/Bloc.** Create one placeholder feature library, e.g. `lib/src/placeholder/`, with `placeholder_screen.dart` (a dumb View `ConsumerWidget` that reads its one string from `l10n.*`), `placeholder_view_model.dart` (the 1:1 ViewModel — a `Notifier`/`AsyncNotifier` stub exposing immutable read-only state and a no-op command; **no** legacy provider), a `widgets/` folder (may hold one leaf view or stay reserved), and `placeholder_providers.dart` (the scoped Riverpod provider for this feature — never global). Barrel `lib/features.dart` exports the placeholder screen. Add `- packages/features` to the root `workspace:` list. State-bearing code uses `flutter_riverpod` Notifier/AsyncNotifier only.
5. **`packages/assets/` — the single socket, fail-closed shape.** `pubspec.yaml`: `name: assets`, `publish_to: none`, `resolution: workspace`, `environment: sdk: ^3.6.0` + `flutter: ">=3.38.0"`, `dependencies: flutter: { sdk: flutter }`, `http: ^1.x`, `crypto: ^3.x`, `path: ^1.x` — **the ONLY `http`/`crypto` in the entire repo** (the network quarantine). Barrel `lib/assets.dart` exposes the `AssetPackService` + `AssetRepository` *shape* (verify-then-expose); reserve `lib/src/pack_downloader.dart` (HTTPS GET to a pinned GitHub Release tag; no auth, no cookies, no identifiers), `lib/src/sha256_verifier.dart` (the **fail-closed** verifier: mismatch ⇒ reject, re-fetch once, then refuse), and `lib/src/pinned_manifest.dart` (baked-in tag + per-file SHA-256) — all empty stubs; the downloader logic and the real manifest content are E05. Declare the one `sealed` I/O error type stub for the boundary (e.g. `sealed class AssetIntegrityError` with `final` subclasses `ChecksumMismatch`/`PackUnavailable`) so the fail-closed surface and typed `catch` shape exist now (eng-write-to-coding-standards §6). Add `- packages/assets` to the root `workspace:` list.
6. **Downward-only is structural, not commented.** `data` declares only `engine`/`profiles` locally and never `features`/`http`; `quran` declares no local dep; `features` is the only package that depends on the leaf packages, and no leaf depends on `features`. The *absence* of `http`/`crypto` in every manifest but `assets`, and of any local dep in `quran`, is the audit proof — `check_no_network.sh`/`check_quran_isolation.sh`/`check_dep_allowlist.sh` (T05) and the analyzer `avoid-banned-imports` path-scoped bans (T02) re-check it.
7. **One barrel, private `lib/src/`.** Each `lib/<package>.dart` exports only the stable surface; no other package may import another's `lib/src/`. `data.dart` exports repositories/DTOs, never the raw DAOs; `assets.dart` never re-exports `pack_downloader.dart` internals; a barrel can start with zero exports for a true stub.
8. **REUSE + standards on every file.** Every `.dart` file (barrel, src unit, the committed generated `app_localizations.dart`, test) carries the two-line REUSE header (`SPDX-FileCopyrightText: 2026 Zakaria Fatahi and Hifz Companion contributors` / `SPDX-License-Identifier: GPL-3.0-or-later`); the `.arb` files are headered via `.reuse/dep5` (T01) since they cannot carry an inline comment. Public declarations get a `///` summary-first doc. `dart format` (page width 80, trailing commas) clean; full-word/unit-bearing names; sacred terms use the fixed transliteration.
9. **Pitfalls to avoid:** declaring `http`/`crypto` anywhere but `assets` (the single most-watched boundary); giving `quran` any local package dep or letting it import `data`/`features` (breaks isolation-golden-testing); exporting a raw DAO from `data.dart` (the boundary is repositories/DTOs); adding `provider`/`get_it`/Bloc, or declaring a **global** provider in `features` (providers are feature-scoped; the only global composition is `app/composition/providers.dart`); a second `l10n.yaml` or a `flutter_gen`/`google_fonts` dependency; `synthetic-package: true` (or omitting it — the generated `AppLocalizations` must be committed in-tree); writing real Drift schema/repository bodies (E03), real glyph render (E05), real downloader/verifier logic or real SHA-256 manifest entries (E05); hardcoding a user-facing string in Dart instead of the ARB; a per-package `pubspec.lock`; forgetting to add a member to the root `workspace:` list (the package then will not resolve).

## Tests

All five suites use `flutter_test`, run with `flutter test`, and read no wall clock. File names mirror their source (`<source>_test.dart` under `<package>/test/`). Widget tests use **in-memory fakes** via Riverpod overrides — never a real Drift DB or real assets (the real stack appears only in an `integration_test` journey, E01-T06 / E05). The shared throwing-`HttpOverrides` offline bootstrap is wired by E01-T06; these cases touch no network by construction.

- **`packages/data/test/data_placeholder_test.dart`** — one asserting case over the placeholder repository/DTO surface: construct the value type via its `const` constructor and `expect` equality, or call the placeholder repository method against an in-memory fake and assert the returned value type. Proves the barrel exports resolve and the `engine`/`profiles` downward edges link; no Drift DB, no `http`, no clock.
- **`packages/quran/test/quran_placeholder_test.dart`** — one widget case pumping the placeholder widget under `Directionality(textDirection: TextDirection.rtl)`, fed a value-type geometry parameter, asserting it builds. No QPC glyph/text byte is rendered yet (E05); structured so the E05 fidelity golden (`@Tags(['golden'])`, real KFGQPC fonts via `FontLoader`, never Ahem) drops in without reshaping. No local dep imported — proves isolation.
- **`packages/l10n/test/l10n_placeholder_test.dart`** — one asserting case that resolves the one placeholder ARB key through `AppLocalizations` for `ar` (and that the `fa`/`ckb` delegates load), plus a `numerals.dart` case asserting the ar/fa/ckb numeral path is wired (placeholder formatter). Proves the generated, committed `AppLocalizations` and the three locales resolve; no hardcoded literal under test.
- **`packages/features/test/placeholder_screen_test.dart`** — one widget case pumping `PlaceholderScreen` inside a `ProviderScope` (with the feature's scoped provider overridden by an in-memory fake) under `Directionality.rtl`, asserting it builds and resolves its one string via `l10n.*`, not a literal. Proves the View+1:1 ViewModel anatomy, the Riverpod wiring, and the downward `l10n` edge.
- **`packages/assets/test/assets_placeholder_test.dart`** — one asserting case over the `AssetPackService`/`AssetRepository` shape and the `sealed AssetIntegrityError` subtypes (construct each `final` subclass, `expect` exhaustive `switch` coverage). This is the **single** suite permitted to opt out of the throwing-`HttpOverrides` (it resets `HttpOverrides.global` to a mock in its own `setUp`) once E05 lands the downloader; the stub case here touches no socket. Proves the fail-closed error shape compiles.

No frozen engine vectors, fidelity goldens, or `integration_test` journeys here — those land with the logic in E04 (vectors/invariants) and E05 (muṣḥaf goldens, asset-hash manifest, journeys); the `@Tags(['golden'])` harness and the `integration_test/` skeleton are owned by E01-T06. The deliberate-violation proofs (a `package:http` planted outside `assets`, a QPC-glyph reference planted outside `quran`, a leaf importing `features`, an undeclared/banned dep) failing their gate scripts and CI jobs are owned by E01-T05/T08 and exercise the boundaries this task makes auditable.
