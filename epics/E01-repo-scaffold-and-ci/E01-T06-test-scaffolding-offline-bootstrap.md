## Goal

The test harness every later suite plugs into exists and exercises each CI lane from commit one. A shared `test/test_setup.dart` installs a **throwing** `HttpOverrides` so a stray network call in any test is a loud, named `StateError` — not a silent 400 — and a one-line `useOfflineTestPolicy()` is wired into the bootstrap of every Flutter package; only the future `assets` downloader test will opt out by resetting `HttpOverrides.global` to a mock in its own `setUp`. Each of the nine packages carries exactly one **asserting** placeholder test (pure `dart test` for `engine`/`models`/`profiles`; `flutter_test` for `data`/`quran`/`l10n`/`features`/`assets`) — never a coverage-only no-op. The `@Tags(['golden'])` golden lane is wired with a `FontLoader`/real-font `setUpAll` skeleton (no masters yet), and an `integration_test/` skeleton runs one trivially-green launch journey after `IntegrationTestWidgetsFlutterBinding.ensureInitialized()`. After this task `dart test` + `flutter test` are green, a deliberately-added network call throws and fails the suite, and the golden and integration lanes run green on the empty set.

## Implementation notes

TEST-FIRST: the offline guard is correctness-critical infrastructure. Before declaring the bootstrap done, add a temporary test that opens an `HttpClient` and confirm it **throws** the named `StateError` and **fails the suite**; that red proof (then reverted) is the acceptance evidence that layer (a) is real, not a no-op. The per-package placeholder *cases* are authored in E01-T03/T04; this task's correctness work is the bootstrap, the golden/integration skeletons, and the negative-network proof.

1. **Shared offline bootstrap — `test/test_setup.dart` (workspace-root `test/`).** Copy `template.dart` BLOCK H verbatim into shape: a private `_ThrowingHttpOverrides extends HttpOverrides` whose `createHttpClient(SecurityContext? context)` is `=> throw StateError('Network access attempted in a test. Hifz is offline-only.');`, and a public `void useOfflineTestPolicy() => HttpOverrides.global = _ThrowingHttpOverrides();` with a `///` doc that names the rule (11 §7) and the single sanctioned opt-out (the `assets` downloader test). REUSE header on top. `import 'dart:io';` is the only import — this is test code, outside the `lib/` §7.2 ban scope. Keep the message string an exact, grep-able literal so a CI log line identifies the offender.

2. **Wire the bootstrap into every Flutter package's tests.** Each Flutter package (`data`/`quran`/`l10n`/`features`/`assets`) gets a thin per-package `test/test_setup.dart` (or imports the root one via a relative path / a tiny `dev_dependency`-free re-export) that calls `useOfflineTestPolicy()` from a top-level `setUpAll`/the test `main`'s first line, so every `flutter_test` case in that package runs under the throwing override. Prefer one small shared helper imported by each package's test bootstrap over copy-paste, but do **not** add a production dependency to ship it — keep it in `test/`. The pure-Dart packages (`engine`/`models`/`profiles`) need **no** override: `package:test` has no `TestWidgetsFlutterBinding`, makes no implicit network client, and the engine is pure by construction — wiring `dart:io` into a pure-package test would itself be a smell. Document that asymmetry in a one-line comment.

3. **The `assets` opt-out is the *only* one.** Confirm (with E01-T04) that `packages/assets/test/assets_test.dart` is the single file permitted to reset `HttpOverrides.global` to a mock in its own `setUp` — and that, because no real fetch exists yet (E05), it makes no network call regardless. No other test resets the override. A grep for `HttpOverrides.global =` outside `test_setup.dart` and the `assets` test is a review tripwire.

4. **Each placeholder asserts a structural fact, never nothing (11 §10).** Re-read the eight placeholder cases authored in E01-T03/T04 and confirm each has a meaningful `expect`: `engine` — the vector-style frozen-identity/`closeTo(_, 1e-6)` slot (E01-T03); `models`/`profiles` — `const`-constructor equality; `data` — the `@DriftDatabase`-shaped placeholder constructs; `quran` — a page-geometry value type round-trips; `l10n` — `AppLocalizations` resolves `appTitle`, `supportedLocales` carries `ar`/`fa`/`ckb`, `numberFormatFor(locale)` is non-null per locale; `features` — the placeholder View, pumped under `Directionality.rtl` in a `ProviderScope` with the `AppLocalizations` delegate, resolves its one string via `l10n.appTitle` (asserting it is *not* a hardcoded literal); `assets` — the `sealed AssetIntegrityError` subclasses are exhaustively pattern-matchable. If any is a no-op, it fails this task's bar — fix it here.

5. **Golden lane skeleton, dormant (11 §5, §8).** Add one tagged golden file — `packages/quran/test/golden/page_render_golden_test.dart` — opening with `@Tags(['golden'])` + `library;`, importing `flutter_test`, and a `setUpAll(_loadRealFonts)` that loops `kBundledFontFamilies`, `FontLoader(family)..addFont(rootBundle.load(fontAssetPath(family)))`, `await loader.load()` (BLOCK D). Because no fonts/masters exist until E05, the file ships with **either** no `testWidgets` body **or** a single `@Skip('muṣḥaf masters land in E05')`-annotated case — so `flutter test --tags golden` runs and is *green on the empty set*, the harness exists, and CI never auto-blesses a master. **Never** load Ahem for a fidelity golden. Mirror the RTL/layout golden shape (BLOCK E) as a commented skeleton or a second `@Skip`'d file so E08/E09's per-locale RTL goldens have a home — but author no master pixels here.

6. **`integration_test/` skeleton, one trivially-green launch journey (11 §6).** Create the top-level `integration_test/` directory with one file, e.g. `integration_test/launch_smoke_test.dart`: `IntegrationTestWidgetsFlutterBinding.ensureInitialized();` then one `testWidgets` that pumps the thin `app/` shell (the T02 placeholder screen) and `expect(find.byType(<PlaceholderScreen>), findsOneWidget)` — a launch-succeeds assertion, nothing more. This is **not** one of the four PRD journeys (J1 cold start / J2 review / J3 teacher sign-off / J4 catch-up) — those land with their feature epics; the comment must say so to forestall scope creep. Keep the network blocked here too (the launch path performs no fetch; the shell carries no downloader call). Add `integration_test: { sdk: flutter }` to the harness package's `dev_dependencies` only where the journey lives (the `app/` or a dedicated harness target, per T02/T07), not to a pure package.

7. **Widget tests use in-memory Riverpod fakes; the real stack only in `integration_test` (11 §6).** No widget/golden test in this task touches a real Drift database or real asset loading — the `features` placeholder uses `ProviderScope` overrides for any store/asset/date provider, with a **fixed** `today` (never `DateTime.now()`). State surfaces through `flutter_riverpod` `Notifier`/`AsyncNotifier` only. Pump explicit durations; never `pumpAndSettle()` an intentionally-indefinite indicator (none exists yet, but bake the habit into the scaffold comment).

8. **Pitfalls to avoid:**
   - **A golden lane that asserts nothing forever.** An empty or all-`@Skip` golden file is the *intended* dormant state here, but it must be a documented, reviewed state (a `@Skip` reason naming E05), not a silent no-op that nobody remembers to fill — the epic risk "the asset-hash and golden gates ship dormant and are forgotten."
   - **Relying on the 400 default alone.** A silent 400 can be swallowed and look "handled"; the *throwing* override is what makes a stray call loud. Keep the throw, not just the default block.
   - **`pumpAndSettle()` on an indefinite indicator** — pump explicit durations.
   - **A real DB/assets in a widget test** — in-memory Riverpod fakes only; real stack is `integration_test`'s alone.
   - **Wiring `dart:io`/the override into a pure-package test** — `engine`/`models`/`profiles` tests stay `package:test`-only and framework-free; the engine-purity gate (E01-T05) would otherwise have a reason to fire.
   - **Growing the journey suite** — exactly one launch-smoke journey here; the four PRD flows are owned by the feature epics, and a fifth needs a decision-log amendment.
   - **`!`/`late`/bare `catch`/`print` in a test** — typed `catch`, full-word names, no force-unwraps on engine/persistence values.

## Tests

This task *is* the test scaffolding; its "tests" are the harness files themselves plus the negative-network proof that the guard works. (The per-package placeholder cases are authored in E01-T03/T04 and merely confirmed-asserting here.)

- **`test/test_setup.dart`** — the throwing-`HttpOverrides` bootstrap (BLOCK H). Not a `_test.dart` file; it exports `useOfflineTestPolicy()` for the package bootstraps. No network, by definition.
- **Negative-network proof (temporary, reverted).** A throwaway case under any Flutter package — `await tester.runAsync(() => HttpClient().getUrl(Uri.parse('https://example.test/')))` (or the bare `HttpClient()` construction) — `expect`'d to throw the named `StateError`; run `flutter test`, confirm the suite goes **red** with the offline message, then revert. This is the acceptance evidence that layer (a) of the no-network gate is real, mirroring the epic's deliberate-violation discipline (the permanent version of this proof lands in E01-T08).
- **`packages/quran/test/golden/page_render_golden_test.dart`** — `@Tags(['golden'])` fidelity-golden skeleton (BLOCK D): `setUpAll(_loadRealFonts)` loading **real** KFGQPC + UI fonts via `FontLoader` (never Ahem), fixed `tester.view.devicePixelRatio`/`physicalSize`/theme, animations off. Ships with **no master** and an `@Skip('muṣḥaf masters land in E05')` body so the `golden` lane is green-on-empty; E05 fills `matchesGoldenFile('goldens/mushaf/page_001.png')`.
- **RTL/layout golden skeleton (BLOCK E)** — a commented or `@Skip`'d `today_rtl_golden_test.dart` shape under `packages/features/test/golden/` that pumps a screen under `Directionality(textDirection: TextDirection.rtl)` for `ar`/`fa`/`ckb` with locale numerals — wired but author no master; E08/E09 own the per-locale RTL masters.
- **`integration_test/launch_smoke_test.dart`** — one trivially-green launch journey (BLOCK G shape, but launch-only): `IntegrationTestWidgetsFlutterBinding.ensureInitialized()`, pump the `app/` placeholder shell, `expect(find.byType(<PlaceholderScreen>), findsOneWidget)`. Net stays blocked. Explicitly **not** J1–J4.
- **Offline/no-network guards.** The throwing override is installed in every Flutter package bootstrap; the `assets` test is the sole opt-out; no engine/models/profiles test imports `dart:io`. No test reaches the network; the airplane-mode acceptance run (E20) and the four journeys (feature epics) build on this guard.

## Definition of Done

- [ ] All acceptance criteria met; `dart test` (pure packages) and `flutter test` (Flutter packages) green locally and in the E01-T07 `fast` lane; `flutter test --tags golden` green-on-empty in the `golden` lane; `flutter test integration_test/` green in the `journeys` lane.
- [ ] **Offline / no-network (non-negotiable):** the throwing `HttpOverrides` is installed by the shared bootstrap in every Flutter package; a stray network call throws the named `StateError` and fails the suite (proven, then reverted); the `assets` downloader test is the *only* sanctioned opt-out; no test reaches the network. This is layer (a) of the three-layer gate (the static dep allow-list and banned-import scope are E01-T05).
- [ ] **No AI / no microphone (non-negotiable):** no ML/ASR/audio-recognition/microphone dependency or API is introduced by any test or harness file; the scaffolding asserts behaviour by reveal-on-tap/value semantics only, never audio.
- [ ] **Quran text fidelity (non-negotiable):** the golden harness loads the **real** KFGQPC fonts via `FontLoader`, never Ahem (which would render every glyph as a solid square and defeat a fidelity golden); no master pixel is committed or auto-blessed here — E05 supplies the page images on the pinned Linux runner, fail-closed.

- [ ] **Deterministic tests:** no test reads `DateTime.now()`; any `today` is a constructed `CalendarDate`/`SerialDay` literal injected via Riverpod override; widget tests use in-memory fakes only; the real Drift/asset stack appears solely in `integration_test`; no `pumpAndSettle()` on an indefinite indicator.
- [ ] **Standards & provenance:** every test file carries the REUSE SPDX header (`GPL-3.0-or-later`); full-word/unit-bearing names; typed `catch`; no `print`/`!`/`late` on engine/persistence values; `dart format` and `dart analyze --fatal-infos` clean; the PR is one concern, ≤ ~400 LOC, with the negative-network proof self-annotated.
