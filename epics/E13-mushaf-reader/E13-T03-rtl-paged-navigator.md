## Goal

The reader can be paged through the 604 muṣḥaf pages right-to-left without ever re-shaping a glyph. `packages/features/lib/src/mushaf/widgets/mushaf_pager.dart` ships a `MushafPager` that wraps E05's immutable `MushafPageView` in an RTL-aware `PageView.builder`/`PageController` whose paging direction is derived from `Directionality.of(context)` (`reverse: true` under RTL) so a swipe from page 1 toward page 2 advances right-to-left in fa/ckb/ar. Each visible page is `MushafPageView(pageNumber: p)` rebuilt with **only** a new `pageNumber`/geometry — the dedicated `QPC_P###` font, the bundled QUL line geometry, and the overlay coordinates are *re-selected* per page, never mirrored, reordered, re-flowed, re-typeset, or re-rendered. The pager binds page position to the E13-T02 reader-state store's `pageNumber` (controller ↔ state are kept in sync both ways) and triggers no engine mutation and no write. RTL applies to the paging direction and the surrounding chrome layout; it never reaches the glyph layer. Jump-to seeking (E13-T04) and the overlay/zoom/theme wiring (E13-T05/T06) hang off the `PageController`/page slot this task establishes.

## Implementation notes

TEST-FIRST: the paging direction and the "page rebuild changes pageNumber/geometry only, never mirrors glyphs" behaviour are correctness-critical for the sacred surface (a flipped or reordered glyph page is a fidelity defect). Write the RTL-direction widget test and the page-rebuild golden (Tests section) **before** the `MushafPager` body.

1. **File / anatomy** (eng-add-feature-module, established by E13-T01): the pager is a leaf widget under the reader feature — `packages/features/lib/src/mushaf/widgets/mushaf_pager.dart`, one primary public type `MushafPager` (`ConsumerWidget` or `ConsumerStatefulWidget` — `StatefulWidget` only because the `PageController` is a disposable resource, not because it holds business state). It is read into `MushafReaderScreen` (the dumb View, E13-T01); it imports E05's `MushafPageView`, the reader-state store (E13-T02), and `flutter`/`riverpod`/`go_router` only — no `drift`/DAO, no `package:http`/`dio`/`HttpClient`, no engine mutation API.

2. **RTL-aware `PageController` + `PageView.builder`** (ui-mushaf-page-view rule 6; eng-rtl-and-bidi-layout §2):
   ```dart
   final isRtl = Directionality.of(context) == TextDirection.rtl; // locale-derived, never assumed
   PageView.builder(
     controller: _pageController,           // initialPage = readerState.pageNumber - 1
     reverse: isRtl,                         // page 1 → 2 advances RIGHT-to-LEFT in fa/ckb/ar
     scrollDirection: Axis.horizontal,
     itemCount: edition.pageCount,           // 604, read from MushafEdition — never hardcoded (§6.1)
     onPageChanged: (i) => ref.read(readerStateProvider.notifier).setPage(i + 1),
     itemBuilder: (context, i) => MushafPageView(pageNumber: i + 1), // E05 primitive, glyph-only
   )
   ```
   The page *index* runs 0..603 internally; the user-facing **page number** is `index + 1` (1..604). `reverse` is the *only* RTL knob the pager turns — it flips the paging/gesture direction, not the page content. Read direction from context so the two permitted manual-`Directionality` islands (and a hypothetical LTR debug locale) still behave; do **not** hardcode `reverse: true`.

3. **The page slot is a pure `pageNumber` rebuild** (ui-mushaf-page-view rules 1/6; domain-mushaf-text-integrity rule 3): `itemBuilder` returns nothing but `MushafPageView(pageNumber: i + 1)` (plus the E13-T05/T06 overlay/zoom/theme parameters threaded through unchanged). Selecting page *p* re-selects its dedicated `qpcFontFamily(p)` font, its bundled QUL geometry, and its coordinate overlays — **all from E05**. The pager performs **no** glyph transform: no `Transform(...flip...)`, no `Matrix4` mirror, no `textDirection` override on the glyph layer, no reordering of lines, no `softWrap`/`TextPainter`/width-wrap. RTL lives in the `PageView.reverse` and the chrome layout, never in the page.

4. **Controller ↔ reader-state binding** (eng-create-riverpod-store; E13-T02): the store's `pageNumber` is the single source of truth.
   - **State → controller:** when `pageNumber` changes from *outside* the swipe (a jump-to seek, a deep-link, a restore), the pager listens (`ref.listen(readerStateProvider.select((s) => s.pageNumber), ...)`) and drives the controller (`_pageController.jumpToPage(p - 1)` for an instant seek, `animateToPage` for a chrome-initiated nav) — guarding against an echo loop (do not re-seek if the controller is already on that page).
   - **Controller → state:** `onPageChanged` writes the *landed* page back via the store's `setPage(p)` setter only. No other path mutates page state. The store mutation is display-only — it writes no `review_log`, mutates no `Card`, and re-derives no `due_at` (E13-T02's contract).

5. **Bounds and lifecycle**: `itemCount` and the clamp on any incoming page come from `MushafEdition.pageCount` (read, never the literal `604`); an out-of-range deep-link page is clamped to `1..pageCount` before seeking (the jump-to resolver in E13-T04 also validates, but the pager defends its own controller). Dispose the `PageController` in `dispose()` (the only reason this is a `StatefulWidget`); keep `keepPage: true` so a tab switch returns to the same page; do **not** introduce `PageController.viewportFraction != 1.0` (a peeking next-page sliver would render a second sacred page partially — keep one full page per viewport).

6. **No-dashboard restraint on the paging surface** (domain-mushaf-text-integrity rule 10; adab): paging adds **no** page-flip sound, haptic fanfare, page-curl/3-D flip animation, "pages read" counter, glow, or ornament — the default horizontal slide is the whole affordance; the words dominate and the chrome recedes (the auto-hiding edge chrome itself is E13-T08). No reduce-motion concern beyond the platform default slide; no celebration on reaching the last page.

7. **Pitfalls to avoid**: hardcoding `reverse: true` instead of deriving it from `Directionality.of(context)` (breaks the moment an LTR island/debug locale is added — eng-rtl-and-bidi-layout §2); mirroring/flipping/rotating the page itself with a `Transform`/`Matrix4`/`textDirection` override to "make it RTL" (the page is already RTL by its glyph layout — the muṣḥaf is **never** mirrored, §2/§8 of ds-12; rule 4 of the RTL skill); introducing `softWrap`/`TextPainter`/width-wrap or any per-page reflow on a page change; adding a `fontFamilyFallback` to dodge tofu while paging (a missing glyph must fail loud — §2); using a `ListView`/scroll view or `viewportFraction < 1` instead of a one-page-per-viewport `PageView` (the muṣḥaf is paged, not scrolled — C-031, rule 10); a state↔controller echo loop (seek → `onPageChanged` → setPage → listen → seek …); writing any engine/review state from `onPageChanged`; hardcoding `604`/`15` instead of reading `pageCount`/`lineCount` from the edition (§6.1); reading a second feature's `src/`; a `DateTime.now()` anywhere in the pager.

## Acceptance criteria

- [ ] `packages/features/lib/src/mushaf/widgets/mushaf_pager.dart` exists with one primary public type `MushafPager`, mounted inside `MushafReaderScreen` (E13-T01); it imports E05's `MushafPageView`, the E13-T02 reader-state store, and `flutter`/`riverpod`/`go_router` only — no `drift`/DAO, no `package:http`/`dio`/`HttpClient`, no engine-mutation symbol (verifiable by grep).
- [ ] The pager uses a `PageView.builder` whose `reverse` is `Directionality.of(context) == TextDirection.rtl`; a swipe from page 1 toward page 2 advances right-to-left in fa/ckb/ar and left-to-right under a forced-LTR `Directionality`; `reverse: true` is never hardcoded.
- [ ] `itemCount` is `MushafEdition.pageCount` (read, never the literal `604`); `itemBuilder` returns `MushafPageView(pageNumber: index + 1)` (with the E13-T05/T06 layer params threaded unchanged) and nothing else — no glyph `Transform`/`Matrix4` mirror, no `textDirection` override on the page, no line reordering, no `softWrap`/`TextPainter`/width-wrap, no added `fontFamilyFallback`.
- [ ] Paging page *p* → *p+1* rebuilds with a new `pageNumber`/font/geometry only; goldens prove the same page renders **identical pixels** under RTL and forced-LTR `Directionality` (the page content is never mirrored or reordered).
- [ ] `onPageChanged` writes the landed page (`index + 1`) back through the E13-T02 store's `setPage` setter and through no other path; an external `pageNumber` change (jump-to/deep-link/restore) drives `_pageController.jumpToPage`/`animateToPage` via `ref.listen`, with no seek↔change echo loop.
- [ ] The pager mutates no engine state — no `review_log` write, no `Card` mutation, no `due_at` re-derivation; reader-state page changes are display-only.
- [ ] The pager opens no socket and fetches nothing at runtime; an out-of-range incoming page is clamped to `1..pageCount`; the `PageController` is disposed in `dispose()`, `keepPage: true`, `viewportFraction == 1.0` (one full page per viewport, paged not scrolled).
- [ ] No page-flip sound, haptic fanfare, page-curl/flip animation, counter, glow, or ornament is added to the paging surface; the chrome carries logical `start`/`end` insets only (no `left`/`right`); every new Dart file carries the REUSE SPDX header (`GPL-3.0-or-later`) and a `///` on each public declaration.

## Tests

Dart tests run in CI on every PR (`flutter test`); muṣḥaf goldens load the **real KFGQPC page fonts** via `FontLoader` (never Ahem, which draws squares) under an RTL `Directionality`. **Write the RTL-direction widget test and the page-rebuild golden first.**

- `features/test/mushaf/mushaf_pager_rtl_test.dart` — **paging direction** (widget): pump `MushafPager` over a fake reader-state store and a stub `MushafPageView` (a cheap labelled placeholder for the *non-golden* direction tests — the glyph fonts are exercised in the golden below). For `Locale('ar')`, `Locale('fa')`, `Locale('ckb')` assert `PageView.reverse == true` and that a forward swipe gesture advances the page number from N to N+1 with the page moving **right-to-left**; under a forced `Directionality(TextDirection.ltr)` assert `reverse == false` and the swipe advances left-to-right. Assert `reverse` tracks `Directionality.of(context)` and is not a constant.
- `features/test/mushaf/mushaf_pager_binding_test.dart` — **controller ↔ state binding** (widget, `ProviderContainer`/`overrideWith` over a fake store): (a) a swipe that lands on page p calls the store's `setPage(p)` exactly once and writes through no other path; (b) an external `pageNumber := q` (simulating a jump-to/deep-link) drives the controller to index `q - 1`; (c) the seek→`onPageChanged`→`setPage`→listen cycle does **not** re-seek (no echo loop), asserted by counting controller `jumpToPage`/`animateToPage` calls; (d) an out-of-range incoming page is clamped to `1..pageCount`; (e) no `recordReview`/engine-mutation method is reachable from the pager (the fake store records zero engine writes).
- `features/test/mushaf/mushaf_pager_no_mirror_golden_test.dart` — **page-rebuild golden** over the **real KFGQPC fonts** (`FontLoader`, never Ahem): render a representative page (e.g. page 1 with the basmala/surah banner, and a mid-muṣḥaf page) inside `MushafPager` under `Directionality(TextDirection.rtl)`, `matchesGoldenFile`; then render the **same** page under `Directionality(TextDirection.ltr)` and assert it matches the **same** golden — proving the glyph content is identical (never mirrored/reordered) regardless of paging direction. Page through to the next page and assert the new page's golden differs only by being page p+1 (a pure `pageNumber` rebuild), not a transform of page p.
- `features/test/mushaf/mushaf_pager_bounds_test.dart` — **bounds** (widget): `itemCount` equals `MushafEdition.pageCount` read from a fake edition (a grep also asserts no literal `604`/`15` in the pager source); first page (index 0) cannot page before page 1 and last page (index `pageCount - 1`) cannot page past `pageCount`; `viewportFraction == 1.0` and `keepPage == true`.
- **Offline / no-network guard** (eng-write-dart-test): a throwing `HttpOverrides` is installed via the shared test bootstrap for every test above (paging must touch no socket — C-048); a banned-import grep over `packages/features/lib/src/mushaf/widgets/mushaf_pager.dart` finds no `http`/`dio`/`HttpClient`/`drift`/DAO symbol, no engine-mutation symbol, and no `DateTime.now()`/`Calendar.current`. (The reader's full real-font theme/zoom/overlay golden matrix is E13-T10; this task supplies the paging-direction and no-mirror goldens E13-T10 builds on.)
