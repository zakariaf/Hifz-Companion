## Goal

Every reader-chrome string lands as a `gen_l10n` key with `ar` (template/base) + transcreated `fa`/`ckb` values, read only through `AppLocalizations` — the riwāyah/edition label and its About/Credits entry (E13-T07), the jump-to unit/index/confirm labels (E13-T04), the zoom and light/sepia/dark control labels (E13-T06), the weak-line and mutashābihāt overlay-toggle labels (E13-T05), and a localized `Semantics` label on every one of those interactive controls. The page/juz/ḥizb/sūrah indices render through `numberFormatFor(locale)` (Extended Arabic-Indic ۰۱۲ for fa/ckb, Arabic-Indic ٠١٢ for ar), and every mixed run ("Page ۲۵۳ · Juz ۷", a sūrah name beside its index, the riwāyah label) is FSI/PDI-isolated as a formatted placeholder. The muṣḥaf is **identical across all three locales** — only the chrome localizes — and the navigator and jump-to advance right-to-left in fa, ckb, and ar alike.

## Implementation notes

TEST-FIRST (gate-critical): the per-locale numeral assertion (fa/ckb emit U+06F0-range, ar emits U+0660-range) and the `SemanticsTester` "every control has a localized label" check are the executable form of the §20 gate. Write them **first**; they must exist and fail (against the un-localized chrome) before the ARB keys and `Semantics` wrappers land.

1. **Author every key in `lib/l10n/app_ar.arb` (the `ar` template/base) first, then transcreate `fa`/`ckb`.** One ARB key per reader-chrome string, each with an `@description` giving translator/scholar context. The keys this task owns (final names per the ARB convention):
   - **Riwāyah / edition (E13-T07):** `readerRiwayahLabel` (the `displayName`/`riwayah` run, e.g. "Ḥafṣ ʿan ʿĀṣim — Madani muṣḥaf"), `readerAboutCreditsLabel` (the attribution-entry label), `readerAttributionBody` (Tanzil/QUL/KFGQPC + byte-for-byte checksum guarantee). The riwāyah/About copy is religious-adjacent — `@description` carries "needs native + scholar review", `ckb` ships provisional; it never names the page "the Quran" absolutely.
   - **Jump-to (E13-T04):** `jumpUnitJuz`, `jumpUnitHizb`, `jumpUnitSurah`, `jumpUnitPage` (the four unit names), `jumpConfirmPageJuz` (the `"{page} · {juz}"`-style confirm message, placeholders only), `jumpToTitle` (the picker title).
   - **Controls (E13-T06):** `readerZoomLabel`, `readerThemeLightLabel`, `readerThemeSepiaLabel`, `readerThemeDarkLabel`.
   - **Overlay toggles (E13-T05):** `readerWeakLineToggleLabel`, `readerMutashabihToggleLabel`.
   - **Semantics labels/hints:** one localized `label` per interactive control (`readerJumpToSemantics`, `readerZoomSemantics`, `readerThemeSemantics`, `readerWeakLineToggleSemantics` with its on/off state, `readerMutashabihToggleSemantics`, `readerAboutCreditsSemantics`, the navigator's page-change announcement `readerPageOfTotalSemantics`). Sensitive labels follow the four-attribute voice (reverent/calm/plain-and-warm/honest); no guilt/loss/"safe to drop"/"mastered", no exclamation, no emoji.

2. **Read every string only through `AppLocalizations` (`l10n.*`) in `features/lib/src/mushaf/`.** No string literal reaches `Text(...)`, `tooltip:`, `semanticsLabel:`, or `Semantics(label:)` in the reader View/widgets — `nullable-getter: false` makes a typo'd key a compile error and the `features/**` hardcoded-string grep (§8) catches a literal. The engine stays locale-free; this task touches only the feature chrome.

3. **Interpolation is one full ICU message with `{placeholder}` slots — never concatenated fragments.** The "Page N · Juz M" confirm and the page-of-total `Semantics` announcement are single ICU messages (`jumpConfirmPageJuz`, `readerPageOfTotalSemantics`) with the numeric values as isolated placeholders, so RTL word order survives and `fa`/`ar`/`ckb` grammar each read naturally. Never `"Page " + n.toString()` and never glue two localized pieces.

4. **Format every index via `numberFormatFor(locale)`, then inject and isolate.** The page/juz/ḥizb/sūrah indices and any surfaced zoom magnitude format through `numberFormatFor(locale)` (`fa`/`ckb` `-u-nu-arabext` → ۰۱۲, `ar` `-u-nu-arab` → ٠١٢) **before** placement into the ICU placeholder; each formatted digit run is then wrapped via the `l10n/bidi.dart` helper — `isolateLtr` for the known-direction digit run, `isolate` for an unknown-direction value, never raw concatenation or legacy LRE/RLE. The sūrah list pairs `surah.name_ar` (a single `Text`/`TextSpan`, isolated) beside its isolated locale-numeral index. **The muṣḥaf's own printed ayah/juz/ḥizb/sajda numerals are never `numberFormatFor`-reshaped** — only chrome indices are.

5. **`Semantics` on every interactive control, localized, ≥48 dp, RTL focus order.** Wrap each reader control (jump-to entry, zoom, each theme option, each overlay toggle, the About/Credits entry, and the navigator's page surface) in `Semantics(button: true, label: l10n.<key>, …)` per §7, sourcing the label from the ARB set so the screen reader speaks the active locale. The two overlay toggles announce their on/off state (a `select`/state-bearing label, e.g. `readerWeakLineToggleSemantics(on/off)`). Targets are ≥48 dp; no `textScaler` clamp on chrome text. Reading/focus order is correct under RTL (a consequence of logical layout, not a manual reorder).

6. **Direction stays locale-derived; layout stays logical.** No hardcoded `Directionality` anywhere in the reader chrome — RTL is the consequence of `supportedLocales: [ar, fa, ckb]` + `GlobalWidgetsLocalizations`. Picker rows, control rows, and the riwāyah label use `EdgeInsetsDirectional`/`AlignmentDirectional`; direction-needing logic reads `Directionality.of(context)`. The jump-to and the navigator advance right-to-left in all three locales because the chrome is laid out logically — this task adds no per-locale branch and mirrors no glyph.

7. **`ckb` is canonical-encoded and flagged provisional.** Author the Sorani values with U+06D5 (ە) for AE and U+06A9 (ک) for kaf; no stray U+200C (ZWNJ), no Teh-Marbuta-for-AE hack (the canonical-`ckb` lint rejects them). Religious/term-set wording (riwāyah, About) carries "needs native + scholar review" in `@description` and ships clearly provisional, never stating a ruling. Confirm the bundled UI font's Sorani coverage (پ چ ژ گ ڤ ڕ ڵ ۆ ێ ە) and that the vendored `CkbMaterialLocalizations` chrome delegate is wired so framework chrome (OK/Cancel, picker labels) resolves.

8. **Adab gate first on every sensitive string.** The riwāyah label, the About/attribution body, and any methodology-adjacent wording pass `domain-adab-and-religious-integrity` before they land: sect/madhhab-neutral, zero tafsīr/translation/commentary, the page never "the Quran" absolutely, no gamification/guilt copy. Transcreate per-locale register (Persian respectful-warm *šomā*; Arabic imperatives softened to statements of readiness; Kurdish provisional).

9. **The muṣḥaf is identical across all three locales.** Switching UI language re-renders only the chrome strings/numerals/`Semantics`; the glyph page, its line/page breaks, its overlay rectangles, and its printed markers are byte-identical in fa, ckb, and ar. This is asserted by the goldens (the page pixels match across locales; only the chrome differs).

10. **Pitfalls to avoid:** a string literal in a `Text`/`tooltip:`/`semanticsLabel:`/`Semantics(label:)` inside `features/**` (grep + compile gate); concatenating an ASCII index (`"Page " + n.toString()`) or splicing a value mid-string instead of an isolated placeholder; emitting the Arabic block (٤٥٦) to a Persian/Kurdish reader (or vice-versa) by trusting a sublocale default instead of the pinned `-u-nu-`; hardcoding `Directionality.rtl` or using `EdgeInsets.only(left:/right:)`/`Alignment.centerLeft/Right` (physical-side grep); an unlabeled control (`SemanticsTester` catch); running `numberFormatFor`/`intl` over the muṣḥaf's own printed numerals (the §1 sacred-boundary violation); shipping the heh+ZWNJ or Teh-Marbuta `ckb` hack, or locking `ckb`/riwāyah terminology before scholar review; a literal-translation that silently shifts tone (transcreate, adab gate first); rendering the goldens with `Ahem` instead of the real bundled fonts.
