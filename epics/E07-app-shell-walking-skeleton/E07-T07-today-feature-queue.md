## Goal

The `today` feature slot stops being the inert E07-T04 placeholder and becomes a real, reactive vertical-slice surface. `packages/features/lib/src/today/` ships a dumb `TodayScreen` (`ConsumerWidget`) reading exactly one `TodayController` (`AsyncNotifier<TodaySession>`, 1:1 with the View), which `await`s a `todayQueueProvider` — a `StreamProvider.family<…, ProfileId>` over a Drift query (`CardRepository.watchTodaySession(profile, today)`) that runs `Engine.buildToday(cards, today)` on the live card set and emits the day grouped **Far (manzil) → Near (sabqi) → New (sabaq)** in recitation order. A committed review (the E07-T05 single write path) re-emits the stream and rebuilds the queue — there is no second cache and `R`/health is computed on read, never stored. The controller reads "today" only from the injected `clockProvider` (`CalendarDate`), never `DateTime.now()`. `session.when(...)` surfaces `loading` as a calm skeleton and `error` as a calm `RetryView` — never a spinner-of-shame. The page-card row internals (track chip, decay indicator) and the one-tap grade command are **E07-T08**; the full Today surface (time-budget cap, Far/Near/New section headers, catch-up banner, honest budget-feedback line, all-done terminal copy) is **E12** — this task renders the engine-ordered queue and wires the read path that everything later hangs off.

## Implementation notes

TEST-FIRST: the queue read model is correctness-critical (a wrong order or a stale stream is a wrong day shown to a ḥāfiẓ). Write the `TodayController` test and the queue-ordering / re-emit test (Tests section) **before** the controller and provider code.

1. **Folder anatomy** (eng-add-feature-module): under `packages/features/lib/src/today/` — `today_screen.dart` (the dumb `ConsumerWidget` entry View, replacing E07-T04's inert body), `today_view_model.dart` (the 1:1 `TodayController extends AsyncNotifier<TodaySession>`), `today_providers.dart` (the **scoped** `todayQueueProvider` + `todayControllerProvider`), and `widgets/` (the minimal queue row + the calm skeleton/retry leaf views). One primary public type per file, `lower_snake_case` (project-structure §4).

2. **`todayQueueProvider` — the `StreamProvider.family` over Drift** (flutter §3, verbatim shape): in `today_providers.dart`,
   ```dart
   final todayQueueProvider = StreamProvider.family<List<ReviewItem>, ProfileId>((ref, profile) {
     final repo = ref.watch(cardRepositoryProvider);
     final today = ref.watch(clockProvider).today();   // CalendarDate — injected, never DateTime.now()
     return repo.watchTodaySession(profile, today);     // Drift stream → engine.buildToday() over live cards
   });
   ```
   `cardRepositoryProvider` and `clockProvider` are the E07-T01/T02 DI providers; this task constructs no live service. The provider is `family`-keyed by `ProfileId` (stable/equatable, §5) and is `autoDispose`-eligible per Riverpod default for a `.family` read model — but the **db/engine app-scope singletons it reads are never `autoDispose`d** (§5).

3. **`CardRepository.watchTodaySession` — the read counterpart of `recordReview`** (data package, eng-add-persisted-model territory; this task adds the *read* method, the schema/DAO body is E03): returns `Stream<List<ReviewItem>>` from `db.<cardDao>.watchMemorizedCards(profile)` (a Drift `.watch()`), mapping each emitted card set through `scheduler.buildToday(cards, today.serialDay)` and projecting the resulting `DayPlan.items` to an immutable `ReviewItem` list **in the engine's returned order** (Far→Near→New). The repository calls the engine and persists nothing on a read; `R` is recomputed inside `buildToday` from `today` and is **never stored** (flutter §3). No DAO is imported into the feature — only the repository is (eng-add-feature-module §4).

4. **`ReviewItem` is an immutable display value, not a `Card`** (eng-create-riverpod-store §5): a small value type carrying only what the row needs — `pageId` (1–604), `juz`, `track` (the `Track` enum: far/near/new), `dueAt` (`CalendarDate`, the trust-clamped engine value — C-016), and the decay/health inputs the E07-T08 indicator will read — never the mutable `Card`, never a stored `R` column. Live in the `models` (or `engine`) package so the boundary stays clean; the View receives primitives/value types, never a `DayPlan`/`Card`.

5. **`TodayController` (the 1:1 ViewModel)** (flutter §1.3 worked example):
   ```dart
   class TodayController extends AsyncNotifier<TodaySession> {
     @override
     Future<TodaySession> build() async {
       final profile = ref.watch(activeProfileProvider);          // the only profile gate (§1.2)
       final queue = await ref.watch(todayQueueProvider(profile).future);
       return TodaySession(queue: queue);                          // immutable UI state
     }
   }
   final todayControllerProvider =
       AsyncNotifierProvider<TodayController, TodaySession>(TodayController.new);
   ```
   `TodaySession` is an immutable value (`copyWith`) holding the ordered `List<ReviewItem>`. The controller holds **no** grade command in this task (that is E07-T08); it must not navigate (eng-create-riverpod-store §11) and must not read `DateTime.now()` — the only "today" is via the queue provider's injected clock.

6. **`TodayScreen` (the dumb View)** (flutter §1.3; ui-daily-session-list §10): a `ConsumerWidget` that reads exactly `todayControllerProvider` and renders `session.when(...)`:
   - `loading:` → a calm `surfaceContainerLow` skeleton (ui-daily-session-list §4) — **never** a bare spinner / "loading…" guilt text;
   - `error:` → a `RetryView(onRetry: () => ref.invalidate(todayControllerProvider))` — calm copy, never a spinner-of-shame (flutter §1.3; eng-create-riverpod-store §6);
   - `data:` → the queue rendered **in the engine's Far→Near→New order**, one `ReviewItem` per minimal row, with a calm empty-data fallback line (the full all-done terminal surface + section headers + budget line are E12). No `try/catch`/business logic in `build`; no engine/repository call from the widget.

7. **The View renders the pre-built order; it never reorders** (ui-daily-session-list §10; engine §7): no `..sort`, no `.where`, no cap, no `loadBalance`, no `buildToday` call in the widget or controller — the ordered, manzil-first list arrives from the engine through the stream. A row is one muṣḥaf page (C-031), placed in muṣḥaf RTL order via `EdgeInsetsDirectional`/`AlignmentDirectional`; it renders **no Quran glyphs** and surfaces **no D/S/R, no percentage, nothing "safe to drop"** (ui-daily-session-list §3/§9).

8. **Minimal row + strings** (this task's row is a placeholder for E07-T08's full page card): one calm row showing the localized "Page N · Juz M" headline (locale numerals, bidi-isolated FSI/PDI) read via `AppLocalizations` (`l10n.*`, `ar` template + fa/ckb) — no hardcoded literal survives the `features/**` grep. The track chip and decay indicator internals are **deliberately deferred to E07-T08**; this task may show the track as a plain localized label so the Far/Near/New grouping is visible and asserted. Adab voice: calm, reverent, no guilt/fear, no "safe to drop"/"mastered", no exclamation/emoji.

9. **Accessibility** (ui-daily-session-list §8): a `Semantics` container announces the list ("Revise today") with section roles so the three tracks are heard as ordered groups in the active locale; each row reads as one labelled item ("Page 253, Juz 13, far-revision"). Stable accessibility identifiers: `today.queue` (the list root) and `today.row` (each row), consumed by the E07-T10 spine journey. Tap targets ≥ 48 dp; Dynamic Type via `type.*` text styles.

10. **Pitfalls to avoid** — calling `buildToday`/`engine` or sorting/capping in the View or controller (the day is pre-built — engine §7; ui-daily-session-list §10); any `DateTime.now()`/`Calendar.current` in the controller or provider (today is the injected `CalendarDate` — flutter §3); persisting or caching `R`/health as a column or a second store (the stream is the cache — flutter §3); importing `drift`/a DAO/`package:http`/`dio` into the feature (eng-add-feature-module §4); an un-keyed "current Today queue" provider, or `autoDispose`-ing the db/engine singletons (§5); reading a second feature's `src/`; a `setState`/manual cache poke or optimistic republish (there is no mutation here, but the read path must not introduce one — the grade command is E07-T08 through `recordReview`); rendering a Quran glyph in a row, re-typesetting an āyah, or showing D/S/R / a percentage / "safe to drop"; a streak/score/badge/confetti on the surface; a hardcoded string instead of `l10n.*`; raw hex/pt/ms instead of E06 token names; building the section headers / budget-feedback line / catch-up banner here (those are E12).

## Acceptance criteria

- [ ] `packages/features/lib/src/today/` has the fixed anatomy: `today_screen.dart` (dumb `ConsumerWidget`), `today_view_model.dart` (`TodayController extends AsyncNotifier<TodaySession>`, 1:1 with the View), `today_providers.dart` (scoped `todayQueueProvider` + `todayControllerProvider`), `widgets/` (minimal row + calm skeleton/retry leaves) — replacing E07-T04's inert Today placeholder body without touching the route table, the redirect guard, or `HomeShell`.
- [ ] `todayQueueProvider` is a `StreamProvider.family<List<ReviewItem>, ProfileId>` that reads `cardRepositoryProvider` + `clockProvider().today()` and returns `repo.watchTodaySession(profile, today)`; it constructs no live service and reads no wall clock.
- [ ] `CardRepository.watchTodaySession(profile, today)` returns a Drift `.watch()` stream that maps each card set through `Engine.buildToday(cards, today)` and projects `DayPlan.items` to `ReviewItem`s **in the engine's returned Far → Near → New order**; it persists nothing and stores no `R`.
- [ ] Changing the live card set (e.g. the E07-T06 seed, or a committed review) re-emits the stream and rebuilds the queue automatically — there is no second cache and no manual republish; the displayed queue can never disagree with the stored cards.
- [ ] `TodayController.build()` watches `activeProfileProvider` (the only profile gate) and `await`s `todayQueueProvider(profile).future`, returning an immutable `TodaySession`; the controller holds no grade command (E07-T08), never navigates, and never reads `DateTime.now()`.
- [ ] `TodayScreen` reads exactly `todayControllerProvider` and renders `session.when(loading/error/data)`: `loading` is a calm skeleton (no spinner theatre), `error` is a `RetryView` (`ref.invalidate(todayControllerProvider)`), `data` is the queue in the engine's order; no engine/repository call or business `try/catch` in `build`.
- [ ] The View renders the engine's pre-built order verbatim — a grep finds no `..sort`/`.where`/cap/`loadBalance`/`buildToday` in the feature widgets or the controller; the queue order is asserted to be Far → Near → New.
- [ ] Each row names one muṣḥaf page as localized "Page N · Juz M" (locale numerals, bidi-isolated) via `AppLocalizations`; the row renders no Quran glyph and surfaces no D/S/R, no percentage, and nothing "safe to drop"; the rendered due date is the trust-clamped engine value (C-016).
- [ ] The list root carries a `Semantics` "Revise today" container with section roles and the stable identifier `today.queue`; each row carries `today.row` and reads as one labelled item in each locale; tap targets ≥ 48 dp; Dynamic Type via `type.*`.
- [ ] No `package:http`/`dio`/`dart:io HttpClient`, no `drift`/DAO import, and no `DateTime.now()`/`Calendar.current` appears anywhere reachable from `TodayScreen`, `TodayController`, or `todayQueueProvider`; tokens are referenced by name (no raw hex/pt/ms); every new Dart file carries the REUSE SPDX header (`GPL-3.0-or-later`) and a `///` on each public declaration.

## Tests

Dart tests run in CI on every PR (`flutter test`); goldens load the **real** bundled UI fonts and pin the runner. **Write the controller and queue-ordering tests first.**

- `features/test/today/today_controller_test.dart` — **controller unit** (`ProviderContainer.test()` + `overrideWith`, faking the **repository**, not the `Notifier` — flutter §7): override `cardRepositoryProvider` with a fake whose `watchTodaySession` emits a fixed `ReviewItem` list, `clockProvider` with a `FixedClock`, and `activeProfileProvider` with a fixed `ProfileId`; assert `TodayController.build()` resolves to a `TodaySession` whose `queue` equals the faked list in order, runs in milliseconds (no widget pump), and that the controller exposes no grade command. This test verifies wiring/UI-state mapping only — **not** the FSRS math (that has its own engine golden/property tests).
- `features/test/today/today_queue_ordering_test.dart` — **read-model ordering** over real `Engine.buildToday`: seed an in-memory Drift store (`NativeDatabase.memory()`, the E07-T01 double) with a deterministic mixed card set (some far/manzil due, some near, some new) under a `FixedClock`; assert `watchTodaySession` emits the items in **Far → Near → New** recitation order, that a manzil-due page is never absent, and that each item's `dueAt` is the trust-clamped engine value (≤ the cycle ceiling, C-016). Reuse the engine's §7/§8 fixtures rather than re-deriving an expected order by hand.
- `features/test/today/today_queue_reemit_test.dart` — **stream re-emit** (the §3 single-write-path completion): with the in-memory store + `FixedClock`, subscribe to `todayQueueProvider`, then commit a card change through `CardRepository.recordReview` (E07-T05) and assert the queue stream emits a **second** list reflecting the new card state — proving the committed write, not a manual republish, refreshes the queue, and that no stale `R` is shown.
- `features/test/today/today_screen_test.dart` — **widget**: pump `TodayScreen` in a `ProviderScope.test`/`MaterialApp` over the faked controller for each of `Locale('ar')`, `Locale('fa')`, `Locale('ckb')`; assert the `loading` skeleton, the `error → RetryView` (and that tapping retry invalidates the controller), and the `data` list rendering rows in order with the `today.queue`/`today.row` identifiers and `l10n` "Page N · Juz M" headlines; assert no interactive grade control exists yet (that arrives in E07-T08).
- `features/test/today/today_screen_rtl_golden_test.dart` — **golden**: `matchesGoldenFile` of the populated queue under fa, ckb, and ar on the real bundled UI fonts, proving rows lay out RTL (leading at start), the Far→Near→New grouping is visible, and locale numerals / Sorani letters render — never `Ahem`.
- **Offline / no-network guard** (eng-write-dart-test): a throwing `HttpOverrides` is installed via the shared test bootstrap for every test above; a banned-import grep over `packages/features/lib/src/today/` finds no `http`/`dio`/`HttpClient`/`drift`/DAO symbol and no `DateTime.now()`/`Calendar.current`. (The kill-and-relaunch crash-safety and the end-to-end `integration_test` spine journey are E07-T09/T10; this task supplies the `today.queue`/`today.row` identifiers and the re-emit behaviour they assert against.)
