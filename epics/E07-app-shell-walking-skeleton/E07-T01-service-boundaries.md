## Goal

The three side-effect boundaries the shell wires in E07-T02 exist, build, and are testable in isolation: the **`Clock`** (`CalendarDate today()`, live `LiveClock` + `FixedClock`), the **Drift `AppDatabase` handle** (live opened in `main`, in-memory `NativeDatabase.memory()` double for tests), and the **`AppReadyGate`** (`core pack verified ∧ a profile exists`, live impl reading E05's verified-asset state + the profile store, and a fake-ready/fake-not-ready double). Each is an `abstract interface class` referencing value types only, declared as a Riverpod `Provider<Interface>` whose IO-opening variants **throw when read un-overridden**, with no `DateTime.now()` / `HttpClient` at any call site. This task ships the boundary *shapes*, *providers*, and *fakes* — not the composition-root override wiring (E07-T02), the engine/repository providers (E07-T02), the router's consumption of `appReady` (E07-T03), or the live Drift schema/repository body (E03/E05).

## Implementation notes

1. **`Clock` interface** — `packages/models/lib/src/services/clock.dart`: `abstract interface class Clock { CalendarDate today(); }`. The file imports `hifz_models` value types only — **no `package:flutter`, no `dart:io`, no `DateTime`** — so `engine/`, `data/`, and the features can name it without importing a framework (`01` §3.1). Re-export from `packages/models/lib/hifz_models.dart`. This is the canonical boundary in `eng-define-service-boundary/template.dart` piece 1.
2. **`LiveClock`** — `packages/app/lib/composition/services/live_clock.dart` (shell boundary code — the **only** `DateTime.now()` outside test fixtures the repo will ever contain): `CalendarDate today() => CalendarDate.fromDateTimeLocal(DateTime.now())` (or E02's exact named constructor). Do **not** hand-roll serial-day math here — delegate to E02's `CalendarDate` (`domain-calendars-and-hifzdate`). Add the `clockProvider = Provider<Clock>((ref) => const LiveClock());` in `composition/providers.dart` (the provider declaration lands here; T02 only confirms it needs no override because it opens no IO).
3. **Drift `AppDatabase` handle** — the "interface" is E03's `AppDatabase` Drift class itself (the live impl is the opened handle, not a new class). Declare `appDatabaseProvider = Provider<AppDatabase>((ref) { throw StateError('appDatabaseProvider must be overridden in main()'); });` in `composition/providers.dart` — a **throwing placeholder** (`04` §1.2), satisfied only by T02's `overrideWithValue(db)`. The deterministic double is `AppDatabase(NativeDatabase.memory())` (an in-memory Drift connection), constructed in the test bootstrap, never in production. Do **not** open a `NativeDatabase` from a file here, and do **not** `autoDispose` this provider — it is an app-scope singleton.
4. **`AppReadyGate` interface** — `packages/models/lib/src/services/app_ready_gate.dart`: `abstract interface class AppReadyGate { bool get coreAssetsVerified; bool hasProfile(); }` (or a single `bool get isReady` composing both — keep the surface minimal and value-typed; no Flutter, no `dart:io`). `appReady == coreAssetsVerified ∧ hasProfile()` ("no Quran screen renders before the core pack is verified and a profile exists" — `01` §6 / PRD R1).
5. **`LiveAppReadyGate`** — `packages/app/lib/composition/services/live_app_ready_gate.dart`: composes E05's verified-asset state (a verified core pack present in app-documents) and the `ProfileRepository`'s "any profile exists" read. **It reads already-verified local state — it opens no socket, downloads nothing, verifies no bytes, and renders no glyph** (asset download/verification is E05's quarantined `assets/` module; PRD C1/C-048). Declare `appReadyProvider = Provider<AppReadyGate>((ref) { throw StateError('appReadyProvider must be overridden in main()'); });` as a throwing placeholder (T02 wires the live impl; T03 watches it).
6. **Deterministic fakes** — `packages/app/test/fakes/`: `FixedClock implements Clock { const FixedClock(this._today); final CalendarDate _today; CalendarDate today() => _today; }`; `FakeAppReadyGate implements AppReadyGate` with constructor flags for the four states (verified+profile = ready; the three not-ready combinations). The Drift double is `AppDatabase(NativeDatabase.memory())` directly (no wrapper fake). **No mock framework, no codegen** — plain hand-written classes installed via `overrideWith`/`overrideWithValue` (`eng-define-service-boundary` piece 5; `11` §2).
7. **TEST-FIRST (correctness-critical: the determinism + no-network covenants).** Write the boundary tests before/with the impls: assert `FixedClock(literal).today()` returns exactly the literal `CalendarDate` (no host clock read anywhere in the suite); assert the in-memory `AppDatabase(NativeDatabase.memory())` opens and round-trips a trivial query without touching disk; assert `FakeAppReadyGate` reports each of the four ready/not-ready states; assert reading any throwing placeholder un-overridden throws a `StateError`. These pin the boundaries' contracts independent of T02's wiring.
8. **What NOT to do** (pitfalls): no `DateTime.now()` / `Calendar.current` / `TimeZone.current` anywhere except `LiveClock` (E01's CI grep bans it on every PR, including these new targets); no `package:http` / `dio` / `dart:io HttpClient` in any of these files (the socket is E05's `assets/` only); no `get_it`, no second DI container, no global mutable "current service" singleton; no business logic inside a `Provider` (it is a thin wire); no live `AppDatabase`/`LiveAppReadyGate` constructed here at module scope — the live wiring is T02's `ProviderScope` overrides; no `autoDispose` on the app-scope DB/clock; no user-facing string, no Quran glyph, no engine math, and no telemetry inside any boundary.

## Acceptance criteria

- [ ] `packages/models/lib/src/services/clock.dart` and `app_ready_gate.dart` exist as `abstract interface class`es; both import `hifz_models` value types only — **no `package:flutter`, no `dart:io`, no `DateTime`, no `HttpClient`** — and `engine/`/`data/` can name them without importing a framework.
- [ ] `Clock.today()` returns a `CalendarDate` (E02's serial-day value type); `AppReadyGate` exposes only `coreAssetsVerified` ∧ `hasProfile()` (or a composed `isReady`) over value types.
- [ ] `packages/app/lib/composition/services/live_clock.dart` is the **only** file in the repo (outside test fixtures) containing `DateTime.now()`; it converts via E02's `CalendarDate` constructor and hand-rolls no day math; a grep for `DateTime.now()`/`Calendar.current` confirms the single site.
- [ ] `LiveAppReadyGate` composes E05's verified-asset state and the profile-exists read; it imports **no** networking package, opens no socket, downloads/verifies no bytes, and renders no glyph.
- [ ] `composition/providers.dart` declares `clockProvider` (`Provider<Clock>`, live default — opens no IO), `appDatabaseProvider` (`Provider<AppDatabase>`, **throwing placeholder**), and `appReadyProvider` (`Provider<AppReadyGate>`, **throwing placeholder**); each provider is a thin wire with no business logic; none is `autoDispose`d.
- [ ] Reading `appDatabaseProvider` or `appReadyProvider` un-overridden throws a `StateError` (a forgotten wiring is a loud startup failure, not silent null data); `clockProvider` resolves to `LiveClock` with no override.
- [ ] Deterministic doubles exist and are plain hand-written classes installed with `overrideWith`/`overrideWithValue`: `FixedClock`, `FakeAppReadyGate` (all four states), and `AppDatabase(NativeDatabase.memory())` — **no mock framework, no codegen** anywhere in the change.
- [ ] No `get_it`, no second DI container, no global mutable "current service" singleton introduced; Riverpod is the sole DI mechanism.
- [ ] Every Dart file carries the REUSE SPDX header (`GPL-3.0-or-later`); every `public` declaration carries a `///` doc comment; the analyzer/lint config and `dart format` pass.

## Tests

`package:test` for the pure `models` interfaces, `flutter_test` for the shell fakes/providers; file names mirror sources; every file carries the REUSE SPDX header; the throwing-`HttpOverrides` offline guard is installed via the shared bootstrap (`11` §7):

- `packages/app/test/composition/services/fixed_clock_test.dart` — `FixedClock(CalendarDate.ymd(2026, 6, 16)).today()` returns exactly that `CalendarDate`; the suite reads no host wall clock; two calls return the identical value (deterministic).
- `packages/app/test/composition/services/app_ready_gate_test.dart` — `FakeAppReadyGate` reports `isReady == true` only when `coreAssetsVerified ∧ hasProfile()`, and `false` for each of the three not-ready combinations (no-pack, no-profile, neither).
- `packages/app/test/composition/providers_test.dart` — in a `ProviderContainer.test()`: reading `appDatabaseProvider` and `appReadyProvider` **un-overridden each throws `StateError`**; `clockProvider` resolves to a `LiveClock` (type assertion only — the suite does not read `today()` against the host clock); overriding all three with the doubles resolves cleanly.
- `packages/app/test/composition/services/memory_database_test.dart` — `AppDatabase(NativeDatabase.memory())` opens and a trivial round-trip query succeeds without touching disk (proves the in-memory Drift double the E07-T05 write-path suite and E07-T10 journey reuse is sound).
- **Offline/no-network guard:** the shared test bootstrap installs a throwing `HttpOverrides` (`11` §7); none of these boundary suites may make a network call — a stray socket is a loud, named failure. No golden, engine-vector, or muṣḥaf-fidelity test is in scope for this task (the boundaries carry no glyphs and no schedule math).
