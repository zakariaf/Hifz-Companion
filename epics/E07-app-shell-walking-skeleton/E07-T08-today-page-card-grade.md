## Goal

The Today queue from E07-T07 stops being a bare list of identifiers and becomes a real page-card row a ḥāfiẓ can read and act on. Each `ReviewItem` renders as one flat `ListTile`/`Card` (elevation Level 0-1) in muṣḥaf RTL order: a **leading** non-interactive `TrackChip` (sabaq/sabqi/manzil, localized term-set) + a **calm decay indicator** (the same fact three ways — single-hue lightness ramp + glyph + text label, never hue alone), a **headline** of localized "Page ۲۵۳ · Juz ۱۳" in locale numerals (bidi-isolated), an optional **supporting** line, and a **trailing** chevron. The whole row is one ≥48 dp tap target. Bound to that row is the one-tap grade command (Again/Hard/Good/Easy) — the View calls `TodayController.grade(...)`, which routes through `CardRepository.recordReview` (the single write path: one `db.transaction`, append `review_log`, upsert the engine-updated `Card`, commit **before** republish), so the `todayQueueProvider` Drift stream re-emits and the page reappears rescheduled in its new track/position with no manual cache poke. No celebration, streak, score, badge, percentage, or `R`/D/S value. Reveal-on-tap, stumble-line capture, and teacher sign-off are explicitly E12.

## Implementation notes

**TEST-FIRST (correctness-critical):** the grade command's wiring through the single write path is the load-bearing seam — write the `TodayController.grade` test (faking the repository, asserting `recordReview` is called with `ReviewSource.self` / `errorLines == const []` and that the stream re-emit drives the rebuild) **before** the command code. The decay-band → indicator mapping and the locale-numeral headline are also pinned by tests below; write those before their widget code.

1. **Files (under the E07-T07 feature target).** Add leaf widgets under `packages/features/lib/src/today/widgets/`, one primary type per file (`docs/engineering/02-project-structure.md` §3.4, §4): `page_card.dart` (`PageCard`, the feature-layer row mapping a `ReviewItem` → slots), `track_chip.dart` (`TrackChip`, domain-blind), `decay_indicator.dart` (`DecayIndicator`, domain-blind), `grade_bar.dart` (`GradeBar`, the four-verb one-tap control). `TodayScreen` (E07-T07) composes `PageCard` per `ReviewItem`; the `grade` command lives on `TodayController` (E07-T07), this task adds its body.

2. **The row is built on the M3 list-item anatomy, flat.** `PageCard` is a `ListTile`/`Card` at elevation Level 0-1 (token by name) — no decorative shadow, no brand-color tint (ui-page-card §1). Leading slot = `TrackChip` then `DecayIndicator`, `space.2` apart, both **labels inside the row**; placement via `EdgeInsetsDirectional` / logical start-end (ui-page-card §2). Headline = the localized "Page N · Juz M" run in `type.body`; optional supporting line ("next: in N days") in `type.caption` / `color.text.secondary`; trailing chevron at the row **end** (left, in RTL). The whole row is **one** ≥48 dp `touch.min` `InkWell`/tap into the (E12) recite flow — the chip, the indicator, and the grade bar are not the row's primary tap; in this epic the row tap is inert/placeholder and the grade bar is the only active control (E12 replaces the row tap with the reveal surface).

3. **`TrackChip` — a non-interactive label that names the tradition (ui-page-card §4-§5).** A label-only `Chip` (or styled `Container` with the same metrics) in `type.label`, pairing a **tradition-tied color family with its localized term-set label**: FAR/manzil → the `color.accent.green` *maintenance* family (green is reverent ground, **never** reward or alarm), NEAR/sabqi → a secondary neutral-tinted family, NEW/sabaq → a tertiary neutral-tinted family. The label is a regional term-set string from the ICU `select`/region-override (المراجعة البعيدة / مرور دور / مەنزڵ), switchable per region, ckb's longer terms **wrap, never truncate**. Color is never the sole channel — color **and** text always. No count, XP, streak, or badge on the chip ever.

4. **`DecayIndicator` — honest, triple-redundant, never an alarm (ui-page-card §6-§7).** A tiny custom-painted swatch + glyph at the row start (after the chip), ≤ `space.4`, encoding the same fact **three ways**: a single-hue lightness ramp (`color.heatmap.strong` → mid → `color.heatmap.faded`) **+** a glyph (filled / half / hollow) **+** a text label ("solid" / "holding" / "needs revision"). The three calm bands roll up from the engine's retrievability `R` (computed on read, min-leaning) — but the user reads "needs revision," **never** "R = 0.83" or a percentage. The indicator ranges only solid → needs-revision: it has **no** "safe to drop"/"mastered"/"stop revising" state (release-blocking honesty contract). `DecayIndicator` takes a `DecayLevel` enum (solid/holding/needsRevision) + localized labels — it never receives `R` or a `Card`.

5. **Headline + supporting in locale numerals, bidi-isolated (ui-page-card §3; eng-add-localized-string §6-§7).** "Page N · Juz M" is **one** ICU message in `app_ar.arb` whose `{page}`/`{juz}` placeholders are formatted via `numberFormatFor(locale)` (`fa`/`ckb` → Extended Arabic-Indic U+06F0; `ar` → Arabic-Indic U+0660) **before** injection, and the mixed run is wrapped through the `l10n/bidi.dart` isolate helper (FSI/PDI) so it never reorders to "30 of 7" or breaks row alignment. Never `"Page " + n.toString()`, never raw ASCII digits. The `ReviewItem` carries Latin-digited `pageId`/`juz`; the locale re-mapping happens **here**, at the presentation boundary, never in the engine.

6. **The grade bar — four verbs, one tap, no reveal (E12 owns reveal).** `GradeBar` renders four buttons whose labels are the active term-set's **traditional grade verbs** for Again/Hard/Good/Easy (ICU `select`/region-override; the engine `Grade` enum is unchanged — eng-add-localized-string §8). A tap fires one callback `onGrade(Grade)`; the View binds it to `ref.read(todayControllerProvider.notifier).grade(item.pageId, grade, source: ReviewSource.self)`. **Calm receipt only** — no confetti, no score, no streak bump, no exclamation; the page simply leaves its slot and re-spreads when the stream re-emits. (The four-button *band styling*, undo, and the disabled-until-revealed state are E12; here the buttons are always enabled because there is no reveal gate yet.)

7. **The grade command routes through the single write path (eng §1.3, §4).** Add to `TodayController` (E07-T07):
   ```
   Future<void> grade(int pageId, Grade grade,
       {ReviewSource source = ReviewSource.self}) async {
     final profile = ref.read(activeProfileProvider);
     await ref.read(cardRepositoryProvider).recordReview(
       profile: profile, pageId: pageId, grade: grade,
       errorLines: const [], source: source,        // stumble-lines + teacher source are E12
     );
     // No manual republish: recordReview commits, todayQueueProvider re-emits (§3).
   }
   ```
   Do **not** `state = ...` optimistically before the `await` returns, do **not** call a DAO or open a transaction in the controller/View, do **not** `setState`/poke a cache — the UI updates **only** because the committed transaction makes the Drift stream re-emit (persist-before-republish). `errorLines` is `const []` and `source` defaults `ReviewSource.self` for this epic.

8. **`MergeSemantics` + one localized phrase (ui-page-card §9; ui-daily-session-list §8).** Wrap the row in `MergeSemantics` so a screen reader hears **one** localized phrase — "Page ۲۵۳, Juz ۱۳, far-revision, needs revision" — in the active locale, the leading glyph-words announced as words; `ExcludeSemantics` on decorative dividers. The grade bar's four buttons each carry their own stable accessibility identifier (`grade.again`/`grade.hard`/`grade.good`/`grade.easy`) and a `Semantics` label in each locale; the row carries a stable identifier (`today.page-card.<pageId>` or a list-index identifier) for the E07-T10 spine journey to address and grade.

9. **Pitfalls to avoid.** Making the chip, the decay indicator, or the headline a **second tap target** inside the row (they are labels); showing `R`, a D/S/R number, a percentage, or **any** "safe to drop"/"mastered"/"done" state; rendering a Quran **glyph** or re-typesetting an āyah in the row (the muṣḥaf lives only in the reader/recite surface); an **alarm-red** decay or a downward-arrow/falling-grade animation; a streak/badge/score/confetti/celebratory tint on the card or after a grade; optimistic republish before the `recordReview` commit; opening a transaction or calling a DAO from the View/controller; `DateTime.now()` anywhere (the row reads streamed state; "today" is the injected clock upstream in E07-T07); `"Page " + n.toString()` / raw ASCII digits / a non-isolated mixed run; hardcoded "Manzil"/"Sabqi"/"Again" literals instead of the term-set `select`; `EdgeInsets.only(left:/right:)` / `Alignment.centerLeft/Right`; inlined hex/pt/dp instead of E06 token names; reaching the network/microphone/a model from this surface.
