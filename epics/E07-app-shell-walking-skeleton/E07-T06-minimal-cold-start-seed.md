## Goal

A **stripped** onboarding sub-step exists as a feature folder `packages/features/lib/src/onboarding/` that turns "I am a ḥāfiẓ" into a seeded schedule without grading 604 pages: a fast juz-level coverage grid (one tap = held / not-held over the 30 juz in muṣḥaf order), then a single per-held-juz **Solid / Shaky / Rusty** pick. On completion the ViewModel maps each `JuzConfidence` to a conservative prior **by calling `Engine.coldStartCard(pageId, confidence, today)`** for every page of every held juz — never inventing `(D, S)` itself — and routes the resulting seeded `Card`s through the **single write path** (a named `data`/`profiles` repository method that opens one Drift transaction and commits **before** republishing). After the commit, the redirect guard's `appReady` condition is satisfied and Today (E07-T07) has real engine-selected due pages. This is the minimal seed only: welcome/privacy, the one-time core-pack download, the optional "when memorized" stale-time input, the named cycle-preset picker, and the daily budget are **E11** — this task seeds a profile so the spine has a non-empty schedule, nothing more.

## Implementation notes

**TEST-FIRST (correctness-critical):** the seed→engine→persist mutation is a *sanad*-touching write — author the `OnboardingController.commitPlacement` write-path/seed-mapping tests (faking the repository; asserting `coldStartCard` is called once per held page with the captured confidence and the injected `today`, and that persist precedes republish) **before** the controller body. The widget capture surface follows.

1. **Feature folder** — `packages/features/lib/src/onboarding/` (eng-add-feature-module anatomy): `onboarding_screen.dart` (dumb `ConsumerWidget` entry View, reads exactly one controller, renders), `onboarding_view_model.dart` (the 1:1 `AsyncNotifier`/`Notifier` holding the immutable capture state + the `commitPlacement` command), `widgets/coverage_grid.dart` + `widgets/juz_confidence_rater.dart` (one primary type per file), `onboarding_providers.dart` (scoped, not global). The `GoRoute('/onboarding')` is declared by E07-T03 — this task supplies the View body the guard routes to, never the route table or the guard.

2. **Capture state (immutable)** — the ViewModel exposes one immutable value, e.g. `OnboardingState({Set<int> heldJuz, Map<int, JuzConfidence> confidence})`. `heldJuz` holds 1…30 juz indices; an un-selected juz is simply absent (`UNMEMORIZED`), never a stored "0%". `JuzConfidence` is the **engine enum** (`solid`/`shaky`/`rusty`) imported from `engine` — the View maps the `SegmentedButton` selection onto it; it never defines a UI-local confidence type or a `(D, S)` table. View-local transient UI (which pass is showing) stays in the widget / a small `autoDispose` feature provider; `useState` is not used.

3. **Coverage grid** — `widgets/coverage_grid.dart`: a `GridView` of 30 selectable cells in **muṣḥaf order, juz 1 at the start/right under RTL** (`EdgeInsetsDirectional`, no `.reversed`, no hardcoded left/right). Each cell carries the juz number in locale numerals **plus** a selected/checked glyph — never selection by color alone (SC 1.4.1). Tapping toggles membership in `heldJuz`. An un-held cell is a calm un-emphasized `color.*` surface — **never** alarm-red, "missing", or "0%". Cells are ≥48dp `touch.min`, `Semantics`-labelled "Juz ۱۳, held" / "Juz ۱۳, not held" with the toggle/selected flag and a visible focus ring (`color.outline`).

4. **Confidence rater** — `widgets/juz_confidence_rater.dart`: for each **held** juz, a single mutually-exclusive Solid/Shaky/Rusty pick (M3 `SegmentedButton` or a row of large `FilledButton`s), ≥56dp, `space.2` apart, in the thumb zone, mirroring the grade band's sizing. Each option pairs a color family **with** a transcreated self-report label ("I hold this firmly / it wobbles / it's gone rusty"), never praise or a score; `Semantics` announces the word and its meaning ("Shaky — needs regular revision"). The selection writes `confidence[juz]`.

5. **Seed mapping — the engine owns `(D, S)`** — on `commitPlacement`, for every held juz the ViewModel expands the juz to its constituent **page ids** (the fixed juz→page span lives in the `quran`/reference data, not invented here — C-031: one card = one of 604 pages) and calls `engine.coldStartCard(pageId, confidence[juz], today)` for each, where `today = ref.read(clockProvider).today()` (injected `CalendarDate`, **never** `DateTime.now()`). **`memorizedOn` is omitted** in this minimal task (stale-time decay is E11). The UI hardcodes no seed table and computes no stability — `coldStartCard` and `_coldStartSeed` are the single source of truth (engine §5), so the calibration `dueAt = today` (every held page due once early) comes for free.

6. **Single write path** — the seeded `Card`s persist through **one named repository method** (e.g. `ProfileRepository.seedColdStart(profileId, cards)` / `commitPlacement`) that opens **one `db.transaction`**, inserts the seed cards (and any append-only seed `review_log`/coverage rows the schema defines), and commits **before** the ViewModel republishes "done" — exactly the persist-before-republish covenant T05 established. The View never writes `Card` rows. A failure propagates (calm `.when(error:)` retry, never a guilt message) and leaves **no half-seeded state**; only on a clean commit does the ViewModel set the profile ready so `appReady` flips and "Done → first day generated" can fire. Per-profile providers are `family`-keyed by `ProfileId`.

7. **Strings & RTL** — every label (the two pass titles, the coverage instruction, the three confidence words, the summary line) and `Semantics` label is authored in `l10n/lib/l10n/app_ar.arb` first, transcreated fa/ckb/ar; ckb canonical-encoded and flagged provisional. Read only via `AppLocalizations` (`l10n.*`) — `nullable-getter: false`. Juz numbers go through `numberFormatFor(locale)` then into an ICU placeholder (never `"Juz " + n.toString()`); a "Juz N" mixed run is FSI/PDI-isolated. The summary frames the conservative bias as **behavior** — "we'll revise everything you hold once, then adjust" — never a seeded D/S/R or a "you're 87% ready" score (C-009; ui-cold-start §5).

8. **Pitfalls to avoid** — hardcoding `D=3/S=60` (or any `_coldStartSeed` value) or computing stability in the View; calling `DateTime.now()` instead of `clockProvider`; rendering an un-held juz as alarm-red / "missing" / "0%"; showing the seeded D/S/R or a "readiness %"; a celebration/streak/badge/confetti when the grid fills; writing `Card` rows from the View or persisting outside the single repository transaction; republishing "done" **before** the commit (the exact inversion the suite must catch) or leaving a half-seeded state on a mid-flow kill; passing `memorizedOn` (that pass is E11); a multi-select or free-form slider for confidence (fakes precision); `EdgeInsets.only(left:/right:)` / `Alignment.centerLeft/Right` or a `.reversed` grid to "fake" RTL; a hardcoded literal instead of `l10n.*`; raw hex/pt/ms instead of E06 token names; any `http`/`dio`/`drift`/DAO import reaching the feature; pulling the full E11 flow (welcome, download, "when memorized", cycle picker, budget) into this minimal sub-step.
