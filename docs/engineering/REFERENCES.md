# Master Bibliography — Engineering

This is the deduplicated master bibliography for the Hifz Companion engineering foundation: every external source that any engineering doc cites — the thirteen synthesis docs (`01`–`13`), the [`README.md`](README.md) and its tech-decision log, and the [`research/`](research/) dossiers they distil — gathered in one place, graded, and grouped by theme. It exists so a developer (or a sceptical reviewer, F-Droid maintainer, or auditing ḥāfiẓ) can trace any architectural decision, API call, constant, schema rule, or build-gate back to a named, dated, web-verified source. Each entry gives the full citation, a working URL, an evidence grade, and a short note on **what the engineering takes from it**. This file introduces no new claims; it indexes the sources behind decisions made elsewhere in `docs/engineering/`. Because the work is *ṣadaqah jāriyah*, the standard is *iḥsān* — every citation here is real and was verified against the official documentation page, the package registry (pub.dev), the source repository, the standards body, or the publisher's record. We never fabricate a citation, version number, author, or URL.

---

## How to read this file

- Sources are **grouped by theme** (matching the synthesis docs) and, within each group, sorted alphabetically by author or — for organisational/web sources — by the publishing body or title. A source used by several docs is listed **once**, under its primary theme, with its broader use noted.
- Every entry ends with its **evidence grade** in the form `— [GRADE]`, per the legend below. Engineering grades follow blueprint §3 (official docs, standards, peer-reviewed SE evidence) rather than the science set's empirical grades.
- Each synthesis doc carries its own `## References` list of only the sources it cites; **this file is the deduplicated superset.** Where a source's exact citation, version, or mirror URL varied slightly across docs, the canonical, most-verifiable form (and the most recent verified version number) is used here.
- In-repo documents (the PRD, the blueprint, the engineering README and its decision log, sibling synthesis docs, and the research dossiers) are **not** listed here — they are cross-referenced inline in each doc and are not external bibliography. The two cross-set foundations ([`PRD.md`](../PRD.md) and [`_DOC-SET-BLUEPRINT.md`](../_DOC-SET-BLUEPRINT.md)) are the authority for every internal claim.
- The citation policy (real, web-verified, never fabricated; unsourced "best practices" rewritten as explicit assumptions or removed) lives in [`../_DOC-SET-BLUEPRINT.md`](../_DOC-SET-BLUEPRINT.md) §2 and is recapped in [`README.md`](README.md). All external URLs were verified to resolve during research (the dated-correctness and localization docs note their checks on 2026-06-16).

### Evidence-grade legend

Engineering claims are graded by the **kind of source**, not by experimental design. Preference among technical sources: official platform/standard docs are primary; package/registry pages and maintainer issue trackers are authoritative for the artefact they describe; expert-community and community write-ups are corroborating and used only where official docs do not cover the point.

| Grade | Meaning |
|-------|---------|
| **[STD]** | Formal standard or specification (Unicode UAX, W3C WCAG, IETF RFC, NIST FIPS, ISO) |
| **[DOCS]** | Official platform / framework / library documentation (Flutter, Dart, SQLite, Drift, GitHub, Apple, Google Play, F-Droid) |
| **[PKG]** | Package-registry or source-repository page for a specific dependency (pub.dev, GitHub repo) — authoritative for version, licence, and API surface |
| **[REF]** | Reference implementation or canonical algorithm source (FSRS repos/wikis, minisign, reproducible-builds.org) |
| **[ISSUE]** | Maintainer issue tracker / changelog / PR — authoritative evidence of a bug, breaking change, or behaviour |
| **[SE]** | Peer-reviewed software-engineering or computing research (ICSE, FSE, MSR, IEEE TSE, EMSE, ACL, Onward!) |
| **[EXPERT]** | Named expert-practitioner writing (Code With Andrea, recognised maintainers' essays, established technical blogs) |
| **[COMM]** | Community write-up / vendor guide / encyclopaedic source — corroborating only |
