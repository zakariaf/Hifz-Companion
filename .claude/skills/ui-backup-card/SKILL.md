# ui-backup-card

The **backup card**: the calm Settings surface where a ḥāfiẓ exports their review history to a file they keep, imports (restores or merges) a `.hifzbackup` file, reads an honest status line, and — separately, out of easy thumb reach — erases everything. The card frames a **user-held local file** as a privacy *feature* (no server can snoop a file the app never moves) while stating the matching responsibility plainly: lose the phone *and* the file and the record is gone, because there is no cloud to recover from. It carries the export action, the import/restore action, the optional-encryption toggle, the "last backup" status line, the stale-backup nudge, the replace-vs-merge confirmation, and the erase action — and **no cloud iconography, no account, no "Sign in to sync."**

This card is where the product's "never trapped" promise and its "delete actually deletes" promise both become visible. It states two honest facts at once — *you own this; therefore you are responsible for it* — and then gets out of the way (Pillar 5, *private & offline by feel*; Pillar 8 of the privacy doc, *local-first ownership + real erasure*). A backup card that shows a cloud glyph, a sync spinner, an account, an "upload," or hides the no-recovery tradeoff is the wrong component.

## When to use

Use when building or placing:
- the **backup / export / restore card** in Settings
- the **Export** ("Save backup") action and the **Import** ("Restore from a file") action
- the **status line** — "Last backup: <localized date>" / "No backup yet"
- the **no-recovery tradeoff** copy (the honest "lose phone + file = data gone" line) and the **plaintext-is-readable** honesty line
- the optional **encryption** toggle (protective-default-on) and its passphrase prompt
- the **stale-backup** nudge (a calm "your last backup is from <date>" note, never a red alarm)
- the **replace vs merge** restore confirmation surface
- the **erase** action and its irreversible-confirmation (sits in the hard-to-reach top corner)

Do NOT use this skill for:
- the `.hifzbackup` **file bytes** — container header, `schemaVersion`, SHA-256, the Argon2id→ChaCha20-Poly1305 envelope, replace/merge set-union semantics → use **domain-backup-format** (this card *drives* `HifzBackup.export/import`; it never re-derives a header offset or a cipher choice)
- the **single-choice Settings preferences** (language, calendar, numerals, term-set, theme, muṣḥaf) → use **ui-settings-picker**
- the **profile switcher / scope** the export covers (this profile vs all) → use **ui-profile-switcher**
- the **double-confirmation gate** for the irreversible erase (the "type/hold to confirm", the cancel-is-primary consequence dialog) → use **ui-destructive-confirm** (this card *places* the erase entry point; that skill owns the confirmation flow)
- the **Drift transaction**, WAL crash-safety, or the `flutter_secure_storage` DB key behind erase/import → use **eng-add-drift-table-or-migration** / **eng-define-service-boundary**
- the **persist-before-republish** controller that runs export/import/erase and republishes the status → use **eng-create-riverpod-store**
- the **localized strings** for every label, the tradeoff line, and the date format → use **eng-add-localized-string**
- the **RTL geometry**, locale numerals on the date, and FSI/PDI isolation of the file name → use **eng-rtl-and-bidi-layout**
- the conscience-check on the **tradeoff / encryption / erase copy** → use **domain-adab-and-religious-integrity**

The card *presents and triggers* backup; the `backup/` package *makes the bytes* and the engine *interprets the restored state*. A backup card that opens a socket, renders a glyph page, computes a `due_at`, or shows a cloud is doing another layer's job.

## The canonical pattern

1. **Frame the local file as a privacy feature AND state both halves of ownership.** The card's leading copy says plainly that the backup is *a file the user keeps* that the app moves nowhere — and, in the same breath, the responsibility: because there is no server, the user holds the only copy. `docs/design-system/10-privacy-and-trust-ux.md` §8 (*Backup as a privacy feature*: "Your backup is a file you keep. We never upload it, and only you can restore from it." — local-first *ownership*, both halves stated) and §3 (honesty is **specific and verifiable**, the few load-bearing facts plainly, never rhetoric). Copy is owned by `docs/design-system/11-voice-and-tone.md` §2 (the *plain & warm* + *honest* voice — "Backup saved. All profiles, on this phone.").

2. **State the no-recovery tradeoff honestly — never hide it, never soften it into a feeling.** Restore copy and the export confirmation both carry the load-bearing fact: lose the phone *and* the file and the record cannot be recovered elsewhere, because there is no cloud and no account. This is stated as a *checkable fact*, not "your data is safe." `docs/design-system/10-privacy-and-trust-ux.md` §8 (erasure copy "says plainly that because there is no server, nothing is recoverable elsewhere — a guarantee most apps cannot truthfully make") and §3 (specific, verifiable honesty over reassurance). The tradeoff is delivered with empathy-then-fact, never as blame or fear — `docs/design-system/11-voice-and-tone.md` §4 (lead with empathy, then the honest fact, then a path; never weaponize a real spiritual fear).

3. **No cloud iconography, no account, no "sync", no spinner-as-theater.** The card has no cloud glyph, no "Sign in," no "Upload," no "syncing…" animation — there is no server to sync with and a theatrical spinner would be a lie. Export progress (only the real CPU/crypto work) and the integrity result are honest; nothing implies an ongoing connection. `docs/design-system/10-privacy-and-trust-ux.md` §2 (never a fake "securing…/syncing…" animation — there is no server) and §8 (never imply a backup is uploaded or synced; the app performs no network transfer for user data). The export/import path itself opens no socket — `docs/engineering/10-backup-format.md` §1, §9 (the OS share sheet moves the file; the app transmits nothing) — driven via **domain-backup-format**.

4. **Encryption is the optional, protective-default-on choice — one plain sentence, and no false recovery promise.** The encryption toggle defaults on, reversible, explained in one sentence; the passphrase prompt states honestly that a forgotten passphrase means the file is unrecoverable (no reset, nothing stored). For an *unencrypted* backup, the card states plainly that the file is readable by anyone who opens it. `docs/design-system/10-privacy-and-trust-ux.md` §9 (the few real choices: protective default, reversible, one honest line — "Locks your backup file with a password only you hold."); `docs/engineering/10-backup-format.md` §2 (the export screen states an unencrypted backup is readable by anyone) and §6 (no recovery — the passphrase is never stored, never logged; a forgotten passphrase means the file is unrecoverable). The toggle persists via **eng-create-riverpod-store**; the actual cipher is **domain-backup-format**.

5. **The status line is plain and dateful — "Last backup: <localized date>" / "No backup yet" — not a streak.** One calm status line states whether and when the last backup was made, in the user's calendar and numerals; "No backup yet" is a neutral fact, not a scold. `docs/design-system/11-voice-and-tone.md` §2 (*honest*, plain & warm — state the fact, then stop) and §7 (no transactional framing, no engagement metric — this is a status, never a score or a streak to maintain). The date renders in the user's calendar (Hijri Umm al-Qurā / Solar-Hijri Jalālī / Gregorian) and locale numerals — `docs/design-system/10-privacy-and-trust-ux.md` §8 (locale numerals + calendar in any backup-date display); date math and the `CalendarDate` value are **domain-calendars-and-hifzdate**; never `DateTime.now()` in the View (**eng-define-service-boundary**).

6. **The stale-backup nudge is a calm note, never a red overdue alarm.** If the last backup is old, the card may show a quiet "Your last backup is from <date>" note that *offers* a fresh export — it never nags, never escalates, never shows a red/danger fill or a guilt line. `docs/design-system/10-privacy-and-trust-ux.md` §10 (the reminder layer stays calm and peripheral — no guilt, no fear, no escalation) and §11 (no **nagging** — the dark-pattern release gate); `docs/design-system/11-voice-and-tone.md` §3 (the tone matrix: hard/old-state news leads with a calm fact + a path, never "you're behind" or an alarm) and §4 (empathy-then-path, never blame).

7. **Restore offers replace vs merge as two explicit, separately-confirmed actions — each with distinct, honest copy.** Import surfaces two choices: **Replace** ("This will replace all data currently in Hifz Companion") and **Merge** ("This will add the imported reviews to your existing history") — each confirmed separately, neither pre-selected as a trap. The merge path is the teacher↔student transfer. `docs/engineering/10-backup-format.md` §7 (replace vs merge, both explicit/confirmed/transactional; the exact confirmation copy) and §0 framing (merge is a deduplicating set-union over the append-only *sanad* `review_log` — never an overwrite). The card only *presents and confirms*; the algorithm is **domain-backup-format**; the merge is the local, no-server "teacher sees student data" path — **ui-profile-switcher** for the halaqa scope.

8. **Erase is real, immediate, honest — and sits out of easy thumb reach behind confirmation.** One erase action wipes all local data; the copy says plainly it is irreversible and that any existing backup file is then the only remaining copy. It uses thumb-zone *difficulty* (top corner, behind a confirmation) as a natural safety margin — never friction-for-friction's-sake, never a buried dark pattern. `docs/design-system/10-privacy-and-trust-ux.md` §8 (erasure is real/immediate/honest; "because there is no server, nothing is recoverable elsewhere"; erase sits in the hard-to-reach top corner behind a confirmation) and `docs/design-system/05-layout-spacing-touch.md` (destructive controls out of easy thumb reach — composed via **ui-settings-picker**'s Settings template). The erase plumbing (WAL siblings + DB key) is **domain-backup-format** §9 / **eng-add-drift-table-or-migration**.

9. **Audit the card against the five dark-pattern strategies — export/import/erase/silence are one tap each.** Export, import, erase, and dismissing the stale nudge are each a single, un-obstructed action; nothing sneaks a network call, no option is pre-checked as a trap, the protective choice is never visually buried, and nothing is gated behind an account or upgrade. `docs/design-system/10-privacy-and-trust-ux.md` §11 (the five-strategy release gate: no **nagging/obstruction/sneaking/interface-interference/forced action** — export/import/erase are one tap each; no hidden network; honest hierarchy) and §9 (no pre-ticked boxes; the safe option is the easy one). No commercial-transaction framing anywhere — `docs/design-system/11-voice-and-tone.md` §7 (no "upgrade/premium/unlock"; worship is served, not managed).

10. **Persist through the single write path; the card computes nothing and opens no socket.** Export/import/erase and the encryption-toggle change route through one controller/notifier method that persists transactionally **before** republishing the card's status; the View never writes state, never reaches a global, never opens a socket, never calls `DateTime.now()`. After import, the engine recomputes — the card just refreshes its status. `docs/engineering/10-backup-format.md` §1 (the shell orchestrates `/data` read → `backup/` serialize → OS share; **zero** network I/O) and §9 (export = build snapshot → serialize/encrypt off-UI isolate → atomic temp file → OS share sheet); the persist-before-republish path is **eng-create-riverpod-store**; the OS share / file-pick / clock are injected boundaries (**eng-define-service-boundary**).

11. **RTL by geometry, the file name bidi-isolated, locale numerals on the date — fully offline, no-AI.** The card is RTL by construction for fa/ckb/ar: actions and status sit on logical start/end via `EdgeInsetsDirectional`, never named left/right; a backup file name (a mixed Latin/number run like `Hifz-2026-06-16.hifzbackup`) is FSI/PDI-isolated so it never flips the line; the date renders in the locale numeral set via `intl`. Every string is a transcreated ARB value (the tradeoff/erase/encryption lines reviewed per locale), never hardcoded English. The whole card works in airplane mode forever and infers nothing. `docs/design-system/10-privacy-and-trust-ux.md` §8 (export/import/erase fully transcreated for fa/ckb/ar, locale numerals + calendar in backup-date display); `docs/design-system/11-voice-and-tone.md` §8 (transcreation, not literal translation; mixed runs bidi-isolated; numerals never concatenated) and §9 (per-locale native + scholar review of the sensitive copy). Layout via **eng-rtl-and-bidi-layout**; strings via **eng-add-localized-string**.

## Do / Don't

| Do | Don't |
|---|---|
| Frame the backup as "a file you keep, we move nowhere" AND state the responsibility (only copy) in the same breath (§8 privacy) | Imply the backup is uploaded, synced, or recoverable from a server |
| State the no-recovery tradeoff as a checkable fact — "lose this phone and the file and your record is gone" | Soften it into "your data is safe with us," or hide it below the fold |
| Use plain Settings affordances — `FilledButton`/`ListTile` rows, a status line, an encryption `Switch` | Show a cloud glyph, an account/"Sign in", an "Upload", or a fake "syncing…" spinner |
| Default encryption ON, reversible, one honest line; warn a forgotten passphrase is unrecoverable | Promise a passphrase reset, store the passphrase, or hide that an unencrypted file is readable |
| Status line: "Last backup: <localized date>" / "No backup yet" in locale calendar + numerals | Turn backup status into a streak, score, or "you're overdue" scold |
| Stale-backup note is a calm fact + an offer to export | Show a red/danger fill, escalate, nag, or guilt the user about an old backup |
| Restore = two explicit, separately-confirmed actions: Replace (replaces all) / Merge (adds to history) | Pre-select replace, auto-resolve a conflict silently, or blur replace and merge into one button |
| Put erase in the hard-to-reach top corner, behind confirmation; state it is irreversible | Place erase in easy thumb reach, or imply a "soft delete" that secretly retains data |
| Route export/import/erase/toggle through one controller (persist-then-republish); refresh status | Mutate state in the View, open a socket, call `DateTime.now()`, or compute a `due_at` here |
| RTL by `EdgeInsetsDirectional`; file name FSI/PDI-isolated; date numerals via `intl`; ARB strings | Hardcode left/right, ASCII digits, English labels, or concatenate the date into a sentence |

## Checklist

Before this card is done:

- [ ] The leading copy frames the backup as **a user-held file the app moves nowhere** AND states the responsibility (the user holds the only copy) — both halves, plainly (`docs/design-system/10-privacy-and-trust-ux.md` §8; `docs/design-system/11-voice-and-tone.md` §2).
- [ ] The **no-recovery tradeoff** appears on export confirmation and restore as a *checkable fact* ("lose this phone and the file and your record is gone — there is no cloud"), with empathy, never as reassurance, blame, or fear (`docs/design-system/10-privacy-and-trust-ux.md` §8, §3; `docs/design-system/11-voice-and-tone.md` §4).
- [ ] **No cloud iconography, no account, no "Sign in/Sync/Upload", no fake spinner** — only real export progress + the honest integrity result; the path opens no socket (`docs/design-system/10-privacy-and-trust-ux.md` §2, §8; `docs/engineering/10-backup-format.md` §1, §9).
- [ ] The **encryption toggle** defaults on, is reversible, and is one honest line; the passphrase prompt states a forgotten passphrase is unrecoverable; an **unencrypted** backup is plainly flagged readable-by-anyone (`docs/design-system/10-privacy-and-trust-ux.md` §9; `docs/engineering/10-backup-format.md` §2, §6).
- [ ] The **status line** is "Last backup: <localized date>" / "No backup yet" in the user's calendar + locale numerals — a plain status, **never a streak/score/scold** (`docs/design-system/11-voice-and-tone.md` §2, §7; `docs/design-system/10-privacy-and-trust-ux.md` §8).
- [ ] Any **stale-backup nudge** is a calm fact + an offer, **never** a red/danger fill, escalation, nag, or guilt line (`docs/design-system/10-privacy-and-trust-ux.md` §10, §11; `docs/design-system/11-voice-and-tone.md` §3, §4).
- [ ] **Restore** offers **Replace** and **Merge** as two explicit, separately-confirmed actions with the distinct copy ("This will replace all…" / "This will add the imported reviews…"); neither is a pre-selected trap (`docs/engineering/10-backup-format.md` §7).
- [ ] **Erase** is one irreversible action behind confirmation in the **hard-to-reach top corner**; copy states it is irreversible and that any backup file is then the only remaining copy — no soft-delete (`docs/design-system/10-privacy-and-trust-ux.md` §8; `docs/design-system/05-layout-spacing-touch.md`).
- [ ] Every action (export, import, erase, dismiss nudge) is **one un-obstructed tap**; nothing pre-checked as a trap; honest hierarchy; nothing gated behind an account/upgrade; no commercial-transaction wording (`docs/design-system/10-privacy-and-trust-ux.md` §11, §9; `docs/design-system/11-voice-and-tone.md` §7).
- [ ] Export/import/erase/toggle route through **one controller method** (persist transactionally **before** republishing the status); no View-level write, no global, no `DateTime.now()`, no socket; after import the card just refreshes status (`docs/engineering/10-backup-format.md` §1, §9; **eng-create-riverpod-store**, **eng-define-service-boundary**).
- [ ] **RTL** by `EdgeInsetsDirectional`; the backup **file name** (mixed Latin/number run) is FSI/PDI-isolated; the date renders locale numerals via `intl`; every string is a transcreated ARB value, reviewed per locale (`docs/design-system/10-privacy-and-trust-ux.md` §8; `docs/design-system/11-voice-and-tone.md` §8, §9; **eng-rtl-and-bidi-layout**, **eng-add-localized-string**).
- [ ] Fully **offline / no-AI**: the card opens no network, runs no model, has no telemetry on backup use; works in airplane mode forever (`docs/engineering/10-backup-format.md` §1).
- [ ] **adab conscience-check**: the tradeoff, encryption, stale-nudge, and erase copy carry no guilt/fear/loss framing and no "safe" reassurance; reviewed via **domain-adab-and-religious-integrity**.
- [ ] Tests: a widget test asserting **no cloud/sync/account** glyph or string in the tree; that export/import/erase route through the controller (persist-before-republish); that restore shows two distinct confirmations; an RTL golden per locale (fa/ckb/ar) with a bidi-isolated file name and locale-numeral date (**eng-write-dart-test**).

This card makes two honest promises visible at once: *your record is never trapped* and *delete actually deletes*. The two outranking lines: it must never imply a cloud, an account, or a recoverable server copy (the file is the only copy, stated plainly), and it must never weaponize the no-recovery tradeoff into fear or dress decay/loss into a scold — honesty here is *adab*, not a disclaimer. If any line drifts toward "your data is safe," a cloud metaphor, a streak, or a guilt nudge, run it past **domain-adab-and-religious-integrity** and the voice-and-tone never-ship lint.
