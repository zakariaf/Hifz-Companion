# references — eng-add-localized-string

The precise governing doc sections for this skill. Only real sections are listed; take exactly what each line says.

## Primary — the ARB / gen_l10n / RTL / numeral / bidi / plural / a11y implementation

- `docs/engineering/12-localization-rtl-accessibility-impl.md` §1 (The ARB / `gen_l10n` pipeline) — **The one thing:** user-facing strings come *only* from `flutter_localizations` + `intl` + `gen_l10n` over ARB with `generate: true`; **`ar` (`app_ar.arb`) is the template/base content language**; the key becomes a getter on `AppLocalizations`, `@description`/`placeholders` carry translator/scholar context, `nullable-getter: false` makes a missing key a compile error; a CI grep bans string literals in `features/**`; the `/engine` is locale-free (no `AppLocalizations`/`intl` imports).
- `docs/engineering/12-localization-rtl-accessibility-impl.md` §6 (Plurals: ICU `plural` and Arabic's six categories) — **The one thing:** every count-bearing string is an ICU `plural`; Arabic's full CLDR set (`zero`/`one`/`two`/`few`/`many`/`other`) is a *translation contract* and a missing category is a release blocker; the count is formatted to locale numerals (§5) before placement; the `pagesDue` six-category template is the model.
- `docs/engineering/12-localization-rtl-accessibility-impl.md` §5 (Locale numerals and regional terminology) — **The one thing:** numerals render via `numberFormatFor(locale)` with the numbering system **pinned** (`fa`/`ckb` → `-u-nu-arabext` Extended Arabic-Indic U+06F0; `ar` → `-u-nu-arab` Arabic-Indic U+0660), injected as an ICU placeholder, never ASCII-concatenated; the sabaq/sabqi/manzil term-sets are a region-override ICU `select` (NEEDS scholar review); calendar `(y,m,d)` is re-mapped to locale digits *downstream* of the conversion.
- `docs/engineering/12-localization-rtl-accessibility-impl.md` §4 (One bidi-isolation helper for every mixed-script run) — **The one thing:** every mixed/opposite-direction run goes through the single `l10n/bidi.dart` helper — `isolate` (FSI…PDI) for unknown direction, `isolateLtr`/`isolateRtl` (LRI/RLI…PDI) when direction is known (prefer over FSI; first-strong mis-detects on leading punctuation); isolate *before* injecting into the ICU placeholder; never the muṣḥaf glyph layer.
- `docs/engineering/12-localization-rtl-accessibility-impl.md` §3 (`ckb` is a custom locale with a vendored delegate) — **The one thing:** `ckb` = `Locale.fromSubtags(languageCode: 'ckb')` + the vendored `CkbMaterialLocalizations` delegate for framework chrome Flutter doesn't ship; bundled-font Sorani glyph coverage (پ چ ژ گ ڤ ڕ ڵ ۆ ێ ە) is a release-gate item; `ckb` copy stays flagged provisional until native + scholar review.
- `docs/engineering/12-localization-rtl-accessibility-impl.md` §2 (Direction is derived from the locale — never hardcoded) — **The one thing:** RTL follows the locale via `GlobalWidgetsLocalizations`; strings carry no hardcoded direction; an LTR island (Latin technical token) is a `Directionality(TextDirection.ltr)` *view* wrapper, not an ARB property.
- `docs/engineering/12-localization-rtl-accessibility-impl.md` §8 (The localization & accessibility gate) — **The one thing:** the layered, mostly-compile-time gate — key coverage (codegen + `nullable-getter:false`), `features/**` hardcoded-string grep, physical-side grep, ASCII-digit grep, Arabic-plural completeness, per-locale RTL/numeral goldens (real bundled fonts, **never `Ahem`**), and a TalkBack/VoiceOver pass; RTL/a11y are always-on, never a "polish" phase.
- `docs/engineering/12-localization-rtl-accessibility-impl.md` §7 (Accessibility implementation) — **The one thing:** localized `Semantics` `label`/`hint` on every control (so the screen reader speaks the active locale) — these strings flow through this same ARB pipeline; ≥48×48 dp targets, honored `textScaler`, and "never color alone" are decided at the view/token layer, not in the string.

## Supporting — the voice/adab and the design RTL/term-set contract

- `docs/design-system/11-voice-and-tone.md` §1 (The governing concept is *adab*) — **The one thing:** *adab* is the **first gate** of copy review, ahead of clarity and brevity; a clear-but-harsh string fails first; the reverent register is the same across every surface and locale.
- `docs/design-system/11-voice-and-tone.md` §2 (Four fixed voice attributes) — **The one thing:** every string is **reverent, calm, plain-and-warm, honest**; sentence case; second-person singular warm; **no exclamation marks, no emoji** in product copy; numerals always in the locale set via `intl`, never raw ASCII.
- `docs/design-system/11-voice-and-tone.md` §6 (Phrase as invitation, never command) — **The one thing:** the mandate words "you must / have to / should / don't" and scolding imperatives are **banned**; default copy suggests and informs, leaving the user's choice intact.
- `docs/design-system/11-voice-and-tone.md` §8 (One voice across fa/ar/ckb requires transcreation) — **The one thing:** **transcreate, never literal-translate**; per-locale register set deliberately (Persian respectful-warm/*taʿārof*-aware; Arabic gender/number forms with imperatives softened to statements of readiness; Sorani register + vocabulary by native reviewers, provisional); never concatenate sentence fragments; numerals/dates via `intl`, mixed runs FSI/PDI-isolated.
- `docs/design-system/11-voice-and-tone.md` §9 (Tone is a per-locale QA gate) — **The one thing:** the **never-ship list** (guilt/fear/loss, controlling mandates, "safe to drop"/"mastered"/"done", exclamation marks/emoji, "upgrade"/"premium"/"unlock") is a release-blocking lint run on every locale, plus native-speaker register review and scholar review for religious terminology; zero missing ARB keys / zero hardcoded strings so no string escapes review.
- `docs/design-system/11-voice-and-tone.md` §3–§4 (Tone matrix; lead with empathy) — **The one thing:** hard news (decay, missed days, budget overflow, lapse) follows the empathy-then-path template — acknowledge → honest fact → concrete path → the user's choice — never opening with blame or amplifying spiritual fear.
- `docs/design-system/11-voice-and-tone.md` §5 (Servant to the teacher) — **The one thing:** the app never issues a fiqh ruling, never says a page is "safe to drop"/"mastered", never speaks *for* the Quran; sign-off copy credits the teacher ("Recorded with your teacher's sign-off"), not the app.
- `docs/design-system/12-localization-and-rtl.md` §6 (The sabaq/sabqi/manzil vocabulary is swappable string data) — **The one thing:** track labels, the four grade verbs, and cycle names are ARB keys (`track.far`, `grade.again`, `cycle.weeklyKhatm`) with a regional override; swapping a whole term-set is one file; never hard-code "Manzil"/"Sabqi"; ckb defaults stay provisional pending native + scholar review.
- `docs/design-system/12-localization-and-rtl.md` §4 (Numerals render in the locale's own digit set, via `intl`) — **The one thing:** Extended Arabic-Indic (۰۱۲۳) for fa/ckb, Arabic-Indic (٠١٢٣) for ar, the `type.numeral` discipline; locale-driven via `intl`; the documented date-vs-number `intl` inconsistency means digit output is **verified per surface**; ayah numbers on the muṣḥaf page are the immutable layer, never re-rendered.
- `docs/design-system/12-localization-and-rtl.md` §3 (Mixed-direction text isolated with FSI/PDI) — **The one thing:** keep the opposite-direction value a **placeholder** so the `intl` formatter can isolate it; never concatenate a raw ASCII number/Latin token ("Juz " + n); use isolates (LRI/RLI/FSI + PDI), never legacy embeddings (LRE/RLE/LRO/RLO); a localized label stays in one `Text`/`TextSpan`.
- `docs/design-system/12-localization-and-rtl.md` §7 (String discipline: 100% ARB coverage, canonical Sorani encoding) — **The one thing:** 100% ARB coverage gate, no hardcoded user strings; author `ckb` with **U+06D5 (ە)** for AE and **U+06A9 (ک)** for kaf — the lint rejects stray U+200C (ZWNJ) and Teh-Marbuta-for-AE; strings are transcreations reviewed for adab and tone, ckb provisional.
- `docs/design-system/12-localization-and-rtl.md` §8 (The Quran is never localized — only the chrome is) — **The one thing:** whatever the UI language, the muṣḥaf is the immutable QPC glyph page; no localization mechanism (mirroring, numerals, calendar, term-sets) ever reaches across the sacred/UI boundary.
- `docs/design-system/12-localization-and-rtl.md` §1 (RTL by geometry, logical start/end) — **The one thing:** fa/ckb/ar are first-class RTL citizens, not a bolted-on mode; logical start/end only; direction is the design baseline.

## Standards (primitives — cite the standard, not a blog)

- [Flutter — Internationalizing Flutter apps](https://docs.flutter.dev/ui/accessibility-and-internationalization/internationalization) — the `flutter_localizations` + `intl` + `gen_l10n`/ARB pipeline, `l10n.yaml` options, ARB key→getter rule, `GlobalWidgetsLocalizations` sets default text direction, ICU `plural`/`select`, custom-delegate recipe for an unsupported locale.
- [Unicode UAX #9 — Unicode Bidirectional Algorithm](https://www.unicode.org/reports/tr9/) — isolates (LRI/RLI/FSI + PDI) are "encouraged in new documents" over embeddings; an isolate "functions like a neutral character"; W4–W5 keep digits in logical order.
- [Unicode CLDR — LDML Part 2: General / Plural Rules](https://unicode-org.github.io/cldr/ldml/tr35-numbers.html) — the six plural categories and Arabic's `zero`/`one`/`two`/`few`/`many`/`other` boundaries.
- [Wikipedia — Eastern Arabic numerals](https://en.wikipedia.org/wiki/Eastern_Arabic_numerals) — Arabic-Indic (U+0660–U+0669) vs Extended Arabic-Indic/Persian (U+06F0–U+06F9): "different Unicode point even if it looks identical"; distinct 4/5/6 glyphs.
- [dart-lang/i18n #197](https://github.com/dart-lang/i18n/issues/197) — Eastern-digit handling "is implemented in dates, but not in NumberFormat"; the basis for pinning the `-u-nu-` numbering system and verifying digits per surface.
- [flutter/flutter #35103](https://github.com/flutter/flutter/issues/35103) — `ckb` (Central Kurdish) is not in built-in `GlobalMaterialLocalizations`; a custom delegate is required.
- [r12a / W3C — Sorani (ckb) orthography notes](https://r12a.github.io/scripts/arab/ckb.html) — U+06D5 (ە) AE over the heh+ZWNJ hack; U+06A9 (ک) kaf; canonical-encoding rationale.
- [WCAG 2.2 SC 1.4.1 Use of Color](https://www.w3.org/WAI/WCAG22/Understanding/use-of-color.html) and [SC 1.4.3 Contrast (Minimum)](https://www.w3.org/WAI/WCAG22/Understanding/contrast-minimum.html) — never color alone; ≥4.5:1 text contrast (decided at the token/view layer, not the string, but the `Semantics` label authored here completes the non-color encoding).
