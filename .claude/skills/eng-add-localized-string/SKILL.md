# eng-add-localized-string

Every user-facing word in Hifz Companion — visible UI, notification body, and screen-reader `Semantics` labels/hints — is an ARB entry routed through the first-party `flutter_localizations` + `intl` + `gen_l10n` pipeline and read **only** through the generated `AppLocalizations` accessor. The base content language is **Arabic (`ar`, `app_ar.arb`)**; `fa` and `ckb` are *transcreations* diffed against it, never literal translations. This skill fixes *how* a string is added so it survives the gates: the compile-time key-coverage gate (`nullable-getter: false` makes a missing/typo key a build error), the CI grep that bans hardcoded user-facing literals in `features/**`, the ASCII-digit grep, the Arabic-plural completeness check, and the per-locale RTL/numeral golden suite (`docs/engineering/12-localization-rtl-accessibility-impl.md` §1, §8). Get the key shape, the `AppLocalizations` lookup, the ICU plural with all six Arabic categories, the `numberFormatFor(locale)`-formatted count, the FSI/PDI-isolated mixed run, and the per-locale transcreation right at write time and the gates stay green; skip one and the build fails.

Three rules outrank everything below, because copy in this app is an act of *adab* and the sacred-text boundary is existential (`docs/design-system/11-voice-and-tone.md` §1; `docs/design-system/12-localization-and-rtl.md` §8):

- **The voice is *adab*, and *adab* is the first gate.** A string that is clear but harsh fails before anything else. The four fixed voice attributes — reverent, calm, plain-and-warm, honest — bind every string in every locale; the never-ship list (guilt/fear/loss framing, "You'll lose your hifz," controlling mandates "you must/should/don't," "safe to drop"/"mastered"/"done," exclamation marks and emoji in product copy, any commercial-transaction word) is release-blocking, not a style note (`docs/design-system/11-voice-and-tone.md` §2, §6, §9).
- **The Quran is never localized — only the chrome is.** This pipeline touches buttons, labels, dates, numerals, and term-sets. It never reaches a single glyph of the muṣḥaf, never runs an ayah number through `NumberFormat`, never passes a glyph run through the bidi helper. The immutable glyph layer is owned by `domain-mushaf-text-integrity`; this skill stops at the chrome (`docs/engineering/12-localization-rtl-accessibility-impl.md` §5; `docs/design-system/12-localization-and-rtl.md` §8).
- **Religious copy ships only behind scholarly review.** The sabaq/sabqi/manzil term-sets, any methodology-adjacent wording, and the Kurdish-Sorani defaults are transcreations that need native-speaker review for register *and* a scholar for terminology before they lock; until then the ARB `@description` carries "needs native + scholar review" and the string stays "aid to revision," never a ruling (`docs/design-system/11-voice-and-tone.md` §9; `docs/engineering/12-localization-rtl-accessibility-impl.md` §3, §5; `docs/design-system/12-localization-and-rtl.md` §6).

## When to use

Use this skill when:

- Adding or changing **any** `Text(...)`, `tooltip:`, `semanticsLabel:`, or `label:`/`hint:` string a user sees, hears (TalkBack/VoiceOver), or reads in a notification — every such string resolves through `AppLocalizations` (`l10n.*`), never a literal.
- Introducing a **count-bearing** string ("N pages due", "N days in your catch-up plan", "N sign-offs") — it must be an ICU `plural` message with Arabic's full CLDR set (`zero`/`one`/`two`/`few`/`many`/`other`), never a `count == 1 ? … : …` ternary or a `"$count pages"` splice.
- Adding a string that **interpolates a value or mixed-script run** — a page number, "Juz N", a surah name beside RTL copy, a date, a percentage, a user-typed profile name — it needs FSI/PDI bidi isolation so the run can't reorder the line.
- Adding a string with a **locale numeral or date** — the count/date is formatted via `numberFormatFor(locale)` / `intl` `DateFormat` *first*, then injected into an ICU placeholder; raw ASCII digits are never concatenated.
- Adding or editing a **regional term-set entry** (sabaq/sabqi/manzil track labels, the four traditional grade verbs, cycle names) — the ICU `select` over a region key / region-override file, never an edited base string. Triggers the scholarly-review gate.
- Adding the **`ckb` (Kurdish Sorani)** value for any key — it is canonical-encoding-sensitive (U+06D5 ە, U+06A9 ک), stays flagged provisional until native + scholar review, and may need the vendored `CkbMaterialLocalizations` delegate for framework chrome.
- Renaming a key, or splitting one phrase into two keys because it means two things in two contexts.

Do NOT use this skill for:

- **Calendar conversion / date math** — Hijri Umm al-Qurā / Solar-Hijri Jalālī / Gregorian conversion and the `CalendarDate` value type are `domain-calendars-and-hifzdate`; this skill only *renders* the already-converted `(y,m,d)` into locale numerals inside an RTL line.
- **What a screen-reader label should *say*** (the sentence, role, traversal strategy) and the RTL mirror / golden-screenshot audit — that is the accessibility/RTL implementation, decided alongside `eng-add-feature-module`; this skill only stores and transcreates the already-decided sentence and keeps its interpolation/numerals/bidi correct.
- **Any muṣḥaf glyph, ayah number, juz/sajda marker, or page text** → `domain-mushaf-text-integrity`. The localization toolkit never crosses the sacred-text boundary.
- **Choosing the never-ship phrase list / the adab verdict itself** on a sensitive string → `domain-adab-and-religious-integrity` owns the conscience-check; this skill enforces that the cleared wording lands in ARB correctly.
- A new **user-facing factual/methodology claim** (a number, range, or guidance) — its *wording* uses this skill, but the claim itself must first land in the claims register via `domain-claims-register-and-science-screen`.

## The canonical pattern

1. **Add the key to `app_ar.arb` first — `ar` is the template/base content language.** Every new string is authored in `lib/l10n/app_ar.arb` (the `l10n.yaml` `template-arb-file`), because the source-of-truth copy must be in the language a reviewer/scholar reads natively, and `fa`/`ckb` are diffed against it. The key becomes a getter/method on the generated `AppLocalizations`; `@`-prefixed metadata carries the `description` (translator/scholar context) and `placeholders`. `docs/engineering/12-localization-rtl-accessibility-impl.md` §1 (Decision + ARB entry shape); `docs/design-system/12-localization-and-rtl.md` §7 (100% ARB coverage, no hardcoded strings).

2. **Read it only through `AppLocalizations` (`l10n.*`); never a literal in `features/`.** A CI grep bans string literals passed to `Text(...)`, `tooltip:`, `semanticsLabel:`, and `label:` inside `features/**`; every user-facing string resolves through `AppLocalizations.of(context)`. `nullable-getter: false` means a missing/typo key is a **compile error**, not a silent empty widget. `docs/engineering/12-localization-rtl-accessibility-impl.md` §1 (Pitfalls: refuse hardcoded strings, refuse silently-missing lookup); `docs/engineering/12-localization-rtl-accessibility-impl.md` §8 (gate: key coverage + hardcoded-string grep).

3. **No user-facing strings in the engine.** `/engine` is pure, deterministic, and locale-free; it returns structured state (grades, day-counts, track enums), and the presentation layer localizes them. A grep bans `AppLocalizations`/`intl` imports from `/engine`. If you find yourself wanting a string in the engine, return an enum/value and map it to `l10n.*` at the feature boundary. `docs/engineering/12-localization-rtl-accessibility-impl.md` §1 (Pitfalls: refuse strings in the engine); cross-ref `domain-scheduling-engine-rules`.

4. **Interpolation = ICU placeholders in one full message; never concatenate fragments.** Any string with a value is a single ICU message with `{placeholder}` slots, never two localized pieces glued together — RTL word order will not survive concatenation, and Arabic/Persian grammar differs. The opposite-direction value stays a **placeholder** (so the generated `intl` formatter can isolate it), never a hard-spliced substring. `docs/engineering/12-localization-rtl-accessibility-impl.md` §4 (Specification: isolated *before* injection into the ICU placeholder); `docs/design-system/12-localization-and-rtl.md` §3 (placeholder, never hard-spliced substring); `docs/design-system/11-voice-and-tone.md` §8 (never concatenate sentence fragments).

5. **Every count-bearing string is an ICU `plural` with Arabic's six CLDR categories.** Cycle pages-due, catch-up days, sign-off counts — each is an ICU `plural` message, and the ARB review treats Arabic's full set (`zero`, `one`, `two`, `few`, `many`, `other`) as a *translation contract*: a missing category is a release blocker, not a cosmetic gap. The count is also formatted to locale numerals (step 6) before placement, so number agreement and digit shaping are both correct. The ARB-completeness CI step asserts every `plural` message in `app_ar.arb` defines all six. `docs/engineering/12-localization-rtl-accessibility-impl.md` §6 (Decision + the `pagesDue` six-category template); `docs/design-system/12-localization-and-rtl.md` §7 (ICU plurals where the language needs it).

6. **Numerals and dates render via `numberFormatFor(locale)` / `intl` — never hand-formatted ASCII.** A digit a user reads (page number, juz/ḥizb index, retention %, budget minutes, a date field) is formatted through `numberFormatFor(locale)` — which pins the numbering system explicitly: **`fa`/`ckb` → Extended Arabic-Indic** (`-u-nu-arabext`, U+06F0 ۰۱۲۳۴۵۶۷۸۹), **`ar` → Arabic-Indic** (`-u-nu-arab`, U+0660 ٠١٢٣٤٥٦٧٨٩) — then injected as an ICU placeholder. Never concatenate `"$count "` or build `"Page " + n.toString()`. This is the `type.numeral` discipline. Calendar `(y,m,d)` values arrive Latin-digited from `domain-calendars-and-hifzdate` and are re-mapped to locale digits **here, downstream of the conversion**. `docs/engineering/12-localization-rtl-accessibility-impl.md` §5 (`numberFormatFor`, pinned `-u-nu-`, ASCII-digit refusal, calendar boundary); `docs/design-system/12-localization-and-rtl.md` §4 (`type.numeral`, two distinct Unicode blocks, always via `intl`).

7. **Mixed-script runs go through the one bidi-isolation helper (FSI/PDI).** A page number, "Juz N", a surah name beside RTL copy, a date, a percentage, a user-typed name — any embedded opposite-direction run — is wrapped in Unicode **isolates** before display via the single `l10n/bidi.dart` helper: `isolate(run)` (FSI…PDI) for *unknown* direction, `isolateLtr`/`isolateRtl` (LRI/RLI…PDI) when direction is *known* (preferred over FSI — first-strong mis-detects on leading punctuation). Raw concatenation of opposite-direction runs is banned. `docs/engineering/12-localization-rtl-accessibility-impl.md` §4 (Decision + helper + isolate-known-direction rule); `docs/design-system/12-localization-and-rtl.md` §3 (FSI/PDI isolation; never legacy LRE/RLE embeddings).

8. **Regional sabaq/sabqi/manzil term-sets are an ICU `select` / region-override file — never edited base strings.** The traditional track labels, four grade verbs, and cycle names are *data*, switched by a region key (`select, region, {levant{…} subcontinent{…} other{…}}`) or a per-region override file the build merges, so swapping a whole vocabulary is one file, never a code change. The four-grade scale shows the localized *traditional verb* from the active term-set; the engine signal is unchanged. The `ckb` defaults stay flagged provisional. `docs/engineering/12-localization-rtl-accessibility-impl.md` §5 (region-override `select`, NEEDS scholar review); `docs/design-system/12-localization-and-rtl.md` §6 (term-sets are swappable string data with a regional override; never hard-code "Manzil"/"Sabqi").

9. **`ckb` is a custom locale with canonical encoding and a vendored chrome delegate.** `ckb` is declared `Locale.fromSubtags(languageCode: 'ckb')` and needs the vendored `CkbMaterialLocalizations` delegate for framework chrome Flutter does not ship (OK/Cancel, date-picker labels). Author Sorani with **canonical Unicode**: U+06D5 (ە) for AE, U+06A9 (ک) for kaf — a lint rejects stray U+200C (ZWNJ) and any Teh-Marbuta-for-AE substitution before build. The bundled UI font's Sorani coverage (پ چ ژ گ ڤ ڕ ڵ ۆ ێ ە) is CI-verified. Direction is *not* part of this work — `ckb` resolving to RTL is automatic. `docs/engineering/12-localization-rtl-accessibility-impl.md` §3 (custom locale + vendored delegate + glyph coverage); `docs/design-system/12-localization-and-rtl.md` §7 (canonical Sorani encoding, reject ZWNJ/Teh-Marbuta hacks).

10. **Transcreate per locale to the four-attribute voice charter; never literal-translate.** Pronoun, register, and the imperative do **not** map across fa/ar/ckb, so a literal translation silently changes the tone. Set register deliberately: **Persian** respectful-warm (*šomā*/honorific, *taʿārof*-aware — never cold over-formality nor over-familiar *to*); **Arabic** number/gender-appropriate forms with **imperatives softened into statements of readiness** (a bare command verb reads curt); **Kurdish Sorani** register *and* religious vocabulary set by native reviewers, defaults provisional. Hard-news strings follow the empathy-then-path template (acknowledge → fact → path → choice). `docs/design-system/11-voice-and-tone.md` §8 (transcreation + per-locale register), §3–§4 (tone matrix, lead with empathy); `docs/engineering/12-localization-rtl-accessibility-impl.md` §3 (ckb copy stays provisional).

11. **Direction stays locale-derived; never assume RTL in a string.** Strings carry no physical-side assumptions and no hardcoded direction. Because all three locales are RTL, direction follows the locale automatically; if a string needs an LTR island (a Latin technical token), that is a `Directionality(TextDirection.ltr)` wrapper at the *view* layer, not a property of the ARB value. `docs/engineering/12-localization-rtl-accessibility-impl.md` §2 (direction derived from locale, never hardcoded); `docs/design-system/12-localization-and-rtl.md` §1 (RTL by geometry, logical start/end).

12. **Run the localization & accessibility gate before you call it done.** Codegen + `nullable-getter: false` (key coverage), the `features/**` hardcoded-string grep, the physical-side grep, the ASCII-digit grep, the Arabic-plural completeness check, the canonical-`ckb` lint, the per-locale RTL/numeral goldens (loaded with the **real** bundled fonts, never `Ahem`), and a TalkBack/VoiceOver pass over the cold-start → recite → grade journey. Religious/methodology wording additionally needs the scholarly review. `docs/engineering/12-localization-rtl-accessibility-impl.md` §8 (the layered gate table); `docs/design-system/11-voice-and-tone.md` §9 (banned-phrase lint + native + scholar review per locale); cross-ref `eng-write-dart-test`.

## Do / Don't

| Do | Don't |
|---|---|
| Author the key in `app_ar.arb` (the `ar` template/base) first, with an `@description`; transcreate `fa`/`ckb`/`ar` against it | Hand-edit `fa`/`ckb` first, or literal-translate so register/tone drifts silently |
| Read every user-facing string through `AppLocalizations` (`l10n.*`) | Pass a string literal to `Text`/`tooltip:`/`semanticsLabel:`/`label:` in `features/**` |
| Keep `/engine` locale-free — return enums/values, localize at the feature boundary | Import `AppLocalizations`/`intl` into `/engine`, or put user-facing copy there |
| Write one full ICU message with `{placeholder}` slots for interpolation | Concatenate localized fragments, or hard-splice a value into the string |
| Make every count an ICU `plural` with all six Arabic categories (`zero`/`one`/`two`/`few`/`many`/`other`) | Write `count == 1 ? … : …`, a `"$count pages"` splice, or omit `few`/`many` |
| Format every numeral/date via `numberFormatFor(locale)` (`fa`/`ckb` arabext, `ar` arab), then inject as a placeholder | Concatenate ASCII digits, `"Page " + n.toString()`, or use the Arabic block for fa/ckb (or vice-versa) |
| Wrap every mixed-script run through the `bidi.dart` helper — `isolate` (FSI) for unknown, `isolateLtr`/`isolateRtl` for known direction | Raw-concatenate opposite-direction runs, use FSI where direction is known, or legacy LRE/RLE embeddings |
| Model sabaq/sabqi/manzil term-sets as an ICU `select`/region-override; show the active term-set's traditional grade verb | Hard-code "Manzil"/"Sabqi" in a widget, or edit a base string to localize a region's vocabulary |
| Author `ckb` with canonical encoding (U+06D5 ە, U+06A9 ک); keep it flagged provisional pending native + scholar review | Ship the heh+ZWNJ AE hack / Teh-Marbuta-for-AE, or lock `ckb` terminology before review |
| Pass every string through the adab gate; transcreate to the four voice attributes; soften Arabic imperatives to statements of readiness | Ship guilt/fear/loss, "safe to drop"/"mastered", a mandate ("you must/should"), an exclamation mark, emoji, or "upgrade/premium" |
| Render goldens with the real bundled UI fonts and run the per-locale RTL/numeral suite + TalkBack/VoiceOver pass | Render goldens with `Ahem`, or defer RTL/accessibility to a "polish" phase |

## Checklist

Before this string is done:

- [ ] Key authored in `lib/l10n/app_ar.arb` (the `ar` template/base content language) with an `@description`; `fa`/`ckb`/`ar` values are **transcreations** diffed against `ar`, not literal translations.
- [ ] Read only through `AppLocalizations` (`l10n.*`); no string literal in `features/**` (`Text`/`tooltip:`/`semanticsLabel:`/`label:`). `nullable-getter: false` keeps a missing/typo key a compile error.
- [ ] No user-facing string in `/engine`; the engine returns enums/values, the feature layer localizes them.
- [ ] Interpolation is one full ICU message with `{placeholder}` slots — no concatenated fragments, no hard-spliced substring.
- [ ] Count-bearing → ICU `plural` with **all six** Arabic CLDR categories (`zero`/`one`/`two`/`few`/`many`/`other`); the count is locale-numeral-formatted before placement; no ternary, no `"$count"` splice.
- [ ] Every numeral/date comes from `numberFormatFor(locale)` / `intl` `DateFormat` (`fa`/`ckb` Extended Arabic-Indic, `ar` Arabic-Indic) and is injected as a placeholder — never hand-formatted ASCII, never the wrong digit block. Calendar `(y,m,d)` is re-mapped to locale digits downstream of `domain-calendars-and-hifzdate`.
- [ ] Every mixed-script run goes through the `bidi.dart` helper: `isolate` (FSI…PDI) for unknown direction, `isolateLtr`/`isolateRtl` (LRI/RLI…PDI) when direction is known — no raw opposite-direction concatenation, no legacy LRE/RLE.
- [ ] Regional sabaq/sabqi/manzil terms are an ICU `select`/region-override (one-file swap), not a hard-coded label or an edited base string; the active term-set's traditional grade verb shows for Again/Hard/Good/Easy.
- [ ] `ckb` value is canonical-encoded (U+06D5 ە for AE, U+06A9 ک for kaf; no stray ZWNJ, no Teh-Marbuta-for-AE), flagged "needs native + scholar review" in the `@description`, and the vendored `CkbMaterialLocalizations` chrome is in place; bundled-font Sorani coverage verified.
- [ ] The string passes the **adab gate first** and the four voice attributes (reverent/calm/plain-and-warm/honest); no never-ship phrase (guilt/fear/loss, "safe to drop"/"mastered"/"done", "you must/should/don't", exclamation marks, emoji, "upgrade/premium"); Arabic imperatives softened to statements of readiness; hard news follows empathy → fact → path → choice.
- [ ] Direction stays locale-derived; the ARB value carries no physical-side or hardcoded-RTL assumption.
- [ ] The gate is green: key coverage (compile), `features/**` hardcoded-string grep, physical-side grep, ASCII-digit grep, Arabic-plural completeness, canonical-`ckb` lint, per-locale RTL/numeral goldens (real bundled fonts, never `Ahem`), and a TalkBack/VoiceOver pass.
- [ ] If the string is a term-set, methodology-adjacent, or any religious wording, the per-locale **native + scholar review** has cleared it (or it ships clearly marked provisional and does not state a ruling).

This skill governs only the *chrome*. If a string ever surfaces a methodology or factual claim, that claim must first land in the claims register (`domain-claims-register-and-science-screen`); if it touches the muṣḥaf, the page is `domain-mushaf-text-integrity` and the localization toolkit stops at the chrome boundary.
